# HAMR v3.2 — post-spike redesign baseline 2026-05-04

## Summary

HAMR v3.2 is the design baseline after the 6-spike validation gate
(EPIC #860) and the 3 client clarifications (Q1 OS-agnostic key store,
Q2 judge quorum independence, Q3 failover/redundancy explicitness).
Substrate (CF Worker + R2 + KV + MCP + Tailscale fleet) survives
unchanged. Trust model, latency contract, cache strategy, compression
gate, key storage, judge gate, and failover semantics are revised.

## Key changes vs v3 input

- **Latency:** ≤80 ms p50 budget scoped to warm cache-hit only; cold
  ≤180 ms p95 disclosed (S3).
- **Cache:** 1-h extended default; bundle-rebuild cadence rate-limited
  to ≥5 min (S4).
- **Compression:** two-stage gate — deterministic Stage-1 (≥99%
  keyword) + reasoning-grounded Stage-2 (≥97%) with judge quorum (S5).
- **Key store:** 4-tier OS-agnostic ladder T1 hardware enclave → T2 OS
  keychain → T3 Age file → T4 ephemeral. ChromeOS LXC, BSD, illumos
  all supported (Q1).
- **Judge gate:** quorum-of-2 across model families with provenance
  tag. Local fleet judges eligible if vendor-attested + family-distinct
  (Q2). Replaces v3.1's "non-fleet cloud judge" wording.
- **Failover:** explicit 3-tier graceful degradation map (Q3); HAMR
  off ≡ today's harness; never gets worse.

## Eight remediations (R1–R8)

R1 signed governance + 4-tier key store; R2 signed mailbox envelopes;
R3 SLSA-L3 + Cosign verification; R4 warm-only ≤80 ms claim + HTTP/2
keepalive + KV edge-cache; R5 1-h extended cache + rebuild rate
limit; R6 two-stage rule-coverage gate; R7 capability-gated 3-tier
deployment; R8 live-measurement closure (Wave 1 #891–#893).

## Revised 9-child MVP + 5-wave order

S1 audit cut MVP from 13 → 9. S6 build-vs-adopt: 2 ADOPT, 4 BUILD, 3
HYBRID, 0 REUSE. New shared module `judge-quorum.js` added per Q2.

Wave 1 (foundation, 4 d): #891–#896.
Wave 2 (substrate, 5 d): children 1, 2, 6.
Wave 3 (mailbox + identity, 3 d): child 5.
Wave 4 (intelligence, 4 d): children 3, 7, 9.
Wave 5 (UX, 2 d): children 4, 8.

Total: 18 working days serial baton.

## Residuals after v3.2

3 of 9 S6 STRIDE MEDIUM/HIGH cells remain (A2-S T4 only, A4-T npm
transitive, A5-S PKCE-beta) — bounded surface area, not single-point
failures. A3-E HIGH eliminated.
