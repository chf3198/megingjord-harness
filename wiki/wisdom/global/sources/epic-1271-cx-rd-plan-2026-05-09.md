# Epic #1271 Codex R&D Plan: Epic-State Truthfulness

## Summary

Codex Team researched Epic-state truthfulness drift for #1274 under Epic #1271.
The plan reframes the problem as deterministic evidence reconciliation between
Epic AC checkboxes, child tickets, dependency state, time-window obligations,
and independent Consultant verification.

## Key Takeaways

- AC checkboxes should be parsed into an evidence table rather than trusted as
  prose.
- Time-windowed ACs need a distinct measuring state and recheck date.
- Cross-Epic dependencies should be parsed from text and native GitHub issue
  dependency APIs, then cycle-checked.
- Manager closeout text must be linted against AC state; unresolved ACs require
  an explicit `EPIC_RESCOPE` artifact.
- Consultant verification should be the only final Epic close authority.
- GHS/self-annealing sensors should detect declared-complete/unmet-AC drift.
- The 2025-2026 external research core supports traceability graphs, dependency
  validation, policy-as-code guardrails, and independent human/agent review.
