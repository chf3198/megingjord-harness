# HAMR v3.2.1 patch 2026-05-05

## Summary

Patch document amending v3.2 (#890) with three Wave-1-derived
deltas. v3.2 stays unmodified; v3.2 + this patch are the combined
input contract for Wave 2.

## R9 (NEW) · Cross-level resource-failure recovery

Four crash-recovery patterns validated during Wave 1:

- **R9.1 Worktree-isolation as crash-survivable execution surface**
  — validated by #895 recovery from VS Code crash.
- **R9.2 Cwd-vs-branch pre-flight for `gh pr create`** — learned
  from #899 mis-target → #900 reopen on #895.
- **R9.3 Sequential dispatch with backoff + family-fallback** —
  validated by #893 quiz with 14/14 Cerebras→Gemini covers.
- **R9.4 Idempotent infrastructure tear-down** — validated by
  #891 wrangler-delete + HTTP-404 verification.

R9 becomes a required architectural input for Wave 2 (HAMR core
Worker + substrate-health probe + SLSA pipeline).

## §R6 update · 3-stage gate replaces v3.2 binary

Per #893 live measurement, v3.2 §R6 ≥97% Stage-2 threshold is not
achievable on free-fleet quorum (direct mean 0.55 / counter-
factual 0.50 / boundary 0%). Replacement:

- **Stage-1** every build, ≥99% deterministic keyword (unchanged).
- **Stage-2a** weekly, free-fleet 2-of-N quorum, **≥80% on direct
  plus counter-factual** (boundary excluded).
- **Stage-2b** monthly OR rule-change PR, paid-tier OR fine-tuned,
  **≥95% including boundary cases**.
- **Stage-3** on-demand operator review for any rule scoring
  <0.50 in Stage-2b.

Trades guarantee strength for operational feasibility on free
path; preserves paid path for high-stakes verification.
