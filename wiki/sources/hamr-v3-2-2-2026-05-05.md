# HAMR v3.2.2 R9.2 hook patch 2026-05-05

## Summary

Patch document amending v3.2.1 §R9.2 (#907) with three sub-patterns
that turn cwd-vs-branch pre-flight from a deploy-script convention
into a harness-level guard. Triggered by the same hazard catching
HAMR PR cycles three times across Waves 1–3.

## Three sub-patterns

- **§R9.2.1** Bash-hook contract — pre-tool guard asserts (a) HEAD
  not detached, (b) no in-progress rebase/merge/cherry-pick/bisect,
  (c) branch matches `EXPECTED_BRANCH` if set.
- **§R9.2.2** `gh pr create` MUST pass explicit `--head <ref>` —
  cwd-resolved head is BANNED. Same principle applies to
  `gh pr merge` PR-number explicitness and `git push` ref
  explicitness in scripts.
- **§R9.2.3** Branch-ops audit log — every cross-branch operation
  writes one line to `~/.megingjord/branch-ops.log` for
  post-incident reconstruction.

## Implementation deferred

This is a docs-research patch only. Actual hook implementation is
a separate development child ticket spawned post-merge so the
patch can ship without code drift.

## Wave 4 prerequisite

Documented as a required architectural input. Wave 4 children 3,
7, 9 do not directly implement R9.2.1–R9.2.3; that work is a
separate ticket tracked alongside Wave 4.
