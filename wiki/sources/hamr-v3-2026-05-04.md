# HAMR v3 — 5-axis optimization 2026-05-04

## Summary

Builds on v1 (#861) + v2 (#863). Acronym formalized as **HAMR — Harness-Aware Mesh Routing**. Optimizes the design across 5 axes with cutting-edge 2026-Q2 primary-source evidence.

## Axis 1 — Security

SLSA Build L3 provenance for the Worker; OIDC-federated CI publishing (zero long-lived tokens); Cosign Bundle 1.0 single-file attestation; MCP OAuth 2.1 + DPoP (RFC 9449); capability manifests in the bundle (explicit per-tool scopes); gitleaks/TruffleHog scan at bundle build; narrowed Anthropic content-blocks via forced tool-use.

## Axis 2 — User-friendliness

`npx megingjord init` one-command install; GitHub device-flow OAuth; progressive disclosure with sensible defaults; never-silent failure surfacing in operator terminal; `hamr:status / doctor / quota` self-check; Diátaxis four-quadrant docs; JSON Schema `$id` versioning for graceful client degradation.

## Axis 3 — Token-use minimization (no quality loss)

4 per-tier sub-bundles (FIM 5KB / coding 15KB / reasoning 30KB / docs 12KB); differential bundles via JSON Patch RFC 6902; LLMLingua-2 distilled constitution with ≥97% rule-coverage gate; structured outputs via JSON Schema (30–60% fewer tokens); JIT wiki retrieval over MCP `resources/read`; sub-agent isolation; compact memory digest replacing full session history. **Estimated cumulative ~80% session-token reduction.**

## Axis 4 — Paid-token min + rate-limit compliance

Anthropic / OpenAI / Gemini Batch APIs (50% off, 24h SLA) for non-interactive baton work; sticky cache routing for ≥80% prompt-cache hit-rate; Anthropic context-editing for >30-turn sessions; header-driven spillover routing across Anthropic/Groq/OpenAI/OpenRouter; token-budget per baton-role; `hamr:quota` polls count_tokens + usage endpoints.

## Axis 5 — Maintenance

Wrangler 4.x drift detection; Tail Workers + Analytics Engine (free 10M ops/month) for observability; R2 lifecycle rules for stale-bundle eviction; JSON Schema `$id` graceful degradation; Dependabot scoped to Worker source; R2-price / CF-outage migration paths documented.

## Decisions: v3 changes vs v2

13 explicit changes / additions tracked in the deliverable. Notable:
- ADD: SLSA L3, OIDC publishing, Cosign Bundle 1.0, DPoP-bound MCP OAuth (security hardening)
- ADD: per-tier sub-bundles, distilled constitution, differential JSON-Patch (token min)
- ADD: Batch API integration, sticky cache routing, context-editing (paid-token min)
- KEEP: CF Worker + R2 + KV + MCP substrate (v1/v2 happy path validated)
- KEEP: Yjs CRDT bidirectional Wiki (v2 still optimal)
- REJECT: IPFS/libp2p federated wiki (ops complexity outweighs benefit at this scale)
- REJECT: Browser-side WASM agents (no concrete near-term win)

## MVP children — re-ordered + 4 new

13 children total (vs v2's 9). 4 new from v3 optimizations: SLSA-L3 build pipeline, per-tier sub-bundle generator, distilled-constitution compressor, Batch API integration. NOT spawned per Manager scope.
