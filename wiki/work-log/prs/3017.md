Phase-1 child of Epic #2707 (mandatory parallel technical writing). Activates the
doc-coverage Tech-Writer gate + the cross-family review check, which were **dead code**.

## Root cause
`collaborator-handoff.js` ran both checks only when `input.lane === 'lane:code-change'`,
but `.github/workflows/baton-gates.yml` calls `validate({ comments, labels })` with no
`lane` scalar — so `input.lane` was always `undefined` and both checks were silently
skipped on every PR, despite #2712 shipping the blocking-capable validator.

## Fix
- Derive lane from `labels` when the scalar is absent (back-compat preserved).
- Accept both `doc-coverage:` and `doc_coverage:` block keys (canonical builder uses underscore).
- **Fail-closed** on matrix load: a throw *or* a null/empty matrix now blocks (was fail-open).
- `LEGACY_DOC_SKIP` escape hatch, line-anchored to a real `BLOCKER_NOTE` (per #2707 AC1).
- Add `cross_family_receipt` (required) + `reviewer_family` to the baton-artifact builder
  schema so the canonical tool can emit a gate-conforming handoff (closes a #2904 drift).
