## Summary
Adds `scripts/global/load-local-env.js` — a shared, secret-safe shim that auto-hydrates the gitignored
repo-root `.env` into `process.env` so a fresh session reaches the **zero-cost** fleet + free-cloud
review/provider lanes with **no manual `set -a; . ./.env` step** (G3). Root cause was verified:
`free-cloud-dispatch.js` read `process.env` only, with no dotenv load and no SessionStart hydration.

Wired into `free-cloud-dispatch.js`, `cascade-dispatch.js`, and `hamr-provider-wrapper.js` (guarded so
injected-env unit tests are unaffected). Contract: fill-don't-override precedence (G2), value-free
redaction-safe audit line (G4), graceful no-op on missing/malformed `.env` (G5/G6), `MEGINGJORD_NO_DOTENV=1`
disable, single shared shim across all 3 dispatchers (G9/G10).

Scope boundary vs **Epic #2291** (global cross-entrypoint hydration + secret-storage strategy) and
**#2569** (prompt-suppression) documented on the ticket; this PR is the dispatch-local G3 slice.
