## Summary
Cross-family generator-critic verification of fleet-authored code with prove-it escalation (Epic #2791
P1-4; Phase-0 #2792 design D4/D8-critic). New pure module `fleet-cross-family-critic.js` dispatches a
fleet-authored pr-diff to N **different-family** critics (reusing #2041 `dispatchRedTeam` / `pr-diff`
artifact), takes a majority panel verdict, and decides mergeability under the invariant that **objective
gates are authoritative**:

- gates fail → **block**; gates pass + panel accept + no flagged flaw → **merge**.
- a critic REJECT/flaw is **prove-it'd**: reproduces → **re-arm** the verifier; an unprovable
  **high-severity** security pattern → **escalate** one tier; otherwise → **advisory**. Never silent-merge.
- scans **every** critic before honoring a majority accept (a minority critic's flaw outranks it).
- every REJECT is prove-it'd **keyword-independently** (prove-it, not a regex, is the arbiter of "is it real").
- AC4 multi-critic panel = majority; exact tie escalates one tier.
- classifier is injectable (`deps.classifyFlaw`) — the seam for the #2888 model/structured-tag upgrade.
