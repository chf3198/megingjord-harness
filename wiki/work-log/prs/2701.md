## Summary
Adds a **deterministic credential pre-prompt guard** (G1/G4) so the operator never asks the client for a
secret already available in the approved local `.env`. New `scripts/global/credential-availability.js`:
`isSecretLocallyAvailable(name)` probes `process.env` + the local `.env` (via #2645's `loadLocalEnv`,
non-mutating, value-free); `classifyCredentialRequest(text)` conservatively distinguishes a credential
request from ordinary clarification (anti-over-block); `preCredentialPromptCheck([names])` returns
`use-local` when any is available else `report-absent-no-prompt` (absent ⇒ report + terminal-entry, never
request the raw secret in chat). New policy `instructions/credential-prompt-guard.instructions.md` (wired
into `CLAUDE.md`) codifies the forbidden behavior.

Boundary: #2645 (closed) owns hydration; Epic #2291/#2292 own the global entrypoint inventory +
storage-strategy. This PR ships only the prompt-suppression guard, consuming #2645's shim.
