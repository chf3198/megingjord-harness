## Summary

scripts/sync.sh copies ~/.copilot/ INTO the canonical main checkout (the inverse of scripts/deploy.sh). When ~/.copilot/ holds stale content (pre-merge runtime state), running sync.sh from main regresses tracked source files to pre-merge versions. Root cause of the 2026-05-27 22:14:10 incident that silently reverted 16 files merged via PRs #2304 + #2308.

This PR adds a refusal guard:
- scripts/sync.sh exits 2 with a redirect message when invoked from ${HOME}/devenv-ops/ unless --allow-canonical-write is passed
- The guard emits a v3 incidents.jsonl event on trip (pattern_id: sync-sh-reverse-direction-regresses-main)
- 6 test cases in tests/sync-canonical-main-refuse.spec.js validate the behavior
- instructions/global-standards.instructions.md (Canonical-main checkout policy) documents the trap
