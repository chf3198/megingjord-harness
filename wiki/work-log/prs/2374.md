## Summary

Tier-2 anneal landing: implements `scripts/global/megalint/cross-team-response-fidelity.js` validator that catches the cross-team signature-forgery pattern observed across rounds 2+3 of #2360. Parses TEAM_RESPONSE artifacts in issue comments, extracts the from-field target team and the signing block, parses Team&Model into team:model@substrate, and emits `team-response-signer-team-mismatch` when the signer-team does not equal the from-team. Also emits `signer-alias-non-derived` when the Signed-by literal does not match the registry-derived alias for the asserted (team, model, role) tuple.

Reuses existing `signer-registry-check.js` helpers so derivation logic stays single-source. Five violation rules; all emit `severity: 'advisory'` per AC5 so the orchestrator can implement advisory-mode-soak by filtering on severity. Promotion to blocking is a CI-workflow-config decision after one cycle of soak, not a code change in this PR.

Documents one residual gap (covered by a test): perfect-forgery (source team forges both alias AND team-model string) cannot be caught by this validator alone; requires the optional Crypto-Signature fields per `instructions/team-model-signing.instructions.md`.
