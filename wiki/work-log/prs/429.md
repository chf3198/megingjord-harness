## How it works

On every push to `main`: release-please scans commits since last release, groups by Conventional Commit type, and opens/updates a Release PR with CHANGELOG diff and version bump. Merging the release PR creates the Git tag and GitHub Release. `feat:` commits → minor bump; `fix:` commits → patch bump; `feat!:` or `BREAKING CHANGE` → major bump.
