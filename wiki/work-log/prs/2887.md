## Summary
Fleet-dev tests (#2795/#2796) were writing ~9k junk records to **production** telemetry
(`~/.megingjord/*.jsonl`) because `defaultEmit` wrote real files even under the test runner — corrupting
the #2800 cost report's escalation/demotion data. This fix:

1. **`MEGINGJORD_NO_TELEMETRY` opt-out** honored at the top of both `defaultEmit` surfaces (governor +
   gates). Default OFF → live emit path unchanged.
2. **Traversal-safe `MEGINGJORD_TELEMETRY_DIR` redirect** — new pure helper `fleet-telemetry-path.js`
   rejects any `..` path segment and preserves the basename (env can choose a directory, never an
   arbitrary file). Lets the isolation spec write to a per-test temp dir → no shared global file, no
   cross-worker flake.
3. **Cleaned the polluted files**; the #2800 cost report now reads clean (escalation-rate 0%,
   demotions `{}`, NET +$0.084).
