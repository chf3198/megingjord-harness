## Summary
The **packaging capstone** for Epic #2791 (P1-8; Phase-0 #2792 design D16). Packages the whole fleet-dev
capability (#2794–#2800) as a **discrete, opt-in, tier-3-gated** HAMR module so a **fleetless (Tier 0–2)
operator carries zero fleet code** and the baseline router stays **byte-for-byte unchanged**.

- `config/fleet-dev-module.json` — the manifest: `requiredTier: 3`, opt-in `MEGINGJORD_FLEET_DEV_ENABLED`,
  the 9 member files.
- `scripts/global/fleet-dev-module.js` — `fleetDevAvailable()` offers the fleet-dev lane **only** when
  installed ∧ enabled ∧ `MEGINGJORD_MINIMUM_TIER ≥ 3` (explicit, via #2637 `assertTier`; unset → fleetless,
  never offered). **Fail-closed**; empty env can never enable-by-default; own-property-only manifest reads.
- The Copilot-owned router (`model-routing-engine.js`) is **not touched** and has zero fleet-dev imports —
  so baseline routing is byte-for-byte unchanged when off (no stubbed dep, no dead path), respecting the G9
  ownership boundary.
- Every member declares `// tier: 3`, enforced by a manifest↔tag contract test (AC4).
