## Summary

Replace `.catch(() => {})` silent swallows on `removeLabel` and `addLabels`
calls in the `auto-transition` block of `label-lint.yml` with explicit
`try/catch` + `core.setFailed()`. Previously, GitHub API failures (rate limits,
permission errors, label-not-found edge cases) were silently swallowed,
leaving the workflow green while the label state diverged.
