## Summary
Wires both fleet dispatch request-timeouts to `config/timeout-policy.json` (the classes already exist) instead
of hardcoding, with a graceful fallback, plus a progress heartbeat:

- `fleet-red-team-dispatch.js` `REQUEST_TIMEOUT_MS` ← `fleet-red-team-rate` (1200s; Phase-0 #2174 p99=907s),
  fallback **600s**.
- `ollama-direct.js` `TIMEOUT_MS` ← `fleet-dispatch-basic` (300s), fallback **120s**.
- Fallback fires on a missing / malformed / incomplete / non-positive policy → reverts to the prior
  known-good value (a corrupt policy can never produce a 0/NaN/negative timeout).
- A 30s `unref`'d stderr heartbeat (cleared in `finally`) so slow Tailscale generations aren't mistaken for
  a hung terminal — preventing premature availability-failure → paid escalation (**G3**).
- Loaders exported + path-injectable with unit coverage.
