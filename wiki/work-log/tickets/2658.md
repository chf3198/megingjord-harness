# #2658 — Make canonical-main read-only enforcement cross-runtime (not Claude-Code-only)

## Body

## Self-anneal origin (Tier-2, recurrence ≥2 in one session)
Surfaced 2026-06-04, **twice this session**. The Copilot Team (Auto mode) left uncommitted tracked-file WIP in the canonical main checkout (`~/devenv-ops/`) — first the #2638/#2640 state-semantics work, then #1948 Phase-1 test-floor work. Both violated the #2107 canonical-main read-only policy and blocked an unrelated session's Stop gate. Incidents: `concurrent-writer-left-uncommitted-tracked-files-in-canonical-main`, `codex-writes-uncommitted-to-canonical-main-recurrence` (mis-labeled "codex"; actually Copilot — see attribution correction).

## Root cause
Epic #2091 C6 shipped `hooks/scripts/canonical_main_enforcer.py`, but it is invoked **only by Claude Code's `pretool_guard.py`** — it intercepts Claude Code's Edit/Write tool calls. **Other runtimes (Copilot, Codex, Antigravity) write to the filesystem through their own extensions, which never call Claude Code's pretool_guard** — so their tracked-file writes to canonical main are not caught. The enforcement is per-runtime (Claude-Code-only), not per-checkout.

This is the **correctly-framed residual** of Epic #2071's retired "cross-runtime write-prevention" concern: the flock-on-shared-checkout mechanism (#2071 C2/C3, cancelled) was the wrong tool, but the *need* — stop ANY runtime from writing tracked files into canonical main — is real and unmet by #2091 (which guards one runtime).

## Candidate approaches (Phase-0 to evaluate)
- A **git-native guard** in the canonical checkout (`pre-commit` + a `pre-add`-equivalent / `core.fsmonitor` watcher) that rejects tracked-file staging on `main`, firing regardless of which runtime made the edit.
- **Per-runtime hook installation**: deploy the canonical-main enforcer into Copilot/Codex/Antigravity tool-call paths too (parity with the Claude Code hook), so each runtime self-enforces.
- A **filesystem watcher / read-only-mount** approach on the canonical checkout's tracked paths.
- Detection-and-revert backstop: a cron/watcher that detects + quarantines stranded tracked WIP in main (the manual remediation done twice this session, automated).

## Acceptance (Phase-0 research-first)
- AC-R1: confirm the enforcement-path gap empirically (which runtimes hit `pretool_guard` vs not).
- AC-R2: design a runtime-agnostic guard; cross-family rater AGREED-A+; Consultant rubric ≥7 (min G1-G10).
