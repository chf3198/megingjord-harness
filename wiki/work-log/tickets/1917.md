## Body

Parent Epic: #1912
Discovered by: #1913 and `governance:orchestrator-parity`

## Problem

Claude Code currently has no repo-owned `hooks` block in `.claude/settings.json`, while Claude Code official docs support hooks through settings. The harness therefore cannot prove hook/gate parity against Copilot and Codex.

## Scope

Add a Claude Code hook/settings adapter for the canonical governance gates without editing live runtime homes directly.

## Acceptance criteria

- [x] `.claude/settings.json` or a generated/template source asset maps supported Claude Code hook events to shared governance scripts.
- [x] The adapter covers ticket-first, prompt gates, pre-tool edit/commit guards, post-tool reminders, and stop reminders where Claude Code supports equivalent events.
- [x] Unsupported or intentionally skipped events are documented in the parity manifest with evidence.
- [x] Runtime deployment remains repo-mediated only.
- [x] `npm run governance:orchestrator-parity` no longer reports `claude-hooks-missing` after this ticket lands.
- [x] Tests cover Claude hook/settings detection.
