## Body

Refs #1271

## Scope

Add advisory Manager-narrative linter on `issues.comment` events for `type:epic` issues. Marker-aware regex prevents false positives. #1211-style advisory→required graduation pattern.

## Wave

Wave 2 (continuous detection)

## Problem

Per Epic #1271 gap 1: "Manager narrative ≠ governance state — no automation reconciles them". Reconciler (C3) and sensor (C4) detect drift after-the-fact; linter prevents drift at narrative-authoring time.

## Acceptance criteria

- [ ] `scripts/global/manager-narrative-lint.js` runs on `issues.comment` events for `type:epic` issues
- [ ] Marker-aware regex: `\bEpic\b\W*(\#\d+\W*)?(complete|done|shipped|finished)\b` — avoids false positives on words like "the build is complete"
- [ ] When match found: invokes reconciler (C3); if `truth_status != READY_TO_CLOSE`, posts advisory comment with delta
- [ ] Default mode: ADVISORY (comments only; does not block)
- [ ] #1211 graduation pattern: advisory for 4 weeks; promote to required gate based on operator decision + false-positive metrics
- [ ] Test fixtures: Manager comment with "Epic complete" + drift (advisory comment posted), Manager comment with "build is complete" (no advisory), Manager comment with "Epic complete" + no drift (no advisory)
