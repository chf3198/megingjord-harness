# #3020 — Copilot baton artifacts on closed/in-flight tickets use non-registry-derived signer aliases

> **Source**: github:issue/3020 | **state: OPEN** | **Labels**: type:bug, status:backlog, priority:P2, area:governance
> Mirror of `gh issue view 3020` (derived; edit the GitHub item, not this page).

## Body

## Problem
Copilot-team baton artifacts sign with aliasSeed **Quinn/Quill Harper**, but the registry (`inventory/team-model-signatures.json`) derives `copilot:claude-sonnet-*` to **Soren** (modelPattern `sonnet` / `^claude-[34]` → Soren). Quinn maps to qwen/openclaw; Quill maps to codex gpt-5.4. So Copilot's manager/collaborator artifacts fail `admin-gate` `analyzeComments` with `signer-alias-not-registry-derived`.

## Impact
`analyzeComments` scans ALL issue comments, so ANY ticket carrying a Copilot artifact with a Quinn/Quill alias can never pass admin-gate — it parks indefinitely at status:testing. Observed on #2945 (4 days stuck). The completing operator cannot fix it without forging Copilot's identity (G1/OA3), forcing an admin-bypass per merge.

## Expected
Copilot runtime derives signer aliases via `scripts/global/agent-signature.js` (→ Soren), OR the registry's Quinn/Quill copilot seeds are reconciled with the deterministic derivation so both resolve consistently.

## Acceptance criteria
- AC1: `agent-signature.js --team copilot --model claude-sonnet-4-5 --role collaborator` and the Copilot runtime's emitted alias AGREE.
- AC2: a regression test asserts copilot:sonnet artifacts pass `baton-independence`/`signer-registry-check`.
- AC3: cross-team note added so Copilot artifacts stop using Quinn/Quill for sonnet models.

Refs #2945. Cross-team (target: Copilot runtime signing + registry reconciliation).
