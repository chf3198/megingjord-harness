# #1571 — Collaborator self-review checklist (deterministic, pre-handoff)

## Body

## Parent

Epic #1568 (AC-2). Research: #1569.

## Goal

Add a fixed deterministic checklist to the `role-collaborator-execution` skill that the Collaborator agent runs *before* posting `COLLABORATOR_HANDOFF`. Each item is a pasted exit code or diff inspection — not a judgment call. Catches the class of trip that the lefthook hook (Epic AC-1) can't (e.g., "spec file referenced in MANAGER_HANDOFF actually present in diff").

## Acceptance Criteria

- [ ] AC1 — Update `skills/role-collaborator-execution/SKILL.md` (or `.claude/agents/Implementer.md` depending on runtime) with a fixed checklist section titled "Pre-handoff verification". The list is bullet-points each agent must paste before composing `COLLABORATOR_HANDOFF`.
- [ ] AC2 — Each checklist item is **verifiable via a single command output** — not phrased as a judgment. Examples:
  - `git diff --name-only origin/main | grep -E "tests/.*\\.spec\\.(js|ts)" || echo "NO_SPEC_IN_DIFF"`  (catches missing-spec-file failures)
  - `git rev-parse --abbrev-ref HEAD | grep -E "^(feat|fix|hotfix)/[0-9]+-|^(chore|skill)/[a-z0-9]"`  (catches branch-prefix failures)
  - `grep -niE "test_strategy:\\s*([a-z-]+)" <handoff body>` (catches markdown-bold trip)
  - `grep -niE "\\bflaw\\b|\\bbug\\b|\\bfailure\\b|\\bincident\\b" <handoff body>` (catches unanchored marker words; if matches, must show #N within ±2 lines)
- [ ] AC3 — Checklist explicitly cross-references `MANAGER_HANDOFF`'s `acceptance:` list and `gates:` list — every box maps to one of those, no orphan items.
- [ ] AC4 — `closeout-schema` workflow updated so that `COLLABORATOR_HANDOFF` without a "Pre-handoff verification" section fails the consultant-gate (advisory at first, required after 7-day soak).
- [ ] AC5 — Tests: golden-file fixture for the regex set + integration test that a baton with a missing checklist trips the new validator.

## test_strategy

`golden-file` for the workflow + checklist template; `tdd-pyramid` for any new JS helper (e.g., a single-file `scripts/global/collaborator-self-check.js`).

## Design note (non-binding)

The checklist deliberately mirrors the structure of [[feedback-baton-artifact-format-pitfalls]] — every regex trap that's been observed in the wild becomes a deterministic check.  Goal: zero same-session recurrences of patterns already documented in memory.
