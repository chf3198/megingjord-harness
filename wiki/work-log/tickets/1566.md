## Body

## Problem

The closeout-schema CI check validates baton completeness only after `git push`.
When violations exist, the result is a failed CI run + empty-commit rerun cycle.
In session 2026-05-14 this happened 3 times, adding ~5 min of wasted cycle time.

A local pre-push preflight that runs the same megalint closeout validators
(manager-handoff, consultant-closeout, merge-evidence-pr-gate) against the linked
issue's existing comments would catch violations before CI sees the push.

## Acceptance Criteria
- [ ] `hooks/scripts/closeout-preflight.js` (or extension of existing pre-push hook)
  runs closeout-schema validators against the issue linked in the current branch name.
- [ ] Reports violations to stdout and blocks push on FAIL (non-zero exit).
- [ ] Integrates into the existing pre-push hook chain alongside format and readability checks.
- [ ] Can be skipped with `SKIP_CLOSEOUT_PREFLIGHT=1` for hotfix/admin-override flows.
- [ ] Tests cover preflight pass and fail cases.
