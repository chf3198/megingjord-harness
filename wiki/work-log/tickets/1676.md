## Body

## Context
Part of Epic #1626. Depends on #1675 (health model unification).

74 of 160 wiki pages are missing from `wiki/index.md`. The root cause: `scripts/wiki/wiki-io.js` `updateIndex()` is only called during ingest, not when stub/concept pages are manually added. No sweep command exists.

## Objective
Automate index maintenance so that `wiki:reindex` can repair drift on demand, and so that writing any page via the harness tools automatically updates the index.

## Implementation notes
- Add `npm run wiki:reindex` script that walks `wiki/**/*.md` (excluding `index.md`, `log.md`, `WIKI.md`) and rebuilds the index from frontmatter — idempotent, safe to run anytime
- Wire `wiki-io.js` `writePage()` to call `updateIndex()` after every write (currently only ingest does this)
- Update `package.json` with the new script
- Index sync miss count target: ≤ 5 after running `wiki:reindex` once on current corpus

## Acceptance criteria
- [ ] `npm run wiki:reindex` exists and succeeds
- [ ] Running `npm run wiki:reindex` on current corpus reduces index sync misses from 74 to ≤ 5
- [ ] `wiki-io.js` `writePage()` calls `updateIndex()` after every page write
- [ ] `npm run wiki:lint` reports ≤ 5 index-sync misses after reindex
- [ ] `scripts/wiki/wiki-io.js` remains ≤ 100 lines
