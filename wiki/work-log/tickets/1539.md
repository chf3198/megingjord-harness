## Incident

Discovered during #1279 pre-push hook: `/home/curtisfranks/devenv-ops/node_modules` is a self-referential symlink.

```
$ ls -la /home/curtisfranks/devenv-ops/node_modules
lrwxrwxrwx ... node_modules -> /home/curtisfranks/devenv-ops/node_modules
                                ↑                        ↑
                                same path                same path
```

Effect: `Too many levels of symbolic links` on any node_modules read.

## Root cause analysis

`scripts/worktree-bootstrap-node-modules.sh` (shipped via #1378) creates the canonical link from each worktree's `node_modules` to the main checkout's. When the main checkout's `node_modules` is itself a broken self-symlink (rather than a real directory), every subsequent bootstrap chains through the broken link.

The script has **no sanity check** that `main_root/node_modules` is a real directory before creating links — see follow-up #NEW for the script hardening.
