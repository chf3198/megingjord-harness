## Body

## Problem

In session 2026-05-14 (#1415), baton artifacts were posted without required schema fields:
- MANAGER_HANDOFF: missing `acceptance:` and `gates:`
- CONSULTANT_CLOSEOUT: missing `rubric:` (G1-9 scores) and `verification-timestamp:`

Both required reactive PATCH fixes after CI failures, costing 3 extra CI reruns.
The schema is defined in `instructions/role-baton-routing.instructions.md` but is not
enforced locally before artifacts are posted or commits pushed.

## Acceptance Criteria
- [ ] `scripts/global/baton-schema-preflight.js` validates a baton comment body against
  required fields for each role (manager, collaborator, admin, consultant).
- [ ] Script is callable standalone: `node baton-schema-preflight.js --role manager --body "..."`
- [ ] Pre-commit hook (or pre-push, whichever fires before posting) surfaces violations.
- [ ] megalint `baton-schema` validator (or integration with existing closeout-schema)
  documents the required field set in one canonical place.
- [ ] Tests cover all 4 role schemas with pass/fail cases.
