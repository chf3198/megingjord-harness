# #1938 — Tier-2 anneal: pre-commit branch-intent check (cherry-pick HEAD-drift trap)

## Body

Parent Epic: #1894

## Problem

When git operations (cherry-pick, rebase, merge) move HEAD between branches, subsequent commits land on whichever branch is currently checked out — which may NOT be the branch the operator intended. This trap recurred **4 times in a single session** on 2026-05-19, despite an existing R9.2.1 validator (`cwd-vs-branch`) that caught 3/4 violations at push time:

1. **Incident 1**: After `git rebase`, HEAD switched from `feat/1912-parity-iteration` to `feat/1924-operator-ownership-scripts`. Commit landed on wrong branch. R9.2.1 caught at push; recovery via cherry-pick.
2. **Incident 2**: After `git checkout feat/1912-parity-iteration`, the branch had stale position; my commit landed on wrong target. Recovery via `git reset --hard <sha>` + cherry-pick.
3. **Incident 3**: After successful cherry-pick to correct branch, HEAD remained on wrong branch. Next commit landed on wrong branch again. R9.2.1 caught at push.
4. **Incident 4**: A commit landed on `main` directly (not even on any feature branch). R9.2.1 did NOT catch this because `cwd-vs-branch` allows commits when cwd matches branch (i.e. operator was somehow on main). Required revert-on-main + cherry-pick-to-correct-branch recovery.

Root cause: the R9.2.1 validator catches MIS-MATCHED cwd/branch (operator standing in branch-X-worktree but committing to branch-Y), but doesn't catch the post-git-operation HEAD-drift class where operator's cwd unchanged but HEAD silently moved.

## Goal

Pre-commit hook that asserts the current HEAD matches the operator's INTENDED target branch, where intent is derived from:

1. The active lease's `branch` field (`scripts/global/cross-team-lease.js list --my-team auto`)
2. The most recent `git checkout`/`git switch` operator command (parsed from `~/.local/share/git-history` or similar)
3. Last branch operated on via `gh pr create` / `gh pr edit`

When pre-commit fires and HEAD doesn't match any of these signals, fail with structured remediation: "HEAD is on <branch-A> but your active lease/last-checkout was <branch-B>. Run `git checkout <branch-B>` or release lease if intentional."

## Why programmatic

Existing R9.2.1 validator covers the cwd-mismatch surface. This Tier-2 anneal covers the COMPLEMENTARY surface (cwd matches HEAD but HEAD is wrong-by-intent). Both together close the branch-correctness gate.

Composes with Tier-2 anneal #1891 (lease-worktree-path-consistency) which catches lease-vs-worktree mismatch at lease-creation time. This new validator catches HEAD-vs-intent at commit-time.
