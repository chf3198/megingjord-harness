## Origin

Issue #1917 ("Add Claude Code governance hook adapter") was managed and executed by the Codex Team, writing `.claude/settings.json` on behalf of Claude Code. Two breaking defects shipped to main before detection:

1. **Hook schema regression** — `.claude/settings.json` hooks block used flat entry format instead of the required `{matcher, hooks:[...]}` wrapper. The VS Code Claude Code extension fatal-parsed the file, disabling all permissions and hooks until #1952 landed.
2. **Path divergence** — Hook scripts referenced `~/.claude/hooks/scripts/` while the deploy pipeline writes to `~/.copilot/hooks/scripts/`. Hooks silently exited 2; `UserPromptSubmit` was blocked. Repaired in commit c857ce8.

Coverage gap at merge: `governance:orchestrator-parity` validated presence-of-references only, not schema validity of the generated config, nor that the referenced script paths existed at deploy-time. No pre-merge smoke test invoked the target runtime against the new settings.

## Goal

Add governance to prevent the #1917-class of regression — where one team authors another team's runtime config — and define a named recovery protocol for breaking changes that escape detection at merge.

## Phase-1 children (implementation)

Filed as separate tickets, picked up via standard baton:

- Child A — JSON-schema validation gate for cross-runtime config files (`.claude/settings.json`, `.codex/settings.json`, equivalents).
- Child B — Cross-team artifact-write contract: when Team A authors Team B's runtime config, Team B owns the schema/contract test that must pass before merge.
- Child C — Breaking-change recovery protocol: named workflow (detect, revert, triage, fix, re-merge with smoke evidence) that defines roles and rollback authority.

## 2026 best-practice anchors

Convergent guidance from current external evidence (web research dated 2026-05-19):

- JSON schema validation as gatekeeper between LLM outputs and tools is high-ROI hardening that eliminates an entire class of production failures.
- Verifier-agent pre-merge gates (Augment Code "Intent" pattern) check specialist work against a living spec before PR stage.
- Smoke tests as PRE_MERGE gate should automatically block deployments when they fail.
- Multi-agent systems need versioned agent registries and coordinated CI/CD rollouts with quick rollback for breaking changes.
- VS Code provides built-in JSON schema validation via `json.schemas` settings entries — usable for any extension config including this one.
