# #2964 — Harden worktree bootstrap diagnostics for missing tooling

> **Source**: github:issue/2964 | **state: OPEN** | **Labels**: type:bug, status:backlog, priority:P2, area:scripts, lane:code-change
> Mirror of `gh issue view 2964` (derived; edit the GitHub item, not this page).

## Body

## Problem
In worktree `devenv-ops-2960`, validation failed with missing `eslint` and `@playwright/test` despite both being declared in `package.json`. The local worktree had no `node_modules`, and the bootstrap path did not surface enough diagnostics to make root cause obvious on first pass.

## Objective
Harden worktree bootstrap observability and preflight validation so missing local tooling in a worktree is detected early with deterministic, actionable output.

## Evidence
- Manifest declarations exist:
  - `package.json` includes `"@playwright/test": "^1.52.0"`
  - `package.json` includes `"eslint": "^9.39.4"`
- Worktree evidence at incident time:
  - `node_modules exists: false`
  - `node_modules/.bin/eslint exists: false`
  - `node_modules/@playwright/test/package.json exists: false`
- Main checkout had `node_modules` present.

## Acceptance Criteria
- [ ] Add a deterministic preflight script/check for worktrees that verifies:
  - `node_modules` presence or valid symlink target
  - eslint binary resolution
  - `@playwright/test` package resolution
- [ ] On failure, emit explicit reason classification (`missing-main-node_modules`, `broken-symlink`, `missing-worktree-link`, `missing-package-install`) and one-command remediation.
- [ ] Integrate preflight into the existing worktree bootstrap flow or gate path used before local validation.
- [ ] Add unit/integration tests covering at least:
  - main present + worktree missing link
  - broken symlink
  - valid symlink / healthy path
- [ ] Update operator docs/runbook with the exact troubleshooting sequence.

## Non-goals
- Changing dependency versions for eslint or Playwright.
- Introducing paid-tier tooling.
