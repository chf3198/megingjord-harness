## Goal

Promote `Crypto-Signature` from optional to default-on. Operators without keys get a clear bootstrap path.

## Acceptance Criteria

- [ ] AC1: `scripts/global/signing-bootstrap.js` generates `ed25519` keypair per team and stores public key in `inventory/team-model-signatures.json` (private key in `~/.megingjord/keys/` with mode 0600).
- [ ] AC2: `npm run signing:bootstrap` invokes the helper; idempotent on re-run.
- [ ] AC3: `npm run signing:rotate` rotates the key; publishes old key to revocation list.
- [ ] AC4: `scripts/global/agent-signature.js` extended to sign baton artifacts by default — opt-out via `MEGINGJORD_CRYPTO_SIGNING_DISABLED=1`.
- [ ] AC5: `scripts/global/governance-artifact-signature.js` `verifyArtifact` becomes a HARD CHECK during the 14-day required-mode soak; advisory-warn during the 30-day signed-but-warn soak.
- [ ] AC6: 8+ unit tests covering: bootstrap idempotency; sign + verify roundtrip; rotation; revocation lookup; opt-out env var; missing key handling; signed-but-warn mode; required-mode hard fail.
- [ ] AC7: Documentation update to `instructions/team-model-signing.instructions.md` reflecting the default-on state.
