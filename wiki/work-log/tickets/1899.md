# #1899 — Build harness skill: cross-team adversarial red-team review (research-first)

> **Source**: github:issue/1899 | **state: CLOSED** | **Labels**: type:epic, priority:P2, area:governance, status:cancelled, lane:code-change, resolution:duplicate, anneal:tier-2, phase-gate:research-first

## Body

## Goal

Build a harness skill that any AI Agent Orchestrator team (claude-code, copilot, codex, future teams) can invoke to perform a structured adversarial red-team review of another team's work on a ticket. Target invocation: `Perform red-team work on ticket #N` produces the same caliber of analysis observed across the recent #1399 / #1874 / #1857 review sessions.

## Why

Adversarial red-team reviews have repeatedly surfaced governance gaps and false-completion claims that the original team's self-review missed. The pattern is reproducible — but currently lives only in the operator's prompting style and the reviewing model's context. Codifying it as a skill makes it:

1. **Discoverable**: any team can find it via `Skill` tool listing.
2. **Consistent**: same depth/structure regardless of which team performs the review.
3. **Auditable**: skill output format is structured so downstream validators can parse findings.
4. **Composable**: integrates with Epic #1894 programmatic-governance Epic and Epic #1612 cross-family second-opinion runner.

## Constraints

- **Provider-neutral**: skill must not assume Claude-specific tooling. Use `gh` CLI + repo state as primary sources.
- **Bounded scope**: skill produces analysis, NOT remediation. The invoking Orchestrator decides what to do with the findings.
- **Out-of-baton**: skill output is a comment, not a baton artifact. No status transitions.
- **Composable with #1894 programmatic-governance**: as #1894 children ship validators, the skill should INVOKE those validators where possible rather than re-implementing checks.
- **Velocity-relative**: no calendar thresholds in the skill (per Epic #1771).

## Success metric

- After ship, time-to-produce a red-team review of similar depth to the #1399 v1-v3 cycle drops from ~30 minutes operator-prompted to ~5 minutes skill-invoked.
- 80%+ of red-team findings on a synthetic adversarial corpus match what an operator-prompted reviewer produces.
- Skill is invoked by ≥2 different Orchestrator teams (universality validation) within 30-PR window post-ship.
