## Body

## Problem
`checkDocCoverageAdvisory()` in `scripts/global/megalint/collaborator-handoff.js` returns `severity: 'advisory'` and its results are NOT fed into the `violations` array. This means the doc-coverage check never causes `ok: false` and never blocks merge.

## Objective
Promote the doc-coverage check from advisory to a hard-blocking violation for `lane:code-change` tickets.

## Acceptance Criteria
- [ ] `checkDocCoverageAdvisory()` renamed to `checkDocCoverage()`
- [ ] Return type changed from advisory-only to `{ rule, severity: 'error', detail }`
- [ ] Result pushed into `violations` array (not just `advisory` output)
- [ ] `validate()` returns `ok: false` when doc-coverage block is missing for `lane:code-change`
- [ ] Lane skip preserved: `lane:docs-research` and `lane:config-only` bypass the check
- [ ] Unit tests: missing block → violation; present block → no violation; docs-research lane → no violation
- [ ] File stays ≤100 lines (lint-enforced)
