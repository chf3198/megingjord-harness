## Problem

The cross-team worktree-isolation protocol exists and is documented:

- `instructions/sandbox-worktree-governance.instructions.md` — protocol spec
- `scripts/global/cross-team-lease.js` — lease create/refresh/expire/close
- `scripts/global/cross-team-conflict-gate.js` — path-overlap detection
- `scripts/worktree-session-start.sh` — clean session bootstrap
- `.github/workflows/worktree-governance.yml` (`worktree-governance-required`) — CI audit
- 4th cross-team invariant in `governance/README.md` — dedicated-worktree per `research/concurrent-agent-worktrees-2026-04-24.md`

**But every enforcement is opt-in or post-hoc:**

1. Lease creation is a manual command — if a team skips it, no gate fires.
2. Conflict gate is a manual command — if a team skips it before editing, no path-overlap detection runs.
3. CI audit runs at PR time — by then the collision has already happened and the agents have already done the work.
4. Pre-commit branch-name guard only blocks commits on `sandbox/*` launchers; doesn't prevent file writes into someone else's working tree.

**Lease registry observation (2026-05-17/18):** 6 leases, all team=codex, all status=closed, all expired >24h ago. No active lease for ANY current work. The protocol is essentially unused.

**Worktree audit (2026-05-17/18):** status=fail; all 3 launcher branches grossly stale — sandbox/claude-code 355 commits behind, sandbox/codex 342 behind, sandbox/copilot 177 behind.

## Proposed AC

- AC1: SessionStart hook auto-creates a cross-team lease for any active ticket; refuses to start if a conflicting lease exists.
- AC2: PreToolUse hook (or equivalent write-intercept) verifies the touched path is covered by the current session's active lease; blocks writes when no lease covers the path.
- AC3: flock-style lock file in the checkout root (`.megingjord/active-session.lock`) acquired at session start; any orchestrator attempting to write while another holds the lock receives an explicit collision error with rebase/rescue guidance.
- AC4: `scripts/worktree-session-start.sh` becomes idempotent + auto-runs on every SessionStart hook fire (closes the "manual command bypassed" gap).
- AC5: lease registry has a heartbeat-based liveness check; stale-but-not-closed leases auto-expire after 24h with a notification posted on the linked ticket.
- AC6: replay-eval (#1771 pattern) against today's two incidents — proposed system correctly blocks both.
