# #2440 — Clarify G7 Throughput definition — fleet latency is not a G7 penalty

## Body

## Problem

G7 Throughput is currently defined as: "acceptable speed after higher-priority goals are satisfied."

This is ambiguous — it can be (mis)read as: "if using a fleet model for G3 takes 60 seconds, that's a G7 penalty." That interpretation is wrong. Latency incurred while satisfying a higher-priority goal is not a G7 violation — the priority order already accounts for the trade-off.

## Objective

Clarify the G7 definition to explicitly state:
- G7 concerns are coding workflow speed inefficiencies: slow CI, sluggish tool calls, unnecessary blocking steps
- Latency from satisfying a higher-priority goal (G3 fleet wait, G2 cross-family review) is NOT a G7 concern
- G7 violations are unnecessary slowdowns that could be removed without compromising G1–G6
