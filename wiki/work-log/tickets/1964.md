## Goal

Translate R1's research findings into the canonical Phase-1 implementation plan. Iterate the plan against a self-evaluation rubric scored against G1-G10. Phase-1 may not start until this plan is rated A+ (mean >= 9.0 / 10 with no goal below 8).

## Iteration protocol

Each plan revision posted as a comment on this ticket with sections:

1. Plan vN — enumerated Phase-1 child tickets with title, scope, ACs, dependencies, lane, test_strategy, target enforcement primitive(s).
2. Self-evaluation table — one row per goal G1-G10; column "score" 0-10; column "rationale citing evidence boxes"; column "improvements needed for A+".
3. Decision: continue iteration | stop and finalize.

Self-evaluation MUST use the same evidence-box style as `inventory/rubric-g1-g9-v2.json`. Boxes for each goal:

- G1: every Phase-1 child has owner-role, lane, test_strategy, baton expectation; mid-flight pivot path identified.
- G2: every Phase-1 child has verifiable AC with file/command-level evidence; no AC is opinion-only.
- G3: every Phase-1 child can be implemented in Free/Fleet lane first; Premium use justified per child.
- G4: every Phase-1 child names any secret/credential surface or declares N/A.
- G5: every Phase-1 child works without optional resources (or has documented opt-out parity with MEGINGJORD_HAMR_DISABLED).
- G6: every Phase-1 child names its degraded-mode behavior or declares N/A.
- G7: every Phase-1 child names its perf budget or declares N/A.
- G8: every Phase-1 child names its evidence signal (log path, metric, artifact) or declares N/A.
- G9: every Phase-1 child names cross-runtime parity expectations.
- G10: every Phase-1 child is ≤ the 100-line cap and has its test plan declared.

A+ threshold: mean >= 9.0 AND no goal < 8 AND every box ticked across at least 80% of Phase-1 children.

## Acceptance criteria

- [ ] AC1: At least 2 plan iterations posted as comments before final.
- [ ] AC2: Final plan ticked A+ per the threshold above and signed off by Consultant peer-review rubric `min(G1..G10) >= 7`.
- [ ] AC3: Plan enumerates 6-12 Phase-1 child tickets with the metadata listed in the iteration protocol section.
- [ ] AC4: Final plan written to `research/goal-enforcement-plan-2026-05-19.md` (≤200 lines split if needed).
- [ ] AC5: No code or runtime changes — planning artifact only.
