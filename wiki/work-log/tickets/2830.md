## Body

## Research Objective

Design an accessible, responsive nav and UX fix strategy for the dashboard covering: tab wrapping, icon-only buttons, activity text overflow, and timestamp consistency.

## Scope

- Nav tab overflow at < 900px viewport (currently wraps to 3 rows)
- 5 icon-only action buttons (↺ ⏱ ✏ 💡 ⊙) — no text labels
- Live Activity detail column text clips without ellipsis
- Timestamp inconsistency: Fleet Health uses full locale date; Live Activity uses time only
- Context Flow SVG text unreadable at mobile widths
- Scenario picker click-target issue on mobile

## Constraints

- No build step — vanilla CSS/JS only
- No breaking changes to local/vscode modes
- Must pass WCAG AA contrast after changes

## Deliverable

A design spec comment on this ticket with:
1. Nav overflow solution (horizontal scroll vs. overflow menu with trade-off table)
2. Icon label approach (text label under icon vs. tooltip-only with aria-label)
3. CSS diff sketch for activity overflow and timestamp normalization
4. SVG viewBox strategy for context-flow responsiveness

## AC

- [ ] Nav solution selected with G5-portability rationale (works in all 3 HOST_ENV modes)
- [ ] Icon label approach selected with WCAG AA justification
- [ ] CSS/DOM approach specified for text overflow
- [ ] Timestamp normalization approach specified (shared format util or CSS)
