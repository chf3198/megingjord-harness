# #1305 — Add portable cross-team consultant pickup protocol

> **Source**: github:issue/1305 | **state: CLOSED** | **Labels**: type:story, status:done, priority:P2, area:governance, resolution:completed, lane:code-change, coordinator:cross-team-needs-hand-off

## Body

## Problem

Cross-team Consultant closeout (Epic-level signing by a non-lead team, e.g. Epic #1271 AC6 — SOX/DORA/ISO 27001 baseline) currently requires the operator to copy a multi-line prompt from a GitHub comment into a separate VS Code Copilot Chat or Codex session. This is friction-heavy and unportable.

Beyond ergonomics, an unbounded queue of cross-team-pending Epics has no first-claim-wins discipline — two teams could pick up the same closeout simultaneously and produce duplicate signatures.

## Goal

Replace the operator-paste workflow with a portable, single-flight pickup protocol:

- The receiving team's session needs only a short trigger phrase (`cross-team consult #N` or `find cross-team work`).
- Duplicate effort across teams is prevented by a first-claim-wins protocol with a signer-substrate check at closeout.
- Zero per-team labels, scripts, or instruction overlays — adding a 4th team requires only a `inventory/team-model-signatures.json` entry.

Per goal-lens: G1 Governance (baton integrity preserved) > G5 Portability (no team hard-coding) > G6 Resilience (concurrency-safe) > G3 Zero Cost (reuses existing CI primitives).

## Background

- Current pain documented during Epic #1271 closeout (operator paste of ~3K-char prompt across substrates).
- Cross-Team R&D Protocol v2 §3 (`research/cross-team-rd-protocol-v2-2026-05-09.md`) establishes substrate-derived team identity from `inventory/team-model-signatures.json` — this protocol reuses that pattern.
- Related but distinct: Epic #1112 productizes the R&D *synthesis* phase prompt-dispatch. This Story addresses the Consultant *closeout* phase only.
- Existing primitives reused: `baton-gates.yml`, `label-lint.yml`, `closeout-lint.yml`, `scripts/global/baton-signing.js`, `npm run deploy:apply`.

## Acceptance Criteria

- [ ] **AC1 — Generic labels.** Repo has `consultant:cross-team-needed` and `consultant:cross-team-in-progress`. No per-team suffixes (`*-cx`, `*-cp`, `*-cc`) anywhere.
- [ ] **AC2 — Single canonical instruction.** `instructions/cross-team-consultant.instructions.md` is the only place the protocol is defined. No additions to `.github/copilot-instructions.md`, Codex `AGENTS.md`, or per-team overlays.
- [ ] **AC3 — Single canonical skill.** `cross-team-consult-pickup` skill defined once; auto-deploys to `~/.copilot/skills/`, `~/.codex/skills/`, and Claude Code via `npm run deploy:apply`. Skill `description:` field declares trigger phrases (`cross-team consult #N`, `find cross-team work`).
- [ ] **AC4 — Substrate-aware queue script.** `scripts/global/cross-team-queue.js --my-team auto` resolves the calling substrate from `inventory/team-model-signatures.json` (Cross-Team R&D Protocol v2 §3 pattern). Lists tickets where lead-team Manager substrate ≠ caller substrate AND label `consultant:cross-team-needed` present.
- [ ] **AC5 — First-claim-wins pickup.** Claimant atomically posts `CROSS_TEAM_CLAIM: substrate=<mine>, alias=<mine>, expires=<UTC+24h>` AND swaps label `consultant:cross-team-needed` → `consultant:cross-team-in-progress`. Re-reads comments after 5s; if an earlier-timestamped claim exists, posts `CROSS_TEAM_CLAIM_YIELD` and aborts.
- [ ] **AC6 — Signer-substrate gate.** `baton-gates.yml` extended: `CONSULTANT_EPIC_CLOSEOUT` signer's `Team&Model` substrate must equal the active `CROSS_TEAM_CLAIM` substrate on the same ticket. Mismatch fails the gate.
- [ ] **AC7 — Label mutual exclusion.** `label-lint.yml` extended: `consultant:cross-team-needed` and `consultant:cross-team-in-progress` are mutually exclusive on the same ticket; at most one unresolved `CROSS_TEAM_CLAIM` (without `_YIELD` or `CONSULTANT_EPIC_CLOSEOUT`) per ticket.
- [ ] **AC8 — Stale-claim reaper.** Daily cron checks `expires:` timestamps in active CLAIMs; expired claims revert `:in-progress` → `:needed` and post `CROSS_TEAM_CLAIM_EXPIRED`.
- [ ] **AC9 — Manager-side automation.** When the lead-team Manager posts the closeout-request comment (e.g., the pattern in Epic #1271's latest comment), automation applies `consultant:cross-team-needed`. Implementation may extend an existing post-merge or label-scan workflow; no new operator action required.
- [ ] **AC10 — Live verification.** Protocol exercised end-to-end against a real or synthetic cross-team Consultant pickup. Evidence: candidate listing, claim comment, signer check pass, label state at each transition. If Epic #1271 is still pending closeout at merge time, use it; otherwise create a synthetic test ticket.

## Constraints

- **G5 Portability:** No team-set hard-coding in label vocabulary, scripts, workflows, or instructions. Validated by grep against the merged change.
- **No per-team overlays:** Single canonical `instructions/cross-team-consultant.instructions.md`. No edits to `.github/copilot-instructions.md` or Codex `AGENTS.md` for trigger phrases (skills auto-discover via `description:`).
- **Scripts location:** All new scripts in `scripts/global/`; deployed via `npm run deploy:apply`.
- **Substrate identity:** Resolved at runtime from `inventory/team-model-signatures.json`; never hard-coded.
- **Operator role boundary:** Operator (`chf3198`) is client only — never assigned as required reviewer, approver, or gate-keeper for any baton step.
- **Goal-lens justification:** Any tradeoff lowering G5 Portability or G1 Governance must be explicitly justified in the Manager handoff or PR body.
