# #1466 — Agent Baton: real-time ticket fidelity and workflow sync hardening

> **Source**: github:issue/1466 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P2, area:dashboard
> Mirror of `gh issue view 1466` (derived; edit the GitHub item, not this page).

## Body

## Summary
Fix Agent Baton so it only shows actually active epics/tickets per team and reflects real-time Agile role workflow transitions without stale or missing entries.

## Incident Evidence
Observed failure mode:
- Agent Baton displayed tickets as "in-progress" when they were not active.
- Tickets actively worked by Claude Code and Copilot teams did not appear in Agent Baton as expected.
- Real-time transition visibility across Manager -> Collaborator -> Admin -> Consultant was unreliable.

## Scope
- Dashboard Agent Baton data source correctness (selection/filtering of active items).
- Real-time update pipeline (event ingest, debounce/cache behavior, render/update timing).
- Workflow-state mapping accuracy across status + role labels.
- Cross-team isolation so one team's feed does not suppress another team's active baton chain.
- Regression coverage for stale/phantom in-progress tickets and missing active tickets.

## Acceptance Criteria
1. Agent Baton shows only tickets/epics that are truly active for each team.
2. Active tickets worked by Claude Code and Copilot teams appear within the expected refresh window.
3. Status/role transitions are reflected in order with no stale badge state after transition.
4. Parallel team workflows render independently and correctly in the same time window.
5. Automated test coverage reproduces and guards against this incident class.
6. A post-fix verification artifact documents before/after behavior and evidence.
