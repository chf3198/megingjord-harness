## Scope

Add `EPIC_RESCOPE` artifact template + closeout-lint validation. When an Epic closes with deferred ACs, a structured RESCOPE block is mandatory; closeout-lint blocks Epic close on unmet undeferred ACs OR missing RESCOPE.

## Wave

Wave 1 (governance certainty)

## Problem

Per Epic #1271 §5: "EPIC_RESCOPE rule honored in spirit, not artifact (no template, no gate)". Cross-team plans converged on adding a structured artifact + gate.

## Acceptance criteria

- [ ] Template added at `.github/ISSUE_TEMPLATE/epic_rescope.md` OR instruction snippet in `instructions/epic-governance.instructions.md`
- [ ] YAML schema specified: `deferred_acs`, `deferred_reason_per_ac` (category enum), `re_evaluate_by` (ISO date for time-windowed), `follow_on_tickets`, `signed_by` (Consultant — Manager forbidden)
- [ ] N≥1 RESCOPE blocks per Epic (one per deferred AC; batch-rescope cleanly supported)
- [ ] `closeout-lint.yml` (or new gate) parses RESCOPE markers; rejects Epic close on (unmet undeferred AC) OR (missing RESCOPE when ACs unmet)
- [ ] Bypass-actor provenance binding: if Epic close uses ruleset bypass, RESCOPE must declare which bypass actor + why (per CP cross-team review note)
- [ ] Test fixtures cover: RESCOPE-with-deferred ACs (passes), no-RESCOPE-but-all-ACs-met (passes), no-RESCOPE-and-unmet-ACs (blocks), RESCOPE-signed-by-Manager (blocks)
