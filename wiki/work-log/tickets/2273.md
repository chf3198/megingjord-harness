## Body

Problem
The Epic #1942 guest-collaborator red-team comment for #2054 contained generic risk language with weak diff-grounding. This creates governance noise and can trigger low-signal follow-on work.

Objective
Add a self-annealing guardrail that enforces evidence quality for AI-authored red-team comments before they are posted to Epic/ticket threads.

Acceptance Criteria
- Introduce a validator script for red-team artifacts requiring:
  - explicit file references present in changed diff
  - at least one concrete exploit/failure chain tied to actual changed code paths
  - explicit out-of-scope classification for access-control claims when no auth surface exists in diff
- Add a CI/advisory check for red-team comment quality on governed review flows.
- Document fallback behavior: if fleet model output is generic, route to consultant/manual synthesis instead of direct posting.
- Add tests/fixtures for pass/fail examples.

Validation
- New validator test suite passes.
- Governance docs updated for reviewer workflow.
