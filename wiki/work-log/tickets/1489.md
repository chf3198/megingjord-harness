# #1489 — Test files using bare assert outside Playwright runner — 18 of 120 invisible to npm test

## Body

## Problem

`npm test` resolves to `npx playwright test`. Playwright collects files in `tests/` matching `**/*.@(spec|test).[jt]s` and registers tests via `test('name', () => {…})` calls from `@playwright/test`.

A scan of `tests/*.spec.js` shows **102 of 120** spec files import `@playwright/test`. The remaining **18 files** use bare `require('assert')` and self-execute via top-level IIFEs. These files:

1. Are not registered as tests by Playwright — `npm test` does not run their assertions
2. Run only on direct invocation (`node tests/foo.spec.js`)
3. Sit in the test directory but fail discoverability — anyone running `npm test` thinks they're covered

Most recent example: `tests/cross-team-edit-warn.spec.js` (commit 59cf6a5, not yet on main — see #1486). Pattern surfaced during 2026-05-13 audit of Copilot Team work.

## Goal

Drive the bare-assert count to zero OR establish an explicit lane (e.g., `tests/scripts/` for non-Playwright bare scripts) so the convention is intentional.

## Acceptance Criteria

- [x] AC1: Inventory all 18 bare-assert files with one-line purpose summary
- [x] AC2: Decide per-file: convert to Playwright OR move to a clearly-named directory (`tests/scripts/`?) OR delete if obsolete
- [x] AC3: Add `npm run lint:test-discoverability` check that fails when a `tests/*.spec.js` file does not import `@playwright/test`
- [x] AC4: Document the convention in CONTRIBUTING.md (\"every spec file under tests/ must register with @playwright/test\")
- [x] AC5: Re-run `npm test` and confirm test count increases by N (where N = converted file count)

## Test strategy

`tdd-pyramid` — the lint rule is itself a pure-function check (parse file, check import).

## anneal_tier

## Out of scope

- Adding new tests beyond what's needed for the lint rule
