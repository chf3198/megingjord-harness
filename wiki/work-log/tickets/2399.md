## Body

## Problem

The harness today has 46 operator-local memory files at `/home/curtisfranks/.claude/projects/-home-curtisfranks-devenv-ops/memory/feedback_*.md` containing rules-of-thumb that have been learned through painful experience but are NOT codified in the canonical `instructions/` tree. Examples:

- `feedback_all_baton_artifacts_before_pr.md` — post all 4 baton artifacts BEFORE `gh pr create` (7 consecutive clean ships once applied)
- `feedback_baton_artifact_format_pitfalls.md` — markdown bold breaks test_strategy parsing; flaw-emission validator matches "flaw" inside identifier names; tdd-pyramid needs a spec file in PR diff
- `feedback_team_model_prose_collision.md` — Team&Model regex matches the FIRST `Team&Model:` colon-bearing string in any comment, including prose
- `feedback_calendar_thresholds_in_agentic_systems.md` — never default to calendar-day thresholds in multi-agent governance (Epic #1827 v1→v2 reframe)
- `feedback_sync_sh_reverse_direction_trap.md` — sync.sh copies ~/.copilot/ INTO main checkout (inverse of deploy.sh)
- `feedback_optimal_with_fallback_pattern.md` — (will exist after this Epic) features should use optimal resource when available, fall back to baseline when not

These rules-of-thumb apply to ALL operators across ALL runtimes (Claude Code / Codex / Copilot / Antigravity) but are currently only available to the Claude Code Team operator that has the local memory directory. Other teams hitting the same trap re-learn it from scratch.

## Goal

Audit the 46 operator-local memory files, identify the subset that:
1. Apply to all operators/runtimes (not Claude-Code-specific operator workflow)
2. Reflect canonical-true facts about the harness (not just personal preferences)
3. Would have prevented observable real-world failures if codified earlier

Promote that subset to canonical `instructions/` files (or to appropriate `wiki/wisdom/global/concepts/` pages) with proper baton.

## Acceptance Criteria

- AC1: Triage matrix at `research/operator-memory-promotion-audit-2026-05-29.md` mapping each of the 46 memory files to:
  - {applies-to-all-teams | claude-code-specific | personal-preference}
  - {canonical-true-fact | situational-guidance | recovery-recipe}
  - {would-have-prevented-X-incident | observability-only | meta-discipline}
- AC2: For each rule classified as "applies-to-all-teams + canonical-true-fact", file a child ticket to promote it to an appropriate instructions/* file or wiki/wisdom/global/concepts/ page
- AC3: A new section "Codified rules-of-thumb" in `instructions/global-standards.instructions.md` that lists the promoted rules with cross-references to their canonical locations
- AC4: The operator-local memory files for promoted rules are updated to reference the canonical location (so future memory consultations point to the authoritative version)
- AC5: A new pre-commit hook check: any new `feedback_*.md` operator-memory file flagged at creation time so operators are prompted to consider promotion (advisory, not blocking)
