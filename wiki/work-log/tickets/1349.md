# #1349 — Epic: Upgrade GitHub ticket governance quality, signing fidelity, and AC traceability

## Body

## Summary
Upgrade GitHub ticket practices and governance quality to enforce high-fidelity baton artifacts, role-correct Team&Model signing, AC completion hygiene, and parent-Epic AC traceability to child tickets; include remediation of existing flawed tickets.

## Why this Epic
Current governance drift includes malformed Consultant notes, anemic Collaborator handoff quality, incorrect Manager signing identity usage, unchecked ACs, and missing Epic AC→child linkage. These failures degrade trust in workflow evidence and baton truthfulness.

## Existing backlog/paused/in-progress coverage check
Assessed open governance work:
- #1271 (Epic-state truthfulness) partially covers AC reconciliation at Epic level, but does not fully cover artifact formatting quality standards, signer-fidelity lint for all role artifacts, and broad remediation backlog for existing flawed tickets.
- #1334 (cross-team consult automation) covers a different cross-team claim flow and does not address AC traceability + artifact quality remediation holistically.

Conclusion: no single active ticket comprehensively covers this scope; a dedicated Epic is required.

## Objective
Establish enforceable, auditable governance so that:
1) baton artifacts are structured and complete,
2) signing is role/model-consistent,
3) AC states are truthful and traceable,
4) existing defective tickets are remediated.

## Acceptance Criteria
- [ ] Define canonical quality schema for `MANAGER_HANDOFF`, `COLLABORATOR_HANDOFF`, `ADMIN_HANDOFF`, `CONSULTANT_CLOSEOUT` (required sections, minimum evidence fields, formatting validity checks).
- [ ] Enforce role-correct Team&Model signing policy in ticket artifacts (including Manager signature identity derivation/verification).
- [ ] Enforce AC truthfulness: checked ACs must have evidence references; open ACs must block close/readiness where required by lane.
- [ ] Enforce parent Epic AC traceability: each Epic AC must link to one or more child tickets or explicit deferred/rescope artifact.
- [ ] Add governance audit/reporting for malformed/anemic artifacts with deterministic findings.
- [ ] Remediate existing ticket flaws via controlled backfill runbook and tracked child tickets.
- [ ] Publish operator-facing remediation checklist and rollout plan.

## Delivery Phases
- Phase 0: Research + drift inventory + policy options
- Phase 1: Governance contract + lint/gate design
- Phase 2: Implementation + migration/backfill
- Phase 3: Verification + closeout evidence

## Gate Policy
No implementation child tickets are allowed until Phase 0 research completes and is Consultant-reviewed.
