# #1336 — Reconcile docs/ branch type + early-handoff pattern + close-label transition

## Body

## Problem

Three small frictions surfaced during the 2026-05-10 session that together cost ~15 minutes of recovery time across CC and CP team work:

1. **`docs/` branch type contradiction.** `instructions/github-governance.instructions.md` says: *"Branch naming: `<type>/<issue#>-<slug>`. Allowed types: feat fix chore content perf refactor docs style test."* But the actual `.git/hooks/pre-commit` regex is `^(feat|fix|chore|skill|hotfix)/...` — rejects `docs/`. CC team's #1309 had to be renamed `docs/...` → `feat/...`.

2. **`evidence-completeness` 60s-predate rule isn't documented operator-facing.** The validator requires `COLLABORATOR_HANDOFF` comment to predate PR creation by ≥60 seconds (retroactive-planting check). Caught CC team on #1310 (had to close + recreate PR #1321 → #1322) and CP team twice (#1325 → #1326, #1318 → #1320). The workaround we adopted — post an early "pickup acknowledgement" comment containing the literal string `COLLABORATOR_HANDOFF` — isn't documented in `role-collaborator-execution`.

3. **Pre-close hook blocks if `status:done` not applied first.** CP team's CONSULTANT_CLOSEOUTs on #1313/#1314/#1315 were valid but the close attempts were all blocked by the label-lint pre-close hook ("Close blocked: add `status:done` or `status:cancelled` before closing"). CC Manager had to perform out-of-band label transitions. The `role-consultant-critique` CLOSEOUT protocol should explicitly include the label-transition step.

## Goal

Eliminate all three sources of friction.

## Acceptance Criteria

- [x] **AC1 — Reconcile docs/ branch type.** Either: (a) add `docs|content|perf|refactor|style|test` to the pre-commit hook regex (`hooks/scripts/validate-branch-name.sh`), or (b) remove non-supported types from `instructions/github-governance.instructions.md`. Pick the more useful option — recommend (a) so docs work doesn't have to use `feat/`.
- [x] **AC2 — Document the early-handoff pattern in role-collaborator-execution.** Add a "Pickup acknowledgement (60s-predate compliance)" subsection explaining: post a comment containing literal `COLLABORATOR_HANDOFF` (e.g., "**COLLABORATOR_HANDOFF (pickup acknowledgement)**") immediately upon pickup, BEFORE implementation begins. Full evidence-bearing handoff follows after.
- [x] **AC3 — Auto-transition labels on close.** Update `label-lint.yml` (or a new workflow) to: when `CONSULTANT_CLOSEOUT` comment is detected and `status:review` is the current status, auto-apply `status:done` + `resolution:completed` and remove `status:review,role:consultant` BEFORE close is attempted. Eliminates the "close blocked" friction.
- [x] **AC4 — Tests.** Each change has a regression test (workflow golden-file, or spec test, as appropriate).
