## Acceptance criteria

- **AC-R1** — Inventory every documentation surface the harness currently
  maintains. Produce a structured manifest: path, description, owner, estimated
  freshness. Covers README, CHANGELOG, AGENTS.md, CLAUDE.md, .codex/AGENTS.md,
  .github/copilot-instructions.md, CONTRIBUTING, SECURITY, ARCHITECTURE,
  docs/howto/*, docs/workflow/*, wiki/wisdom/global/*, skills/*/SKILL.md,
  instructions/*, and any surface not yet in doc-coverage-matrix.yml.

- **AC-R2** — Audit `config/doc-coverage-matrix.yml` against the AC-R1 manifest.
  For each `area:*` label: (a) surfaces correctly required, (b) surfaces missing
  from required that should be required, (c) surfaces to promote from suggested
  to required. Produce a gap table.

- **AC-R3** — Design the governed N/A reason enumeration. Define the complete
  approved set with: the reason key, when it is legitimate, what evidence it
  implies, and what abuse it prevents. Cross-reference with how other documentation
  governance frameworks (e.g., Kubernetes, Rust, CNCF projects) handle N/A
  justifications. Produce the proposed `config/doc-coverage-na-reasons.json`
  schema.

- **AC-R4** — Research and spec the diff-based surface-update verification
  approach. Investigate: GitHub Actions diff APIs, git diff filtering, handling
  of generated/compiled files, symlinks, and indirect changes. Identify the
  canonical pattern used in mature CI documentation gates. Produce a design spec
  with pass/fail semantics.

- **AC-R5** — Synthesize all Phase-0 findings into a rescope comment on Epic
  #2707. Define Phase-1 child AC definitions and priority order. This ticket
  closes only after the AC-R5 comment is posted on #2707 AND a red-team rating
  of ≥94/100 is achieved on this research ticket.
