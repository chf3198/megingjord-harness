# #2869 — Harden baton pre-code gate and post-merge issue lifecycle

> **Source**: github:issue/2869 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P1, area:hooks, phase-gate:research-first, resolution:research-delivered

## Body

## Problem

Three related session-continuity failures surfaced during Epic #2827 Demo Mode Quality Sprint audit:

1. **Pre-code gate miss**: MANAGER_HANDOFF was not posted before the first file edit on C5/C6/C9 (#2837/#2838/#2841). The `pretool_guard.py` governance hook did not block the commit, despite the G1 ticket-first rule requiring scope definition before implementation.

2. **Post-merge lifecycle gap**: After PRs #2843–#2857 merged, all 9 linked issues remained `status:triage` (never advanced to `status:done`). No guardrail detected the mismatch between "PR merged on GitHub" and "issue still open with active-phase labels".

3. **Stale worktree re-push**: After PRs were merged, stale worktrees pushed the same branches again, creating 7 duplicate open PRs (#2859–#2866). No pre-push hook compared branch to already-merged PRs.

## Root causes (hypothesized — Phase-0 to verify)

- `pretool_guard.py` checks `flags.code_touched` but may not validate that MANAGER_HANDOFF exists on the linked issue before allowing the first write
- No post-merge lifecycle gate: nothing reads merged PR list + open issues to detect stale baton states between sessions
- Worktree pre-push hook does not compare against already-merged branches on origin

## Acceptance Criteria

### Phase-0 (Research)
- AC-R1: Audit `pretool_guard.py` MANAGER_HANDOFF ordering check — does it exist? Why did it not fire?
- AC-R2: Survey post-merge lifecycle options — GitHub Actions trigger vs hook vs session-start scan
- AC-R3: Survey worktree pre-push stale-branch detection options

### Phase-1 (Implementation — gated on Phase-0)
- AC-I1: If `pretool_guard` lacks MANAGER_HANDOFF ordering check, add it: deny file writes on a ticket branch when no MANAGER_HANDOFF comment exists on the linked issue
- AC-I2: Add a session-startup scan: detect open issues with `status:in-progress` or `status:testing` whose linked PRs are already merged and flag them for baton continuation
- AC-I3: Add pre-push worktree check: warn and optionally block when the current branch already has a merged PR on origin
