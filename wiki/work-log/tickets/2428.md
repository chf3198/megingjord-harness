# #2428 — Add Dangerfile.js file-diff check: changelog fragment required when code files changed

> **Source**: github:issue/2428 | **state: OPEN** | **Labels**: type:task, priority:P2, area:governance, status:triage, lane:code-change, phase-gate:phase-1
> Mirror of `gh issue view 2428` (derived; edit the GitHub item, not this page).

## Body

## Problem
Dangerfile.js enforces ticket-first and branch naming but has no doc-coverage check. A PR can merge scripts/skills/hooks changes without a changelog fragment (`.changes/unreleased/<N>.md`).

## Objective
Add a Dangerfile.js check that fails the build when code files under `scripts/`, `skills/`, or `hooks/` are changed but no changelog fragment for the linked issue number exists in the PR diff.

## Acceptance Criteria
- [ ] New block in `Dangerfile.js` after existing checks
- [ ] Detects if any file in `scripts/**`, `skills/**`, or `hooks/**` appears in `danger.git.modified_files` or `danger.git.created_files`
- [ ] If code files detected: checks that `.changes/unreleased/<N>.md` exists in `danger.git.created_files` (where N is the linked issue number from PR body)
- [ ] Uses `fail()` for hard block; message cites the expected fragment path
- [ ] Skips check if PR title or body contains `[skip-changelog]`
- [ ] Skips check if no code files (docs-only PR)
- [ ] `Dangerfile.js` stays ≤100 lines (may need helper extraction)
- [ ] Unit test or comment-based test documenting expected behavior
