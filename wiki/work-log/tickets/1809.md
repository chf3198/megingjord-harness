# #1809 — Self-anneal: prevent or translate soak language into replay-based eval references

## Body

## Why this self-anneal is needed

A recurring operator-side flaw was observed: governed artifacts (baton handoffs, PR bodies, Epic scopes, closeout rubrics) continue to reference **calendar-bound "soak"** language ("14-day soak", "30-day window", "multi-day measurement", "needs N days to validate") even though **Epic #1771 (closed)** explicitly replaced the soak pattern with **replay-based eval gates** that produce promotion decisions in hours, not weeks.

### Recent recurrence evidence (within last session)

| Instance | Quote | Source |
|---|---|---|
| 1 | "#1793 (cache-hit SLO — **needs multi-day soak**)" | #1797 ship summary, this session |
| 2 | "#1794 (premium 12% governor — **needs 30-day window**)" | #1797 ship summary, this session |
| 3 | "promote to required after a 14-day soak per Path D rollout" | docs/howto/escalation-coverage-gate.md shipped in PR #1806 |

This is a **Tier-2 anneal pattern** per `instructions/workflow-resilience.instructions.md` — recurrent (≥2 instances in 7 days), affects multiple artifacts, contradicts canonical guidance that was shipped specifically to remove the pattern.

## Impact

- Reintroduces calendar latency that Epic #1771 was filed to eliminate (40,000× compression demonstrated).
- Sets bad pattern for next operator/agent reading the artifacts.
- Composition risk: if a downstream gate trusts "30-day window" language and enforces it as a hard wait, the entire G3 cost-minimization Epic #1792 promotion path stalls for a calendar month per child.
- Violates the structural fix from #1684 cancel-trim incident (calendar soaks create cancel-trim incentive).

## Root-cause hypothesis

Training-data prior for SRE-style validation phrasing is much stronger than the new replay-based-eval phrasing. Without an instrumented translator at artifact-emission time, the operator/agent defaults to the older idiom.

## Proposed remediation

Two complementary controls (prefer both; second is fallback if first overfires):

### 1. Pre-merge lint (preferred)

`scripts/global/megalint/soak-language-guard.js`:

- Detects soak-style phrases in baton artifacts, PR bodies, `.changes/` fragments, `docs/howto/*.md`, and `research/*.md`:
  - `N-day soak`, `N day soak`, `\d+-?day window`, `multi-day soak`, `multi-day window`, `calendar soak`, `\d+ days? to (validate|measure|promote|sample|observe)`
- For each hit, suggests the replay-based alternative referencing Epic #1771's `scripts/global/soak-replay-runner.js` and `scripts/global/adversarial-fixture-gen.js`.
- Two modes:
  - `--check`: exit 1 if any hit; CI gate.
  - `--annotate`: pass-through but emit advisory comments on PR.
- Honors `<!-- soak-language-override: <rationale> -->` HTML-comment override on the same line.

### 2. Documented translation rubric

`docs/howto/soak-to-replay-translation.md`:

- Table mapping common soak phrases to their replay-based equivalents.
- Decision tree for when calendar exposure is genuinely necessary (high-novelty operator environments per Epic #1771 §"Out of scope" exception).
- Worked examples taken from this session's mistakes (#1793, #1794, #1797 doc) and their corrected forms.
