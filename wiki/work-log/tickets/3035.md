## Body

## Problem (F-WT2, F-WT3, F-WT4, F-WT5, F-GT2)
No pre-flight one-ticket-per-worktree in the bash entrypoint (only post-hoc JS throw). state-store cwd fallback can load the wrong worktree state. Stale-worktree cleanup races active leases + squash-merge stale-base -> deletes a branch another team extends. node_modules symlink check-then-link race. REST-merge detaches main label -> new worktrees branch off stale base.

## ACs
- AC1: pre-flight CROSS-TEAM one-ticket-per-worktree check in the bash entrypoint (intra-team multi-worktree still allowed).
- AC2: hooks derive cwd from git toplevel (no wrong-worktree state).
- AC3: cleanup respects active leases + a squash stale-base guard before removal.
- AC4: atomic node_modules link; main-label freshness check before new-worktree creation.

## Success metric: a cross-team same-ticket worktree collision is refused pre-flight; intra-team still works.
