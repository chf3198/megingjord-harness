# #1830 — Self-anneal: coordinator label auto-strip on cross-team conflict resolution

## Body

## Why this self-anneal is needed

The label `coordinator:cross-team-needs-hand-off` is auto-applied by `.github/workflows/cross-team-edit-warn.yml:77` (and `cross-team-pr-parallel-check.yml:79`) when a cross-team sibling-role conflict is detected on an Epic's child tickets. **No automation removes it when coordination resolves.** Result: stale labels accumulate; tickets carry `coordinator:cross-team-needs-hand-off` long after the handoff has actually happened.

Operator surfaced this 2026-05-17 on Epic #1792 — the label was applied during early cross-team coordination, but the Codex + Copilot Teams had since handed off the work. Label was orphaned with no remove path. Manager hygiene stripped it manually, but the pattern will recur on any future cross-team Epic.

## Proposed remediation

Add an auto-strip rule to the same workflows (or to `governance:cross-team-check` already-existing) that REMOVES `coordinator:cross-team-needs-hand-off` when ALL of the following are true:

1. The sibling-role conflict that originally triggered the label no longer exists (the conflicting sibling moved past the conflicting role OR closed).
2. No new sibling-role conflict has been detected in the last N hook executions (velocity-aware threshold, not calendar — per `[[feedback-calendar-thresholds-in-agentic-systems]]` / Epic #1827 v2).

Implementation:

- Extend `.github/workflows/cross-team-edit-warn.yml` to ALSO run a remove-path: when checking for conflicts and finding none, AND the label exists, remove it.
- Same for `cross-team-pr-parallel-check.yml`.
- Add a Manager-runnable `npm run governance:coordinator-cleanup` that scans for orphaned coordinator labels across all open tickets and removes them in batch.
