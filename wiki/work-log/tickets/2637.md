## Objective
Phase-1 code child of Epic #2398 — the tier-enforcement machinery. Delivers AC3, AC6, AC10 in one coherent code diff (all the tier-detection/assertion surface; shared test surface).

## Epic ACs delivered
- AC3: per-script tier knowledge + CI lint that validates tags are present and accurate. Implementation (Manager scoping per G10/G2): `scripts/global/megalint/tier-tag-lint.js` treats untagged scripts as the **tier-1 baseline** (matches Phase-0 #2412: 95.1% are tier-0/1) and REQUIRES an explicit `// tier: N` (N in 0..5) tag on any `scripts/global/*.js` that references a tier>=2 resource (workers.dev/HAMR, Tailscale 100.91/100.78, paid-provider key env). Fails closed on missing-tag-for-tier>=2-script or out-of-range tag. Explicit tags applied to the above-baseline scripts. Rationale: blanket tier:1 tags on 341 files is low-value churn; the validator catches exactly the cases where tier is non-baseline.
- AC6: `scripts/global/tier-assert.js` — reads `MEGINGJORD_MINIMUM_TIER` and fails closed when a feature's required tier exceeds the asserted minimum; advisory when unset.
- AC10: extend `tests/orchestrator-compatibility.spec.js` to verify each orchestrator operates at the asserted MEGINGJORD_MINIMUM_TIER (tier-assert honored).
