# #1948 — Bulletproof testing governance for AI-agent-produced code shipping

## Body

## Why

The #1682 ship surfaced an over-confident "stress not required" judgment in a COLLABORATOR_HANDOFF for a script that writes shared mutable state (`logs/wiki-metrics.json`), claims atomic tmp+rename semantics, and shipped with zero concurrent-writer assertion. The [test-methodology-matrix](instructions/test-methodology-matrix.instructions.md) prescribes when stress is required, but the floor is **agent-declared** (`test_strategy` is chosen by the authoring path and consumed by `test-evidence`). An agent that mis-classifies the surface can ship an incomplete suite unless the harness derives a minimum floor objectively from the change itself.

## The precise, unique gap

The harness already ships substantial test infrastructure and this Epic must not duplicate it:
- `test-evidence` CI gate consumes the declared strategy
- `scripts/global/megalint/test-discoverability.js` and the `stress:*` surface suite already codify discovery and evidence checks
- Epic #1875 already established stress as a composable strategy
- Epic #1771 already established replay-eval as the promotion model

The only uncovered delta is an **objective, surface-signal-derived test-floor classifier** that computes the required strategy floor from the diff itself and reconciles it against the declared strategy before merge.

## Scope

| In scope | Out of scope |
|---|---|
| Objective surface-to-required-floor classifier | Replacing `test-evidence` or `test-discoverability` |
| Deterministic declared-vs-derived reconciliation before merge | Creating new strategy enum values |
| Test-drift detection for changed code with stale tests | Historic backfill migration beyond advisory replay |
| Mutation / coverage-delta / replay evidence as sufficiency inputs | Visual-regression governance |
| Cross-runtime parity across all orchestrators | Per-runtime bespoke CI forks |

## Explicit Testing Governance Protocol (Phase-1 Execution Contract)

### Required protocol steps for each implementation child

1. Surface classification step
- Classify each diff into risk tier using objective signals: shared mutable state, concurrency, adversarial input, persistence/irreversible writes, externally visible contract deltas, and perf-SLO-sensitive paths.
- Record classifier result in child evidence as `required_floor_candidate`.

2. Declared-vs-derived reconciliation step
- Compare declared strategy against derived floor candidate.
- If declared strategy is below required floor, child cannot advance beyond collaborator stage until reconciled.

3. Sufficiency evidence step
- Attach all three where applicable:
  - changed-lines coverage delta evidence,
  - mutation sample score evidence,
  - replay regression evidence.
- Any missing required evidence must be explicitly justified as N/A with reason.

4. Drift controls step
- Run drift checks for changed code vs stale tests using multi-signal policy:
  - flaky trend delta,
  - mutation trend delta,
  - replay regression delta,
  - failure clustering delta.
- Blocking/advisory classification is determined by replay-calibrated thresholds only.

5. Cross-runtime parity step
- Confirm identical floor derivation and reconciliation outcome across runtime paths (Copilot, Codex, Claude Code, orchestrator compatibility harness).
- Any parity mismatch is a hard block for advisory->blocking promotion.

6. Audit-schema compatibility step
- Validate emitted decision artifacts against canonical schema contract.
- Missing required fields or incompatible versions hard-block promotion.

### Promotion gate policy (advisory -> blocking)

Promotion is permitted only when all are true:
- replay-eval thresholds met on labelled corpus,
- cross-runtime parity proof present,
- audit-schema contract proof present,
- collaborator/admin/consultant fleet-backed review evidence present,
- measured false-positive rate remains below the defined guardrail.

### Mandatory evidence bundle fields

Every implementation child must produce a single promotion-evidence bundle with:
- `required_floor_candidate`
- `declared_strategy`
- `reconciliation_outcome`
- `coverage_delta`
- `mutation_sample_score`
- `replay_regression_result`
- `drift_signals`
- `cross_runtime_parity_result`
- `audit_schema_contract_result`
- `role_review_scores` (collaborator/admin/consultant)
