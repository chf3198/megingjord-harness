## Body

## Background

The #1641 MCP migration audit ranked \`issue-transition.js\` as the **highest value** migration target (priority #1 of 8). It runs on every baton transition — dozens of calls per session. Migrating to MCP provides schema validation, preventing label typos like \`lable\` vs \`label\`, and brings it into alignment with the MCP-first governance contract.

## Current behavior

\`scripts/global/issue-transition.js\` uses \`gh issue edit --add-label\`, \`gh issue edit --remove-label\`, and \`gh issue close\` via shell-exec calls.

## Target behavior

Replace with MCP equivalents:
- \`gh issue edit --add-label / --remove-label\` → \`mcp__github__update_issue\`
- \`gh issue close\` → \`mcp__github__update_issue\` (state field)

Keep \`gh\` CLI as opt-out fallback path when \`MEGINGJORD_MCP_DISABLED=1\` is set (G5 Portability per #1629 contract).
