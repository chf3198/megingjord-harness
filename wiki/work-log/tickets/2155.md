## Body

## Scope (Phase-1 child of Epic #2148)

Author a small YAML doc-coverage matrix at `config/doc-coverage-matrix.yml` mapping `area:*` labels to the set of doc surfaces that must be updated when a ticket carrying that area-label ships. The Tech-Writer sub-phase validator (C1) consumes this matrix.

Refs Epic #2148
Refs #2149 (Phase-0 source)

## Why agent-speed-appropriate

Round-5 of Phase-0 research kept this child as a load-bearing minute-scale win. ~30 lines of YAML; trivial to author; cheap to consume; eliminates per-ticket guesswork on which docs apply.

## Schema

```yaml
version: 1
surfaces:
  area:hooks:
    required: [docs/howto/hooks.md]
    suggested: [wiki/code/hooks.md, README.md]
  area:scripts:
    required: [README.md]
    suggested: [wiki/code/scripts.md]
  area:instructions:
    required: [docs/workflow/learnings.md]
    suggested: [wiki/wisdom/project/]
  area:governance:
    required: [CHANGELOG fragment, docs/workflow/learnings.md]
    suggested: [wiki/wisdom/global/concepts/]
  # ... one entry per area-label
```

## Acceptance criteria

- AC1: `config/doc-coverage-matrix.yml` exists with one entry per active area-label
- AC2: Golden-file test in `tests/doc-coverage-matrix.spec.js` validates schema (required fields, valid surface paths)
- AC3: YAML is human-readable and fleet-model-parseable (no nested anchors, no merge keys)
- AC4: `scripts/global/megalint/doc-coverage.js` reads the matrix and emits a coverage block in advisory mode
- AC5: Lint clean
