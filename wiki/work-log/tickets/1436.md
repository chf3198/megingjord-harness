# #1436 — Epic: Worker mid-flight self-anneal recognition + emission

## Body

## Problem
The harness's self-anneal protocol has three tiers (per `instructions/workflow-resilience.instructions.md`):
- **Tier 1**: passive observation, append to `~/.megingjord/incidents.jsonl` (no ticket)
- **Tier 2**: nightly cron detects recurrence ≥2/7d → auto-files Manager ticket
- **Tier 3**: Consultant rubric below 7 → escalation + remediation ticket (gap addressed by #1376)

But there is **no mechanism for a worker (any role) to recognize during work that a Tier-2 recurrence pattern is happening RIGHT NOW** and emit proactively. The current options are:
1. Wait for the nightly cron (`anneal-tier2-autofile.yml`) to detect via incidents.jsonl — but the cron runs once per day at 03:37 UTC, so any patterns that recur within a single session are not detected until the next cron window.
2. Hope the agent self-files retroactively (relies on agent discipline; demonstrably fails).

## Observed evidence
Epic #1407 (closed 2026-05-12) had TWO recurring patterns that hit Tier-2 criteria mid-session but received NO ticket until the user explicitly asked "Were any self-anneal tickets created?" — filed retroactively as #1433, #1434. The agent (me, Claude Code Manager) self-rated G1-G9 above threshold (9.4-9.6) and assumed no anneal escalation needed. That assumption skipped Tier-2 (recurrence) which is independent of Tier-3 (goal-failure).

Filed `feedback_anneal_emission_during_implementation.md` to fix the personal-discipline gap, but the systemic gap remains: **the harness gives the worker no in-session signal that a pattern has crossed the Tier-2 recurrence threshold.**

- **#1114** (Phase-0 R&D for #1113) — Goal Health Score formula, actuator escalation matrix; not specific to mid-flight worker awareness.

**Conclusion**: no existing ticket addresses mid-flight worker awareness of Tier-2 recurrence. New Epic warranted.

## Goal
Design and implement a worker-awareness mechanism so that during a session, when a pattern recurs ≥2 in 7d at severity ≥ medium, the harness:
1. Detects the recurrence in real-time (not next-cron)
2. Signals the active worker (skill prompt, hook output, dashboard banner, or similar)
3. Either auto-emits the Tier-2 event/ticket OR prompts the worker to confirm emission
