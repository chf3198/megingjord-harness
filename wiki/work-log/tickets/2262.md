## Phase-0 synthesis

### 1. The problem being solved

Memory-reinforcement governance (operator memory notes + prompt instructions + advisory hooks) is **soft**: the rules live INSIDE the agent's readable context, so they can be forgotten, ignored, compacted away, prompt-injected around, or simply skipped. Recurring traps accumulate `feedback-*` notes (28+ in current memory index) without being structurally eliminated. The 2026 industry term for the corrective discipline is **"harness engineering"** — designing the environment AROUND the model so misbehavior is physically impossible.

```
SOFT (agent CAN forget/ignore/jailbreak):
  System prompt instruction
  feedback-* memory notes
  CLAUDE.md / instructions/
  "governance anchor" reminders
              │
              ▼  "You cannot govern X with X"
              │
HARD (agent CANNOT, period):
  PreToolUse hooks (exit-2 deny)
  Policy-as-code (OPA / Cedar) in the tool path
  Constrained decoding (token mask)
  Capability sandbox (gVisor / Firecracker / WASM)
  Pre-action authorization (APort passports)
```

### 2. The 5-tier guardrail strength spectrum

| Tier | Mechanism | Bypassable by… | Current Megingjord |
|---|---|---|---|
| **T0** | Capability sandbox (microVM, gVisor, WASM) | kernel exploit only | ❌ none |
| **T1** | Constrained decoding (XGrammar, JSON-schema strict, structured output) | swapping inference engine | ❌ none |
| **T2** | Pre-tool hook + policy engine (OPA, Cedar, PreToolUse) | disabling/editing the harness itself | ⚠️ partial (hooks exist but advisory on many surfaces) |
| **T3** | External judge / classifier (NeMo, Llama Guard, Bedrock Guardrails, Constitutional Classifiers) | classifier false-negative | ⚠️ partial (megalint validators) |
| **T4** | Prompt instruction + memory feedback | context compaction, jailbreak, forget, model-version swap, literally just ignoring | ✅ HEAVY use |

Megingjord is heavy at T4 and T3, partial at T2, and absent at T1/T0 — which exactly matches the symptom: rules get forgotten because the only place they live is in the agent's head.

### 3. The five hard-guardrail families (2026 state-of-the-art)

#### Family A — Policy-as-code in the tool path (T2)

**Open Policy Agent (OPA + Rego)** and **AWS Cedar** are the de-facto pattern for agent governance in 2026. The agent makes a request; a deterministic policy engine evaluates it; the engine — not the agent — decides allow/deny. Even a fully jailbroken agent cannot bypass it because the model isn't the thing executing the policy. Industry calls this "decoupled authorization."

```
   Agent ──"call tool X with args Y"──▶ Harness
                                          │
                                          ▼
                                   ┌──────────────┐
                                   │   OPA/Cedar  │ ◀── policy.rego
                                   │              │     (versioned, auditable, signed)
                                   │   allow?     │
                                   └──────┬───────┘
                                          │ deny → block + emit signed decision
                                          │ allow → forward to tool runtime
                                          ▼
                                       Tool exec
```

OPA scope: infrastructure-level (service-to-service, K8s admission). Cedar scope: application-level authorization. For Megingjord's tool-call hooks, OPA fits better (already pattern-matches `pretool_guard.py`).

#### Family B — Pre-action authorization (T2, agent-native)

**APort / Open Agent Passport** (open spec, Apache 2.0) formalizes T2 for agent frameworks: every agent carries a signed JSON passport declaring allowed capabilities; every tool call goes through a `before_tool_call` hook (~40ms overhead); every decision produces a signed audit record. Companion to OPA — same principle, agent-shaped API.

#### Family C — Constrained decoding (T1)

**XGrammar** is the default structured-generation backend in vLLM, SGLang, and TensorRT-LLM as of March 2026 — sub-40-microsecond per-token overhead, up to 80× throughput improvement over older approaches. **OpenAI/Anthropic `strict: true` structured-output mode** uses the same approach: a finite-state automaton over the JSON Schema masks the vocabulary so invalid tokens are physically un-emittable.

```
Without:  model generates → parse → if invalid: retry / fix-up / hope
With:     vocabulary mask → only schema-valid tokens are ever produced
          ┌────────────────────────────────────────────────┐
          │ valid grammar state → allowed_tokens ⊂ vocab    │
          │ sample only from allowed_tokens                 │
          │ advance grammar state                           │
          └────────────────────────────────────────────────┘
```

**Limitation**: semantic constraints ("output must be factually accurate") cannot be enforced by token masking — but structural constraints ("a CONSULTANT_CLOSEOUT artifact must contain a Role: consultant line and must not contain the substring MANAGER_HANDOFF outside an escaped block") absolutely can.

#### Family D — Sandbox isolation (T0)

2026 isolation hierarchy:

| Tech | Startup | Isolation | Use when |
|---|---|---|---|
| **WASM isolate** | ms | strong (capability-typed) | trusted code, hot path |
| **gVisor** (user-space kernel) | hundreds of ms | syscall interception | semi-trusted |
| **Firecracker microVM** | 150 ms | dedicated kernel | untrusted code (DEFAULT in 2026 per Northflank) |
| **Container** | 500 ms+ | shared kernel | trusted code only |

For Megingjord: fleet codegen execution and adversarial-red-team output execution should run in microVM. Daily Bash hooks/scripts that operate on tracked governance state can stay in-process.

#### Family E — External classifier / judge (T3)

2026 shipped products: **AWS Bedrock Guardrails** (blocks ~88 % of harmful content, 99 % hallucination detection per AWS metrics), **Azure AI Content Safety**, **NVIDIA NeMo Guardrails** (Colang dialogue policies), **Guardrails AI** (output structure), **Llama Guard 3/4**, **Anthropic Constitutional Classifiers**. Pair 2–3 of these — common shape:

```
   request ─▶ NeMo (dialogue policy) ─▶ model ─▶ Guardrails AI (structure)
                                                  └─▶ Llama Guard (safety)
                                                  └─▶ Bedrock GR (content)
```

### 4. Megingjord drift map — current failures and the right tier for each

Each row is a recurring trap from operator memory or recent sessions:

| Symptom | Today | Right tier | Concrete fix |
|---|---|---|---|
| `roles.admin` state-store drift after PR merge → Stop blocks | manual patch; memory note | T2 | OPA post-merge transition policy. Agent doesn't write the flag; the policy does, atomically on merge event. |
| Cross-artifact prose collision (CONSULTANT_CLOSEOUT substring inside other artifacts) | memory note `feedback-team-model-prose-collision` | T1 | Constrained-decoding grammar for handoff artifacts. Peer-artifact name tokens physically un-emittable outside header position. |
| Silent stop after Bash dispatch (Pattern B, recurred ≥3 sessions) | memory note `feedback-bash-sleep-block-recovery` | T2 | PreToolUse/PostToolUse policy that refuses turn-end while a tracked background task is outstanding without an emitted ack token. |
| Stub cross-team-lease (missing `--ticket` arg) | manual purge; memory note | T2 | OPA pre-check on `cross-team-lease.js create` — reject if required fields missing, before file write. |
| Python `json.dump` escapes em-dashes → prettier CI fail | manual prettier rerun; memory note | T2 | Pre-commit hook running prettier on staged JSON. (already common, just standardize.) |
| Local main 39 commits behind origin during merge-evidence check | reflex `git fetch && git log origin/main` | T2 | OPA check `merge-evidence` rule references `origin/main`, not `HEAD`. |
| Validator regex prose collision (markdown bold breaks `test_strategy` parsing; `failure` substring matches flaw-emission) | memory notes | T1 | Constrained grammar with reserved keywords outside designated fields. |

**Every one of these is a T2 problem solved with prompt-instruction / memory and would be eliminated by a tool-path hook with policy-as-code.**

### 5. Recommended additive evolution (no rewrite)

```
Today                            +6 months                       +12 months
─────                            ─────────                       ──────────
T4 instructions ───────────────▶ T4 instructions ──────────────▶ T4 instructions
T3 megalint validators ────────▶ T3 + Llama Guard for fleet ───▶ T3 + Constitutional
T2 some pretool_guard hooks ───▶ T2 OPA + Cedar in tool path ──▶ T2 OPA + APort passport
                                 T1 XGrammar on baton artifacts  T1 strict everywhere
                                                                 T0 fleet exec in microVM
```

**Recommended Phase-1 sequencing** (highest-leverage first):

1. **OPA at the tool-call boundary** for: `cross-team-lease` create/refresh/close, baton-state transitions, label flips, merge-evidence resolution, state-store role flips. Single change retires ~70% of recurring memory notes.
2. **XGrammar grammar for baton artifacts** (MANAGER_HANDOFF / COLLABORATOR_HANDOFF / ADMIN_HANDOFF / CONSULTANT_CLOSEOUT). Eliminates prose-collision trap class entirely.
3. **Bypass-block default-on for governed sessions** (extend #2029 `MEGINGJORD_FLEET_DIRECT_BLOCK` semantics to all paid providers; ship default-on in CI).
4. **microVM (Firecracker) for fleet-codegen execution** (T0 — when untrusted code is run, run it in a disposable VM).
5. **Llama Guard inline judge for cross-family review outputs** (T3 — defense-in-depth on the read path).

### 6. Headline insight

> **"A deterministic policy in the tool hook does not get bypassed by prompt injection because the model is not the thing running it."** — the throughline across every 2026 reference reviewed.

The recommendation is unambiguous: **stop adding `feedback-*` notes for recurring trap classes and instead invest the same effort in moving each trap class from T4 (prompt) down to T2 (hook) or T1 (grammar).** Industry calls this "Tier-0 governance" / "deterministic guardrailing." Open-source tooling (OPA, XGrammar, NeMo, APort, gVisor, Firecracker) is mature in 2026 — adoption cost is implementation time, not research risk.
