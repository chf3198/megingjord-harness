# #2825 — deferred-final-terminal-close auto-closes multi-slice tickets on each increment closeout

> **Source**: github:issue/2825 | **state: OPEN** | **Labels**: type:bug, status:backlog, priority:P2, area:scripts
> Mirror of `gh issue view 2825` (derived; edit the GitHub item, not this page).

## Body

## Problem

`scripts/global/deferred-final-terminal-close.js#reconcile` (run by the `deferred-final-terminal-close` job in `.github/workflows/post-merge-automation.yml`, triggered on any `issue_comment` containing `CONSULTANT_CLOSEOUT`) closes the linked issue when **any** merged PR carries `merge-evidence-deferred-final: #<issue>` in its body.

For a ticket built across **multiple deferred-final PRs** (the P1-0 slice pattern on #2802: PRs #2815, #2817, #2820, #2823), this means **every slice's CONSULTANT_CLOSEOUT auto-closes the parent ticket** — because the search `is:pr is:merged in:body "merge-evidence-deferred-final: #N"` matches a *prior* slice's merged PR, not the finalizing one. The "slice N increment — #N stays OPEN" language in the closeout is ignored.

Slices 1+2 survived only by an ordering artifact: at their closeout-time no prior deferred-final PR was merged yet (`action: no-merged-deferred-final`). Slice 3 (#2823) was auto-closed mid-flight and had to be reopened.

## Recurrence

Confirmed recurring: structural, fires on every increment after the first merged deferred-final PR. Affects any multi-PR-per-ticket workflow (incremental builds, slice patterns).

## Candidate fixes (Manager to scope)

1. **Distinct increment marker**: non-final slices use a token that satisfies `merge-evidence-pr-gate` but is NOT matched by `deferred-final-terminal-close` (e.g. `merge-evidence-slice-increment: #N`); only the final slice uses `merge-evidence-deferred-final: #N`.
2. **Finalizing-PR correlation**: `reconcile` should only close when the *triggering* closeout correlates to the PR that finalizes the ticket (e.g. closeout cites the finalizing PR#, or the closeout omits a `keep-open`/`increment` marker), not any historical deferred-final PR.
3. **Explicit increment opt-out**: honor a `keep-open: <reason>` / "stays OPEN" marker in the closeout body as a skip signal in `reconcile`.

Recommend option 1 or 3 (deterministic, no search-correlation fragility). Prefer the harness-blessed alternative long-term: build large tickets via **Sub-issues children** (each a normal single-PR ticket that `Closes #child`), keeping the parent open until all children close — which sidesteps deferred-final-terminal-close entirely.

## Goal-lens

G1 (governance correctness) — the auto-closer violates the deferred-final contract's own intent ("Consultant retains explicit terminal-finalize authority") by closing prematurely on a non-final increment.

Surfaced building Epic #2791 P1-0 (#2802) slice 3 (#2823). Flaw decision: file-ticket (structural/repeatable).
