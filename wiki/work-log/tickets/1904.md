## Body

### Objective
`hamr-activate.sh` step 3 checks for provider keys (`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, etc.) but never sources `.env`. Provider keys are present in repo-root `.env` but are unavailable to the activation script, causing false "missing env" warnings and leaving cloud/escalation routing paths unavailable.

### Expected Outcome
`npm run hamr:activate` completes with zero "missing env" warnings when `.env` contains the required keys. Cloud escalation paths (Haiku/Premium) become available. Secrets are never logged or echoed.

- [ ] AC3: No secret value is ever printed, logged, or echoed — sourcing is silent.

- [ ] AC5: `npm run lint` passes (file stays ≤100 lines).

- Change is minimal and localized to `scripts/global/hamr-activate.sh` only.
