## Body

## Problem
Deployed runtime artifacts (`~/.copilot/`, `~/.codex/`, `~/.agents/skills/`) have no signed attestation (Gap G-09). The deploy pipeline runs `npm run deploy:apply` which copies files without generating a SHA-256 manifest or SLSA provenance record. A compromised hook script could be deployed undetected.

This is ASI07 (Supply Chain) + ASI10 (Code Execution, because a deployed hook can execute arbitrary shell commands) + EU Art.15 (Cybersecurity).

## Acceptance Criteria
- [ ] Create `scripts/global/deploy-manifest.js`: generates SHA-256 manifest of all files in a deploy target directory
- [ ] Create `scripts/global/verify-deploy.js`: compares manifest against expected hashes, exits 1 on mismatch
- [ ] Integrate manifest generation into `npm run deploy:apply` (post-deploy step)
- [ ] Add `verify-deploy` as a pre-deploy check in `npm run deploy:apply`
- [ ] Store manifests in `inventory/deploy-manifests/` (gitignored runtime state)
- [ ] Add tests: `tests/deploy-integrity.spec.js`
