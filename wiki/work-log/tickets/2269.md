## Body

Problem
Tavily credentials are present in local .env, but managed Codex runtime config does not register a Tavily MCP server, so the harness cannot consume Tavily through the runtime MCP surface.

Expected outcome
Codex managed runtime config includes Tavily MCP wiring using environment-based auth (no secret material in repo files), plus a guardrail check that flags config/key drift.

Acceptance criteria
- Add Tavily MCP server table to managed `.codex/runtime.config.toml` with env-based auth.
- Add activation-time drift warning when Tavily key exists but Tavily MCP server is not configured.
- Validate deploy/sync paths remain functional.
