# #2974 — Fix harness bootstrap and fleet/cloud access degradation blocking zero-cost routing

> **Source**: github:issue/2974 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P1, area:infra, lane:code-change
> Mirror of `gh issue view 2974` (derived; edit the GitHub item, not this page).

## Body

## Problem
Harness-level bootstrap/access is partially degraded in local sessions and causes avoidable paid-token fallback risk:
- Local env hydration depends on fragile default-path assumptions.
- Fleet/OpenClaw gateway routing can point to non-reachable loopback values.
- Hook troubleshooting UX is poor when scripts are run directly without piped JSON.

This is harness infrastructure scope, not app-repo scope.

## Impact
- Increases risk of non-free model escalation despite available fleet/cloud credentials.
- Creates operational friction for Copilot/Claude runtime sessions.
- Can block or delay governance-compliant workflows.

## Scope
- Megingjord Harness workspace only.
- Local ignored config and harness runtime scripts only.
- No product-app repository changes.

## Acceptance Criteria
1. Default local env hydration resolves from canonical local runtime secrets source without manual per-command overrides and without logging secret values.
2. `node scripts/capability-probe.js --json` reports:
   - tailscale available
   - cloudflare account available + authenticated
   - r2 available
   - wrangler available
   - providers (anthropic/openai/groq/cerebras/google_ai_studio/openrouter) available when keys exist
3. OpenClaw preflight succeeds when remote service is healthy and configured endpoint is reachable over Tailscale.
4. Hook troubleshooting path is non-blocking for direct terminal invocation (either documented wrapper usage or script guard behavior), while normal hook-runner behavior remains unchanged.
5. All local-path/credential edits remain outside tracked repo files (ignored/local runtime files only).

## Verification Commands
- `node scripts/load-local-env.js`
- `node scripts/hamr-doctor.js`
- `node scripts/capability-probe.js --json`
- `node scripts/openclaw-preflight.js`
- `printf '{}' | python3 ~/.copilot/hooks/scripts/pretool_guard.py`
- `printf '{}' | python3 ~/.copilot/hooks/scripts/stop_reminder.py`
