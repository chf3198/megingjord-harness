## Body

Follow-on from red-team analysis on Epic #1962 (comment 4509891153) — Attack #3 (Adversarial Fixture Injection).

## Origin

Red-team proposed: "Inject malicious adversarial fixtures that mimic legitimate operations, exploiting gaps in the 10 fixture loader." Currently, `scripts/global/goal-hijack-fixture-loader.js` reads JSON from `tests/fixtures/goal-hijack/` with no integrity check.

## Scope

Add a manifest + SHA-256 integrity check so the fixture loader rejects modified/injected fixtures at runtime.

## Acceptance criteria

- [ ] AC1: `tests/fixtures/goal-hijack/MANIFEST.sha256` lists each fixture filename + SHA-256 of its canonical content
- [ ] AC2: loader verifies hash on load; rejects with descriptive error on mismatch
- [ ] AC3: tests cover positive (canonical content passes) + negative (modified content fails)
- [ ] AC4: optional signing path via inventory/team-model-signatures for cross-team-authored fixtures
- [ ] AC5: G4 box — no credential surface
