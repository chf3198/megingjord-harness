# #2805 — Research dual-mode (local + GitHub Pages demo) dashboard patterns

## Body

## Objective

Research dual-mode dashboard deployment: how can the same codebase serve both
as a **local web-app** (connected to live harness state) and as a **GitHub-hosted
static demo** (GitHub Pages) for education and feature demonstration?

Refs Epic #2803

## Research Questions

1. **Dual-mode architecture patterns** — how do real OSS projects ship a local
   dev-tool web-app AND a public GitHub Pages demo from the same codebase?
2. **GitHub Pages demo-mode conventions** — what patterns do popular developer
   tool repos (like Langfuse, OpenHands, PostHog, Netdata, etc.) use to publish
   a hosted demo/educational version of their local-first tool?
3. **Runtime environment detection** — how do web-apps detect whether they are
   running locally (connected to live backend) vs. hosted (demo/mock-data mode)
   and switch behavior accordingly?
4. **Mock/fixture data strategy** — how do demo-mode dashboards simulate live
   data (websockets, fleet activity, governance events) so that GitHub Pages
   visitors get a realistic experience without a real backend?
5. **Educational overlay patterns** — what UX conventions (e.g. tours, annotated
   panels, explanatory sidebars) do developer tool docs/demo pages use to teach
   concepts like MCP, agents, and governance to newcomers?

## Acceptance Criteria

- [ ] Identify ≥3 real OSS repos that ship dual-mode local+hosted dashboards
- [ ] Document the branching/environment-detection pattern each uses
- [ ] Identify best mock/fixture data strategy for the harness fleet simulation
- [ ] Identify the educational UX pattern most appropriate for harness onboarding
- [ ] Post findings as a structured comment
