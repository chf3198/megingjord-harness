## Body

## Problem

During #2249 closeout, Consultant checks surfaced baton artifact issues. The initial response used additive audit comments, but when the original issue comments are editable and the defect is fully remediable, additive comments can leave future governance checks exposed to the same malformed artifact.

## Scope

Add a guardrail so fixable baton artifact defects require source artifact remediation instead of additive audit comments as the primary fix.

## Acceptance Criteria

- [x] Consultant/governance feedback distinguishes source-fixable baton artifact defects from non-editable historical audit defects.
- [x] For source-fixable signer-role or missing-field defects, remediation guidance requires updating the offending comment/artifact in place before adding audit commentary.
- [x] Additive audit comments are allowed only when the source artifact cannot be edited or rewriting would violate audit/history policy.
- [x] Tests cover malformed Collaborator/Admin artifacts where early lowercase `role:` fields cause signer-role mismatches and verify source remediation guidance.
- [x] Documentation or inline help names the expected behavior: fix the source artifact first, then rerun the check.
