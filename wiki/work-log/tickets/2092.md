### D3 — Why the alert filter is unnecessary (operator question answer)

Direct answer to "is there a true need to remove the alert filter" with:
- Root-cause analysis showing the filter only existed because state was conflated
- Demonstration that per-session state file isolation eliminates the conflated-data condition
- Demonstration that canonical-main read-only enforcement eliminates the cross-team-mutates-shared-path condition
- Demonstration that per-worktree hooks eliminate the cwd-confusion condition
- Conclusion: when all three root-cause fixes are in place, NO false alarm conditions exist for the filter to suppress; the filter is redundant.
