## Body

## Problem

Red-team adversarial review of Epic #2707 Phase-1 (PR #2726) identified six confirmed bugs and two quality gaps that prevent the implementation from meeting the harness >93/100 threshold on all goals.

Six issues are confirmed from source code inspection:

1. **G6 CRITICAL — fail-open N/A enum**: `loadNaReasons()` in `doc-coverage-helpers.js` swallows all errors and returns `null`. `valueViolation()` only enforces the N/A enum when `loadNaReasons` succeeds. File deletion or corruption silently disables AC3 (the entire N/A governance check). Current score: 30/100.

2. **G9 CRITICAL — advisory bypass not removed from caller**: `collaborator-handoff.js` line 74 still guards the doc-coverage call with `process.env.DOC_COVERAGE_GATE_ADVISORY !== '1'`. Phase-1 removed the env check from `doc-coverage.js` but not from the caller. The hard gate can still be bypassed. Current score: 65/100.

3. **G4 MEDIUM — shell injection in diff-verify**: `getChangedFiles()` uses `execSync` with a template-string `"${base}...HEAD"`. Unvalidated `base` input is a shell injection vector (OWASP A03). Fix: `spawnSync` with args array. Current score: 65/100.

4. **G2 MEDIUM — diff-verify violations are non-blocking**: All `doc-diff-not-changed` violations use `severity: 'warning'` which the `ok` calculation ignores. A developer can declare `DONE: README.md` without touching README.md and the gate passes. Current score: 75/100.

5. **G2 MEDIUM — covered-by-sibling-pr format not enforced**: The N/A reason config (`doc-coverage-na-reasons.json`) specifies `validator_check: regex: covered-by-sibling-pr:#\\d+` but the validator never enforces this. Freeform `covered-by-sibling-pr: arbitrary text` passes. Current score: 75/100 (same goal).

6. **G7 MEDIUM — no local validate CLI**: Every N/A reason iteration requires an 8-12 min CI round-trip. No `--check` entrypoint exists. Current score: 60/100.

## Root Cause

Phase-1 was delivered as a batch PR (#2726) covering 8 tickets. The cross-file interaction between `doc-coverage-helpers.js` and `collaborator-handoff.js` was not tested end-to-end. The `loadNaReasons` fail-open pattern and the unremediated caller bypass were not caught by unit tests because each file was tested in isolation.

## Acceptance Criteria

- **AC1**: `loadNaReasons()` throws on missing/corrupt/malformed file. `valueViolation()` catches and returns `doc-coverage-config-error` violation (severity: error). Setting `DOC_COVERAGE_GATE_ADVISORY=1` has no effect on enum enforcement. Test: file removed → gate blocks with `doc-coverage-config-error`.

- **AC2**: `collaborator-handoff.js` line 74 `DOC_COVERAGE_GATE_ADVISORY !== '1'` guard removed. The doc-coverage gate is called unconditionally for `lane:code-change`. Test: env var set → gate still blocks on missing block.

- **AC3**: `getChangedFiles()` in `diff-verify.js` uses `spawnSync(['git', 'diff', '--name-only', ...])` with args array, not template-string `execSync`. Test: verify no string interpolation in git invocation.

- **AC4**: `verifyDeclaredSurfaces()` returns `ok: false` when any `doc-diff-not-changed` violation is present (violations.length === 0, not just error-severity filter). Test: surface declared DONE but not in diff → ok=false.

- **AC5**: `valueViolation()` enforces `covered-by-sibling-pr:#N` format when reason is `covered-by-sibling-pr`. Missing or malformed `#N` returns `doc-coverage-invalid-na`. Test: `covered-by-sibling-pr` without `#123` → error.

- **AC6**: `doc-coverage.js` has a `require.main === module` CLI entrypoint: `node scripts/global/megalint/doc-coverage.js --body <file> [--labels <json>] [--lane <lane>]`. Exits 1 with structured stderr on violation, 0 on pass. Test: CLI invocation with invalid body exits 1.

- **AC7**: All existing 68 tests continue to pass. New tests added for AC1-AC6 (minimum 7 new tests).

- **AC8**: All modified files remain ≤ 100 lines after changes.
