## Body

## Problem

The harness's G5 portability contract (`instructions/harness-goals.instructions.md` lines 22-31) requires every feature that depends on resources beyond a documented minimum to ship with either (a) an env-var/label opt-out OR (b) a minimum-resource fallback. The contract is in place; what is NOT in place is an Epic-level definition of the resource TIERS the harness recognizes, and a per-feature audit of which tier each feature requires.

This gap surfaced today during the #1112 Phase-0 R&D synthesis (#2397): the proposed cross-team R&D productization plan recommended HAMR R2 Mailbox as the cross-team channel, but HAMR is Cloudflare-Workers-dependent — outside the documented minimum environment a typical operator carries. The plan unintentionally violated G5 by failing to specify a local-only fallback.

Per client direction at 2026-05-29, the harness should explicitly define its minimum environment and tier features against it.

## Acceptance Criteria

- AC1: Resource tier taxonomy documented at `instructions/resource-tier-portability.instructions.md`; tier definitions, escape hatch env vars, and "minimum environment" baseline ratified
- AC2: Hugging Face Hub support added as an alternative Tier 1 remote-repo-management surface (currently GitHub-only); the harness's ticket-IS-the-baton contract abstracted to work over Hugging Face Discussions or Issues
- AC3: Per-script audit: every `scripts/global/*.js` script tagged with its minimum tier in a frontmatter comment; CI lint validates the tags are present and accurate
- AC4: Feature matrix at `docs/howto/resource-tier-feature-matrix.md` listing each major harness feature × required tier × fallback availability
- AC5: For every feature requiring tier ≥ 2, document the Tier-1 (GitHub-only) fallback OR explicitly mark it as tier-gated with operator-visible degradation message
- AC6: New env var `MEGINGJORD_MINIMUM_TIER=<N>` that asserts the operator's intended tier and fails closed if any feature requires a higher tier than asserted
- AC7: Audit existing escape hatches (`MEGINGJORD_HAMR_DISABLED`, `MEGINGJORD_MCP_DISABLED`, `MEGINGJORD_REBASE_DISCIPLINE_DISABLED`) for consistency with the new tier model; consolidate or extend as needed
- AC8: Update `#1112` Phase-0 synthesis (#2397) and Phase-1 plan to specify Tier-1 fallback for every HAMR-dependent surface (mailbox, cron, mcp)
- AC9: Update `instructions/harness-goals.instructions.md` G5 paragraph to reference the new tier taxonomy and resource matrix
- AC10: Cross-orchestrator compatibility test suite (`tests/orchestrator-compatibility.spec.js` from #2388) extended to verify each orchestrator works at the asserted MEGINGJORD_MINIMUM_TIER
