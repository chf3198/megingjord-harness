# #2050 — Guard leaseRegistry calls in branch-cleanup-plan.js against registry unavailability

> **Source**: github:issue/2050 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P3, area:scripts, resolution:completed, lane:code-change
> Mirror of `gh issue view 2050` (derived; edit the GitHub item, not this page).

## Body

## Problem

`orphanedLeases()` in `scripts/global/branch-cleanup-plan.js` calls `leaseRegistry.read()` and `leaseRegistry.active()` without error handling. If the registry file is absent, malformed, or locked, the exception propagates through `plan()` and crashes the entire tool — including the branch classification output that has no dependency on leases.

## Expected Outcome

Wrap the registry calls in a try/catch and return an empty array on any error, with a warning logged to stderr.

## Acceptance Criteria

- AC1: `orphanedLeases()` catches any exception from `leaseRegistry.read()` or `leaseRegistry.active()`
- AC2: On error, returns `[]` and writes a warning to `process.stderr`
- AC3: Branch classification output is unaffected when the registry is unavailable
- AC4: Unit test covering the case where lease registry throws

## Context

Found during Consultant red-team on #1610. The lease registry is a frequently-mutated live state file; a corrupt or absent registry is a plausible operational failure.

Refs #1610
