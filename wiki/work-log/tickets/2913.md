# #2913 — Add real-time session behavioral anomaly detection to tool_activity.py

## Body

## Problem
No real-time session behavioral anomaly detection exists (Gap G-15). An agent can execute individually benign tool calls that form a dangerous sequence (e.g. read config → read secrets → write → push) without any individual step triggering a deny. Sequence-level detection is entirely absent.

This is ASI05 (Cascading Failures) + EU Art.14 (Human Oversight). Detection requires human-in-the-loop trigger on anomalous session patterns.

## Acceptance Criteria
- [ ] In `hooks/scripts/tool_activity.py`, add session-level aggregate counters: total_writes, total_reads_sensitive_paths, total_pushes, estimated_cost_usd
- [ ] Define anomaly thresholds in `config/governance-rules.yaml` (e.g. writes_in_session > 50, sensitive_path_reads > 10)
- [ ] On threshold breach: emit `ANOMALY_DETECTED` event to `incidents.jsonl` and return human-review-required deny
- [ ] Add tests: `tests/session-anomaly-detection.spec.js`
