# #2682 — Closure-block guard and BLOCKER_NOTE emission for Phase-0-complete Epics

## Body

## Objective

Ship the closure-block guard that prevents a research-first Epic from closing when Phase-0 is complete but Phase-1 children are absent. Converts the current advisory-only behavior to a hard block. Depends on the phase0_green_complete predicate from child A.

## Acceptance Criteria

- [ ] AC1: `scripts/global/label-lint-close-protection.js` extended: when an Epic with phase-gate:research-first label attempts to close, invoke phase0_green_complete; if `{ complete: true, missingPhase1Children: true }`, post EPIC_PHASE_GATE_PAUSE comment and return blocking violation
- [ ] AC2: `.github/workflows/closeout-lint.yml` updated to invoke the new guard path for research-first Epics
- [ ] AC3: EPIC_PHASE_GATE_PAUSE comment includes: structured BLOCKER_NOTE with owner, unblock_condition (Phase-1 children must be created), and pattern_id `phase0-complete-no-phase1`
- [ ] AC4: Guard exits non-zero (core.setFailed) causing CI to block Epic close
- [ ] AC5: Guard is skipped (no-op) if Phase-1 children already exist
- [ ] AC6: Regression test: recreates the #2661 scenario (Phase-0 closed, Epic attempts close, no Phase-1 children) and verifies CI blocks it

- Tests: `tests/label-lint-close-protection.spec.js` -- #2661 regression scenario
