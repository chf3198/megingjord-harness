## Body

## Symptom

Main checkout at `/home/curtisfranks/devenv-ops` persistently shows `M` (modified) entries in `git status --short` across many session starts. Files include:

- `hooks/scripts/validate-branch-name.sh`
- `scripts/global/megalint/{admin,collaborator,manager}-handoff.js`
- `scripts/global/megalint/{merge-evidence,merge-evidence-pr-gate,index}.js`
- `scripts/global/test-strategy-enum.js`
- `scripts/global/{constitution-compressor,consultant-checks-lib,cross-team-lease,github-dispatcher,issue-transition,worktree-cleanup-plan,worktree-inventory}.js`
- `wiki/index.md`

These persist across Claude Code session restarts and cause the Stop hook to repeatedly fire `"Stop blocked: uncommitted changes; Admin baton incomplete"` even when the active session made no code edits.

## Empirical evidence (2026-05-28)

`git diff` on these files shows working tree content is OLDER than HEAD on files shipped by recent merged PRs:
- `validate-branch-name.sh`: working tree has `#1336 AC1` enum comment; HEAD has the `#2304` canonical-source-of-truth comment + reordered enum (the version that shipped via PR #2307 merged 2026-05-27)
- `scripts/global/megalint/manager-handoff.js`: working tree is MISSING the `checkTestStrategy` function that HEAD has (shipped via PR #2308 merged 2026-05-27)

This means SOMETHING in this Chromebook's environment is keeping the main-checkout's working tree pinned to PRE-merge content even after the merges land on main HEAD.

## Hypothesis space (to be narrowed by investigation)

1. **Sync-back script overwrites main from ~/.copilot/**: a sync script (e.g., `scripts/sync.sh` or hook-deployed equivalent) copies content from ~/.copilot/hooks/scripts/ or ~/.copilot/scripts/global/ INTO main checkout, and ~/.copilot/ holds the older versions.
2. **VS Code auto-restore**: VS Code's file-history or auto-save mechanism restores files to pre-merge content on session start.
3. **Pre-tool hook write-back**: pretool_guard.py or another hook may write deployed-runtime content back into main on certain triggers.
4. **Cross-team writeback**: another team's session on this Chromebook (Codex/Copilot) running concurrently writes their canonical content into main, regressing my merges.
5. **Branch-switch artifact**: a `git checkout` operation (manual or scripted) at some point copied the working tree to pre-merge HEAD content.

## Acceptance criteria

- AC1: Root cause identified with concrete reproduction evidence
- AC2: Working tree on main is restored to match HEAD (one-shot cleanup)
- AC3: Guardrail implemented in the harness that prevents recurrence (per-session check + auto-reset, OR pre-tool hook that detects+blocks writes-to-main-tracked-files, OR sync script that no longer writes back to canonical-main)
- AC4: Test covering the guardrail (drift-lint or stress-test depending on mechanism)
- AC5: Documentation update in `instructions/global-standards.instructions.md` "Canonical-main checkout policy" section explaining the prevention mechanism
- AC6: Stop hook stops firing the false-positive "uncommitted changes" warning when no real edits happen in-session
