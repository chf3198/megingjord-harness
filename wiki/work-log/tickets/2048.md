# #2048 — Fix shell injection in branch-cleanup-plan.js via branch-name interpolation

> **Source**: github:issue/2048 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:scripts, area:governance, resolution:released, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 2048` (derived; edit the GitHub item, not this page).

## Body

## Problem

`isMergedToMain()` and `prState()` in `scripts/global/branch-cleanup-plan.js` interpolate `${branch}` directly into `execSync` shell strings. A branch name containing `"` or shell metacharacters closes the quote and allows arbitrary command injection.

```js
// current (vulnerable)
sh(`git merge-base --is-ancestor "refs/heads/${branch}" origin/main && echo yes`)
sh(`gh pr list --head "${branch}" --state all --json number,state --jq '.[0]'`)
```

## Expected Outcome

Replace string-interpolated `execSync` calls with `child_process.spawn` (argument array) or validate branch names against `/^[a-zA-Z0-9/_.-]+$/` before use. Output cleanup commands should also quote branch names.

## Acceptance Criteria

- AC1: `isMergedToMain()` uses spawn-with-args or validated input — no shell interpolation of branch names
- AC2: `prState()` same fix
- AC3: `commandsFor()` output strings quote branch names (e.g. `git branch -d '${branch}'`)
- AC4: Unit test covering a branch name with special characters returns keep-active (safe classification)
