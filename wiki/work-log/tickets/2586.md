# #2586 — Derive active-ticket from edited-file worktree branch in pretool_guard

## Body

Refs Epic #2585

## Problem (Finding 1 — worktree-blind gate)
`pretool_guard.py` denies a file edit when `state.get("active_ticket")` is unset. But `state` is keyed on the **session cwd**, which is always the canonical main checkout (`~/devenv-ops`) — never the per-ticket worktree where the edit actually lands. After `reset_on_branch_change` clears main's state, *legitimate* edits in `~/devenv-ops-<N>/` are falsely denied "no active ticket." Evidence: hit live this session editing `~/devenv-ops-2564/...`; worked around by hand-writing `active_ticket` into main's state. Memory: `feedback_worktree_edit_ticket_gate_reads_main_state`.

## Remediation (runtime-neutral, strictly-additive)
When the active-ticket check is about to deny, attempt a **path-derived** resolution before denying:
1. Read edited path(s) via `admin_patterns.iter_paths(tool_input)` (the #2371-hardened key normalizer — handles `file_path`/`TargetFile`/`apply_patch` across all 4 runtimes; **no hardcoded keys**).
2. For each path, compute the enclosing git toplevel: `git -C <dirname(path)> rev-parse --show-toplevel`.
3. **Mitigation A (governed-root guard):** accept the toplevel ONLY if it matches the governed checkout pattern (`~/devenv-ops` or `~/devenv-ops-*`), reusing the `canonical_main_enforcer` path logic. This rejects unintended nested repos/submodules and prevents a false-allow from an arbitrary outside repo.
4. Read that worktree's branch: `git -C <toplevel> rev-parse --abbrev-ref HEAD`.
5. **Mitigation B (tolerant extractor):** extract the ticket via one consolidated helper accepting `type/<N>-slug`, `#<N>-slug`, and bare `<N>-slug`; return `None` for anything else (main, dependabot/*, detached HEAD). (Fixes existing `extract_from_branch` which only matches `^#?(\d+)-` and misses `fix/<N>-`.)
6. If a ticket is derived → treat as the active ticket (allow; optionally reconcile `state["active_ticket"]`). Otherwise → **fall back to existing behavior** (no regression, no crash).
