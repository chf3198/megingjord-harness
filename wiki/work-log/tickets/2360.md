# #2360 — Remediate Antigravity Governance Parity & Hook Validator Blind-Spots

## Body

## Discovered Governance Gap
During the critical audit of ticket #2054, we identified a systematic **Governance Accessibility and Parity Gap** affecting the Antigravity agent runtime.

Historically, the Megingjord Harness's hooks and enforcers were designed around the toolsets of Claude Code and Copilot. Because Antigravity is a newer addition (Epic #2014), the validators suffered from a severe blind spot:
- **Canonical-Main Write Protection ()**: Only checked tool names like `Write`, `Edit`, and `create_file`. Antigravity's native tools (`write_to_file`, `replace_file_content`, `multi_replace_file_content`) bypassed this check, permitting direct edits on `main` without redirection to worktrees.
- **Lease Verification (`worktree-write-intercept.js`)**: Ignored Antigravity tools and failed to extract path metadata since Antigravity uses `TargetFile` instead of `file_path`.

## Remediations Applied
We have successfully resolved these gaps to establish 100% process parity:
1. **Hook Validation**: Updated `pretool_guard.py` and `tool_activity.py` to support Antigravity's write and terminal tools.
2. **Write Intercept**: Added Antigravity write tools and mapped `TargetFile` dynamically to path checks.
3. **Parity Matrix**: Added `antigravity` to `inventory/orchestrator-governance-parity.json` as a first-class supported runtime and updated the digest checksums.
4. **Test Coverage**: Authored new Python hooks tests (`tests/hooks/test_pretool_guard_antigravity.py`) and Playwright tests verifying complete alignment.

All checks are 100% green and verified.
