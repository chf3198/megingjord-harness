## Root cause hypothesis

The workflow uses `actions/github-script` to read PR comments + issue body via the default `GITHUB_TOKEN`. When the API call hits a rate limit OR the token lacks the right scope at that exact moment, the script throws `HttpError: Resource not accessible by integration` instead of being caught + reported as a structured violation. The error appears identical to a permanent permissions issue but is actually transient.

## Proposed remediation

- **AC1**: Wrap `github.rest.issues.listComments` (and similar API calls) in `closeout-schema.yml` with a small retry-on-403/422 helper. Limit to 2 retries with 2-second backoff.
- **AC2**: On final retry exhaustion, emit a structured `closeout-schema: api-transient` advisory comment instead of `core.setFailed`. Operator can then rerun manually OR the next workflow trigger picks it up.
- **AC3**: Add operator runbook note in `docs/howto/` or instruction file describing the transient pattern + how to re-run without admin override.
