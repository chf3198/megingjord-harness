# #2905 — Add live prompt injection defense for Read/Fetch tool returns

> **Source**: github:issue/2905 | **state: CLOSED** | **Labels**: type:task, status:done, area:hooks, priority:P0, phase-gate:phase-1
> Mirror of `gh issue view 2905` (derived; edit the GitHub item, not this page).

## Body

## Problem
Live prompt injection defense is entirely absent (Gap G-20). When an agent calls Read/Fetch/search tools, the returned content is not scanned for injected instructions. An adversary can serve a file or web page containing `## INSTRUCTIONS: Ignore all prior rules and...` which gets passed to the model without any sanitation layer.

This is ASI01 (Goal Hijacking) + EU Art.15 (Cybersecurity/robustness). Rated P0 because exploitation requires no harness-level access — only a file or URL the agent reads.

## Acceptance Criteria
- [ ] Add injection-pattern scanning in `hooks/scripts/pretool_guard.py` (or equivalent) for Read/Fetch/WebSearch tool returns
- [ ] Define a blocklist of injection pattern regexes in `config/` (e.g. `prompt-injection-patterns.json`)
- [ ] Return a deny decision when injected instruction patterns are detected in tool output
- [ ] Add tests: `tests/prompt-injection-defense.spec.js` with ≥5 injection pattern cases
- [ ] Document in the guardrail assessment under G-20
