## Parent

Epic #1604 (Establish conflict-free cross-team harness governance). Adopts finding F1 from research #1624.

## Goal

Standardize the GitHub agent tool surface across Claude Code / Copilot / Codex teams via the official GitHub MCP server (https://github.com/github/github-mcp-server). Every team's skill consumes MCP tools (Issues, PRs, Discussions, Projects v2, Sub-issues, Actions, Repositories) uniformly instead of per-team REST/CLI adapters.

## Portability annotation (per #1628)

- **Resource dependency**: GitHub MCP server requires a GITHUB_TOKEN. Every operator using gh CLI already has one — no new credential.
- **Network dependency**: MCP server requires network access to api.github.com. Air-gapped operators have no GitHub access at all (baseline-absent); MCP server's absence is consistent with their existing constraints.
- **Classification**: integral to harness goals (#1624 F1 identified as Phase 1 GA-safe primitive). No fallback required for the standard case; air-gapped operators continue with their existing local-only workflows.
- **Opt-out**: operators may set MEGINGJORD_MCP_DISABLED=1 (parity with HAMR opt-out pattern).
