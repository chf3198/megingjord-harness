# #1344 — Epic: Research-first stress test realism for orchestrator workflow, dashboard liveness, and low-cost HAMR coverage

## Body

## Summary
Examine and redesign the Stress Test so it faithfully replicates real orchestrator Agile role workflow across Epics and Tickets, drives visibly active live behavior across all dashboard sections intended for animation/live telemetry, and validates harness + HAMR end-to-end under a minimum-cost profile that maximizes free resources.

## Why this Epic
Current stress behavior appears insufficient to prove real-world baton execution, cross-panel liveness, and low-cost governance routing at scale. We need a research-first initiative before defining implementation ACs.

## Objective
Produce a high-confidence research foundation that specifies what “real orchestrator workflow stress realism” must include, what dashboard liveness evidence is required, and how to exercise harness + HAMR comprehensively while minimizing paid usage.

## Scope (Research-Only in this Epic phase)
- Orchestrator Agile workflow realism for Epics + child Tickets (Manager → Collaborator → Admin → Consultant).
- Dashboard liveness realism for all sections that should show active animations and/or live information under stress.
- Harness-wide and HAMR-wide coverage under low-cost/free-first routing strategies.
- External research into cutting-edge, out-of-the-box stress/testing/governance patterns applicable to this repository.

## Non-Goals (until research completes)
- No development implementation ACs yet.
- No architecture lock-in yet.
- No irreversible automation changes yet.

## Acceptance Criteria — Research Gate (must complete before any development ACs are authored)
- [ ] **AC-R1: Current-State Audit** — Document the existing Stress Test architecture, event sources, orchestration simulation behavior, and panel activation behavior.
- [ ] **AC-R2: Workflow Realism Gap Matrix** — Provide a gap analysis comparing current stress behavior vs. expected real orchestrator baton lifecycle for both Epics and Tickets.
- [ ] **AC-R3: Dashboard Liveness Coverage Map** — Enumerate every dashboard section expected to animate or stream live data; define observable activity signals and pass/fail criteria per section.
- [ ] **AC-R4: Harness + HAMR Coverage Inventory** — Identify all harness and HAMR pathways that must be exercised in stress runs, with explicit free-tier/low-cost alternatives where available.
- [ ] **AC-R5: Cost-Minimization Strategy** — Define a free-first execution strategy and cost guardrails (budgets, fallbacks, rate controls, cache/reuse patterns) with quantified expected spend profile.
- [ ] **AC-R6: External Research Dossier** — Produce a curated synthesis of current best practices and novel techniques from credible sources, mapped to actionable options for this repo.
- [ ] **AC-R7: Research Synthesis Decision Brief** — Deliver prioritized options with tradeoffs, risks, and recommendation confidence scoring.
- [ ] **AC-R8: Phase Gate Enforcement** — Record a governance rule in this Epic: no implementation child tickets or development ACs until AC-R1..AC-R7 are complete and Consultant-reviewed.
