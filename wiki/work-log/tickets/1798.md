# #1798 — Fix baton-phase-blind Stop hook + Manager-gate false-positive (Copilot/Codex)

## Body

## Epic Summary

Fix the **baton-phase-blind Stop hook + Manager-gate false-positive** defect identified in the 2026-05-17 governance audit. Copilot and Codex runtime adapters emit premature "ADMIN ROLE INCOMPLETE" warnings during Manager-phase sessions when uncommitted unrelated files exist in the worktree, and emit false "Manager handoff missing ticket reference" warnings when Manager operates via `gh` CLI without typing `#N` in the user prompt. Claude Code is unaffected because its runtime wires no Python hooks.

## Audit reference

See the governance audit report on this Epic for full evidence (F1–F5 findings). Patches are all small (3–10 lines each); core problem is shared `hooks/scripts/stop_checks.py` and `hooks/scripts/manager_ticket_gate.py` lacking phase awareness.

## Five findings to address

| # | Severity | Location | Defect |
|---|---|---|---|
| F1 | HIGH | `stop_checks.py:check_uncommitted` | Fires "ADMIN ROLE INCOMPLETE" on any uncommitted `.sh/.js/.py/.ts/.json/.md` without checking baton phase |
| F2 | HIGH | `stop_checks.py:check_admin_ops` | Reports missing Admin steps when `flags.code_touched=True`, regardless of whether Collab finished |
| F3 | MEDIUM | `manager_ticket_gate.py` | Triggers on prompt words `work`/`task` (too broad); requires literal `#N` in prompt; ignores `state.active_ticket` |
| F4 | LOW | `state_store.py:_default_state` | No `current_phase` tracker — only 4 independent role-completion booleans |
| F5 | LOW | Adapter wiring | Copilot + Codex wire 7 hook events; Claude Code wires zero — parity hazard either way |

## Proposed contract

A **phase-aware Stop hook**: Admin-related warnings (uncommitted state + missing Admin ops) only fire when `roles.collaborator=True`. Before that, the session is in Manager or Collab phase and Admin-completion enforcement is silent.

A **state-aware Manager gate**: trigger only on explicit handoff markers (`MANAGER_HANDOFF`, `scope:`, `acceptance:`); accept ticket reference from `state.active_ticket` as well as prompt-embedded `#N`.

A **`current_phase` field** in governance state: deterministic phase tracking so future hooks can stop guessing from indirect signals.

An **adapter parity decision**: either deploy the hook config to Claude Code (`~/.claude/settings.json` extension) or document the intentional exemption in `instructions/canonical-governance-anti-duplication.instructions.md`.
