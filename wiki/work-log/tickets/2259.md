## Objective

Research the best governance design for a constrained Manager -> Consultant-only lane used solely for ticket-drift remediation when no code changes, branch work, PR creation, CI, or merge actions are required.

## Research Questions

1. What is the strongest way to preserve separation-of-duties when Collaborator and Admin are intentionally absent?

3. What anti-abuse controls prevent operators from misclassifying real delivery work as remediation-only?

5. How should the workflow interact with existing `config-only`, `docs-research`, and terminal-state governance rules?
