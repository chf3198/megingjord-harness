# #2891 — Governance guardrails assessment and improvement

> **Source**: github:issue/2891 | **state: OPEN** | **Labels**: type:epic, priority:P1, area:instructions, role:manager, status:triage, phase-gate:phase-1

## Body

## Objective

The harness governance model has a systemic gap: many governance rules exist only as instructions (markdown files loaded into context) rather than as machine-enforced guardrails. When an AI agent bypasses or misses an instruction, there is no enforcement mechanism to catch it before artifacts are produced or PRs are merged.

**Evidence trigger:** #2785 COLLABORATOR_HANDOFF posted without `collaborator:preflight` — missing `cross_family_rating`, `cross_family_reviewer`, `cross_family_findings`. Caught only post-hoc by operator review. Incident: `collaborator-preflight-skipped` (incidents.jsonl 2026-06-10).

## Problem

Governance that is instruction-only has 100% bypass rate when the agent does not load or follow the instruction. Guardrails (pre-commit hooks, CI gates, pre-push checks, schema validators) enforce governance deterministically regardless of agent behavior.

Current split (approximate):
- **Guardrailed**: branch naming, PR title, commit #N reference, lint (100-line), baton CI gates (collaborator/admin/consultant), evidence-completeness, changelog fragment structure
- **Instruction-only (no guardrail)**: cross-family preflight before COLLABORATOR_HANDOFF, test_strategy field presence in MANAGER_HANDOFF, README sync on script adds, docs-compile automation, various MANAGER_HANDOFF required fields, lease creation before edits, conflict gate execution, worktree governance for edits

## Scope

- AC-R1: Enumerate every governance rule in `instructions/**` and classify each as guardrailed, partially guardrailed, or instruction-only. Identify the highest-risk instruction-only rules.
- AC-R2: For each high-risk instruction-only rule, propose a guardrail mechanism (pre-commit hook, CI gate, schema validator, or script check) with effort estimate.
- AC-R3: Produce a prioritized implementation roadmap with Phase-1 child tickets drafted.
- AC-Rn: Terminal research child (gate closes Phase-0).
