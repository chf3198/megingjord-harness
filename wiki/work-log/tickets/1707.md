## Body

## Research Question

How can the harness provide canonical single-source governance to 5 orchestrators (Claude Code, Copilot, Codex, Cline, Continue) without maintenance overhead spiraling as we add support for more tools?

## Key Findings

### Top 5 Orchestrators (Market-Ranked)
- **Copilot** (73.5M VS Code installs, format stable since 2024)
- **Claude Code** (Anthropic, web-based, explicit dedup strategy)
- **Codex** (Legacy, sunsetting slowly, zero breaking changes expected)
- **Cline** (44.9k GH stars, high release velocity, stable format)
- **Continue** (33.2k GH stars, highest release velocity, 822+ releases, backward-compatible format changes only)

**Why not Cursor, Windsurf, Aider, GPT-Engineer?**
- Cursor, Windsurf: Single-product, less ecosystem integration (Phase 2 candidates)
- Aider: CLI-first, different governance model; separate handling needed
- gpt-engineer: Archived April 2026; community moved to Lovable.dev

### Format Stability (6-Month Analysis)
| Tool | Last Breaking Change | Trend |
|---|---|---|
| Copilot | 2024 (now stable) | Zero changes in 2026 |
| Claude Code | Stable since inception | Additive changes only |
| Cline | 2024 (now stable) | Additive changes only |
| Continue | NEW: March 2026 (.continue/configs) | **Backward compatible** |
| Codex | Frozen (read-only) | No new changes |

**Result:** All format changes in last 6 months are **additive** (new optional fields, new directory support). Zero breaking changes. Adapters can safely emit to all versions.

## Critical Gap in Epic #1701

Epic is strong on technical architecture (schema, adapters, pipeline) but **missing two critical ACs:**

1. **Team Coordination Protocol** — Define governance decision flow, approval gates, role assignments (Governance Author, Tool Lead per orchestrator, CI gate enforcement)
2. **Runtime Behavior Parity Tests** — Golden tests proving identical canonical rules enforce identically across all 5 tools (not just syntactically correct)

## Recommendation

- Update Epic #1701 with two new ACs covering team coordination + parity tests
- Proceed with Phase 1: Canonical schema + Copilot/Claude/Continue adapters (80% of use cases)

- Establish frozen governance spec until all 5 tools align on format
