## Context
Part of Epic #1626. Independent.

`logs/wiki-metrics.json` contains a `pages` map that is always empty `{}`. The root cause: the access-tracking function fires with a section name but no page slug. Page-level popularity has zero effective signal.

## Objective
Fix the access tracking call sites so that each wiki page read records its slug in the `pages` map. Surface page-level popularity in the dashboard wiki panel.

## Implementation notes
- Audit all callers of the access-tracking function in `scripts/wiki-metrics.js` — identify where slug is not passed
- Fix callers to pass the page slug; fall back to section/filename if title unavailable
- Update the dashboard wiki panel to display a Top pages section sourced from the `pages` map
- `pages` object stores `{ [slug]: count }` with atomic read-modify-write

## Acceptance criteria
- [ ] After 5 wiki read operations, `logs/wiki-metrics.json` `pages` map contains >= 1 slug entry
- [ ] Dashboard wiki panel displays a Top pages section
- [ ] No existing section-level telemetry is removed
- [ ] `npm run wiki:lint` passes
