# #2917 — Add per-session blast-radius cap (file/push/cost limits) to pretool_guard

> **Source**: github:issue/2917 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:hooks, lane:code-change, phase-gate:phase-1
> Mirror of `gh issue view 2917` (derived; edit the GitHub item, not this page).

## Body

## Problem
No per-session aggregate blast-radius cap exists (Gap G-06). An agent session can edit unlimited files, make unlimited pushes, and incur unlimited cost with no session-level enforcement. A runaway or compromised session has no hard stop.

This is ASI05 (Cascading Failures) + EU Art.15 (Cybersecurity/robustness).

## Acceptance Criteria
- [ ] In `hooks/scripts/pretool_guard.py`, add session aggregate counters: `files_edited_count`, `push_count`, `estimated_cost_usd`
- [ ] Define caps in `config/governance-rules.yaml`: `max_files_per_session: 200`, `max_pushes_per_session: 20`, `max_cost_usd_per_session: 5.00`
- [ ] On cap breach: emit `SESSION_BLAST_RADIUS_CAP` to `incidents.jsonl` and return deny
- [ ] Provide `MEGINGJORD_BLAST_RADIUS_DISABLED=1` operator override (with audit log)
- [ ] Add tests: `tests/blast-radius-cap.spec.js`
