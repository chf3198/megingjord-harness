# #1804 — Reconcile Copilot/Codex hook-event wiring drift (PreCompact + SubagentStart)

## Body

## Objective

Reconcile the hook-event wiring drift between Copilot and Codex runtime adapters surfaced by the Epic #1798 stress test (Quill Vale / Codex consultant report, 2026-05-17).

## Observed drift

| Hook event | `~/.codex/hooks.json` | `~/.copilot/hooks/global-standards.json` |
|---|:---:|:---:|
| SessionStart | ✓ | ✓ |
| UserPromptSubmit | ✓ | ✓ |
| PreToolUse | ✓ | ✓ |
| PostToolUse | ✓ | ✓ |
| Stop | ✓ | ✓ |
| PreCompact | — | ✓ wired to `precompact_anchor.py` |
| SubagentStart | — | ✓ wired to `subagent_inject.py` |

Both `hooks/scripts/precompact_anchor.py` and `hooks/scripts/subagent_inject.py` exist in repo source; only Copilot's deployed adapter wires them. The scripts deploy to `~/.codex/devenv-ops/hooks/` via `scripts/deploy.sh --target codex` but the event registration in `~/.codex/hooks.json` doesn't reference them.

## Decision required

Three options (must choose one):

1. **Wire on Codex too** — extend `.codex/runtime-hooks.json` (the deploy template) to register `PreCompact` and `SubagentStart`. Achieves parity-up.
2. **Remove from Copilot** — if those events aren't valuable, remove from Copilot adapter. Achieves parity-down.
3. **Document the asymmetry** — record in `instructions/canonical-governance-anti-duplication.instructions.md` the per-runtime intentional differences (Codex doesn't natively fire `PreCompact` or `SubagentStart` events; Copilot does).

## Acceptance Criteria

- [ ] AC1: Investigate whether Codex CLI emits `PreCompact` / `SubagentStart` lifecycle events natively (read Codex docs or test).
- [ ] AC2: Decision made and recorded (option 1, 2, or 3 above) with goal-lens rationale.
- [ ] AC3: If option 1: `.codex/runtime-hooks.json` updated and `npm run deploy:codex:apply` integration tested.
- [ ] AC4: If option 2: scripts/global-standards.json updated to remove from Copilot.
- [ ] AC5: If option 3: `instructions/canonical-governance-anti-duplication.instructions.md` §"Adapter parity" extended with the per-runtime event registration matrix.
- [ ] AC6: Stress-test parity check (Codex Step 4.8 from Epic #1798 stress test) updated to reflect the decided contract.

## Constraints

- Smallest-correct fix preferred. If Codex CLI doesn't natively fire `PreCompact` events, wiring them on Codex would be no-op overhead — option 3 (document) is the right answer.
- No edits to deployed runtimes directly; all changes flow through repo source + `npm run deploy:*:apply`.
- Preserve baton invariants per `governance/README.md`.
