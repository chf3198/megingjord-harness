## Goal

Restructure `CHANGELOG.md` to the Keep-a-Changelog 1.1.0 standard hierarchy (H1 title + H2 releases + H3 categories) and fix the existing MD001 heading-increment violation. Resolves ~88 of the 89 baseline lint errors blocking `lint-required` CI on every PR. The remaining MD001 + future-prevention is C2's scope.

## Background

Per the Phase-0 synthesis at `wiki/wisdom/project/research/changelog-aggregator-2120.md` (root-cause section 2), the current CHANGELOG.md uses H1 release headers (`# 1111`) contrary to the Keep-a-Changelog 1.1.0 H2 standard (`## [1111] - YYYY-MM-DD`). This trips MD024 `siblings_only` due to a documented markdownlint edge case (issues #136 / #175 / #300 / #1591) when H1-rooted documents have repeated H3 children across H1 siblings.

## Acceptance criteria

- **AC1**: Every release header in `CHANGELOG.md` converted from `# <num>` (H1) to `## [<num>]` (H2). Title `# Changelog` stays at H1.
- **AC2**: An `## [Unreleased]` H2 section added at the top (between `# Changelog` and the first release H2) per the Keep-a-Changelog convention.
- **AC3**: All existing `### Added/Changed/Fixed` H3 category headers preserved unchanged.
- **AC4**: MD001 heading-increment violation at line 13 eliminated (was H1 → H3 skip; becomes H1 → H2 → H3 valid increment).
- **AC5**: New snapshot test `tests/changelog-structure.spec.js` asserts:
  - `CHANGELOG.md` starts with `# Changelog` (exactly one H1)
  - First H2 is `## [Unreleased]`
  - Every other H2 matches pattern `## [<numeric-version>]` (with optional ` - YYYY-MM-DD` suffix)
  - No H1 appears below line 1
  - All H3 headers are children of an H2 release section
- **AC6**: `npm run lint:md` reports zero MD024 + zero MD001 errors on `CHANGELOG.md` after the restructure.
