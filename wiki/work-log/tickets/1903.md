# #1903 — Tier-2 anneal: goal-failure-emission validator CI-accessibility (accept Tier-3 ticket evidence)

## Body

Parent Epic: #1894

## Problem

`scripts/global/megalint/goal-failure-emission.js` enforces Tier-3 escalation when CONSULTANT_CLOSEOUT rates any G1-G9 score below 7. The validator requires an `event:goal-failure-escalation` entry in `~/.megingjord/incidents.jsonl` with specific schema.

**Structural bug**: when this validator runs as part of `closeout-schema` CI workflow on a GitHub Actions runner, `~/.megingjord/incidents.jsonl` does NOT exist (the runner has a fresh home directory). `loadIncidents()` returns `[]` and ALL sub-threshold goal scores fail the gate, regardless of whether Tier-3 has actually been escalated.

Real-world impact (observed 2026-05-18):
- claude-code Team takeover on #1888 honestly rated G8=6 per cross-family fleet rater feedback
- Filed Tier-3 anneal #1900 via auto-trigger
- Logged event to local incidents.jsonl
- CI ran closeout-schema → `missing-goal-failure-event: G8=6 (sub-threshold) requires event:goal-failure-escalation ... No such event found in incidents.jsonl` (failure)
- Operator had to revise G8 to 7 (artificially) to pass the gate

The validator is preventing the honest behavior it's supposed to incentivize.

## Goal

Make `goal-failure-emission` validator CI-accessible by checking for Tier-3 evidence in artifacts the runner CAN read:

1. Linked Tier-3 ticket in `anneal_tickets_filed:` field of the CONSULTANT_CLOSEOUT (verifiable via `gh issue view`)
2. `mid_flight_flaws:` entries citing the sub-threshold goal + ticket reference
3. Fallback: existing local incidents.jsonl check for operator-local runs

## Acceptance Criteria

- [ ] AC1: Extend `enforceTier3Emission()` in `scripts/global/megalint/goal-failure-emission.js` to accept multi-source evidence:
  - Primary: parse `anneal_tickets_filed: #N` from the CONSULTANT_CLOSEOUT body itself
  - Verify each cited `#N` exists via passed-in resolver (callback / API)
  - Verify cited ticket has `anneal:tier-3` label
  - Verify ticket body references the sub-threshold goal (e.g. "G8 disagreement")
- [ ] AC2: Fallback to `~/.megingjord/incidents.jsonl` check for local-dev compatibility (existing behavior preserved when file exists).
- [ ] AC3: Workflow `closeout-lint.yml` updated to pass ticket-resolver callback to validator (`gh issue view <N> --json labels,body`).
- [ ] AC4: Tests covering: (a) sub-threshold + Tier-3 ticket cited + ticket-exists + correct-label passes, (b) sub-threshold + no Tier-3 cited fails, (c) sub-threshold + Tier-3 cited but ticket missing fails, (d) sub-threshold + Tier-3 cited but wrong-label fails, (e) above-threshold no-op.
- [ ] AC5: Stress test: 20 fixture closeouts with mixed sub-threshold + Tier-3-cited combinations.
- [ ] AC6: harness:self-test registry entry; existing self-tests preserved.

## Why this matters

Per memory note `feedback_consultant_tier3_emission.md`: *"When Consultant rubric scores G1-G9 below 7, MUST emit Tier-3 event + file remediation ticket — skipping defeats the auto-anneal protocol."* The current validator implementation makes honest sub-threshold rating impossible in CI. Operators are incentivized to inflate scores rather than file Tier-3 anneals.

Composes directly with Epic #1612 second-opinion-runner + second-opinion-tier3-trigger (Tier-3 auto-file on max_abs_delta > 1.0). If the validator accepted the Tier-3 ticket reference as evidence, the auto-file pattern would complete the loop end-to-end.
