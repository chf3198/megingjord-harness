## Problem
To make the governance guardrail impassible, we must integrate the drift checks directly into the local and remote pre-push and pre-PR validation gates.

## Scope
1. Integrate the `lint-epic-drift.js` validator script into `scripts/global/pre-pr-gate.js` and `scripts/global/pre-push-gates.js`.
2. Add support for the emergency bypass environment variable `SKIP_DRIFT_LINT=true`.
3. Force bypass events to log warning telemetry to `~/.megingjord/incidents.jsonl` under `pattern_id=ticket-drift-lint-bypass`.
4. Ensure files stay within the strict 100-line cap.

## Acceptance criteria
[ ] Gate blocks git push/commit/PR creation when active Epics contain status sync or linkage drift.
[ ] Emergency bypass variable `SKIP_DRIFT_LINT=true` successfully bypasses the gate.
[ ] Telemetry logging validated on bypass events (written to incidents.jsonl).
