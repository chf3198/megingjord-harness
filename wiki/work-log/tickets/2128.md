## Goal

Wire `scripts/global/changelog-fragment-validator.js` (from C2) into the `lint-required` CI workflow as a fragment-time enforcement step. After this child ships, PRs that add malformed CHANGELOG fragments fail CI BEFORE aggregation — the structural fix is now end-to-end enforced rather than just available.

## Acceptance criteria

- **AC1** New CI step in `.github/workflows/lint.yml` (or `lint-required` equivalent) that runs `node scripts/global/changelog-aggregate.js --validate-only` against `.changes/unreleased/*.md` for every PR that modifies any file under `.changes/unreleased/`.
- **AC2** Step exits 0 when all fragments validate cleanly; exit 1 with clear violation message otherwise.
- **AC3** A golden-file test `tests/fixtures/changelog-fragment-invalid/*.md` plus `tests/changelog-fragment-ci.spec.js` simulates the CI invocation and asserts the expected failure on malformed fragments + pass on conforming fragments.
- **AC4** Release-cut runbook section added to `docs/howto/changelog-fragments.md` (or new file) explaining (a) the new validator gate, (b) how to fix common T3/T4 violations, (c) the `--validate-only` CLI usage for local pre-PR check.
- **AC5** After C3 ships and operator pulls `main`, `lint-required` CI gate stays green naturally for all subsequent PRs with conforming fragments; malformed-fragment PRs fail with clear error message.
