## Body

## Why this self-anneal is needed

Self-audit performed 2026-05-17 during a long operator session (Claude Code / claude-code:opus-4-7@anthropic) revealed three structural drifts in how this session executed cross-model review:

1. **Made-up 0–10 G1–G9 sub-score rubric** with no calibration corpus, no inter-reviewer reliability check, and no documented derivation. Anchor-bias-clustered tightly in the 9.0–9.4 range across 7 ships of very different sizes, lanes, and surfaces.
2. **No score→tier escalation map applied**. The canonical contract designed in dormant Epic #1745 specifies "<70 = F = P1 self-anneal". I never mapped any closeout rubric to a self-anneal action. Rubric values landed in baton comments and went nowhere.
3. **Cross-family review shallow / rubber-stamp pattern**. Qwen 7B on `36gbwinresource` was invoked on every closeout (8 of 8). Every single review returned `verdict: Approve`. None blocked or downgraded a ship; informational follow-ons surfaced by Qwen were never acted on or filed as tickets.

## Impact

- **Evidence theater**: rubric values appear in artifacts but feed back into nothing. Operators reading closeouts trust the 9.3/10 as a signal when it is unmoored from any calibration corpus.
- **Failure to self-trigger**: a working score→tier map would have caught the soak-language drift at the FIRST closeout it appeared in. Instead, three recurrences accumulated before the user filed #1809.
- **Cross-family rotation theater**: Qwen review counts as Rule 3 compliance under #1716, but if the review is rubber-stamp it provides no actual independent critique. The cross-family contract assumes the reviewer is substantively independent, not just from a different family.
- **Hidden risk**: future ships may continue to emit 9.3/10 for genuinely defective code. The current system has no detection.

## Root-cause hypothesis

The 0–10 G1–G9 rubric was inherited verbal convention from prior baton artifacts and instruction text (`global-standards.instructions.md` mentions G1–G9 goal-lens priority order). Translation from "9 priority-ordered goals" to "9-dimension 0–10 rubric" is operator drift, not documented contract. The canonical 1–100 unified rubric design lives in dormant Epic #1745 but has never been built, so operators improvise.
