# #1378 ‚Äî Fix worktree-session-start.sh: install or symlink node_modules for pre-push hooks

## Body

## Problem

`scripts/worktree-session-start.sh` (and equivalent worktree-creation paths) does not install or link `node_modules` in the new worktree. The pre-push hook runs `npm run format:check` and `npm run lint:readability:ci`, both of which require `prettier`, `eslint`, and other binaries from `node_modules`. The first push from any fresh worktree fails with `prettier: not found`.

## Live evidence

1. **Codex Team session (2026-05-09)** ‚Äî Codex explicitly reported: *"Push was blocked only because that fresh worktree needed npm dependencies for the pre-push hook."* They had committed work for #1286 but could not push without manual intervention.
2. **CC Team session (2026-05-11)** ‚Äî same blocker hit on `feat/1310-anneal-trigger-router` and subsequent CC worktree pushes. Worked around by `ln -s /home/curtisfranks/devenv-ops/node_modules /home/curtisfranks/devenv-ops-cc-1308/node_modules`.

This is a recurring friction across teams and across sessions. Every new worktree hits it.

## Goal

Eliminate the bootstrap gap so every new worktree can push immediately.

## Acceptance Criteria

- [x] **AC1 ‚Äî Auto-link approach.** `scripts/worktree-session-start.sh` (and any other worktree-creation entry point) creates a symlink: `<new-worktree>/node_modules` ‚Üí `<main-checkout>/node_modules`. Symlink chosen over `npm install` because it's near-instant and shares the dependency tree (no duplication).
- [x] **AC2 ‚Äî Verify symlink not committed.** `.gitignore` already ignores `node_modules`; verify the symlink stays untracked.
- [x] **AC3 ‚Äî Detect+fix retroactively.** Add a one-liner repair script (`npm run worktree:bootstrap` or similar) that ensures all existing worktrees have the symlink. Run it as part of a CI weekly sweep.
- [x] **AC4 ‚Äî Document the pattern.** Update `CLAUDE.md` (or `instructions/repo-health-onboarding.instructions.md`) with a one-line note: *"New worktrees auto-link `node_modules` from main checkout via `scripts/worktree-session-start.sh`; no manual `npm install` needed."*
- [x] **AC5 ‚Äî Test.** Spec test that simulates fresh worktree creation, then runs `npm run format:check` ‚Äî should succeed without manual `npm install`.

## Test strategy

`golden-file` for the worktree-creation script + `tdd-pyramid` for any new helper. Manual verification: create a fresh worktree, immediately attempt push, verify it doesn't fail on missing prettier.
