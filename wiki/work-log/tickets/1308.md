# #1308 — Add distributed pull-based self-anneal with tiered escalation

## Body

## Problem

Epic #1133 builds the *pattern-detection* leg of self-anneal — `anneal-goal-sensor.js`, `recurring-patterns.json`, suppression-registry (#1220), auto-file proposals (#1219), governance-audit integration (#1222), scheduler resilience (#1224). What it does NOT cover:

1. **Distributed any-role pull** — only pattern-recurrence triggers anneal today. A Collaborator who notices a violation mid-implementation has nowhere to put it; the Andon-cord principle is missing.
2. **Tiered escalation** — every drift becomes a ticket the same way; no severity classification, no observation-only channel for trend tracking.
3. **Mid-flight pivot orchestration** — if anneal fires during active baton work, no protocol exists for snapshotting current role/ticket and restoring after the anneal-ticket is filed.
4. **Consultant goal-failure escalation** — Consultant rubric finds harness-goal violations post-implementation, but has no formal authority to (a) reopen failed ACs/tickets via the baton, (b) file a new self-anneal Epic for systemic patterns.
5. **Bounded-loop guards specific to anneal** — kill switches, step limits, single-flight per session — none codified.

## Goal

Build the distributed pull layer + tiered escalation + bounded pivot semantics on top of #1133's pattern-detection pipeline. Achieve: any role can flag drift; severity classifier routes signals to the appropriate tier; mid-flight pivots are snapshotted and reliably restored; Consultant has formal goal-failure escalation authority; runaway feedback loops are impossible by construction.

Per goal-lens: **G1 Governance > G2 Quality > G6 Resilience > G3 Zero Cost > G8 Observability**.

## Architecture (the contract — both workstreams build to this)

### Event schema v2 — `~/.megingjord/incidents.jsonl`

```json
{
  "version": 2,
  "timestamp": "ISO-8601 UTC",
  "tier": 1,
  "trigger_role": "manager|collaborator|admin|consultant|system",
  "trigger_type": "pattern-recurrence|manual-pull|goal-failure|sensor-driven",
  "pattern_id": "string-or-null",
  "severity": "low|medium|high|critical",
  "evidence": ["url-or-file-ref", "..."],
  "ticket_ref": "#N-or-null",
  "epic_ref": "#N-or-null",
  "session_id": "string",
  "schema_compat": "v1-readers-must-ignore-fields-not-in-v1"
}
```

**Backward compatibility**: v1 readers (anneal-goal-sensor.js as it stands today) must continue to work. New fields are additive-only. The `version:2` field is the discriminator.

### Tier transitions

| From → To | Conditions |
|---|---|
| (none) → Tier 1 | Any role appends an event with `tier:1`. No threshold. Pure observation. |
| Tier 1 → Tier 2 | `severity ≥ medium` AND (recurrence ≥ 2 in 7-day window OR `trigger_type == manual-pull`) AND no active session-pivot AND no matching suppression entry |
| Tier 2 → Tier 3 | Authority: Consultant ONLY. Triggered when consultant rubric finds G1–G9 goal violation post-implementation. Effect: invokes Manager to reopen failed AC/ticket OR file new self-anneal Epic. |

### Pivot semantics (mid-flight Tier 2)

```
on_pivot_request:
  1. snapshot = {role, ticket_ref, baton_phase, last_event_id, timestamp}
  2. emit event:pivot-start with snapshot
  3. assume Manager role; run workflow-self-anneal protocol (max 3 doc proposals, max 50 step counter)
  4. file Manager ticket(s) per anneal output (docs to anneal scope; code goes to standard baton)
  5. emit event:pivot-end with files-created list
  6. assert role == snapshot.role; restore baton to snapshot.ticket_ref / snapshot.baton_phase
```

### Bounded-loop kill switches

- Max **1 active pivot** per session (single-flight)
- Max **3 pivots** per 24h per session (rate-limit)
- Max **5 anneal tickets** auto-filed per 7-day window per `pattern_id` (suppression-registry cooperation with #1220)
- Anneal protocol step counter: **abort with `decision: defer`** if >50 tool calls
- All kill-switch trips emit `event:kill-switch-trip` with reason — observable in dashboard

### Tier-router contract (for the new skill)

`anneal-trigger-router` skill receives a drift signal and returns a routing decision:

```json
{
  "tier": 1 | 2 | 3,
  "action": "log-only" | "request-pivot" | "request-consultant-escalation",
  "rationale": "...",
  "estimated_severity": "low|medium|high|critical",
  "kill_switch_trip": null | "rate-limit|step-counter|suppression"
}
```

## Acceptance Criteria

- [ ] **AC1 — Three-tier protocol codified** in `instructions/workflow-resilience.instructions.md`. Tier definitions, transition rules, authority matrix all explicit and testable.
- [ ] **AC2 — Any-role pull works end-to-end.** Collaborator/Admin/Consultant can append a Tier-1 event mid-implementation via `cross-team-anneal-pull` skill OR a `gh` CLI helper. Trigger phrases: `pull anneal`, `andon`, `drift anneal #N`, `report drift`.
- [ ] **AC3 — Severity classifier ships and matches contract.** Misclassification rate <5% on a 50-event golden-file fixture covering low/medium/high/critical examples.
- [ ] **AC4 — Tier-2 auto-ticket on threshold crossing.** When Tier-1 events accumulate to threshold, Tier-2 workflow auto-files a Manager ticket with proper labels, `MANAGER_HANDOFF`, and links to evidence events. Cooperates with #1220 suppression registry — does NOT re-file a suppressed pattern.
- [ ] **AC5 — Pivot snapshot + restore is atomic.** Golden-file fixtures: pivot-during-Collaborator-step → restore-to-Collaborator-step; pivot-during-Admin-merge → restore-to-Admin-merge. Pivot failure (e.g., kill-switch trip) preserves original baton state.
- [ ] **AC6 — Consultant goal-failure escalation.** Consultant skill, on rubric < threshold against any G1–G9, can invoke Manager to (a) reopen the failed AC/ticket, (b) file new self-anneal Epic. Both actions append `event:goal-failure-escalation` with the rubric evidence.
- [ ] **AC7 — Kill switches verifiably hold.** Synthetic load test: attempt 100 simultaneous pivot requests in one session → exactly 1 succeeds, 99 emit `event:kill-switch-trip` with `reason:single-flight`. Step-counter test: anneal protocol with deliberately-runaway logic aborts at step 50 with `decision: defer`.
- [ ] **AC8 — Event schema v2 is backward-compatible.** Existing v1 readers (`anneal-goal-sensor.js`, `anneal-review.js`) continue to work against v2 events without modification. v2-aware readers extract new fields. Add a contract test that lints both directions.
- [ ] **AC9 — Dashboard anneal queue panel.** Live view: per-tier event count (24h + 7d windows), kill-switch trip rate, pivot success/failure rate, top patterns by frequency. WCAG 4.5:1 contrast; <2s render.
- [ ] **AC10 — Wiki concept pages anchor the design.** New pages `[[distributed-self-anneal]]` and `[[andon-pull-protocol]]` plus updates to `[[self-annealing]]` reflect the three-tier model. Linked from `wiki/index.md`.
- [ ] **AC11 — Live verification end-to-end.** A real or synthetic drift event walks all three tiers: Collaborator pulls Andon (Tier 1) → recurrence + severity triggers Tier 2 auto-ticket → Consultant goal-failure rubric triggers Tier 3 reopen. Evidence: events.jsonl trail, ticket links, Manager-handoff comments at each step.
