# #2876 — Add MANAGER_HANDOFF first-edit ordering gate to pretool_guard.py

## Body

## Summary
Add a MANAGER_HANDOFF first-edit ordering gate to `pretool_guard.py` so that code edits are blocked if no MANAGER_HANDOFF artifact exists on the linked issue before the first file touch in a session.

## Problem
`pretool_guard.py` resolves `active_ticket` from branch name and blocks edits. However, it never checks if a MANAGER_HANDOFF artifact exists on the linked issue before the first file edit. A Collaborator can start editing files before the Manager has posted scope, violating the Manager→Collaborator baton order. Confirmed by AC-R1 research (#2871).

## Expected Outcome
On the first file edit in a session (detected by `code_touched == False`), pretool_guard.py checks whether the linked issue has a MANAGER_HANDOFF comment. If absent, the edit is blocked with a clear deny message. Subsequent edits in the same session are not rechecked.

## Acceptance Criteria
- AC-I1.1: New function `linked_issue_has_manager_handoff(cwd)` exists in `hooks/scripts/live_checks.py`, following the same pattern as `linked_issue_has_collab_handoff(cwd)`. Fail-open on gh CLI error.
- AC-I1.2: `pretool_guard.py` imports `linked_issue_has_manager_handoff` from `live_checks`.
- AC-I1.3: Guard in `pretool_guard.py` after `active_ticket` resolution, conditioned on `not flags.get("code_touched")`: if `active_ticket` is set and `linked_issue_has_manager_handoff(cwd)` returns False, emit deny with message referencing the missing MANAGER_HANDOFF.
- AC-I1.4: Unit test for `linked_issue_has_manager_handoff()` covering: present case, absent case, gh CLI error (fail-open).
- AC-I1.5: Integration test covering the deny path when code_touched is False and MANAGER_HANDOFF is absent.

## Implementation Entry Point
- `hooks/scripts/live_checks.py`: add `linked_issue_has_manager_handoff(cwd)` after `linked_issue_has_collab_handoff()`.
- `hooks/scripts/pretool_guard.py`: line 13 import update; guard at first-edit check (after active_ticket resolved, before block exit).
