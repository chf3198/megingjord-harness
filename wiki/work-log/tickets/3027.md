## Body

## C2 — Automated Tier-2 Ticket Promotion

Refs Epic #3026

### Problem
`incidents.jsonl` accumulates recurrence data but nothing reads it to auto-file a Manager ticket when a `pattern_id` crosses Tier-2 threshold (≥2 events in 7 days). This leaves ticket creation dependent on operator judgment, which has proven insufficient.

### Acceptance Criteria
- [ ] AC1: `scripts/global/anneal-ticket-promoter.js` reads `~/.megingjord/incidents.jsonl`, groups by `pattern_id` within a rolling 7-day window, and calls `gh issue create` when recurrence ≥ 2 and no open issue for that pattern already exists (idempotent — uses GH search before create)
- [ ] AC2: Script is invokable as `npm run anneal:promote` and also as a scheduled action (cron trigger in `.github/workflows/anneal-promote.yml`, daily)
- [ ] AC3: Filed tickets carry labels `type:task,priority:P1,anneal:tier-2` and body includes pattern_id, recurrence count, and linked incident timestamps
- [ ] AC4: `tests/anneal-ticket-promoter.spec.js` covers: promotion fires at threshold, skips when open ticket exists, skips stale events outside 7d window, handles empty/missing incidents file

### Out of Scope
- Automatic ticket close on resolution (manual only for now)
- Promotion to Tier-3 (Consultant authority only)
