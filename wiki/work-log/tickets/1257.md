# #1257 — AC3: wire 6 sensors into Goal Health Score

> **Source**: github:issue/1257 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:scripts, lane:code-change

## Body

Parent: #1113. Implements AC3 per Phase-0 R&D (#1247) §6.AC3.

## Scope

Wire 5 real sensors into Goal Health Score so it leaves the permanent-stale state. Each sensor is a pure-function module returning `{value: 0..1 | null, evidence: [...]}`.

The 6th sensor (`ga` governance-audit violations) is already wired as a placeholder in #1253; this child upgrades it to a real module too.

## Sensors to wire

| Key | Source | Failure-rate definition |
|---|---|---|
| `ga` | `/tmp/governance-audit.json` violation count | `min(1, weighted_count / 14)` |
| `ll` | `gh api workflows/label-lint runs` 7d | `failed_runs / total_runs` |
| `cf` | `gh search` for `CONSULTANT_CLOSEOUT` matching `goal-misorder` | `flagged / total_closeouts` |
| `pr` | PR review comments matching `goal-lens|goal-priority` | `flagged / total_PRs_with_review` |
| `rp` | Reopened issues with priority-cause closeout | `priority_reopens / total_reopens` |
| `oo` | `~/.megingjord/operator-flags.json` `goal_governance_failing` | binary 0/1 |

## Deliverables

- `scripts/global/sensors/ga.js`, `ll.js`, `cf.js`, `pr.js`, `rp.js`, `oo.js` (6 pure-function modules)
- `scripts/global/sensors/index.js` (aggregator: calls each, returns `{ga, ll, cf, pr, rp, oo}`)
- `scripts/global/governance-audit.js` updated: replace placeholder `sensorValues` with aggregator output
- `tests/sensors/{ga,ll,cf,pr,rp,oo}.spec.js` (one spec per sensor)

## Acceptance criteria

- [ ] Each sensor module exports `compute()` (or `compute(opts)` for testability)
- [ ] Each sensor handles null-data case (no data in window → null)
- [ ] Aggregator returns object compatible with `computeGHS({sensorValues})`
- [ ] Running `governance-audit.js` produces non-null GHS score (active sensors ≥ 0.5 weight floor)
- [ ] Each sensor has ≥3 tests covering: happy path, empty data, error path
- [ ] No live network calls in tests (use fixtures + mocks)
