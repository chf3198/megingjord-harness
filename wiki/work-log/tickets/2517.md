# #2517 — Enforce admin-merge bypass exception gate and batch-cancel reconciliation

> **Source**: github:issue/2517 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P1, role:manager, area:governance, resolution:released, lane:code-change

## Body

## Problem

Two structural enforcement gaps identified by cross-family adversarial red-team review (fleet: qwen2.5-coder:7b, 2026-05-31):

**Gap 1 — `--admin` merge bypass treated as routine (G1/G7 violation)**
Every PR in both reviewed sessions (#2504 #2505 #2506) was merged via `gh pr merge --squash --admin` with no BLOCKER_NOTE exception, no CI-green evidence, and no escalation comment. The governance contract requires a formal exception but no automated gate enforces it.

**Gap 2 — Batch-cancel without individual CANCELLATION comment (G1 violation)**
Tickets #2375–#2382 and #2199 (9 tickets) were closed as "not planned" in a batch with no individual `CANCELLATION: <reason>` comment per ticket, as required by `role-baton-routing.instructions.md` (`any → cancelled` transition).

## What is NOT covered by existing tickets

- #2511/#2510/#2192 cover cross-family signer-independence — not the --admin bypass exception gate
- #2265 covers no-code remediation eligibility — not the --admin bypass pattern
- No existing ticket requires BLOCKER_NOTE before `--admin` merge or validates individual CANCELLATION comments at batch-close time

## Acceptance Criteria

- **AC1**: `baton-gates.yml` or equivalent gate blocks `--admin`-merged PRs unless the linked issue carries an explicit `merge-bypass:admin-exception` label OR the PR body contains a `BLOCKER_NOTE` with `bypass_reason:` and `approver:` fields
- **AC2**: `consultant-closeout.js` validator emits blocking violation when any PR closes >1 issue and any of the closing issues lacks a `CANCELLATION: <reason>` comment from the Manager role before close
- **AC3**: Both validators ship with unit tests (tdd-pyramid) achieving ≥80% branch coverage
- **AC4**: `docs/howto/admin-merge-bypass.md` documents the exception procedure

## Red-team evidence

Session 1 (G1=50 REJECT) and Session 2 (G1=20 REJECT) from fleet adversarial review — both sessions failed on G1 partly due to these enforcement gaps.
