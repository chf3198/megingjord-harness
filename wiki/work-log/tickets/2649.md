## Objective
Research AC-R3 for Epic #1948 only: how to detect and prevent test drift when AI-agent-authored code changes but the test suite does not keep pace.

## Research questions

1. Which signals best detect test drift on changed code: mutation testing, coverage delta, diff-to-test locality, golden snapshot rot, or other methods?
2. Which signals are strong enough to gate PRs versus advisory only?
3. How should drift controls behave for AI-agent workflows where code and tests are often updated in the same turn?
