# #2005 — Fix false-positive post-merge warning in Stop hook after admin cycle complete

> **Source**: github:issue/2005 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:hooks, governance:close-without-merge
> Mirror of `gh issue view 2005` (derived; edit the GitHub item, not this page).

## Body

## Problem

The Stop hook (`stop_reminder.py`) emits a persistent "Warning from Stop hook" governance checklist after every agent response turn once any code has been committed in a session — even after all admin ops (commit, push, PR, merge, issue close) are complete.

## Root Cause (two distinct gaps)

### Gap 1 — `post_merge_messages()` has no completion guard
`stop_checks.py:post_merge_messages()` fires the post-merge checklist whenever `"code-changed"` is in signals. It does not check `ops.get("merge")` to see whether admin work is already done.

### Gap 2 — `detect_session_signals()` uses permanent git history, not session activity
`git_checks.py:detect_session_signals()` uses `git diff HEAD~1 HEAD` to detect code changes. This is a **permanent repo-state signal** — it reflects the last commit on the branch, not whether the current session is actively doing admin work. After a branch switch (post-merge reset to `main`), `reset_on_branch_change()` clears `ops` and `flags`, but `git diff HEAD~1 HEAD` still returns `.js` files from the previous merge commit. So the warning fires indefinitely on every new session after any code-changing PR is merged.

## Impact
- Warning appears after every response turn regardless of session state
- Creates noise that obscures genuine governance alerts
- Cannot be suppressed by completing the listed steps (ops state is reset on branch change)

## Acceptance Criteria

- [ ] AC1: `detect_session_signals()` only appends `"code-changed"` when `"recent-commits"` is also present (i.e., commits within the last 2 hours)
- [ ] AC2: `post_merge_messages()` accepts an `ops` parameter and suppresses the checklist when `ops.get("merge")` is True
- [ ] AC3: `stop_reminder.py` passes `ops` dict to `post_merge_messages()`
- [ ] AC4: Unit tests added for both new guards
- [ ] AC5: G5 Portability — fix applies equally when hooks are deployed for Claude Code and Codex (shared source via `hooks/scripts/`; both runtimes share identical deployed code)
- [ ] AC6: Deployed to `~/.copilot/hooks/scripts/` via `npm run deploy:apply`; warning no longer appears in a clean post-merge session

## Files to change
