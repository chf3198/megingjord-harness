# #2342 — Prevent Claude Code file-history from overloading VS Code file watchers

> **Source**: github:issue/2342 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P1, area:infra
> Mirror of `gh issue view 2342` (derived; edit the GitHub item, not this page).

## Body

## Problem

The repo-local `.claude/file-history/` directory accumulated 13,053 files (99 MB) during normal Claude Code usage, flooding VS Code inotify file watchers and preventing the Claude Code extension from starting.

Root causes:
1. `.claude/file-history/`, `.claude/sessions/`, `.claude/backups/`, `.claude/shell-snapshots/`, `.claude/ide/` are NOT in `.gitignore` — only in `.git/info/exclude` (local-only, not propagated)
2. `cleanupPeriodDays` is not set in `.claude/settings.json` (defaults to 30 days)
3. No VS Code `files.watcherExclude` for these directories
4. No periodic purge of the global `~/.claude/file-history/`

## Acceptance Criteria

- [ ] AC1: All Claude Code runtime cache dirs added to `.gitignore` (committed, applies to all operators/worktrees)
- [ ] AC2: `cleanupPeriodDays: 7` set in `.claude/settings.json`
- [ ] AC3: `.vscode/settings.json` `files.watcherExclude` and `files.exclude` cover `.claude/file-history/**` and sibling cache dirs
- [ ] AC4: `hooks/scripts/prune_global_file_history.py` SessionStart hook prunes `~/.claude/file-history/` entries older than 7 days
- [ ] AC5: `.git/info/exclude` entries superseded by `.gitignore` (document, optionally clean up)
