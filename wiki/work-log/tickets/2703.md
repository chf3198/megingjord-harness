# #2703 — Flaky CI: api-smoke-load retry-amplification 3000ms budget fails on slow runners

## Body

## Problem
`tests/api-smoke-load.spec.js:62` (#749 AC5) asserts a retry completes in `<3000ms`, but on current GitHub-hosted runners the retry backoff takes ~5000ms (observed 4999ms / 4980ms across two consecutive `quality-required` runs on PR #2701). The assertion is **wall-clock-timing-based** and deterministically fails when the runner is slow — independent of the PR under test.

## Evidence
- PR #2701 (credential-prompt-guard, #2569) — diff does NOT touch any API/retry surface; `quality-required` failed twice solely on this test.
- Error: `retry took 4999ms` / `Expected: < 3000`.
- Last functional change to the file: #746 (2a643b4a).

## Root-cause framing (Manager rescope)
The defect is a **class**, not a single line. `api-smoke-load.spec.js` carries three wall-clock `toBeLessThan(ms)` assertions. Two use wall-clock as an *accidental proxy for behavior* and are environment-fragile; one is a *legitimate latency metric* under test and must be preserved as a real perf gate:

| Line | AC | Asserts | Wall-clock role | Remediation |
|---|---|---|---|---|
| L62 | AC5 | retry recovers `<3000ms` | accidental proxy for "does retry recover?" | convert to behavioral (attempt-count / recovery) |
| L43 | AC4 | render not blocked `<3000ms` | accidental proxy for "render does not await slow API" | convert to behavioral (visibility/ordering) |
| L32 | AC2/AC3 | p95 cache-miss latency `<2000ms` | **legitimate** metric under test | preserve as perf gate; CI-calibrate + variance-tolerate |

Per harness guidance: root-cause over detection-only band-aids; **no calendar/timing thresholds as behavioral proxies**. The "raise the budget" stopgap from the original draft is explicitly **rejected** — raising a wall-clock budget re-arms the same flake on the next slow runner.

## Acceptance Criteria
- **AC1 — Root-cause (L62/AC5):** Replace `expect(result.elapsed).toBeLessThan(3000)` with a deterministic behavioral assertion. Drive the retry path with Playwright route interception so attempt-1 fails and a later attempt (≤3) succeeds; assert `result.ok === true` AND bounded `attempt` (≤3). No measured-elapsed gating; no raised-budget stopgap.
- **AC2 — Sibling defect class (L43/AC4):** Convert the render-not-blocked assertion from wall-clock `<3000ms` to a behavioral assertion: `header` is visible while the routed `/api/wiki-health` is still delayed (render does not await the slow API). Assert visibility/ordering, not elapsed ms.
- **AC3 — Preserve legitimate perf gate (L32/AC2-AC3):** Keep the p95 cache-miss latency assertion as a measured budget (it IS the metric under test), but make it CI-robust: recalibrate to a defensible CI-safe budget with an inline rationale comment, and tolerate runner variance (p95 over a sufficient sample; documented headroom) so it does not flake on slow runners. Document why this one stays measured while AC1/AC2 do not.
- **AC4 — Anti-regression invariant:** Add an in-file documented invariant block distinguishing *behavioral* assertions (never gate on wall-clock elapsed) from *latency-metric* assertions (measured budget, CI-calibrated), so future edits do not reintroduce an L62-class flake.
- **AC5 — Cross-runner determinism evidence:** Show the retry + render tests pass deterministically under injected slow-runner conditions (route-level delay) — behavior holds regardless of wall-clock. Cite local run output in COLLABORATOR_HANDOFF.
- **AC6 — Governance normalization:** Treated as an independent ticket (parent Epic #749 is closed/terminal); progress `backlog→triage→ready→in-progress→testing→review→done`; project board + full label set applied; `test_strategy: tdd-pyramid`.
