## Body

## Problem
Fixing the cross-framework test pollution in #3166 (Playwright was collecting `node:test` specs, producing the `forEach`-on-undefined cluster) makes the `node:test` specs *actually run* under the new split-test-runner (`npm test`). This surfaced ~19 **pre-existing** baseline failures that were previously masked inside the global pollution. They are confirmed failing on clean `main` (e.g. `node --test tests/timeout-policy.spec.js` → 3 fail; `tests/cross-team-contract-check.spec.js` → expected 5 / actual 6), so they are NOT regressions from #3166 and were out of its scope.

Nothing in CI or the git hooks runs bare `npm test` (CI uses explicit `npx playwright test <files>`; pre-push runs `closeout-preflight.js`), so these do not block pushes or redden CI today — but `npm test` is now honestly red and they should be triaged/quarantined.

## Surfaced failing tests (pre-existing baseline)
- `cross-team-contract-check.spec.js` — "entry-point list covers exactly 5 runtimes" (stale: Cursor added → 6; see cursor-adapter.spec.js)
- `timeout-policy.spec.js` — getTimeout 32b/1200s budget + lane multipliers (3 fail; config drift)
- `github-native-client.spec.js` / substrate selection — GitHub-native mode / HAMR mode / readMailbox / dispatchMcp / fetchBundle / readTelemetry (env-dependent: `MEGINGJORD_HAMR_ENABLED`)
- `governance-rule-parity.spec.js` — rule parity drift classes / blocking introduced conflict (pr mode)
- golden-file workflow specs — "golden-file byte-equality: workflow matches fixture snapshot" (x2), "workflow still triggers on same pull_request paths", "workflow paths include all cross-runtime config surfaces" (fixture drift)
- multi-judge specs — "buildPrompt: caps long input", "loadRubric returns g1-g9-v2 with all 9 goals"
- stress — "recordAccess p99 stays under 5ms without contention (AC3)" (perf flake under parallel load)

## Objective
Triage each surfaced failure: (a) env-gate the network/HAMR-dependent specs so they skip cleanly offline, (b) refresh stale golden-files + config assertions (runtime count 5→6, timeout-policy budgets), (c) quarantine/stabilize the perf p99 flake. Goal: `npm test` (split-runner) deterministically green offline.

## Goal lens
G2 Quality, G6 Resilience (offline determinism), G8 Observability.
