## Objective

Implement the first Phase-1 deliverable from #2295: a deterministic parity validator that compares governance rule statements across documentation/template surfaces and enforcement surfaces, then reports contradictions with severity-aware blocking behavior.

## Scope

- Build a parity validator module that extracts structured rules from selected doc surfaces and gate surfaces.
- Compare equivalent rule IDs and classify conflicts: doc-vs-enforcement, enforcement-vs-enforcement, enum drift, doc-vs-no-enforcement, carve-out.
- Emit PR-time findings and nightly-scan findings with machine-readable conflict records.
- Block PRs only for newly-introduced hard conflicts; keep pre-existing conflicts advisory.

## Acceptance criteria

- [ ] AC1: Validator produces structured conflict output with conflict class, rule id, source citations, and severity.
- [ ] AC2: PR-time gate fails on newly introduced hard conflicts and remains advisory for pre-existing conflicts.
- [ ] AC3: Nightly run emits drift report without duplicating already-open drift issues.
- [ ] AC4: Closes-vs-Refs origin conflict is represented as an explicit conflict record (reconciled or exception-tagged).
- [ ] AC5: Includes fixtures/tests for at least: lane-enum drift, merge-evidence linkage drift, lightweight-set drift.
- [ ] AC6: Includes carve-out handling so approved exceptions do not produce false hard failures.

## Constraints

- Keep implementation minimally invasive to existing megalint gates.
- Use a single canonical conflict schema shared by PR-time and nightly outputs.
- Preserve backward compatibility for existing lightweight lane bypass behavior.
