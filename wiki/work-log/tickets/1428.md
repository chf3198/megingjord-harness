## Summary
Conduct Phase-0 R&D to baseline token spend across orchestrating services, identify the highest-impact waste vectors, and produce an implementation-ready design package for Phase-A/B/C tickets.

## Scope
- Instrument session logs to measure tokens consumed per service per category (context injection, governance output, telemetry, prompt boilerplate)
- Classify waste vectors by type: redundant, compressible, cacheable, scopeable
- Evaluate existing harness tooling (constitution-compressor.js, prompt-reduction-playbook.md, ide-proxy-quality-parity.js, cost-telemetry.js) for gaps
- Survey state-of-the-art context compression and prompt scoping patterns applicable to the Megingjord architecture
- Define measurable success criteria (e.g., target % token reduction per session type)
- Design control options with engineering complexity and risk estimates

## Required Outputs
- Token waste taxonomy with severity and frequency estimates (at least 6 categories)
- Baseline measurement methodology (how to instrument + what to measure)
- Control matrix: preventive (context scoping), detective (waste sensor), corrective (post-session compressor)
- Gap analysis against existing harness tooling
- Proposed Phase-A/B/C implementation tickets with explicit ACs
- Epic #1427 updated with R&D conclusions and delivery sequence

## Acceptance Criteria
- R&D findings documented and actionable
- At least 6 concrete waste vectors identified with baseline estimates
- Controls mapped to harness enforcement points (scripts/global/, hooks/, instructions/)
- Phase-A/B/C tickets defined with measurable ACs
- Epic #1427 body updated with delivery sequence
- Team&Model signing included
