## Problem

The harness correctly uses isolated git worktrees for parallel agent work, but recent diagnosis found stale worktrees 59-63 commits behind `main` plus one detached temporary checkout. This points to a lifecycle governance gap: creation and detection exist, but retirement, refresh, and rescue are not yet enforced strongly enough.
