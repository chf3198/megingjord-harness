## Body

## Problem

The harness has a **mature but fragmented** cross-model review ecosystem (cross-family-review, role-red-team-critique, research-redteam-loop, fleet-red-team-dispatch, cascade-dispatch, free-cloud-dispatch, fleet-decision-oracle, governance-bundle, signer-fidelity, fleet-review-required, red-team-model-matrix, review-score-contract, and Epics #2192/#2041/#2511/#2926/#2791/#2094/#2709). A live 7-round consensus run (Epic #3063) exposed concrete inefficiencies that this Epic optimizes and consolidates — it does NOT rebuild what already works.

## Evidence (from the #3063 consensus run + 2026 web research)

1. **Model suite is stale / not diverse on-host**: 36gbwinresource runs qwen2.5-coder (not the 2026-best Qwen3-Coder-32B) and is all-Qwen + one small Llama; no GLM/DeepSeek locally for jury family-diversity. 3b/7b models are too weak to be final judges (falsely-inflated early scores). Second host (<IPV4_REDACTED>) was down during the run.
2. **Reviewers run under-grounded**: dispatch passes prompt templates (fleet-red-team-prompts.json) but NO source/wiki grounding and an under-specified rubric — research shows this causes inconsistent strictness + overconfidence (observed: scores bouncing 87-90).
3. **The premium orchestrator does the expensive work**: in #3063 the orchestrator (Opus) hand-authored 7 design revisions; only rating was offloaded. Generation should be fleet-offloaded; peer paid teams (Copilot/Codex/Antigravity) must never be a review pool.
4. **Reviewers are stateless across iterations**: round-3 judges did not see round-1 critiques, so resolved points were re-litigated (a rebutted "use Makefiles" critique reappeared). Need a Reflexion-style critique+resolution LEDGER passed as context while the reviewer IDENTITY stays fresh (preserving the no-in-family-coaching invariant).
5. **No anti-reward-hacking / scope guard**: iterative self-refinement can inflate judge scores while true quality drops (arXiv 2407.04549); the #3063 score jump came partly from softening the rubric + adding scope text, and the panel kept requesting out-of-scope gold-plating (verbosity/scope creep).

## Objective

Consolidate the fragmented review surface and apply five evidence-based optimizations (model suite, grounding, generator-offload, iteration memory, anti-reward-hacking/scope-guard) so cross-model review is **diverse, grounded, cheap-on-the-orchestrator, memory-aware, and resistant to gaming** — and so the goalpost is a well-calibrated `median(capable cross-family jury) >= 92` against a FROZEN rubric.
