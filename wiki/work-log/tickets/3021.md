# #3021 — Audit and harden the full parallel-team coordination surface (identity, worktree, git, logging, baton)

## Body

## Epic Summary

Budget-driven **Copilot Auto mode** (cheaper/alternate models) has exposed that the harness's **parallel-team coordination surface** is fragile: when two or more AI teams (Claude Code, Copilot, Codex, …) work the same board, defects in identity/signing, worktree isolation, git operations, logging attribution, and baton commenting can **structurally block or silently corrupt** the workflow. The trigger instance was a signer mismatch (#2945 parked 4 days), but the user has scoped this Epic to a **deep, systemic audit-and-harden of the entire parallel-team coordination surface**, not a single symptom.

**Trigger instance:** #2945 — Copilot Auto-mode artifacts signed `Quill`/`Quinn Harper` (codex/qwen seeds) while declaring `Team&Model: copilot:claude-sonnet-*` (registry-correct: `Soren Harper`). `admin-gate` `analyzeComments` scans ALL comments → hard-fails → structural park. The completing other-team operator cannot fix without identity forgery (OWASP OA3). Incident pattern `copilot-automode-model-family-signer-mismatch` (high). Child defect #3020.

## Goal

The harness must let **multiple AI teams work the same board concurrently without structural deadlock, silent corruption, or forced governance bypass** — while preserving signer-integrity, isolation, and auditability (G1 non-negotiable). Defects must degrade gracefully and surface actionable remediation, never permanent parks.

## Scope — coordination surfaces to audit (deep analysis, Phase 0)

1. **Identity & signing** — runtime `Team&Model` vs actual routed model family under Auto mode; aliasSeed↔derivation reconciliation; `analyzeComments` scanning ALL comments; non-forgery completing-operator path.
2. **Worktree isolation** — one-ticket-per-worktree; concurrent-writer detection; shared `node_modules` symlinks; canonical-main read-only enforcement under cross-team absolute-path writes; stale-worktree cleanup.
3. **Git operations** — force-push denial on cross-team branches; REST-merge detaching main label with worktrees present; stale-base branching; squash-merge detection; rebase discipline; concurrent commits to the same ticket.
4. **Logging & observability attribution** — per-team/per-role attribution in `events.jsonl`/`incidents.jsonl` (`trigger_role`); cross-team event collisions; redaction; goal-coverage per team.
5. **Baton commenting & artifact integrity** — bold-markdown/prose-collision parse traps; last-vs-all comment selection; stale-artifact accumulation; cross-team artifact-write sign-off; multi-team comment ordering races.
6. **Lease / claim coordination** — cross-team-lease, merge-claim, fleet-claim; first-claim-wins atomicity; pre-push blocking gates; GitHub-native vs HAMR substrate (Tier-1 fallback).
7. **Role/baton routing across teams** — cross-team baton pickup; signer-independence across teams; one-active-role invariant under concurrency.
