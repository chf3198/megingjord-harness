## Problem
GOV-009 (Engineering Design Doc Before Implementation) exists as a governance control in `instructions/governance-controls.instructions.md` but has no CI enforcement (Rule 3.14). Implementation tickets can proceed without an EDD artifact. The control is instruction-only.

## Acceptance Criteria
- [ ] Create `.github/workflows/edd-required.yml`: on PR open/edit, check for EDD artifact in linked issue comments or PR body
- [ ] EDD artifact format: at minimum `scope:`, `acceptance:`, `risk:`, `implementation-plan:` fields
- [ ] Add `edd-required` as a required status check in branch protection / ruleset
- [ ] Exempt `lane:trivial` and `lane:config-only` tickets (no EDD required for simple config changes)
- [ ] Add tests: `tests/edd-gate.spec.js`
