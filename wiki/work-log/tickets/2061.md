## Gap analysis (what exists vs what is missing)

### What EXISTS today

`instructions/epic-governance.instructions.md`:
- "Research-First Epic Phase Gate" section (clauses 1-5): Phase-0 closeout requirements, EPIC_RESCOPE requirement, Phase-1 source-citation, re-arm on reopen
- "Epic Progress Comment Protocol" section: template for "Epic Progress Update -- #N Complete" comment on any child close

`.claude/commands/role-manager-execution.md`:
- Generic Manager scope-and-handoff protocol for a single ticket

`scripts/global/baton-comment-build.js`:
- MANAGER_HANDOFF template builder (single-ticket scope)

### What is MISSING (the gap)

1. **No formal Manager-post-research-child checklist** — the explicit step-by-step actions Manager takes after a Phase-0 child closes (translate plan into Phase-1 ACs, file children with v2 amendments from any cross-family rater, advance Epic labels, build EPIC_RESCOPE template, etc.).
2. **No "Manager Epic contemplation cadence" guidance** — operator stated: "The completion of child-tickets through their own, complete Agile role workflow should probably always trigger Manager contemplation of updates to the parent ticket (Epic)." Currently implicit; no automation prompts Manager to revisit Epic state on each child close.
3. **No formal "Phase-0 plan to Phase-1 AC translation" pattern** — Manager currently invents the translation (per the Epic #1942 EPIC_RESCOPE just posted, which authored 10 child ACs from the Plan v3 child slate manually). Could be templated.
4. **No auto-comment on Epic when child closes** suggesting Manager contemplation (similar to how `EPIC_AUTO_PAUSE` auto-comments on inactivity).
5. **No formal handling of "cross-family rater AGREED-A+" as Phase-0 phase-gate evidence** — currently a memory + per-Epic interpretation. Should be codified.

- [ ] AC-R3: Design formal Manager-post-research-child checklist. Concrete step-by-step the Manager runs when a Phase-0 child closes. Include: read closeout rubric; check AC-R8 (cross-family rater) status; post EPIC_RESCOPE with template; translate Phase-0 plan into Phase-1 child ACs; file children with phase-gate:phase-1; advance Epic labels.
- [ ] AC-R4: Design "Epic contemplation cadence on child close" automation. GitHub Action that posts a "Manager contemplation needed" advisory comment on Epic when ANY child closes, with template that prompts Manager to consider: (a) does Epic AC require ticking; (b) are new follow-on children warranted; (c) does Epic scope need rescope.
- [ ] AC-R5: Design "Phase-0 plan → Phase-1 AC translation" pattern. Whether translation lives in (a) updated Epic body, (b) per-child AC author-time, (c) a synthesis-to-AC programmatic map.
