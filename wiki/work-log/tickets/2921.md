## Body

## Problem
No governance adversarial regression suite exists (Gap G-13). Current tests cover functional behavior but do not actively attempt to bypass governance rules. As guardrails are added, there is no automated test that tries known bypass patterns and verifies the guardrail catches them. This is the testing equivalent of a security red-team for the harness itself.

## Acceptance Criteria
- [ ] Create `tests/governance-adversarial.spec.js`: test file with ≥15 adversarial scenarios
- [ ] Scenarios include (non-exhaustive): fabricated cross_family_rating accepted, COLLABORATOR_HANDOFF without cross-family receipt accepted, lane:trivial on 500-line diff accepted, CONSULTANT_CLOSEOUT with G1=2 + verdict:approve accepted, prompt injection in fetched content passed through, memory write without watchdog fired
- [ ] Each scenario: setup (bypass attempt) → assert (guardrail fires, deny returned)
- [ ] Add to main CI test suite
- [ ] Document adversarial test catalog in `research/governance-adversarial-scenarios.md`
