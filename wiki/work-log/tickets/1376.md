# #1376 — Enforce Tier-3 event emission on CONSULTANT_CLOSEOUT with goal-shortfall scores

> **Source**: github:issue/1376 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P1, area:governance, lane:code-change
> Mirror of `gh issue view 1376` (derived; edit the GitHub item, not this page).

## Body

## Problem

Epic #1339 close (2026-05-11) surfaced a latent gap in the Tier-3 self-anneal protocol shipped by Epic #1308 (#1311 + #1314):

The Consultant role on Epic #1339 performed a rubric review against G1-G9 but no `event:goal-failure-escalation` records were emitted to `~/.megingjord/incidents.jsonl` (zero records reference #1339). Tier-2 auto-file workflow (#1314) is event-driven, so even if scores HAD been sub-threshold no remediation tickets would have auto-filed. Operator caught this manually.

This is a single-point-of-failure: the protocol relies on Consultant discipline. The skill (`role-consultant-critique.md`) mentions Tier-3 emission as a "may invoke" option; nothing makes it blocking.

## Goal

Close the enforcement gap: when a CONSULTANT_CLOSEOUT comment names goal scores below 7/10, the closeout-lint workflow must require corresponding `event:goal-failure-escalation` entries before the ticket can close.

## Linkage rule (AC2 design)

A `event:goal-failure-escalation` event is "corresponding" to a sub-threshold goal score iff:
1. Event has `ticket_ref` matching the issue number being closed
2. Event has `evidence` array containing string matching `goal:G<n>` (where `<n>` is the sub-threshold goal number)
3. Event has `tier: 3` and `trigger_role: consultant`

Per `scripts/global/anneal-event-schema.js` v2 schema.

## Acceptance Criteria (refined)

- [x] **AC1 — Score extractor**: Extend `scripts/global/megalint/consultant-closeout.js` (reuse, don't duplicate) with `extractGoalScores(body)` that returns `[{ goal: 'G7', score: 6 }, ...]`. Format-aware: matches `G<n>=<score>`, `G<n>: <score>/10`, `G<n>: <score> —`.
- [x] **AC2 — Threshold check**: Add `checkGoalFailureEvents(scores, ticketRef, incidentsPath)` that verifies for each sub-7 score, a corresponding event exists per the linkage rule above.
- [x] **AC3 — Closeout-lint wiring**: existing `.github/workflows/closeout-lint.yml` (per #1421) already dispatches to megalint consultant-closeout validator. Add the new check inline; no new workflow needed. Sub-threshold-without-event fails the gate.
- [x] **AC4 — Skill enforcement**: `.claude/commands/role-consultant-critique.md` — promote Tier-3 emission language from "may invoke" to "MUST emit before close" when any G<n> < 7. Include a concrete `event:` JSONL template snippet.
- [x] **AC5 — Synthetic verification**: Add fixtures: (a) closeout with `G7=6` + no event → validator fails. (b) Same closeout + matching event → validator passes. (No live #1339 verification — closeout doesn't actually contain sub-7 scores; verification would be vacuous.)
- [x] **AC6 — Golden-file tests**: `tests/megalint/goal-failure-emission.spec.js` covering score extraction, linkage rule matching, and the synthetic fixture pair.

## Test strategy

`tdd-pyramid` (parser unit tests) + `golden-file` (fixtures).

## Out of scope

- Changing the Tier-3 authority (Consultant-only; remains)
- Auto-emitting events from prose scraping (anti-pattern; would re-introduce false positives)
- Changing the 7-point threshold
