## Scope

Add a non-destructive classifier that reports lifecycle state for each git worktree in the repository.

## Acceptance Criteria

- [x] Reports each worktree with path, branch/ref, HEAD, linked ticket when inferable, dirty state, untracked state, unpushed commits, locked state, detached state, missing-directory/prunable state, behind/ahead counts, and last activity signal where available.
- [x] Classifies worktrees into active, ready/parked, stale-warning, stale-safe, stale-risky, abandoned, detached-temp, prunable-metadata, or rescue-needed.
- [x] Treats `behind main` as warning evidence only, never as sufficient safe-removal evidence.
- [x] Emits JSON suitable for governance checks and human-readable summary output.
- [x] Includes focused tests for clean, dirty, untracked, unpushed, detached, locked, prunable, and missing-ticket cases.
