# #1580 — feat: harden collaborator-self-check from heuristic patterns to schema-validated baton artifact checks

> **Source**: github:issue/1580 | **state: OPEN** | **Labels**: type:story, status:backlog, priority:P3
> Mirror of `gh issue view 1580` (derived; edit the GitHub item, not this page).

## Body

## Context

Opened as follow-up from CONSULTANT_CLOSEOUT on #1571 (merged PR #1579).

The collaborator-self-check rules (G5/G6/G9 gap) currently use regex heuristics to inspect baton artifact text (e.g., `COLLABORATOR_HANDOFF` body pattern matching for AC ticks, flaw-marker citations, etc.). This is functional but brittle compared to schema-validated structured fields.

## Problem

- `allAcceptanceCriteriaTicked` — counts `[x]` checkboxes in the handoff body; fails silently if the AC section is absent or uses a non-standard format
- `flawMarkerCitations` — regex scan within ±2 lines; not tied to the baton schema contract
- `noProseColonCollision` / `noMarkdownBoldOnTestStrategy` — text pattern checks that could produce false positives on legitimate prose

## Acceptance Criteria

- [ ] AC-1: Define a machine-readable schema for COLLABORATOR_HANDOFF structured fields (AC table, pre-handoff section, test-strategy value)
- [ ] AC-2: Update `collaborator-self-check-rules.js` to validate against schema fields rather than raw text patterns where a schema field is available
- [ ] AC-3: Maintain full backward compatibility — waiver label `collaborator-self-check:waived` must still silence all checks
- [ ] AC-4: 15+ tests; all existing tests must continue to pass

## Notes

- Low urgency (P3); current heuristics are passing and catching real issues
- Do not regress the 15/15 test baseline from #1571
- Refs #1571, Epic #1568
