# #2264 — Define no-code remediation lane contract

## Body

Refs Epic #2258

Problem:
The baton contract does not currently define a dedicated lane for issue-only ticket-drift remediation. Operators are left to improvise role skipping, which is governance-fragile.

Objective:
Extend the baton/instruction contract with an explicit no-code remediation lane that routes Manager -> Consultant only.

Acceptance criteria:
- Define the lane name, intended scope, and role path.
- Specify exact eligibility conditions and exact exclusions.
- Specify required Manager and Consultant artifacts for the lane.
- Specify how status transitions differ from the default path.
- Preserve compatibility with existing lanes and validator enums.
