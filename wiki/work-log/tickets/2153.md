## Body

## Scope (Phase-1 child of Epic #2148)

Build a deterministic markdown-wikilink graph-builder that scans the repo for `[[wikilink]]` syntax and produces a JSON adjacency-list at `generated/doc-graph.json`. Re-runnable in <5 seconds.

Refs Epic #2148
Refs #2149 (Phase-0 source)

## Why agent-speed-appropriate

Round-5 of Phase-0 research rejected tree-sitter / Graphify / FalkorDB as over-engineering at AI-agent speed. A 200-LoC markdown scanner that rebuilds deterministically beats a stateful graph database because agents can rebuild from source in seconds when they need a fresh view.

## Scan surfaces

- `docs/**/*.md`
- `wiki/**/*.md`
- `README.md` + `vscode-extension/README.md`
- `instructions/**/*.md`
- `CHANGELOG.md`

## Output schema

```json
{
  "ts": "2026-MM-DDTHH:MM:SSZ",
  "version": 1,
  "nodes": [{"id": "file:path/to/foo.md", "type": "doc"}, ...],
  "edges": [{"from": "file:a.md", "to": "wikilink:bar", "kind": "wikilink"}, ...]
}
```

## Acceptance criteria

- AC1: `scripts/global/doc-graph-builder.js` exists and accepts no arguments (reads from CWD)
- AC2: Output written to `generated/doc-graph.json` deterministically (same input ⇒ same output bytes)
- AC3: Runs in <5s on current repo size (assert via `time` in test)
- AC4: `npm run doc-graph` script alias added to `package.json`
- AC5: Unit test in `tests/doc-graph-builder.spec.js` covering wikilink-parse edge cases (escaped brackets, nested links, no-link files)
- AC6: Lint clean (`npm run lint`)

## Test strategy

tdd-pyramid — pure function over file-system input; deterministic; unit-testable.

## Effort budget (agent-speed metrics)

- Token cost: <2K incremental per cycle
- Clock minutes: <1 min added to baton (script runs in seconds)
- Spurious-reject rate: <1% (deterministic output)
