# #2904 — Add comment integrity receipt token to prevent cross-family review fabrication

## Body

## Problem
The cross-family review fabrication gap (G-02): `cross_family_rating: 97` in a COLLABORATOR_HANDOFF is accepted by the CI gate even if no fleet call was ever made. The field is presence-checked (H*) not semantically verified (H). A baton artifact can be fabricated without any actual cross-family review occurring.

Related: comment content integrity (G-01) — a baton artifact COLLABORATOR_HANDOFF can be edited after the timing gate passes with no content hash on record.

## Acceptance Criteria
- [ ] Design a fleet-call receipt token mechanism: when `collaborator:preflight` runs, it embeds a receipt (e.g. `cross_family_receipt: <hash-of-model-response>`) in the artifact
- [ ] Update `scripts/global/megalint/collaborator-handoff.js` to validate receipt presence AND format, not just field presence
- [ ] Add a receipt-format regression test in `tests/`
- [ ] Document H* → H promotion for rules 2.3/2.4 in the guardrail assessment
