# #2895 — Remove || true from advisory-silent lefthook hooks

## Body

## Objective (Sprint 0 of Epic #2891)

Remove `|| true` from 4 lefthook hooks that are currently silently absorbing failures, providing zero user-visible feedback. These hooks were designed to emit advisory output but `|| true` causes their exit codes and output to be discarded.

## Changes

`lefthook.yml` lines 17, 21, 23, 37:
- Line 17: `npm run lint:js || true` → `npm run lint:js` (restore visible JS lint output)
- Line 21: `npm run lint:py || true` → `npm run lint:py` (restore visible Python lint output)
- Line 23: `npm run lint:sh || true` → `npm run lint:sh` (restore visible shell lint output)
- Line 37: `node scripts/global/git-freshness-check.js || true` → `node scripts/global/git-freshness-check.js` (restore freshness blocking per collaborator-rebase-discipline.instructions.md)

## Why no soak period

These are not new rules — the underlying scripts and instructions already specify the blocking/advisory behavior. The `|| true` was suppressing *existing* intent. Removing it restores the behavior described in the instructions. This is a correction, not a promotion.

Exception: if a lint step consistently fails on a clean repo due to environment variance, the operator may re-add `|| true` with a `BLOCKER_NOTE` explaining the specific environmental issue.

## Acceptance criteria

- [ ] All 4 `|| true` removed from lefthook.yml
- [ ] `npm run lint` still passes (100-line rule unchanged)
- [ ] No regressions in CI gates
