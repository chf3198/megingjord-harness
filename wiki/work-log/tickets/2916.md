## Body

## Problem
Cross-team lease creation (`CROSS_TEAM_LEASE_CREATE`) before editing shared files is currently advisory (Rule 5.1 from the guardrail assessment). Agents can edit cross-team paths without claiming a lease, leading to concurrent-edit conflicts and broken governance handoffs.

## Acceptance Criteria
- [ ] Add `cross-team-lease-check` as a required step in `lefthook.yml` pre-push
- [ ] `scripts/global/cross-team-lease.js` should exit 1 on pre-push if editing cross-team paths without an active lease
- [ ] Cross-team path definitions in `config/governance-rules.yaml`
- [ ] Add test: pre-push to cross-team path without lease → blocked
- [ ] Preserve MEGINGJORD_IT_OPS=1 bypass for IT operations
