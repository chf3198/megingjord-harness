## Body

## Scope

Extract `test_strategy` enum to a single source-of-truth module so the matrix doc, validator, and PR template stop drifting.

## Current duplication

- `instructions/test-methodology-matrix.instructions.md` — strategy enum in prose
- `scripts/global/test-evidence-validator.js` — `ALLOWED_STRATEGIES` array
- `.github/PULL_REQUEST_TEMPLATE.md` — strategy enum in template comment

If a future change adds a strategy (e.g. `mutation-testing`), all three need editing or the validator rejects it as `unknown-strategy`.

## Proposed solution

- Create `scripts/global/test-strategy-enum.js`:
  ```js
  module.exports = {
    ALLOWED_STRATEGIES: ['tdd-pyramid', ...],
    NONE_PERMITTED_LANES: [...],
    PEER_REVIEW_RUBRIC_THRESHOLD: 7,
  };
  ```
- Validator imports from this module instead of declaring inline
- A pre-commit hook verifies the matrix doc's enum table matches the module's array (regex extraction)
- PR template stays as a hand-maintained nudge (acceptable drift risk for a UI hint)

## Acceptance criteria

- [ ] Single-source module exists
- [ ] Validator imports from it
- [ ] Drift-detection test verifies matrix doc enum table = module array
- [ ] No regression on `test-evidence` gate (validator behavior unchanged)
