# #1874 — Copilot Team pauses before full task completion - requires explicit user finish signal

> **Source**: github:issue/1874 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:governance, resolution:completed, lane:code-change
> Mirror of `gh issue view 1874` (derived; edit the GitHub item, not this page).

## Body

## Problem
Copilot Agent exhibits pattern of stopping at implementation/planning phases and awaiting explicit user direction to proceed with testing, release, and closure. This prevents full baton workflow execution and requires user to manually signal "complete the Epic" rather than inferring completion responsibility from task scope.

## Expected Behavior
When assigned an Epic or feature ticket with "complete" intent, agent should:
1. Execute full Manager → Collaborator → Admin → Consultant → Closure workflow
2. Not pause between baton phases
3. Ship code, tests, CI green, reviews, and ticket closure as single deliverable
4. Only escalate to user for UAT sign-off or blocking decisions

## Actual Behavior
Agent completes implementation, validates locally, then stops awaiting "admin phase" or "run tests" instructions. Requires explicit user signal to advance through baton phases.

## Impact
- Adds friction to feature delivery workflow
- Requires user context-switching to manage agent progression
- Delays release of validated work

## Suggested Fix
- Retrain agent on "completion" semantic (means ship, not implement)
- Establish default baton-phase routing without pause
- Add timer/auto-escalation if phase gates take >30min

## Acceptance Criteria
- Copilot executes full baton workflow without pause when "complete" signal given
- All governance gates pass (ci_green, consultant review, closure)
- Delivery time reduced to single session
