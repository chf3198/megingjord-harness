## Problem

Epic #2488 (Tier-1 GitHub-native Layer-2 primitives) shipped 8 PRs with `[skip-doc-gate]` applied to AC1–AC7 and a single how-to guide (AC8). The following documentation surfaces were not updated and are now stale or missing content:

1. **`ARCHITECTURE.md`** — two-tier model section describes Global/Workspace layers but says nothing about the HAMR vs GitHub-native Layer-2 coordination distinction. A reader cannot learn from this file that Cloudflare is optional.
2. **`README.md`** — "How it works — two-tier model" and "Quick start" sections have no mention of the zero-Cloudflare operating mode or the `MEGINGJORD_HAMR_ENABLED` toggle.
3. **`wiki/wisdom/global/concepts/mailbox.md`** — describes only the HAMR R2 mailbox. The new `github-mailbox.js` GitHub Issues-based fallback is not referenced.
4. **`wiki/wisdom/global/concepts/hamr-failover-map.md`** — still a stub. Epic #2488 is exactly the content this doc should capture: GitHub-native as Tier-1 fallback before HAMR degraded modes.
5. **`docs/provider-neutral-routing.md`** — no mention of the GitHub-native routing path.
6. **Missing: `wiki/wisdom/global/concepts/github-native-layer2.md`** — no wisdom-layer entry exists for the new primitives (how-to guide is in docs/howto/, not wiki).

## Root Cause

Collaborator Handoffs for AC1–AC7 declared README N/A as "internal scripts; no user-visible change." This was incorrect: the zero-Cloudflare operating mode is a user-visible architectural change that affects onboarding, quick-start, and architectural understanding. Wiki and design doc surfaces were not assessed at all.
