## Objective

Phase-0 research for Epic #1112 AC-R1. Investigate the right architecture for the prompt dispatcher that initiates cross-team R&D synthesis runs.

## Research questions (from #1112 body)

- Should the prompt-dispatcher be operator-driven (current pattern: human pastes prompt into each session) or harness-driven (an agent that knows how to invoke each team)?
- Should the dispatcher integrate with VS Code, Codex CLI, Copilot Chat, and Antigravity CLI directly, or through a generic \"new session bootstrap\" pattern?
- What's the right intermediate layer? (mailbox-outbox / cross-team-event-listener / HAMR /mcp mailbox:read)
- How should the dispatcher signal team-specific runtime constraints (e.g., Antigravity Apollo aliases, Copilot Orion aliases)?
