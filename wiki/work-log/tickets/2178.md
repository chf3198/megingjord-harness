## Scope (Phase-1 child of Epic #2041; from Phase-0 fleet dog-food finding)

Register canonical `tier='fleet-local'` in HAMR's sticky-route registry so `wrapProviderCall('ollama', ..., {tier:'fleet-local'})` does NOT mis-pick a paid provider via sticky-route + then emit cost-stats under the wrong provider name.

Refs Epic #2041
Refs Phase-0 source #2174

## The defect (from Phase-0 dog-food)

During Phase-0 self-validation, `wrapProviderCall('ollama', cb, {tier:'fleet'})` returned `sticky:{provider:'groq', sticky:false, reason:'first_pick_groq_for_tier_fleet'}` — sticky-route picked `groq` for tier `fleet`. The actual call went to Ollama (my callFn), but `emitStatSafe(effectiveProvider=groq, response)` recorded `provider:'groq'` in cache-stats.jsonl. **The cost-tracking entry was mis-labeled.**

## Acceptance criteria

- AC1: `scripts/global/sticky-route.js` recognizes `tier='fleet-local'` and returns `{provider:'ollama', sticky:true, reason:'fleet-local-pinned'}`
- AC2: `scripts/global/hamr-provider-wrapper.js` uses `provider` arg (not sticky pick) when `tier === 'fleet-local'` so emit-stat records ollama not the sticky-pick
- AC3: 4-6 unit tests cover happy path, tier-mismatch fallthrough, ollama-emit verification
- AC4: HAMR docstring updated to list 'ollama' in canonical providers list
- AC5: Lint clean
