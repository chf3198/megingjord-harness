# #2556 — Update package.json deploy scripts to chain xteam-mcp-register.js after each deploy

> **Source**: github:issue/2556 | **state: OPEN** | **Labels**: type:task, status:backlog, priority:P2, area:scripts, lane:code-change, phase-gate:phase-1
> Mirror of `gh issue view 2556` (derived; edit the GitHub item, not this page).

## Body

## Problem
Even after C1 creates `xteam-mcp-register.js`, it will not run automatically unless the deploy npm scripts invoke it.

## Objective
Update the deploy npm scripts in `package.json` so MCP registration runs automatically after deploy, while remaining safe to re-run.

## Refs
Refs #2553 (Epic), Refs #2554 (Phase-0 research), Refs #2555 (C1)
phase_0_sources: [#2554]

## Acceptance Criteria
- [ ] `deploy:apply` chains `bash scripts/deploy.sh --apply && node scripts/global/xteam-mcp-register.js --target copilot --root . --apply`
- [ ] `deploy:claude:apply` chains MCP registration for `claude` target
- [ ] `deploy:codex:apply` chains MCP registration for `codex` target
- [ ] `deploy:antigravity:apply` chains MCP registration for `antigravity` target
- [ ] `deploy:both:apply` chains MCP registration for `copilot` and `codex` targets
- [ ] `deploy:all:apply` chains MCP registration for all supported targets
- [ ] Dry-run scripts (`deploy`, `deploy:all`, etc.) also chain register dry-run mode (no `--apply`)
- [ ] New helper script added: `mcp:register` for explicit operator-triggered registration
- [ ] Registration chain is safe on re-run: if `megingjord-xteam` is already registered, script exits 0 and reports no-op
- [ ] Registration failure surfaces a targeted error without modifying `scripts/deploy.sh`
- [ ] `scripts/deploy.sh` remains unchanged (still exactly 100 lines)

## Design Notes
- C1 owns idempotency; C2 must preserve it when wiring registration into deploy.
- Deploy chaining is acceptable only because C1 now requires no-op behavior on re-run.
- `package.json` wiring must not reintroduce runtime-specific path knowledge; all path/schema logic stays inside `xteam-mcp-register.js`.
