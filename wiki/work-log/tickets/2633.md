## Objective
Analyze Epic and ticket state semantics deeply and produce an implementation-ready enforcement plan that guarantees predictable, query-safe GitHub state.

## Research plan
1. Baseline current behavior
- Enumerate current state labels, role transitions, and close conditions
- Map all programmatic enforcement points (hooks, megalint, label-lint, close-readiness, workflow automations)

2. Failure taxonomy
- Identify drift/mismatch classes (for example: closed epics with open execution scope, stale role labels, ambiguous deferred vs dormant usage)
- Quantify impact on issue filtering correctness and governance trust

3. Enforcement design
- Define canonical state machine for epics and child tickets
- Define deterministic transition guards and disallowed transitions
- Specify machine-checkable evidence required for each transition

4. Implementation plan
- Propose phased rollout with backward-compatible migration
- Include required validators, workflow updates, and tests
- Include dry-run mode, promotion path (advisory to required), and rollback

5. Verification strategy
- Define filter-truth test suite: expected GitHub query outputs by scenario
- Define monitoring and incident signals for state integrity regressions
