# #1826 — Epic: Self-anneal harness stress test — per-team granular pass/fail with remediation pointers

## Body

## Epic Summary

Build a **self-anneal harness stress test** any team (Claude Code, Copilot, Codex) can run on their checkout. Output: granular pass/fail per check, **plus concrete remediation pointers** (which command to run, which file to edit, which doc to read). Anchored in 2026 AgentOps + OpenTelemetry GenAI semantic conventions. Composes with the meta-anneal Epic chain (#1798 → #1815 → #1821 → #1824) by promoting ad-hoc operator stress-test prompts into a first-class harness command.

## Problem

Operator-side stress-test prompts (used during the Epic #1798 / #1815 / #1821 audits and the 2026-05-17 Copilot Team run) are valuable but:

1. **Hand-crafted per session** — every audit re-derives the test list; no canonical regression suite.
2. **Binary signal** — pass/fail without remediation guidance (the 2026-05-17 Copilot Team report listed 8 failures with no machine-recommended fix; #1824 partially closed this for hook-parity specifically).
3. **No persistence** — results aren't stored, no trend tracking, no anneal-event emission.
4. **Cross-team inconsistency** — each team can run a different version of the prompt; results aren't comparable.
5. **No OpenTelemetry surface** — bypasses the harness's instrumentation contract (`instructions/observability.instructions.md` GenAI namespace).

Frontier 2026 evidence (sources at bottom):

- AgentOps platform built on seven interconnected principles where "observability drives evaluation, which informs optimization, governance, and security."
- Agent harness pattern: "eval harness that scores every PR catches 80% of regressions before they hit staging."
- Critical separation: "**capability evals push agents forward**; **regression evals protect what already works**." Both are required.
- 2026 OpenTelemetry GenAI semantic conventions cover LLM client spans, agent spans, events (prompt/completion content), and metrics. Stress-test results should emit as GenAI events.

## Proposed contract

A `npm run harness:self-test` command + an `instructions/self-anneal-harness-stress-test.instructions.md` contract that:

1. **Discriminated diagnostics**: every check produces a `(name, expected, observed, diagnosis, recommend_action)` tuple. `recommend_action` is a concrete command, file path, or doc reference (extending #1824's pattern across the whole stress test).
2. **Two test classes** per 2026 frontier pattern:
   - **Capability evals** — does the harness DO the right thing? (e.g., does `governance:duplicate-check` correctly flag known historical dup pairs?)
   - **Regression evals** — does the harness STILL do what it did yesterday? (e.g., does `stop_checks.check_uncommitted` still silence pre-collab? Each prior self-anneal becomes a regression test.)
3. **Tiered output**: human (color terminal), JSON (machine-readable), markdown table (paste-back to operator).
4. **Telemetry**: emit each result as an OpenTelemetry GenAI event to `~/.megingjord/incidents.jsonl` (tier 1 observation, per `instructions/observability.instructions.md` event-schema-v3).
5. **Replay-based**: per Epic #1771 contract, NO calendar wait between runs. Validation against fixture set + live state, decided in seconds.
6. **Pre-check Step 0**: `governance:hook-parity` (from #1824) runs FIRST and short-circuits with rebase/deploy guidance if branch is stale, so test results aren't polluted by branch-staleness.
7. **Cross-team parity**: same `harness:self-test` runs on Claude Code, Copilot, Codex. Adapter exemptions (e.g., Claude Code no-hooks per Epic #1798 F5) are recognized; not flagged as failures.

## Research findings (cited inline so this Epic is pick-uppable cold)

### F1 — AgentOps observability-driven evaluation pattern

[AgentOps platform](https://github.com/agentops-ai/agentops) (active 2026): Python SDK with time-travel debugging, session replay, multi-agent workflow visualization. Built on 7 principles where observability → evaluation → optimization → governance. Composes with 400+ LLMs and major agent frameworks (CrewAI, Autogen, OpenAI Agents SDK, LangChain). **Pattern to adopt**: instrument every harness check as a span/event with GenAI semantic-convention attributes, then aggregate spans into the regression suite.

### F2 — Capability vs regression eval separation

Per [Augment Code 2026 evaluation tools roundup](https://www.augmentcode.com/tools/best-ai-agent-evaluation-tools): "Teams need to separate capability evals from regression evals because they serve different purposes. Capability evals push agents forward by measuring progress on hard tasks, while regression evals protect what already works. Without the separation, teams will either stop improving or ship regressions." **Apply**: split `harness:self-test` into two subcommands (`--capability`, `--regression`); default runs both with clear segmentation in output.

### F3 — Eval harness as PR gate (80% regression catch)

Cited 2026 pattern (multiple sources): "an eval harness that scores every PR catches 80% of regressions before they hit staging" via PromptFoo/Braintrust/LangSmith-style harnesses. **Apply**: wire `harness:self-test --regression` into pre-merge CI (advisory at first, promoted via #1771 replay-based eval gate).

### F4 — OpenTelemetry GenAI conventions (April 2026 state)

Per [OpenTelemetry GenAI April 2026 Guide](https://callsphere.ai/blog/td30-fw-opentelemetry-genai-conventions-april-2026-guide): conventions cover LLM client spans, agent spans, events, metrics. Stable enough for production telemetry by Q2 2026. Datadog v1.37+ has native support; Grafana collects LLM traces in Loki. **Apply**: emit harness:self-test events with GenAI attributes (`gen_ai.system`, `gen_ai.operation.name`, `gen_ai.response.model`, etc.) — composes with existing `instructions/observability.instructions.md` event-schema-v3.

### F5 — Self-healing diagnostics with remediation pointers

Per [TestQuality 2026 agentic QA architecture](https://testquality.com/agentic-qa-architecture-autonomous-testing-2026/): self-healing test architectures embed the remediation action in the failure signal. **Apply**: extend #1824's pattern (every diagnosis carries `recommend` field) across every check in the stress test.

### F6 — Time-travel / session replay for failure diagnosis

AgentOps pattern: rewind and replay agent runs with point-in-time precision. **Apply**: when `harness:self-test` flags a regression, output the replay-runner invocation that reproduces the failure against fixture data (composes with `scripts/global/soak-replay-runner.js` from Epic #1771).

### F7 — Cross-team tool surface unification

Per [2026 Agent Evaluation Stack guide](https://medium.com/@Micheal-Lanham/why-your-agent-evaluation-stack-is-about-to-get-weirder-and-better-dc9f8cfb9b07): cross-team consistency requires standardized tool surface. **Apply**: ship `harness:self-test` as a deterministic CLI; results across teams (Claude, Copilot, Codex) become directly comparable. Composes with `instructions/canonical-governance-anti-duplication.instructions.md` per-runtime adapter contract.

## Composition with prior session work

This Epic is the **meta-fix for the meta-anneal pattern** that ran through this session:

- #1798 → #1815 → #1821 → #1824 surface a recurring shape: a hook misfires; operator audits; patches one hook; misses sibling hooks; new symptom reveals missed surface.
- A canonical stress test that EVERY team can run trivially would have caught the userprompt_gate.py miss (#1815) and the visual_qa over-scope (#1821) at the time #1798 was first audited.
- This Epic codifies the stress-test prompt into a maintained suite so future audits don't re-derive the test list and don't miss surfaces.

## Out of scope

- Building a new observability platform — uses existing `~/.megingjord/incidents.jsonl` + GenAI semantic conventions.
- LLM-as-judge composition for stress-test results — defer to Epic #1814 (cross-family review quality).
- Auto-remediation (the test EXECUTES the recommend_action automatically) — Phase 1 limits to surfacing the action; auto-execute is a future Epic with much higher safety review.

## Risk register

| Risk | Mitigation |
|---|---|
| Stress-test suite drifts from real bugs (test rot) | Each fixed self-anneal ticket adds a regression case — directly tied to real defects |
| Cross-team parity breaks when runtimes diverge | Adapter-exemption manifest documents intentional differences; recognized as PASS not FAIL |
| Telemetry overhead bloats incidents.jsonl | Per-event size ~500 bytes; 100 checks × daily = 50KB/day acceptable per existing event schema |
| Promotion to CI required-mode causes false-positives at scale | Replay-eval validation (#1771 pattern) requires ≥85% historical agreement before promotion |
| Operators ignore recommend_action and file low-quality tickets | Output template strongly encourages running the action FIRST; tickets pre-pre-empt with the remediation reference |
