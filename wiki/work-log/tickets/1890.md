# #1890 — Tier-2 anneal: signer-alias-canonical extended to all comments

## Body

## Problem

The signer-alias-canonical check (`scripts/global/megalint/signer-registry-check.js`) only validates baton artifact comments (MANAGER_HANDOFF / COLLABORATOR_HANDOFF / ADMIN_HANDOFF / CONSULTANT_CLOSEOUT). Non-baton comments with `Signed-by:` fields slip through, allowing invented aliases.

**Pattern instances** (across multiple sessions):
- "Quinn Mason" on Copilot Team Epic #1857 ADMIN_HANDOFF (per earlier red-team)
- "Cole Mason" on Epic #1113 + #1399 original ticket bodies (artifact persisted; canonical seed not enforced retroactively)
- "Rowan Harper" on #1399 remediation comment (2026-05-18 17:02:14Z) — Copilot collaborator role; "Rowan" is not in canonical seed for copilot:gpt-5.3-codex@openai
- The auto-`megalint-signer-fidelity` lint at 17:02:05Z flagged a related issue body field, but the comment signer escaped detection

Memory note `feedback_signer_alias_derivation` codifies the rule. Less-intelligent models in Copilot auto-mode generate plausible-sounding given-names without checking the registry.

## Goal

Extend signer-alias-canonical enforcement to ANY comment containing structured signature fields (`Signed-by:` + `Team&Model:` + `Role:`), not just baton artifacts.

## Acceptance Criteria

- [x] AC1: `scripts/global/megalint/signer-registry-check.js` extended (or new sibling validator) that scans ALL comment bodies for signature triples.
- [x] AC2: When `Signed-by` + `Team&Model` + `Role` all present on a single comment, the alias MUST resolve via `agent-signature.js` registry-derive — non-canonical aliases fail.
- [x] AC3: Comment-level enforcement runs as a workflow on `issue_comment.created` and `issue_comment.edited`.
- [x] AC4: Advisory-mode initially per Epic #1612 promotion model; promote to required after replay-eval against historical 50-comment corpus shows < 5% FP rate.
- [x] AC5: Auto-emits advisory comment naming the invented alias and the canonical alias derived from registry.
- [x] AC6: Tests covering: (a) baton artifact validation unchanged, (b) non-baton comment with invented alias detected, (c) prose mention of names not flagged (e.g. "thanks Rowan for the suggestion" should pass).
- [x] AC7: Stress test (adversarial corpus): 50 historical invented-alias comments + 50 canonical comments, precision/recall measurement.
- [x] AC8: Wired into harness:self-test as regression check.

## Test strategy

`tdd-pyramid+stress-test` per Epic #1875 (adversarial-input parser).

## Why programmatic

Same rationale as #1889: instructional governance is insufficient when the lower-capability model generates plausible aliases. Programmatic registry-derive is the only durable defense.
