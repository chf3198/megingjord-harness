# #1380 — Strip pre-close status labels in closeout-lint when applying terminal status

> **Source**: github:issue/1380 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P3, area:governance, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 1380` (derived; edit the GitHub item, not this page).

## Body

## Problem

Three tickets closed in this session as duplicates (#1221, #1223, #1225) carry **both** `status:backlog` (legacy) AND `status:cancelled` (closeout-applied). That's a Rule 1 multi-status drift — Rule 1 requires exactly one `status:*` label.

The closeout workflow applied `status:cancelled` but did not strip the pre-existing `status:backlog`. Same class of drift surfaced on #1117 and #1316 earlier this session (those carried `status:in-progress` + `status:done` and `status:review` + `status:done` respectively).

## Goal

Make terminal-status transitions (`status:done`, `status:cancelled`, `status:archived`) implicitly remove all other `status:*` labels.

## Acceptance Criteria

- [ ] AC1: `closeout-lint.yml` workflow detects pre-close status labels at close time and strips them automatically
- [ ] AC2: Or a new pre-close hook in `label-lint.yml` ensures terminal-status application is mutually exclusive with non-terminal status
- [ ] AC3: Golden-file test: ticket with `status:backlog + status:cancelled` triggers cleanup; resulting state has only `status:cancelled`
- [ ] AC4: Retrospective sweep: identify and clean all closed issues with multi-status drift (likely small list — #1117, #1221, #1223, #1225, #1316 known)

## Test strategy

`golden-file` for workflow + retrospective sweep evidence.
