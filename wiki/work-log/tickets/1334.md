## Body

## Problem

Follow-up to #1305 core delivery. The three deferred ACs (6, 8, 9) strengthen the cross-team Consultant pickup protocol with enforcement + automation that the core flow doesn't strictly need but should land before declaring the protocol production-grade.

## Goal

Complete AC6/AC8/AC9 of Epic-#1305 scope. Core protocol (labels, instruction, skill, queue script, mutual-exclusion lint) already shipped in #1305 / PR #1333.

## Acceptance Criteria

- [ ] **AC1 — Signer-substrate gate** (replaces #1305 AC6): extend `.github/workflows/baton-gates.yml` so that for Epics carrying `consultant:cross-team-in-progress`, the `CONSULTANT_EPIC_CLOSEOUT` signer's `Team&Model` substrate must equal the active `CROSS_TEAM_CLAIM` substrate. Mismatch fails the consultant-gate.
- [ ] **AC2 — Stale-claim reaper** (replaces #1305 AC8): add `.github/workflows/cross-team-claim-reaper.yml` running on a daily cron. Reads each Epic with `consultant:cross-team-in-progress`, scans for `CROSS_TEAM_CLAIM` comments, compares `expires:` timestamp to current UTC. If expired: posts `CROSS_TEAM_CLAIM_EXPIRED` comment AND swaps label `:in-progress → :needed`.
- [ ] **AC3 — Manager-side auto-apply** (replaces #1305 AC9): add automation (likely extending `post-merge-automation.yml` or a new workflow on `issue_comment`) that, when a Manager comment on an Epic contains both `CONSULTANT_EPIC_CLOSEOUT` AND `cross-team Consultant required`, auto-applies the `consultant:cross-team-needed` label.
- [ ] **AC4 — Live verification**: exercise the full flow end-to-end (real or synthetic Epic). Demonstrate: Manager request → label auto-applied → receiving team claim → 24h pass with no signoff → reaper expires → re-claim available → signer-substrate gate enforces match.
- [ ] **AC5 — Tests**: golden-file fixtures for each new workflow + spec test for any new JS helper.

## Test strategy
