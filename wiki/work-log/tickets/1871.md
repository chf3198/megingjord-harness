## Scope

Build a robust stress-test suite for the cross-team conflict-prevention chain. **All 9 harness goals (G1–G9) must be reflected in the test design**, not just G2 Quality.

### Targets

1. **Epic #1854 worktree-isolation** — `worktree-active-session-lock.js`, `worktree-write-intercept.js`, `worktree-lease-heartbeat.js`
2. **Epic #1855 anneal-decision detector** — `anneal-decision-detector.js`, `anneal-decision-audit.js`
3. **Epic #1827 rebase discipline** — `git-freshness-check.js`, `git-conflict-predict.js`, `collab-handoff-rebase-freshness.js`

### Test classes per goal

| Goal | Test class | Examples |
|---|---|---|
| **G1 Governance** | CI-runnable + governance-evidence-emitting | `npm run stress:test` produces structured artifact; harness:self-test integration as a regression check |
| **G2 Quality** | Concurrency + adversarial corpus | 100 simultaneous `acquire()` → exactly one wins; 50-pattern adversarial corpus for anneal-detector with ≥90% precision |
| **G3 Zero-Cost** | Pure-local + fleet-routed | No paid API calls; perf-bench uses local clock + git only |
| **G4 Privacy** | Synthetic-only data | No PII in fixtures; redaction verified on output |
| **G5 Portability** | Cross-platform conditional | Windows/macOS/Linux fs semantics; opt-out env vars verified across runtimes |
| **G6 Resilience** | Chaos + fault injection | Kill mid-acquire, corrupt lock file, network unavailable for `gh pr list`, file-system permission denied |
| **G7 Throughput** | Performance benchmarks | acquire latency under load; freshness-check on 10k-commit history; conflict-predict on 100 open PRs |
| **G8 Observability** | Telemetry emission | Stress events emit to incidents.jsonl per event-schema-v3; metrics captured |
| **G9 Interoperability** | Composes without API breakage | Existing 44 unit tests still pass; harness:self-test still 9/9 |

## Proposed ACs

- AC1 (G2/G6): concurrency test — 100 parallel processes call `acquire()`; exactly 1 succeeds; no corrupted lock file
- AC2 (G6): chaos test — kill a process mid-acquire; lock auto-recovered on next attempt; no permanent lockout
- AC3 (G6): corruption test — manually corrupt lock file mid-session; system gracefully re-acquires
- AC4 (G7): performance test — `acquire()` < 50ms p99 under no load; < 200ms p99 under 100-process load
- AC5 (G7): freshness-check perf — < 100ms on 10k-commit branch history
- AC6 (G7): conflict-predict perf — < 2s on 100 open PRs (uses mock `gh` to control variance)
- AC7 (G2/G6): adversarial-corpus stress for anneal-decision-detector — 50 recognition patterns + 30 decision patterns + 20 false-positive traps; precision ≥ 0.90, recall ≥ 0.85
- AC8 (G2/G6): freshness-check tier classification fuzz — 1000 random `{behind, velocity, branch_commits}` tuples; tier transitions monotonic
- AC9 (G6): chaos test for heartbeat — simulate 10 parallel heartbeat refreshes; final state consistent
- AC10 (G6): conflict-predict chaos — `gh` command fails / times out / returns malformed JSON; tool degrades gracefully not crashes
- AC11 (G8): stress runs emit telemetry to `~/.megingjord/incidents.jsonl`; stress event schema documented
- AC12 (G1/G9): `harness:self-test` registry extended with stress-test entries as regression checks
- AC13 (G5): cross-platform CI matrix — Linux (default) + macOS conditional (`process.platform === 'darwin'`) + Windows-skip with explicit reason
- AC14 (G3): full stress suite runs in < 60s on default profile (longer for full-fuzz profile)
