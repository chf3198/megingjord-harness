## Scope (Phase-1 child P1-4 of Epic #2029; adopted by Epic #2246)

Goal-keyword expansion at the provider layer per Epic #2029 AC3. When goal-context
injection is requested AND the prompt contains a goal-keyword (e.g. "cost",
"privacy"), inject the relevant G-definition (the provider-layer parallel of
`goal_lens.py` for hooks). Builds on P1-1 (#2221, CLOSED) — `GOAL_DEFINITIONS` is
already defined + exported in `scripts/global/governance-context.js`.

**Matching semantics (remediated, Refs red-team #2231):**
- Case-INsensitive.
- Whole-word matching via regex word boundaries (`\bkeyword\b`) — `costume` must NOT match `cost`.
- Synonym expansion (e.g. "expense"→cost) is explicitly OUT of scope for Phase-1.
- No keyword found → return empty → NO additional context injected.
- Multiple keywords / a keyword mapping to multiple goals → return the DEDUPLICATED
  set of all unique relevant G-definitions (additive; goal-conflict resolution out of scope).

## Acceptance criteria

- **AC1**: `scripts/global/governance-context.js` gains `expandKeywords(prompt)` returning a
  **deduplicated array of unique relevant G-definition strings** (case-insensitive, `\b`
  word-boundary matching; no match → empty array), formatted consistently with `GOAL_DEFINITIONS`.
- **AC2**: Reuses `GOAL_DEFINITIONS` from P1-1 (#2221, CLOSED — dependency already satisfied; no new defs).
- **AC3**: 10 unit tests covering: positive keyword matches, case-insensitivity, word-boundary
  rejection (`costume`↛`cost`), multi-goal, duplicate/dedup, and no-match→empty.
- **AC4**: Lint clean.
- **AC5 (integration/wiring)**: `injectGoalContext(opts)` consumes `expandKeywords` output and
  **appends** the matched G-definitions AFTER the existing `PRIORITY_SENTENCE` + `DECISION_CHECK`
  block, gated behind a new opt (`opts.expand_goal_keywords`, default off). At least 1 test asserts
  the append placement + ordering. End-to-end `wrapProviderCall` opts plumbing beyond
  `injectGoalContext` is DEFERRED to a named follow-on if non-trivial (state which in COLLABORATOR_HANDOFF).
