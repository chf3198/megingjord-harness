## Objective

Build the auto-materialize script that creates Phase-1 child ticket templates when a Phase-0 research child closes on a research-first Epic. Depends on the phase0_green_complete predicate from child A.

## Acceptance Criteria

- [ ] AC1: Script `scripts/global/phase0-promote.js` takes epicNumber as input and invokes phase0_green_complete predicate from child A
- [ ] AC2: When predicate returns `{ complete: true, missingPhase1Children: true }`, script creates Phase-1 child tickets from template bundle via gh issue create
- [ ] AC3: Template bundle creates tickets with: correct labels (type:task, phase-gate:phase-1, priority:P1, lane:code-change), body containing Refs to Epic and Phase-0 child, placeholder ACs populated from Epic AC list
- [ ] AC4: GitHub Actions workflow `.github/workflows/phase0-promote.yml` triggers on `issues.closed` event when closed issue has `phase-gate:research-first` label; invokes the script
- [ ] AC5: Script is idempotent: if Phase-1 children already exist, it exits 0 with no-op log
- [ ] AC6: Script emits observability event to incidents.jsonl with pattern_id `phase0-complete-phase1-created`
