# #1536 — Enforce canonical Team&Model signature block; reject client-name provenance

## Body

## Objective
Prevent non-canonical provenance lines (e.g., "Manager: <client name> | ...") from being accepted in governed artifacts.

## Why
Recent Epic #1526 tickets accidentally included client identity in provenance. Canonical format requires:
- Signed-by: <human alias>
- Team&Model: <team>:<model>@<substrate>[/device]
- Role: <role>

## Acceptance Criteria
- [x] Add lint check in scripts/global/lint-team-signature.js (or new lint) that fails when Team&Model line does not match canonical regex
- [x] Add check that rejects provenance lines beginning with Manager:/Collaborator:/Admin:/Consultant:
- [x] Add tests for valid canonical signature and invalid client-name format
- [x] Wire into CI lint gate
