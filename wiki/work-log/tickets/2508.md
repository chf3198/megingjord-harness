# #2508 — Epic: Megingjord as VS Code/Antigravity marketplace extension (Hybrid Option C)

## Body

## Goal

Architect the Megingjord harness as a published VS Code (and compatible IDE: Antigravity, Cursor) marketplace extension. Hybrid Option C: `megingjord-core` always-on + opt-in companion extensions (`megingjord-xteam`, `megingjord-dashboard`, `megingjord-wiki-publisher`).

## Problem

Current install model (npm + per-runtime deploy.sh) works for power-users but is inaccessible to typical IDE users. To realize the harness's broader value (per client framing of 'industry excellent VS Code extension'), it needs marketplace distribution.

## Proposed architecture: Hybrid Option C

- megingjord-core (always-on harness governance + extension API surface)
- megingjord-xteam (opt-in companion; consumes core API)
- megingjord-dashboard (opt-in panels)
- megingjord-wiki-publisher (opt-in)
- Extension API contract defines what core exposes, what companions consume

## Phase-0 acceptance criteria (research-first)

- AC-R1: Synthesize 2026 marketplace publishing patterns for VS Code + Antigravity + Cursor
- AC-R2: Design extension API contract (what core exposes to companions)
- AC-R3: Per-IDE marketplace eligibility audit (which IDEs accept marketplace submissions)
- AC-R4: Migration plan from current npm-deploy model to marketplace model (preserve workspace-overlay contract)
- AC-R5: Cross-family rater verdict via qwen-32b on Tailscale (G3 fleet)

## Phase-1 (after Phase-0 accepted)

To be enumerated based on Phase-0 outcome. Initial sketch:
- Extension scaffolding (vsce / extension manifest)
- Core extension implementation
- xteam companion extension
- Dashboard companion extension
- Marketplace publication workflow

## Out of scope

- Standalone CLI distribution (npm install path remains for power-users)
- HAMR substrate refactor (separate Epic #2488)
- Tier 2/3 cross-workspace coordination (#2490)
