# #2345 — Refactor ticket ownership model for baton and accountability

> **Source**: github:issue/2345 | **state: OPEN** | **Labels**: type:epic, priority:P1, role:manager, area:governance, status:triage
> Mirror of `gh issue view 2345` (derived; edit the GitHub item, not this page).

## Body

## Objective
Separate persistent accountability ownership from transient baton execution ownership so backlog, active, and terminal states remain semantically clean and governance-safe.

## Problem
Current label usage overloads role labels for two different concepts:
- persistent ownership/accountability
- active baton holder for workflow execution

This causes confusion in board views, status semantics, and audits (especially around backlog and closed tickets).

## Outcomes
- Clear split between accountability metadata and baton-role state
- Backlog and terminal non-epic tickets no longer carry execution-role ambiguity
- Board columns and filters reflect real execution state without losing ownership attribution
- Validators and automation align with the new model
