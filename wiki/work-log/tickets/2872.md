## Body

## Research Objective

Survey options for detecting open GitHub issues with active-phase labels (`status:in-progress`, `status:testing`) whose linked PRs are already merged. Three candidate approaches: GitHub Actions trigger, pre-tool hook, session-start scan. Select and specify the best approach for AC-I2.

## Scope

- Enumerate open issue detection options:
  1. GitHub Actions on `pull_request.closed` (merged) → auto-advance issue status
  2. Session-start scan in hook: fetch merged PRs from origin, cross-reference open issues with stale labels
  3. Manager tool: CLI script to detect/report stale baton states
- Evaluate each on: reliability, cost (API calls), G5 portability (offline/air-gapped), implementation complexity
- Select preferred approach with rationale

## Deliverable

Findings comment on this ticket:
1. Comparison table: GitHub Actions vs hook vs CLI scan
2. Selected approach with rationale
3. Implementation sketch for selected approach (entry point, data flow, output)
