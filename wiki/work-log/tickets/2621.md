# #2621 — Execute free-cloud failover in cascade-dispatch (not just signal)

> **Source**: github:issue/2621 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:scripts, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 2621` (derived; edit the GitHub item, not this page).

## Body

## Problem (G3 — complete #2619's value)
#2619 made `cascade-dispatch` *suggest* the `free-cloud` tier when the local fleet is unreachable, but the suggestion is **advisory only** — cascade-dispatch still executes nothing beyond Ollama, so on a fleet outage the session must manually route to a free cloud provider (observed repeatedly this session: tailscale fleet DOWN → every review hand-routed to free gemini). There is no programmatic free-cloud client in `scripts/global`. Until cascade-dispatch can actually *execute* a free-cloud call, the zero-cost failover requires manual operator intervention.

## Objective
Add a bounded, gracefully-degrading free-cloud dispatch client and wire it into cascade-dispatch's fleet-unavailable path, so a fleet-lane task whose fleet host is down is served automatically by a $0 cloud provider — no manual routing, no paid-tier escalation.

## Acceptance Criteria
- AC1: `scripts/global/free-cloud-dispatch.js` — `dispatchFreeCloud(prompt, opts)` tries the `model-routing-policy.json` `free-cloud` providers in order (gemini / openrouter-free / groq / cerebras), each routed through `wrapProviderCall` (#1160) for HAMR cost/observability. Returns `{ok, content, provider}` on success; `{ok:false, reason}` when no key is configured or all providers fail. Injectable fetch/dispatch impl for deterministic tests. Short per-provider timeout.
- AC2: `cascade-dispatch.js` — on an availability failure (escalationTier → free-cloud), attempt `dispatchFreeCloud`; on success return its content as a `tier:'free-cloud'` result (escalation_needed:false); on failure fall back to the existing advisory `suggested_tier:'free-cloud'` signal (graceful degradation, no behavior regression).
- AC3: Tests (`tests/free-cloud-dispatch.spec.js`) — provider-order + graceful no-op when no key (injected impl, no network); cascade fleet-down with free-cloud success → returns content; with free-cloud unavailable → advisory signal preserved.
- AC4: Doc note (`global-task-router.instructions.md` free-cloud failover paragraph: signal → now auto-executed) + CHANGELOG fragment.

## Goal Lens
- G3 (primary): fleet-down failover becomes actual $0 execution — eliminates manual free-cloud routing and any paid-tier escalation for routine fleet-lane work during a fleet outage.
- G6: graceful degradation — missing keys / all-providers-fail cleanly falls back to the advisory signal (no regression vs #2619).
- G7 (acknowledged trade): adds a bounded external call on the fleet-down execute path; mitigated by short timeout + only on availability failure. G3 > G7 justifies.
