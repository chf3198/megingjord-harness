# #2198 — Research governance control-plane model for ticket-state integrity

## Body

Refs Epic #2197

## Research objective

Design a control-plane architecture that prevents and repairs ticket-governance drift with deterministic, testable behavior.

## Critical analysis (current harness)

### What works

- Shared rule evaluation exists (`label-rules.js`) and is used by label linting.
- Close-time protection exists and can auto-transition/reopen at close boundary.
- Drift scanners and reports provide observability.

### What fails repeatedly

1. **Open-state terminal drift persists**
   - 21 open issues currently carry `status:done`.
   - Root gap: closed-state terminal rules exist, but open-state terminal prohibition is not codified as a hard invariant.

2. **Instruction dependence exceeds control dependence**
   - Governance depends on role comments/manual cleanup to stay correct.
   - Validators often detect/comment rather than deny/repair at write time.

3. **Event-scoped enforcement leaves blind spots**
   - Close-protection logic only runs meaningfully for close transitions.
   - Open-state drifts can survive long periods, increasing backlog entropy.

4. **Invariant fragmentation**
   - Some rules live in docs, some in validators, some in workflow glue.
   - This increases policy drift between intended model and enforced model.

## Research synthesis (best-practice architecture pattern)

The durable pattern should follow a control-plane model with three layers:

1. **Admission control (prevent invalid writes)**
   - Normalize and deny invalid `state/status/role` combinations at mutation time.
   - Single canonical policy module consumed by all mutation paths.

2. **Reconciliation loop (eventual correction)**
   - Periodic/idempotent reconciler scans all issues and repairs safe classes automatically.
   - Unsafe classes are escalated with machine-readable drift artifacts.

3. **State machine + observability**
   - Explicit finite-state transition map with allowed transitions and actor constraints.
   - SLOs: drift count, correction MTTR, recurrence rate by class.

## Proposed technical direction

- Add explicit rule: open issue must not carry terminal labels (`status:done`, `status:cancelled`).
- Add explicit rule: non-epic `status:backlog` must not carry any `role:*` label.
- Introduce `governance-actuator` script:
  - `--dry-run` shows proposed fixes
  - `--apply` performs idempotent repairs for safe classes
- Enforce same policy module in:
  - label-lint workflow
  - daily label-scan workflow
  - close-protection workflow
  - actuator
- Add fixture corpus and replay tests for known drift classes.
