# #2258 — Define Manager-Consultant drift-remediation lane

> **Source**: github:issue/2258 | **state: OPEN** | **Labels**: type:epic, priority:P2, role:manager, area:governance, status:triage
> Mirror of `gh issue view 2258` (derived; edit the GitHub item, not this page).

## Body

## Objective

Define and ship a governed Agile workflow for ticket-drift remediation when there is no code revision, no git branch work, and no PR/merge requirement. In this narrow class of work, remediation must still use baton governance, but the active roles should be reduced to Manager -> Consultant.

## Problem

Current baton contracts assume either:
- full delivery work: Manager -> Collaborator -> Admin -> Consultant, or
- reduced non-code/config work that still routes through Admin.

That leaves a governance gap for already-shipped ticket remediation work where the only actions are issue-body normalization, label normalization, state correction, and consultant validation of the manager's remediation. The gap encourages ad hoc cleanup instead of an explicit governed lane.

## Scope

- Define the exact entry criteria for Manager -> Consultant-only drift remediation.
- Define hard exclusions that force the standard 4-role lane.
- Define validation evidence required from Consultant when only Manager work occurred.
- Add the necessary instruction / validator / ticketing guardrails in follow-on children.

## Acceptance Criteria

- [ ] AC1: Research and planning child completes with cutting-edge external and repo-local evidence.
- [ ] AC2: Epic-level decision specifies exact eligibility criteria for Manager -> Consultant-only remediation.
- [ ] AC3: Development children created for instruction, validator, and workflow/ticketing enforcement.
- [ ] AC4: Final implementation forbids skipping Agile workflow entirely, while allowing omission of Collaborator/Admin only for the constrained remediation class.
- [ ] AC5: Development child slate explicitly covers false-positive protection and anti-abuse boundaries.

## Constraints

- No weakening of G1 governance.
- No broadening into a generic "skip roles for docs/research" loophole.
- Must be framed as a tightly-scoped remediation lane for already-shipped/no-code/no-git ticket drift.
- Consultant in this lane validates Manager-authored remediation; it does not validate absent evidence.
