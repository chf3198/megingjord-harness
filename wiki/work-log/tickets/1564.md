# #1564 — D1: Prevent raw Cloudflare API token in shell command arguments (G4 Critical)

## Body

## Problem

During #1415 execution, `CLOUDFLARE_API_TOKEN` was passed as an inline shell prefix:
```sh
CLOUDFLARE_API_TOKEN=<token> npx wrangler secret put ...
```
The raw token value appeared in terminal output, command history, and log context.
This violates G4 (secrets never in shell/env/logs) and the secret-exposure-prevention skill.

## Root Cause

No harness-level wrapper exists for wrangler authentication. The operator sourced a `.env`
file but then passed the token inline anyway, bypassing isolation.

## Acceptance Criteria
- [ ] `scripts/global/wrangler-auth.sh` (or equivalent) sources token from `.env` into
  a subshell without echoing value, wraps wrangler commands, and exits subshell after.
- [ ] All wrangler invocations in harness scripts/runbooks use the wrapper.
- [ ] docs/howto/hamr-push-auth.md updated to reference wrapper.
- [ ] Test or smoke check verifies wrapper does not leak token to stdout/stderr.
