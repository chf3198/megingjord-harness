# #1381 — Epic: Programmatic Worker Naming & Signature Algorithm

> **Source**: github:issue/1381 | **state: CLOSED** | **Labels**: type:epic, area:governance, status:cancelled

## Body

## Problem

Current work artifacts inconsistently identify the active AI worker. Human identity (`Curtis Franks`) is sometimes conflated with execution sign-off, even though client identity should only represent design consultation and rare UAT.

This causes governance drift in baton accountability (Manager → Collaborator → Admin → Consultant), weakens auditability, and obscures model-routing decisions.

## Goal

Define and implement a **programmatic naming/signature algorithm** that always creates worker names conveying:

1. Agile role (`manager`, `collaborator`, `admin`, `consultant`)
2. Model identity (`provider.model`)
3. Collaborator specialization when role = collaborator (e.g., `research`, `design`, `coding-simple`, `coding-problem-solving`, `coding-research`)
4. Human-readable form suitable for ticket readers
5. Deterministic, parseable structure for tooling and governance checks

`Curtis Franks` must be reserved strictly for **client consultation/UAT attribution**, never routine worker sign-off.

## Scope

- R&D and design for a canonical worker naming convention
- Role + model + specialization taxonomy and normalization rules
- Deterministic algorithm + parser contract + collision handling
- Integration plan for router/model-matrix ticket assignment
- Governance guardrails to prevent human-name misuse in worker sign-off

Out of scope for this Epic: full rollout across every historical ticket.
