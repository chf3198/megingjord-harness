# #1701 — epic(governance): canonical cross-orchestrator governance without redundancy

## Body

## Objective
Design and implement a **single-source governance system** that supports broad orchestrator compatibility (Copilot, Cline, Claude Code, Continue, and future adapters) **without redundant hand-maintained file sets**.

## Problem statement
Current compatibility expansion requires adding tool-specific governance files in multiple formats and paths. This creates content duplication, drift risk, and high maintenance overhead.

## Completed research input
- **Research ticket (complete):** #1692
- Outcome: adopt **Canonical Governance Graph + Adapter Emitters + Drift Gates**.

## Scope of this Epic
- Define canonical governance manifest and schema contract.
- Generate orchestrator-specific artifacts from canonical source.
- Add deterministic sync/drift controls in CI.
- Validate parity across orchestrator outputs.
- Migrate existing governance content to canonical source.
- Publish operator docs and onboarding playbook.

## Acceptance criteria (Epic complete when ALL are green)
- [ ] **AC-1** Canonical governance manifest schema exists, validates in CI, and supports orchestration target metadata.
- [ ] **AC-2** Adapter emitters generate deterministic outputs for Copilot, Cline, Claude, and Continue targets.
- [ ] **AC-3** `governance:generate` and `governance:sync-check` exist; CI fails on stale generated governance artifacts.
- [ ] **AC-4** Compatibility matrix/golden tests cover all supported orchestrators and fail on parity regressions.
- [ ] **AC-5** Existing governance corpus is migrated to canonical source with no material behavior loss.
- [ ] **AC-6** Operator docs clearly define authoring flow, adapter onboarding, and drift recovery.
- [ ] **AC-7** All child tickets are closed and merged to main.
