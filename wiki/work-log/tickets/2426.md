## Body

## Problem
`scripts/global/megalint/doc-coverage.js` `validate()` always returns `{ ok: true, violations: [] }`. It only computes an advisory string. The required-surface check never blocks.

## Objective
Wire `doc-coverage.js` to verify each `required` surface from `config/doc-coverage-matrix.yml` appears in the `doc-coverage:` block with status `DONE` or `N/A` + reason, and return violations when missing.

## Acceptance Criteria
- [x] `validate(input)` accepts `{ body, labels }` (COLLABORATOR_HANDOFF body text + issue labels)
- [x] Parses `doc-coverage:` YAML block from body using regex or lightweight YAML parse
- [x] For each `required` surface in the matrix (filtered by ticket's area labels): emits `{ rule: 'doc-coverage-missing', severity: 'error', detail: ... }` if surface is absent or has bare N/A without reason
- [x] `ok: false` returned when any required surface violation exists
- [x] `suggested` surfaces: advisory only (no violation)
- [x] Bare `N/A` without reason text emits a violation (reason required per schema)
- [x] Lane skip: docs-research and config-only bypass entirely
- [x] Unit tests covering: all surfaces DONE → ok; missing required → violation; bare N/A → violation; N/A with reason → ok; config-only lane → ok
