## Body

## Parent

Epic #1604. Adopts finding F2 from research #1624.

## Goal

Create a Projects v2 board as the live cross-team state plane. Custom fields: claimed-by (team), locked-paths (text), in-flight-since (date), expected-completion (date), cross-team-stage (text). Every team's baton transition queries + updates this board via GraphQL, giving the harness a real-time "who is working on what" view that replaces VS Code Source Control clutter.

## Portability annotation (per #1628)

- **Resource dependency**: Projects v2 requires GitHub org-level + project-level access. Same baseline as Issues/PRs — every operator working with this repo has it via gh CLI.
- **Network dependency**: GraphQL endpoint at api.github.com. Air-gapped operators fall back to local file-based state per #1624 F10 (decisions.md pattern) — to be addressed in a separate child if needed.
- **Classification**: integral to harness goals when GitHub-connected; **opt-out fallback documented** for air-gapped operators.
