# #1159 — Dashboard HAMR panel: 4-widget coverage view

> **Source**: github:issue/1159 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:dashboard, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 1159` (derived; edit the GitHub item, not this page).

## Body

## Problem
The harness has no consolidated view of HAMR **coverage / cost-control** state. The zero-cost-lane
governors — the premium-share governor (">20% premium over 7d forces fleet") and the cost-ascending
mandate (Free → Fleet → Haiku → Premium) — depend on signals that today are scattered across
`~/.megingjord/cache-stats.jsonl`, `substrate-health`, `header-spillover`, and `/api/quota-probes`.
Without a single auditable surface, an operator cannot *see* whether the harness is actually staying in
the free lanes (G3) — so G3 cannot be governed (G8-in-service-of-G3).

Origin: #1130 D-1148-008. Dependencies #1149/#1150/#1153/#1154 are **all closed** (data producers exist).

## Goal-lens framing
Primary **G3** (make zero-cost-lane adherence visible + flag breaches) in service via **G8**
(auditable/attributable). Secondary: G4 (no secret values rendered), G6 (graceful when a source is stale/
absent), G9 (browser-portable, no runtime coupling), G10 (single shared aggregator, no per-widget refetch).

## Boundary vs. existing panels (de-conflict — required)
This panel owns **exactly the G3 signals no other panel covers**, all derived from one source
(`~/.megingjord/cache-stats.jsonl`) behind one endpoint `/api/hamr-coverage` (single source, G10):
**cache-hit-rate-7d**, **per-provider call mix**, **premium-share governor**, **tier-mix** (free/fleet/
haiku/premium). It does NOT re-render quota/cost (owned by `quota-tracker.js`/`quota-live.js`) or the
LLM-choice log (owned by `router-tracker.js`); a footer link defers there. (Live `/quota` freshness +
spillover-frequency widgets are explicitly OUT of scope here — separate sources/panels — noted so the
single-source G10 invariant holds.)

## Development plan
1. **Aggregator (G2/G8/G10)** — `dashboard/api/hamr-coverage-handlers.js#handleHamrCoverage`: reads
   `~/.megingjord/cache-stats.jsonl` (hit-rate-7d + per-provider counts), `substrate-health` (tier/health),
   `header-spillover` (spillover frequency), and `/api/quota-probes` freshness; returns a single JSON
   `{ coverage_rate, providers:[{name,calls,share}], quota:{fresh,age_ms,push_failures_24h}, spillover:[...],
   premium_share_7d, governor:{breached,threshold} }`. Pure shaping functions exported for unit tests.
2. **Dispatch** — one route line in `scripts/dashboard-server.js` (mirrors `/api/goal-coverage`).
3. **Panel (G3/G4/G9)** — `dashboard/js/hamr-panel.js`, 4 widgets, mirroring `goal-coverage-panel.js`:
   coverage-rate gauge, per-provider call-rate (stacked), /quota staleness+push-failure flag, spillover
   heatmap. **Active G3 element**: the gauge/flag turns warning when `governor.breached` (premium-share
   >20%/7d) or cache-hit < SLO. Renders NAMES/counts only — never a secret value (G4).
4. **Mount + live** — `<script src="js/hamr-panel.js">` in `dashboard/index.html` + `render-panels.js`
   hook; live via `subscribePanelSSE('cache-stats', …)` (shared EventSource, <500ms), not 6h polling.
5. **a11y + motion** — WCAG 4.5:1 (brightness/shape, not color-only) + `prefers-reduced-motion: reduce`
   honored (snap, no 1.6s animation) per `visual-qa-governance` + `observability.instructions`.

## Acceptance Criteria (testable, goal-mapped)
- [ ] **AC1 (G3)** Four widgets render from `/api/hamr-coverage`: cache-hit-rate-7d gauge, per-provider
      call-mix, premium-share governor flag, tier-mix (free/fleet/haiku/premium) breakdown.
- [ ] **AC2 (G3 active)** The coverage gauge/flag shows a WARNING state when `governor.breached` (premium-share
      >20% over 7d) OR cache-hit-rate < SLO; OK state otherwise. Asserted by a unit test over the shaper.
- [ ] **AC3 (G10 single-source)** All four widgets consume the one `/api/hamr-coverage` endpoint; grep proves
      no widget re-implements quota/cost rendering already owned by quota-tracker/router-tracker (AC8 boundary).
- [ ] **AC4 (G8 live + auditable)** Panel updates via shared-EventSource SSE (`subscribePanelSSE`), ≤500ms
      p95 append-to-render; each widget shows its source + last-update age.
- [ ] **AC5 (G4 secret-safe)** No secret value is ever rendered; only provider NAMES, counts, rates, ages.
- [ ] **AC6 (G6 graceful)** A missing/stale/empty source degrades to a labeled "no data / stale" state, never
      a broken panel or thrown error. Asserted by unit test over the aggregator (empty + malformed inputs).
- [ ] **AC7 (a11y + motion)** WCAG 4.5:1 contrast (non-color-only signaling) and `prefers-reduced-motion`
      honored. Captured in the VISUAL_QA_EVIDENCE block.
- [ ] **AC8 (G1 de-conflict)** Boundary section above reflected in code: panel consumes shared signals; does
      not duplicate quota-tracker/quota-live/router-tracker. Cross-checked at review.
- [ ] **AC9 (lint/cap)** Every new/edited file ≤100 lines; readability ≤ repo baseline.
