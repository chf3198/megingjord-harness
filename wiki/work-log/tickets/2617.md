# #2617 — Enforce cross-ticket overlap check on every new ticket

> **Source**: github:issue/2617 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P1, area:governance, lane:code-change

## Body

## Parent
Refs Epic #2623

## Objective
Make cross-ticket conflict/redundancy analysis a required gate for every newly created ticket.

## Problem
Overlap checks are currently ad hoc and operator-memory dependent. This allows duplicate or conflicting tickets to be filed under adjacent epics.

## Acceptance Criteria
- [ ] Add a deterministic pre-ticket check step that scans open/closed issues for semantic overlap and explicit dependency candidates
- [ ] Require `related_tickets:` and `overlap_decision:` fields in MANAGER_HANDOFF schema for all new tickets
- [ ] Enforce handoff block when overlap is high and no boundary decision is recorded
- [ ] Emit machine-readable evidence on-ticket showing links chosen (blocked by / blocking / related)
- [ ] Add tests covering duplicate detection, false-positive guardrails, fallback behavior when GitHub search is degraded

## Enforcement-Now Contract
- This ticket is the implementation vehicle to promote overlap checks from advisory to enforced manager gate.
- Until merged, every new ticket under Epic #2623 must include explicit overlap decision against #2574, #2091, and #2071 in MANAGER_HANDOFF.
