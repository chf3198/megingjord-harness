## Problem

The harness uses a graduated 4-tier enforcement model:
- Tier 1: compiler/type — impossible to violate
- Tier 2: runtime hook — refused at action time
- Tier 3: CI gate — blocked at PR time
- Tier 4: honor-system documentation

Per `instructions/role-baton-routing.instructions.md` "Flaw-recognition anneal decision (required)", whenever an agent recognizes a flaw, it MUST record one of: `file-ticket` | `log-incident-only` | `memory-note-only` | `no-action-justified`. Per `instructions/workflow-resilience.instructions.md` Tier-2 trigger, recurrence ≥2 in 7d at severity ≥ medium MUST result in a filed ticket.

**But the anneal-decision contract itself is tier 4.** No script verifies that recognized flaws produced a recorded decision. The only catcher is operator scrutiny. When the operator doesn't catch it, the contract silently decays and the recursive self-improvement loop the harness depends on quietly stops.

The worktree-isolation contract this session's Epic #1854 will fix is also tier 4 — BUT #1854 itself depends on the anneal-decision contract being honored to file it. If the anneal-decision contract decays first, #1854 never gets filed, and the worktree contract never gets upgraded. The brittleness cascades through the whole self-improvement chain.

## Proposed AC

- AC1: `SessionEnd` hook (or `Stop` hook) scans session transcript for flaw-recognition markers: "I notice", "this is a recurrence", "trap class", "violation", "drift", "should we", "if you want", "perhaps we should file", "worth filing", "could be filed".
- AC2: For each marker matched, hook checks `~/.megingjord/incidents.jsonl` (or equivalent governance state) for a corresponding decision record (`file-ticket`/`log-incident-only`/`memory-note-only`/`no-action-justified`) within ±2 minutes of the marker timestamp.
- AC3: If marker count > recorded-decision count, session-end emits a checklist of unmatched recognitions + blocks clean session exit until each is resolved.
- AC4: Cross-runtime: implement the hook contract for Claude Code (`~/.claude/hooks/`), Copilot (`~/.copilot/hooks/`), Codex (`~/.codex/devenv-ops/hooks/`). Adapter-exemption matrix per cross-team contract.
- AC5: Replay-eval (#1771 pattern): replay session transcripts where recognition was followed by missing-decision (today's "if you want" exchange + historical sessions per `feedback_anneal_emission_during_implementation.md`). Proposed hook should flag all of them.
- AC6: `npm run governance:anneal-decision-audit` standalone command for ad-hoc audit.
