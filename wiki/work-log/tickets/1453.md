# #1453 — Epic CONSULTANT_CLOSEOUT must reference substantive evidence (not just schema fields)

> **Source**: github:issue/1453 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:governance, lane:code-change
> Mirror of `gh issue view 1453` (derived; edit the GitHub item, not this page).

## Body

## Problem

Current `scripts/global/megalint/consultant-closeout.js` checks that an Epic's CONSULTANT_CLOSEOUT comment contains required schema fields (Signed-by, Team&Model, Role, G1-9 rubric, verdict, verification timestamp). It does NOT check that the closeout has any substantive content.

A phantom closeout like the one on Epic #1436 — rubric numbers + verdict + timestamp + signer, with **zero** mention of children, PR, research artifacts, or shipped work — passes the validator entirely.

## Observed evidence

Epic #1436 (Worker mid-flight self-anneal — research-first Epic per #1397 Phase Gate Rule) received a CONSULTANT_CLOSEOUT minutes after filing, with no Phase-0 children filed, no research artifacts produced. The closeout passed all current schema checks because it had the right fields, just no real content. Surfaced during 2026-05-12 critical analysis.

## Goal

Require Epic-level closeouts (CONSULTANT_CLOSEOUT or CONSULTANT_EPIC_CLOSEOUT) to reference substantive evidence:
1. List of completed children (`#N` refs in body)
2. OR linked PR(s) with merge state
3. OR linked research artifact paths under `research/`

If none of the above present, the closeout is rejected as substance-empty.

## Acceptance Criteria

- [x] AC1: Add `checkSubstantiveContent(body, isEpic)` to `scripts/global/megalint/consultant-closeout.js`. Only fires when input is an Epic (`labels` includes `type:epic`).
- [x] AC2: Validator requires AT LEAST ONE of: (a) `#NNN` ref to a child issue, (b) `PR #NNN` or `pull/NNN` reference, (c) `research/*.md` path reference.
- [x] AC3: Violation rule `epic-closeout-no-substantive-evidence`; detail names which evidence types were missing.
- [x] AC4: Test fixture pair — synthetic closeout with no evidence → fails; same closeout + `#100 closed` reference → passes.
- [x] AC5: 5+ golden-file tests covering each evidence type and the no-evidence rejection.
- [x] AC6: Wired into existing `closeout-lint.yml` (already calls consultant-closeout); promotes to required after 2-week soak.
