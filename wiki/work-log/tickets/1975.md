## Trigger

## Root cause (newly identified)

The #1960 fix correctly addressed:
- `.claude/` path exclusion in `stop_checks.py:check_uncommitted`
- BASH_TOOLS suppression in `tool_activity.py` to prevent path classification on Bash inputs
- Clean-tree-no-commit early return in `stop_checks.py:check_admin_ops`
- `uncommitted` parameter wiring through `stop_reminder.py`

But the fix did NOT address **state persistence across cross-task boundaries**. The state store at `~/.copilot/hooks/state/repo-<sha1>.json` retains flags and admin_ops counters across:

- Branch changes within a session (no reset on `git checkout <new-branch>`)
- Cross-task work (no reset when moving from one ticket to another in the same session)
- Sessions (no expiry; flags from prior session reapply on next session start)

Live state at time of recurrence:
```
flags.code_touched: true       (set during earlier #1963 work)
admin_ops.commit: true         (commit from #1963 work)
admin_ops.push: true           (push from #1963 work)
admin_ops.pr_create: false     (PR creation was blocked by cross-team collision)
admin_ops.ci_green: false      (no PR → no CI)
admin_ops.merge: false         (no PR → no merge)
roles.collaborator: true       (lingering)
roles.consultant: true         (lingering)
admin_ops.issue_linked: false  (despite many gh-issue references)
```

Resulting recurring misfire: "Admin baton incomplete — missing: pr_create, ci_green, merge" — true for #1963 (its PR was blocked), but spuriously applied to ALL subsequent tool invocations in the session, including pure-research tool calls on unrelated tickets.

## Recurrence evidence

- pattern_id candidate: `stop-hook-state-persistence-cross-task`
- 2026-05-19 single-session count: 8+ post-#1960-merge
- 2026-05-19 pre-#1960-fix count: 2 (filed as #1960 origin)

## Scope

1. State reset on branch change: when `git rev-parse --abbrev-ref HEAD` changes between hook invocations, clear `flags.code_touched`, `admin_ops.*`, `roles.*` (except keep `routing.*` and `repo_type`).
2. State reset when working tree is clean AND HEAD has advanced past the last-recorded commit: this indicates the previous task's work merged + the session is now on a different sub-task.
3. Per-ticket state scoping: replace single-state-per-cwd with one state per (cwd, active_ticket) tuple; active_ticket derived from branch name regex per existing `RE_BRANCH_ISSUE` pattern.
4. Session boundary marker: SessionStart hook posts a `session-id` into state; subsequent hooks can detect session change and prompt for reset.
