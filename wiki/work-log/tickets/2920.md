## Body

## Problem
Per `instructions/epic-governance.instructions.md` Rule E5, Epics at `status:dormant` require a 90-day EPIC_REVIEW comment. This review is manual — no cron automation exists to trigger the review or alert when the 90-day window passes. Epics silently sit dormant without the required review comment.

## Acceptance Criteria
- [ ] Create `.github/workflows/epic-dormant-review.yml`: runs on schedule (daily), checks all open Epics with `status:dormant` label
- [ ] If an Epic has been dormant > 90 days without an `EPIC_REVIEW` comment: post a reminder comment tagging `role:manager`
- [ ] Configurable via `EPIC_DORMANT_AFTER_DAYS` env var (per Epic #1342)
- [ ] Add tests: `tests/epic-dormant-cron.spec.js`
