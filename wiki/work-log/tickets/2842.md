## Problem

`cascade-dispatch.js` via HAMR-wrapped `callOllamaOnce` produces **zero stdout/stderr** during the 3–8 minute inference window for a 32b model. The operator cannot distinguish:
- Still running (model loading + generating tokens)
- Silently failed (connection dropped, OOM on fleet host)
- Blocked (HAMR rate-limit or queue)

This caused the operator to cancel the first iteration prematurely, then re-dispatch with `--json` which escalated to free-cloud (Gemini) instead of fleet — a G3 cost-routing violation triggered by the observability gap.

Additionally, `fleet-red-team-dispatch.js` is a module with no CLI entry point (no `main()`). Its file comment implies it is CLI-invocable, but `node fleet-red-team-dispatch.js --help` exits silently with no output.

## Root Cause

1. `callOllamaOnce` calls `/api/generate` with `stream: false` — entire response buffered server-side; no incremental progress
2. HAMR `wrapProviderCall` resolves only when the full envelope arrives — no intermediate signals
3. `fleet-red-team-dispatch.js` exports functions but has no `if (require.main === module) main()` guard

## Acceptance Criteria

- [ ] `cascade-dispatch.js` emits a stderr heartbeat line every 30s during Ollama inference: `[cascade] inference in progress: <elapsed>s elapsed, model=<model>`
- [ ] When inference completes, emit: `[cascade] inference complete: <elapsed>s, <tokens> tokens`
- [ ] `fleet-red-team-dispatch.js` gains a `main()` CLI entry point with `--content`, `--artifact-type`, `--ticket` args and `if (require.main === module) main()`
- [ ] `npm run lint` passes on both modified files (≤100 lines each; split if needed)
