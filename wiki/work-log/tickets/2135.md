# #2135 — Research and specify goal-precedence model for rubric scorer

## Parent Epic

Refs #2134

## Objective

Evaluate the three candidate precedence models for `rubric-score.js` and
recommend one with documented rationale.

## Candidates to evaluate

1. **Lexicographic minimum** — score = min(G1..G10); any single goal at 0
   produces a score of 0 regardless of others. Simple, strict.
2. **Priority weights** — G1 weight=10, G2 weight=9, ..., G10 weight=1;
   weighted mean. Preserves partial credit but gives G1 failures more impact.
3. **Hard-fail flag per goal** — each goal in the schema carries
   `hard_fail: true|false`; any hard-fail goal scoring 0 makes
   `precedence_verdict: fail`. Flexible, explicit per-goal control.

## Acceptance Criteria

- AC1: Each candidate evaluated against at least 3 real closeout examples
  (good, borderline, failing) and scored under each model.
- AC2: Recommendation documented with rationale citing the goal constitution
  and observed failure modes.
- AC3: Schema changes required by the chosen model specified (fields, types,
  defaults).
- AC4: Findings posted as a comment on parent Epic #2134 before this ticket
  closes.

## Constraints

- Do not implement — output is a specification only.
- Backward compatibility with existing `boxes_checked / boxes_total` per-goal
  scoring must be preserved (the new model adds a `precedence_verdict` field,
  does not replace existing output).
