# #2659 — Attribute team by runtime, not only by model family (auditability)

> **Source**: github:issue/2659 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P2, area:governance, resolution:released, lane:code-change, anneal:tier-2
> Mirror of `gh issue view 2659` (derived; edit the GitHub item, not this page).

## Body

## Self-anneal origin (Tier-2)
Surfaced 2026-06-04. Cross-team stranded-WIP work was mis-attributed to the "Codex Team" when the actual writer was the **Copilot Team (Auto mode)**. Two compounding causes: (a) operator inferred authorship from an idle process name (operator-memory `feedback_verify_cross_agent_authorship_not_process_name`); (b) **the harness derives "team" from MODEL FAMILY, not RUNTIME**.

## Root cause
`inventory/team-model-signatures.json` + `scripts/global/agent-signature.js` map team by model pattern — e.g. `gpt-5.4` / `gpt-5*codex` ⇒ team `codex`. So a single **runtime** (Copilot in Auto mode) routing to different models is attributed to **different teams** (gpt-5-codex⇒codex, claude⇒claude-code, gemini⇒google/unmapped). "Codex Team" in a baton artifact or incident can therefore actually be "Copilot runtime on a codex-class model" — the **runtime is conflated with the model-derived team**, undermining G8 "who did this?" auditability.

## Need
Distinguish **runtime/agent-surface** (which extension: Claude Code / Copilot / Codex / Antigravity) from **model-team** (the model family). Either:
- Add a runtime field to signing/attribution (sourced from the active extension / HAMR_TEAM / an explicit runtime marker), recorded alongside model-team, OR
- Make team attribution runtime-primary with model as a secondary qualifier.

## Acceptance (Phase-0 research-first)
- AC-R1: audit every attribution surface (signatures, incidents, baton artifacts, dashboards) for runtime-vs-model-team conflation.
- AC-R2: design a runtime-aware attribution contract; Consultant rubric ≥7 (min G1-G10); cross-family rater AGREED-A+.
