# #2601 — Keep fleet review model warm via configurable keep_alive (cut cold-load paid fallback)

> **Source**: github:issue/2601 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:scripts, area:infra, resolution:released, lane:code-change
> Mirror of `gh issue view 2601` (derived; edit the GitHub item, not this page).

## Body

## Problem (G3 — Zero Cost)
Fleet review dispatch (`fleet-red-team-dispatch.js` `callOllamaOnce`) calls Ollama `/api/generate` with only `{ temperature, num_predict }` — **no `keep_alive`**. So after a review the model unloads at Ollama's default TTL (~5 min), and the *next* review cold-loads it (slow on the shared host), times out, and the caller falls back to a **paid cloud** reviewer.

This is the direct complement to #2599: #2599 made `selectModel` *prefer* an already-resident model, but nothing keeps a model *resident* between the back-to-back reviews of a baton cycle (collaborator → admin → consultant, plus the next ticket's reviews). Firsthand: repeated cold-load timeouts → paid fallback across this multi-ticket session.

## Objective
Keep the fleet review model warm between reviews via a configurable `keep_alive`, so resident-preference (#2599) consistently has a warm model and review work stays on the **free fleet**.

## Acceptance Criteria
- AC1: `callOllamaOnce` includes `keep_alive` in the `/api/generate` request body.
- AC2: `keep_alive` is settings-driven (G5): `process.env.FLEET_KEEP_ALIVE` overrides a sensible default (e.g. `30m`); invalid/empty → default.
- AC3: Unit test (mocked fetch) asserts the request body carries the resolved `keep_alive` (default + env-override cases).
- AC4: No change to selection/cross-family/retry semantics; default chosen to balance warmth vs. shared-host VRAM (documented).
- AC5: CHANGELOG fragment.

## Goal Lens
- G3 Zero Cost (primary): sustains free-fleet residency → fewer cold-load timeouts → less paid-cloud fallback for governed review work.
- G5 Portability: keep_alive is env/settings-driven, not hard-coded per host.
- G6 Resilience: invalid value → safe default.
