## Body

## Objective

Build a cross-orchestrator compatibility test suite that ANY AI Agent Orchestrator (Claude Code, Codex, Copilot, Antigravity, future entrants) can execute to self-verify it is compatible with all the harness' features. Per client direction at 2026-05-29 after the Antigravity 100%-parity sequence completed.

## Background

The Antigravity 100%-parity sequence (closed today) shipped 4 PRs adding antigravity recognition across signer-registry, routing-policy, deploy-target, plugin-manifest, and dashboard surfaces. During the gap-check discipline that informed each ticket, 9 of 13 originally-proposed tickets turned out to be symmetric absences (no orchestrator has X today) rather than asymmetric gaps (only Antigravity is uncovered). That insight motivates this suite: a single test runner that surfaces, for any candidate orchestrator, exactly which harness surfaces recognize it and which do not — so future onboarding of new orchestrators uses gap-check discipline by default.

## Acceptance Criteria

- AC1: `tests/orchestrator-compatibility.spec.js` enumerates the 6+ harness surfaces that REQUIRE per-orchestrator registration today: signer-registry team entry, routing-provider-adapters runtimeKinds, deploy.sh target regex, validate-plugin-compat.js (optional path), dashboard VENDOR_ICONS + VENDOR_COLORS + CSS class, and orchestrator-governance-parity.json runtimes array
- AC2: Each test takes an `ORCHESTRATOR` env var (e.g., `ORCHESTRATOR=antigravity npm run test:compatibility`) and asserts the named orchestrator is registered at each surface
- AC3: When env var is unset, the suite runs against all 4 known orchestrators (claude-code, codex, copilot, antigravity) and produces a parity matrix
- AC4: Documentation `docs/howto/orchestrator-compatibility-suite.md` describes the suite, when to run it, and how to add a new orchestrator
- AC5: Suite ships through the normal four-role baton with cross-family fleet adversarial review
- AC6: Test outputs a parity-report JSON to `~/.megingjord/orchestrator-parity-<timestamp>.json` for observability
