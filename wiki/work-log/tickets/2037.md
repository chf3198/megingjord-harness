## Body

## Origin

Operator observation 2026-05-21: Across all three runtime teams (Claude Code, Copilot, Codex), the AI models perform many Agile-role-workflow checklist tasks via non-deterministic LLM generation. Examples:

- Manager handoff text (scope, acceptance, gates, anneal_tickets_filed) — LLM generates the prose each time
- Collaborator handoff (per-AC verification summary) — LLM generates
- Admin handoff (independence statement, deploy notes) — LLM generates
- Consultant closeout (rubric scores, anneal_decision) — LLM generates (rubric-score.js IS deterministic but the assembled comment isn't)
- Commit message construction (subject + body + signing trailers) — LLM generates
- PR body construction (Refs/Closes lines, baton-artifact reference, lane declaration) — LLM generates
- Changelog fragment text — LLM generates

By contrast, certain tasks ARE programmatic:
- Label edits via `gh issue edit --add-label / --remove-label`
- SHA-256 manifest generation
- Rubric scoring via `scripts/global/rubric-score.js`
- OWASP coverage audit (`owasp-coverage-audit.js`)
- Cyclomatic complexity report (`complexity-report.js`)

The hybrid pattern (some programmatic, some LLM-generated) creates variability: models of differing capabilities and intelligence produce inconsistent baton artifacts. Smaller models on auto-mode (gpt-3.5-mini, gpt-5.3-codex-mini) generate malformed handoffs that fail validators; larger models (opus-4.7, gpt-5.5-codex) generate well-formed but stylistically divergent artifacts.

Operator hypothesis: **if all Agile-role-checklist tasks were programmatically performed (template-rendered from structured input, not LLM-generated), behavior would be consistent across all model capabilities.**
