## Body

## Problem

`merge-evidence-reconciler` workflow applies the `governance:close-without-merge` advisory label to closed tickets whose linked PR used the `merge-evidence-deferred-final: #N` form instead of the GitHub-native `Closes #N` form. The deferred-finalize form is the PREFERRED merge-evidence pattern per `instructions/global-standards.instructions.md` Deferred-finalize merge-evidence contract (Epic #2295 P1.3):

> Preferred — `merge-evidence-deferred-final: #N`: satisfies `merge-evidence-pr-gate` WITHOUT triggering GitHub auto-close on merge.

The reconciler bot applies the advisory anyway, generating false-positive noise on every deferred-finalize close.

## Observed false-positives this session

- #2355 (sync.sh guardrail; PR #2358 merged; deferred-final form used) — advisory applied, manually removed by Consultant
- #2360 (Antigravity parity; PR #2361 merged) — advisory applied, still showing on issue

## Acceptance Criteria

- AC1: `merge-evidence-reconciler` parses linked PR body for `merge-evidence-deferred-final:` token (case-insensitive); if found and PR is MERGED, classify the ticket as merge-evidence-present and do NOT apply `governance:close-without-merge`.
- AC2: Existing `Closes #N` / `Fixes #N` / `Resolves #N` recognition preserved (backward-compat per Deferred-finalize merge-evidence contract).
- AC3: Unit test fixtures cover: deferred-final form + merged PR (no advisory), Closes form + merged PR (no advisory), no form + merged PR (advisory applied), deferred-final form + unmerged PR (advisory applied).
- AC4: Existing tickets carrying false-positive advisory (#2355, #2360 at minimum) cleared as collateral cleanup.
- AC5: Validator-test promotion to required-mode after one cycle of advisory soak.
