## Governed MCP tool adapters for fleet models (P1-0 child of #2802)

Parent: #2802 (P1-0 fleet context-access layer). Phase-0 source: #2792 (D14, G4). Sibling: #2844
(sandbox test-exec trust gate). Client design direction (UAT 2026-06-09) locks the permission boundary.

### Problem
The fleet context loop (#2815/#2817/#2823/#2831) grounds a fleet model and #2844 validates its proposed
changes. To run an agentic loop the model needs **tools** (live GitHub reads, wiki/repo-map). Exposing
tools to a fleet model is an OWASP **OA2 (Tool Misuse)** + **OA8 (Insecure Communications)** + **OA9
(Human-Agent Trust)** surface — an ungoverned tool could mutate GitHub state, leak the operator token,
or impersonate a baton governance artifact.

### Client-decided permission boundary (UAT)
- **Read + self-comment-write**: fleet tools = github_read, wiki_search, repo_map (read) + a single
  self-comment write (post the model's OWN analysis). Labels/close/merge/file-writes are operator-only
  (via the #2844 propose→sandbox→apply path).
- **Read-only GitHub network OK; token brokered** — the local adapter brokers GitHub calls with the
  operator token; the token is NEVER exposed to the fleet model.

### Objective
A governed MCP tool adapter layer: a typed tool catalog with per-tool permission classes, an OA2
authorization gate that runs BEFORE every tool execution (deny unknown / forbidden-class / malformed),
and a broker that executes authorized read + self-comment tools with the token brokered and side-effects
injectable. Self-comment bodies are guarded against impersonating baton artifacts (OA9).

### Acceptance criteria
- AC1: typed tool catalog — github_read, wiki_search, repo_map (perm:read), github_self_comment
  (perm:self-comment-write); each declares its permission class.
- AC2 (OA2): `authorizeToolCall` denies unknown tools, any not-allowed permission class, and malformed
  args — runs before any execution; default-deny.
- AC3 (OA9): github_self_comment refuses a body that impersonates a baton artifact
  (MANAGER/COLLABORATOR/ADMIN/CONSULTANT handoff/closeout or signing fields) or is empty.
- AC4 (OA8/G4): the broker brokers GitHub calls with the operator token; the token is never returned to
  the caller / never in a tool result. Side-effects (gh exec, wiki, fs) injectable → tests network-free.
- AC5: read tools reuse shipped primitives (repo_map ← fleet-context-bundle.repoMap; wiki ← wikiContext);
  no parallel RAG (D13 non-redundancy).
- AC6: each new file <=100 lines; readability <=475.
- AC7: tdd-pyramid spec + stress spec (>=1 chaos/privilege-escalation path, >=1 p99 budget).

### Gates
lint, lint:js, lint:readability:ci, test-evidence (tdd-pyramid+stress), baton-gates, closeout-schema,
merge-evidence-pr-gate.

### Lane / strategy
lane:code-change. test_strategy: tdd-pyramid+stress-test.

### overlap_decision
no-overlap. Reuses fleet-context-bundle primitives (repo_map, wiki) and the #2844 trust gate; adds the
NEW governance primitive (typed tool catalog + OA2 authorization gate + token-brokering broker). Not a
parallel MCP server — adapters + permission boundary only; MCP-server lifecycle/embeddings are follow-on.
