# #2885 — Governor stress test pollutes production telemetry (no injected emit)

> **Source**: github:issue/2885 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P3, area:scripts, lane:code-change
> Mirror of `gh issue view 2885` (derived; edit the GitHub item, not this page).

## Body

## Problem

The #2796 governor stress test (`tests/stress-fleet-dev-governor.spec.js`) and any test that calls
`governClass` without an injected `emit` writes Tier-2 demotion records to the **production** telemetry
file `~/.megingjord/fleet-dev-governor.jsonl` via `defaultEmit`. The #2796 p99 loop (3000 iterations,
each demoting class `c`) wrote **~9000 junk demotion records** of `task_class:"c"` to the real file.

Discovered while shipping #2800 (the fleet-cost observability report): the human-audit surface read the
polluted file and reported `demotions_by_class: {"c": 9012}`.

## Impact

- Pollutes production fleet-dev telemetry → the #2800 cost/audit report shows test garbage.
- Test-isolation violation: unit/stress tests must not write to real `~/.megingjord/*` surfaces.

## Acceptance criteria
- AC1: every `governClass` call in `tests/*governor*.spec.js` injects `{ emit: () => {} }` (no real write).
- AC2: a guard so a future governor/cost test can't silently write prod telemetry (e.g. default emit no-ops
  under a test env flag, OR a lint/test asserting no test writes to `~/.megingjord`).
- AC3: clean the existing `~/.megingjord/fleet-dev-governor.jsonl` pollution (IT-ops, fleet-local).
