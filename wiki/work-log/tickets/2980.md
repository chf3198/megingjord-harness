# #2980 — test-evidence tdd-pyramid check does not recognize Python pytest files

> **Source**: github:issue/2980 | **state: OPEN** | **Labels**: type:bug, status:backlog, priority:P3, area:scripts, lane:code-change
> Mirror of `gh issue view 2980` (derived; edit the GitHub item, not this page).

## Body

## Problem
`scripts/global/test-evidence-validator.js` tdd-pyramid check matches only `tests/**.{spec,test}.{js,ts}` — but the test-methodology matrix maps `hooks/scripts/*.py` to **tdd-pyramid (pytest)**. So a Python-only hook fix (e.g. #2978) fails `test-evidence` with `missing-spec-file` unless the author adds a JS shim wrapper (the current de-facto workaround, e.g. injection-guard-2905.spec.js, and #2978's shim).

## Impact
- Every Python-hook ticket must hand-add a JS shim or eat a false `test-evidence` failure (recurrence: Tier-1 `pattern_id=test-evidence-tdd-pyramid-rejects-python-pytest`). G2/G10 friction; risks authors mis-declaring strategy to dodge it.

## Fix direction
- Extend the tdd-pyramid checker to also accept `tests/**/test_*.py` (or `tests/hooks/test_*.py`) when the PR diff touches `hooks/scripts/**` / `*.py`. Keep the JS spec path for JS surfaces.

## Evidence
- Surfaced on #2978 (Epic #2647). Tier-1 incident emitted at failure time.
