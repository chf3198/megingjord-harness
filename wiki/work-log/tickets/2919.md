# #2919 — Add dynamic least-privilege per-tool enforcement based on active baton role

## Body

## Problem
An agent in Collaborator phase retains all tool permissions available in Manager or Admin phase. There is no dynamic capability revocation as the baton advances (Gap G-16). A baton-phase boundary should limit available tools: e.g. Collaborator cannot push to main; Admin cannot create new branches.

This is ASI02 (Tool Misuse) + ASI06 (Rogue Agents) + EU Art.15.

## Acceptance Criteria
- [ ] Define role-scoped tool allowlists in `config/authorization-profiles.json` (or new `config/role-tool-allowlist.json`)
- [ ] In `hooks/scripts/pretool_guard.py`, check active baton role against tool allowlist before permitting the call
- [ ] Manager phase: read/create/triage allowed; no direct push to main
- [ ] Collaborator phase: edit/read/create allowed; no merge, no deploy
- [ ] Admin phase: push/merge/deploy allowed; no new branch creation from admin
- [ ] Add tests: `tests/role-tool-allowlist.spec.js`
