## Body

## Problem
The `collaborator-gate` job in `.github/workflows/baton-gates.yml` only checks that the COLLABORATOR_HANDOFF comment exists (via `trail.includes('COLLABORATOR_HANDOFF')`). It does not verify that the comment contains a valid `doc-coverage:` block with all required surfaces.

## Objective
Add a CI step to `collaborator-gate` that reads the COLLABORATOR_HANDOFF comment body, extracts area labels from the linked issue, and verifies the `doc-coverage:` block covers all required surfaces with DONE or N/A+reason.

## Acceptance Criteria
- [ ] New step in `collaborator-gate` job after existing COLLABORATOR_HANDOFF presence check
- [ ] Step reads linked issue's area labels via GitHub API
- [ ] Step queries `config/doc-coverage-matrix.yml` for required surfaces per label
- [ ] Step parses `doc-coverage:` block from COLLABORATOR_HANDOFF comment body
- [ ] `core.setFailed()` called if any required surface is missing or has bare N/A
- [ ] Step is skipped for lane:docs-research and lane:config-only (check lane from comment body)
- [ ] `DOC_COVERAGE_GATE_ADVISORY=1` env var makes gate non-blocking (emits warning instead) per G5 portability escape hatch
- [ ] Error message cites the specific missing surface(s) and links to template docs
