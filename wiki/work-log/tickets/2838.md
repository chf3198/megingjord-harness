## Problem
`runHealthChecks` always returns `penguin-1` as degraded regardless of scenario, causing the page header to always show `Megingjord degraded` even in the Healthy Agent Session demo. The status should reflect the active scenario.

## Acceptance Criteria
- [ ] `fixture-runner.js` writes `window.demoConfig = window.demoConfig || {}; window.demoConfig.currentScenario = scenarioName` on scenario load
- [ ] `runHealthChecks` reads `window.demoConfig.currentScenario`: returns all-healthy for `healthy-session`, `cost-routing`, `mcp-discovery`, `multi-agent`; returns one degraded device for `governance-alert`
- [ ] `fetchFleetHealthLog` similarly varies latency/status by scenario
- [ ] Page header shows `Megingjord healthy` when Healthy Agent Session is active
- [ ] Page header shows `Megingjord degraded` when Governance Alert is active
