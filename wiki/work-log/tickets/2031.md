## Problem
Developers and agents need a CLI tool to programmatically repair and reconcile any ticket governance drift (auto-linking, status updates, progress update logging) without performing manual GraphQL queries.

## Scope
1. Create `scripts/global/actuator-epic-sync.js` that parses a target Epic number.
2. The script must:
   * Perform native sub-issue associations via GraphQL.
   * Auto-transition Epic status based on sub-issue progress.
   * Auto-post structured `## Epic Progress Update` comments on closed child tickets.
3. Keep the file within the strict 100-line cap.

## Acceptance criteria
[ ] `scripts/global/actuator-epic-sync.js` fully implemented and adheres to 100-line limit.
[ ] Interactive command `npm run epic:sync -- --epic <number>` runs cleanly.
[ ] Reconciles parent-child status and links programmatically.
