# #1912 — Unify governance parity across orchestrators

> **Source**: github:issue/1912 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P1, area:governance
> Mirror of `gh issue view 1912` (derived; edit the GitHub item, not this page).

## Body

## Epic intent

Critically examine whether the Megingjord harness provides the same governance outcomes across the Claude Code Team, Copilot Team, and Codex Team. The portability goal requires that all three orchestrators receive the same protected governance features, with runtime-specific adapters allowed only when they are explicitly mapped, tested, and shown to preserve equivalent outcomes.

## Triggering concern

A portability audit on 2026-05-19 found material differences between local adapter behavior and the project goal that governance should be equivalent for all three teams.

## Initial findings

1. Claude Code is documented as intentionally hook-exempt even though current Anthropic documentation supports Claude Code hooks in `settings.json`.
2. Copilot and Codex hook registrations are not event-equivalent: Copilot wires `PreCompact`, `SubagentStart`, and `goal_lens.py`; Codex wires neither `PreCompact` nor `SubagentStart`, and does not wire `goal_lens.py` or Codex's native `PermissionRequest` event.
3. Deployment/sync language uses `both` to mean Copilot+Codex, not all three teams. Claude deploy/sync is a narrow `.claude/` copy path, while Copilot and Codex receive richer runtime asset deployment.
4. Claude command coverage appears incomplete relative to repo skills; 13 repo skills did not have matching `.claude/commands/*.md` adapters during the audit.
5. Existing hook parity verification reports OK while comparing only a subset of scripts, not event coverage, registry coverage, Claude settings, deployment semantics, or skills/commands parity.
6. Current documentation codifies intentional asymmetry, which may have been acceptable for prior Epic decisions but conflicts with the stricter portability interpretation requested here.

## Research basis

- OpenAI Codex docs: Codex supports lifecycle hooks, `AGENTS.md`, skills, managed hooks, config layers, and multi-agent controls.
- Anthropic Claude Code docs: Claude Code hooks are configured through user, project, local project, and enterprise settings.
- GitHub Copilot docs: Copilot supports repository instructions, agent instructions, skills, custom agents, MCP, and hooks for coding-agent workflows.
