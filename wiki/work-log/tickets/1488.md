## Body

## Problem

`.github/workflows/cross-team-pr-parallel-check.yml` (commit 9952877, NOT YET ON MAIN — see #1486) filters parallel-PR candidates via:

\`\`\`js
if (hit.number !== pr.number && hit.user.login !== author) { … }
\`\`\`

The filter compares **GitHub login**. The parent investigation #1435 was filed precisely because a single human operates multiple agent teams (Claude Code, Copilot, Codex) from one workstation — all PRs end up authored by the same human GitHub user. The filter therefore **never fires in the exact scenario it was designed to detect**.

Proof: commits 9952877 + 59cf6a5 + 32ce291 are all `Author: Curtis Franks <<HASH:42b7a7259ce6>>` — three agent teams' work, one human author. The workflow would not have flagged any of it as parallel.

## Goal

Replace login-based team detection with **Team&Model** identity parsing.

## Acceptance Criteria

- [ ] AC1: Workflow extracts Team&Model from PR body via `Team&Model:\\s*([^@]+)@` (or commit trailers `AI-Team-Model:`); falls back to `pr.user.login` only when no Team&Model field is present

- [ ] AC3: Add unit test simulating two PRs from the same GitHub user AND same Team&Model — confirm NOT flagged
