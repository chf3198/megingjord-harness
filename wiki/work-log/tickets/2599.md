# #2599 — Prefer resident fleet model in review dispatch to cut paid-cloud fallback

> **Source**: github:issue/2599 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:scripts, area:infra, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 2599` (derived; edit the GitHub item, not this page).

## Body

## Problem (G3 — Zero Cost)
Cross-family / fleet review dispatch is **residency-blind**. `fleet-red-team-dispatch.js` `selectModel()` picks a model by stakes (`default_for_stakes`) then `fallback_chain`, never considering which model is *already resident* on the fleet host (`/api/ps`). When the requested model is cold or the host is contended, `callOllamaOnce` times out; retries hit the same cold model; the caller then falls back to a **paid cloud** reviewer.

Firsthand evidence: across this multi-ticket session, **every** cross-family review fell back from free fleet (qwen-32b timed out / another team's model resident) to paid `gemini-2.5-flash` — a ~100% fleet→paid fallback for review work. That is a direct G3 regression (`global-task-router`: Free > Fleet > Haiku > Premium; `routing-fallback` telemetry flags this).

## Objective
Prefer an already-**resident** cross-family fleet model so free-fleet reviews succeed instead of cold-loading and falling back to paid cloud — without weakening high-stakes quality.

## Acceptance Criteria
- AC1: New shared helper `scripts/global/fleet-resident.js`: `residentModels(host, fetchImpl?)` (parses `/api/ps`) and `preferResident(candidateIds, host, fetchImpl?)` → first resident candidate or null. Fetch-injectable; degrades to null on any error (graceful).
- AC2: `selectModel(taskContext, opts)` accepts optional `opts.residentModels: string[]`. When provided AND `stakes !== 'high'`, prefer a resident cross-family candidate over the stakes default (avoids cold-load). High-stakes selection is unchanged (quality first). Absent `opts.residentModels` → current behavior (backward compatible).
- AC3: Unit tests: `residentModels`/`preferResident` (mocked `/api/ps`); `selectModel` residency-preference for low/medium stakes; high-stakes unchanged; no-resident-info unchanged.
- AC4: No weakening of cross-family rule — only `cross_family_ok` candidates are eligible (unchanged).
- AC5: CHANGELOG fragment.

## Goal Lens
- G3 Zero Cost (primary): raises free-fleet hit rate, cuts paid-cloud fallback for governed review work.
- G6 Resilience: graceful (no resident info / fetch error → current behavior).
- G2 Quality: high-stakes still selects the best model regardless of residency.
