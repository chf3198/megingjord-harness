# #1211 — Test methodology matrix + per-lane test-evidence enforcement

> **Source**: github:issue/1211 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P1, area:governance
> Mirror of `gh issue view 1211` (derived; edit the GitHub item, not this page).

## Body

## Epic scope

Add a deterministic per-surface test methodology matrix to the harness governance, and enforce it via baton workflow + CI gate.

**Why now**: Epic #1130 critical-review flagged that 5 of 7 children shipped governance-signal code (lint detectors, sensors, provider wrappers) without tests. The harness has no instruction governing *when* TDD vs. contract-test vs. visual-regression vs. peer-review applies. Adding this matrix closes a coherence gap between `lane:*` labels (which exist) and test-evidence requirements (which don't).

## Background

Per 2025–2026 systematic literature review:
- TDD has small positive effect on external quality (~88% of studies)
- Productivity drops in industrial vs. academic studies
- Modern shapes (Trophy, Diamond, Skyscraper, Crab) are architecture-driven, not religion-driven
- LLM-authored code benefits from after-the-fact test generation as backstop

Reference: ScienceDirect 2025 SLR-of-SLRs on TDD; Kent C. Dodds Trophy; web.dev "Pyramid or Crab".

- **Strict launch** (Required-from-day-one gate)
