# #2263 — Research/planning lane workflow shape — does it need Collab + Admin? (research-first)

## Body

## Goal

Determine whether the research/planning lane (`lane:docs-research` and `lane:research`) requires its own simplified baton workflow that **skips the Collaborator and Admin phases** (which assume git work) and is enforced by deterministic guardrail rather than per-session memory.

This Epic is **scoped to the workflow-design question**, not implementation. Once the right shape is decided, the enforcement work itself becomes a Phase-1 child of [[Epic #2261]] (deterministic guardrail harness) so the chosen shape is cemented as a non-bypassable rule, not a memory note.

## Motivating observation

Today's multi-lane Definition of Done in `instructions/role-baton-routing.instructions.md` carries:

| Lane         | Role sequence                            | N/A markers                  |
|--------------|------------------------------------------|------------------------------|
| code-change  | Manager → Collaborator → Admin → Consultant | none                       |
| research     | Manager → Collaborator(analyst) → Admin → Consultant | Admin = doc reviewer |
| config-only  | Manager → Admin → Consultant             | COLLABORATOR_HANDOFF: N/A    |

The pattern config-only already drops Collaborator entirely. Research lane keeps both Collaborator and Admin, but ceremonially: Collaborator is renamed "analyst," Admin is renamed "doc reviewer," neither writes code, neither touches CI. Recent example: Phase-0 ticket [[#2262]] of [[Epic #2261]] required four baton artifacts (Manager / Collaborator-analyst / Admin-doc-reviewer / Consultant) when the substantive work was contained in two: scope (Manager) and peer-review (Consultant).

The user's specific question: **is this ceremony adding governance value, or just operator friction?**

## Hypotheses to test in Phase-0

- **H1**: Research lane should skip Collaborator AND Admin. Workflow: Manager → Consultant. Justification: no code, no CI, no merge. Manager scopes, Consultant peer-reviews. Symmetric to config-only's Manager → Admin → Consultant skip.
- **H2**: Research lane should skip Collaborator but keep Admin (as doc-reviewer). Workflow: Manager → Admin → Consultant. Justification: doc-review is a real independent-signer check; Consultant is the rubric judge, not the doc reviewer.
- **H3**: Research lane should keep both with stronger N/A markers. Workflow unchanged but Collaborator-analyst and Admin-doc-reviewer artifacts shrink to single-line ack. Justification: signer-independence preservation across 4 distinct aliases is a security property worth retaining.
- **H4**: Research lane should use a new "research-baton" with explicit phases not modeled on code lane. Workflow: Manager(scope) → Researcher(synthesis) → Adversary(red-team) → Consultant(rubric). Justification: adversarial red-team is becoming standard for research artifacts (per [[#2262]] adversarial dispatch pattern); making it a first-class phase formalizes practice.

## Scope

Phase-0 research-first Epic. Output: one chosen hypothesis (with operator UAT) cemented as a guardrail in Epic #2261's Phase-1 work.
