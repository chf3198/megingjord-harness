## Body

## Problem

The harness defines a **Three-Wiki** knowledge system (`instructions/wiki-knowledge.instructions.md`), but two of the three data-stores have **never been populated** and are not kept in sync with their intended sources:

| Wiki | Intended data-store | Actual (2026-06-16) |
|---|---|---|
| **A — Code-Base** (`wiki/code/`) | mirror source symbols + concepts (dev-state) | **0 symbols / 0 concepts** |
| **B — Work-Log** (`wiki/work-log/`) | mirror tickets + PRs (ticket-state) | **0 tickets / 0 PRs** (stubs only, untouched since 2026-05-22) |
| **C — Wisdom** (`wiki/wisdom/`) | research/governance | 169 global / 4 project (maintained — but **manually**, via `npm run wiki:ingest`) |

**Root cause:** [`.github/workflows/wiki-auto-update.yml`](.github/workflows/wiki-auto-update.yml) — the 11-stage pipeline (`stage3ExtractCode`, `stage4ExtractWorkLog`) designed to mirror dev-state and ticket-state on every merge — fails on **every run** as a GitHub **startup_failure** ("This run likely failed because of a workflow file issue"). Last 5 runs (incl. on `main`) all `failure`. The YAML parses locally (112 lines), so the defect is a workflow-level issue (invalid action ref / expression / context access) to be root-caused. Net effect: **the pipeline that would keep A & B current has never executed.**

## Impact (goal-lens)

- **G8 Observability**: dev-state and ticket-state are not reflected in their knowledge stores; the wiki retrieval-routing table points at empty stores.
- **G2 Quality**: agents querying Wiki A/B get nothing and silently fall back to live `gh`/`Read`, defeating the design.
- **G3 Zero Cost**: the wikis are the zero-cost retrieval layer meant to reduce token cost (≥30% reduction is the pipeline's own replay-eval gate); a dead pipeline forces premium re-derivation.

## Objective

Make Wikis **A (code / development-state)** and **B (work-log / ticket + PR state)** reliably and **continuously** auto-update — guardrailed so they cannot silently drift from their data-stores again — matching the Three-Wiki design and the bi-directional drift gate (#2058).
