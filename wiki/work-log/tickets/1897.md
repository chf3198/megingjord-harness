> **Source**: github:issue/1897 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:governance, resolution:completed, lane:code-change, governance:close-without-merge, anneal:tier-2

## Body

## Problem

The harness `closeout-schema` validator requires `Closes #N` keyword in PR body to commit the PR to closing the linked issue on merge. Multiple recent PRs have shipped `Refs #N` only and tripped the gate:

- PR #1880 (#1874) — failed
- PR #1885 (#1612) — caught and fixed mid-flight
- PR #1895 (#1888) — currently failing for this exact reason

`Refs #N` is required FIRST by the validator regex (per memory `feedback_refs_ordering_in_pr_body.md`), but `Closes #N` is ALSO required separately by the closeout-schema's merge-evidence-pr-gate. The two requirements are documented in different memory notes and instruction surfaces, so models keep tripping.

Sibling validator to #1892 (commit-message-ticket-consistency) but a different surface (PR body, not commit message).

## Goal

Pre-PR validator that scans the PR body draft for both `Refs #N` and `Closes #N` keywords matching the branch's lead ticket.

## Acceptance Criteria

- [ ] AC1: `scripts/global/pre-pr-closes-keyword.js` validator that:
  - extracts branch lead ticket from `feat/<N>-...`
  - reads PR body from `gh pr view --json body` OR draft file
  - asserts presence of both `Refs #N` and `Closes #N` (or close-equivalent: `Fixes`, `Resolves`, etc.)
  - rejects on missing keyword
- [ ] AC2: Honors lightweight lanes (`lane:docs-only`, `lane:trivial`) where `Closes` may be substituted with explicit `merge-evidence-override:approved` label.
- [ ] AC3: Tests covering: both-present pass, Refs-only fail, neither-present fail, Closes-only fail (Refs is canonical primary), override path.
- [ ] AC4: Integrated into `lefthook.yml` pre-push.
- [ ] AC5: harness:self-test registry entry.

## Why programmatic

Memory notes catch this for human-attended sessions. Less-intelligent models in Copilot auto-mode skip past memory annotations. Local validator removes the model-attention dependency.

## Composition

- Memory note `feedback_refs_ordering_in_pr_body.md`
- Existing `closeout-schema` (post-PR) — this validator is the local-pre-PR sibling
- Composes with #1896 (pre-PR baton-completeness)
