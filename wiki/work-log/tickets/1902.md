# #1902 — Tier-2 anneal: pre-PR COLLAB temporal-ordering check (≥60s predate)

> **Source**: github:issue/1902 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:governance, resolution:completed, lane:code-change, governance:close-without-merge, anneal:tier-2

## Body

## Problem

The evidence-completeness CI gate enforces a **retroactive-planting check**: COLLABORATOR_HANDOFF must predate PR creation by ≥60s. Posting baton artifacts AFTER `gh pr create` is detected and blocks merge with the error: `COLLABORATOR_HANDOFF posted -NNNs before PR — must predate PR creation by ≥60s (retroactive planting check)`.

This trap class has now recurred FIVE TIMES in 24 hours across both teams:
- PR #1880 (#1874) — Copilot Team
- PR #1885 (#1612) — claude-code Team
- PR #1887 (#1259) — claude-code Team
- PR #1895 (#1888) — Copilot Team (caused PR closure + full takeover by claude-code Team — see #1888 history)
- PR #1901 (#1888 takeover) — claude-code Team (avoided the trap by sequencing artifacts first)

#1896 (pre-PR baton-completeness) addresses the PRESENCE side: all 4 artifacts must exist before push. This sibling anneal addresses the TIMING side: COLLABORATOR_HANDOFF must predate the new PR's createdAt by ≥60s.

## Goal

Pre-PR validator that:
1. Reads the linked issue's COLLABORATOR_HANDOFF createdAt timestamp
2. Asserts current time ≥ COLLAB_createdAt + 60s before allowing `gh pr create`
3. Rejects with structured message if the window hasn't elapsed
4. Recommends specific wait time

## Acceptance Criteria

- [ ] AC1: Extend `scripts/global/pre-pr-baton-completeness.js` (from #1896, pending implementation) OR create sibling `scripts/global/pre-pr-baton-temporal-ordering.js` to query COLLABORATOR_HANDOFF createdAt via `gh issue view --json comments`.
- [ ] AC2: Compute `now - max(COLLAB_createdAt for claude-code/copilot/codex artifacts on issue)`. Reject if < 60s.
- [ ] AC3: Output structured remediation: `"COLLAB posted Xs ago; wait Ys for predate window."`
- [ ] AC4: Honors `merge-evidence-override:approved` label as escape valve.
- [ ] AC5: Tests covering: (a) artifact posted >60s ago passes, (b) artifact posted <60s ago fails with wait-recommendation, (c) no artifact present defers to #1896 sibling check.
- [ ] AC6: Wire into lefthook pre-push hook alongside #1896.
- [ ] AC7: harness:self-test registry entry.

Memory note `feedback_all_baton_artifacts_before_pr.md` documents the timing requirement, but less-capable models in Copilot auto-mode (and claude-code under haste) regress to the same trap. Programmatic pre-push enforcement eliminates the recurrence.
