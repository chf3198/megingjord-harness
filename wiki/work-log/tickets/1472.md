## Body

## Problem
#1336 AC3 added auto-transition logic to `.github/workflows/label-lint.yml`: when a closed issue has `status:review` + a CONSULTANT_CLOSEOUT comment, the workflow should remove status:review/role:consultant and add status:done/resolution:completed.

Tested by closing #1336 itself + #1342 after merging the fix — both required manual label fix-up. Labels remained at status:review/role:consultant after close.

## Hypothesis
- A: workflow ran on stale main HEAD (just-merged workflow update not yet picked up by Actions infra)
- B: `labels.includes('status:review')` returned false because the labels array was already mutated by Phase 1 of the same workflow
- C: header-regex `closeoutRe` failed to match the actual closeout comment format

- [ ] AC2: If B → reorder Phase 1 (close-protection) vs Phase 2 (auto-transition); ensure auto-transition runs FIRST when CONSULTANT_CLOSEOUT detected
- [ ] AC3: If C → tighten or loosen the regex as needed
