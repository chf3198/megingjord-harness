## Scope
Route the 17 JS credential consumers that read `process.env` without hydrating, plus migrate the 14 ad-hoc `require('dotenv')` loaders, through `scripts/global/load-local-env.js` `loadLocalEnvOnce()`. Mechanical, lowest-risk first step.

## AC
- AC1: every credential-consuming JS entrypoint calls `loadLocalEnvOnce()` before first `process.env` key read; the 17 no-hydration files (Phase-0 D3 report) are covered.
- AC2: fill-don't-override preserved (existing exports/CI secrets win); no value logged (G4).
- AC3: `npm test` green; no regression for already-hydrated callers.

test_strategy: tdd-pyramid. Phase-0 cross-family review flagged fill-don't-override edge cases on the dotenv migration.

## Red-team hardening ACs (#2775)
- AC4 (G2): tests are unit (per-consumer hydration call present) AND integration (spawn a child process with EMPTY env; assert keys resolve from `.env`).
- AC5 (G3, fail-closed): a missing REQUIRED key after hydration returns a typed credential-absent error via `credential-availability.js#preCredentialPromptCheck` (#2569) and NEVER prompts the client; optional keys degrade silently.
- AC6 (G7): one-shot hydration adds < 10 ms to startup (cached after first `loadLocalEnvOnce`); guarded in the integration test.
