## Problem
We need an automated validator that programmatically detects ticket governance and epic lifecycle drift (Status Sync, Linkage, and Progress Logging) using the GitHub GraphQL API.

## Scope
1. Create `scripts/global/lint-epic-drift.js` that checks for:
   * **Class A**: Epic status is not `status:review` or `status:done` when all its sub-issues are closed.
   * **Class B**: Issue contains `Parent Epic: #N` in body but lacks a native GraphQL `Sub-issues` relationship.
   * **Class C**: Parent Epic comments lack a progress update for closed child tickets.
2. Write unit and integration tests in `tests/validate-epic-drift.spec.js` asserting the correctness of the engine under mock API shapes.
3. Keep script within the strict 100-line limit.

## Acceptance criteria
[ ] `scripts/global/lint-epic-drift.js` fully implemented and adheres to 100-line limit.
[ ] Integration tests in `tests/validate-epic-drift.spec.js` cover all three drift classes (A, B, C) and pass cleanly.
[ ] Registered in `inventory/harness-self-test-registry.json`.
