# #2626 — Guardrail: enforce bounded timeout for direct fleet model calls

## Parent

## Objective
Prevent session stalls caused by long-running ad-hoc fleet model requests by enforcing bounded timeout, structured failure output, and CI lint detection for paths that bypass existing wrappers (cascade-dispatch.js, HAMR routing).

## Problem
A session stalled on a direct `curl .../api/generate` call to a fleet host and required manual cancel. Root cause: ad-hoc direct call paths have no bounded timeout and bypass cascade-dispatch.js controls. When a fleet call stalls, operators fall back to paid providers — a direct G3 (Zero Cost) violation.

## Goal Lens
- **G3 CRITICAL**: Fleet stalls force paid fallback. Bounded timeouts + structured fallback classification prevent G3 violations by keeping fleet calls within defined latency budgets before declaring failure and returning a `fallback_tier` signal to the caller.
- **G1 Governance**: Lint guardrail blocks new unbounded direct fleet calls at CI (`megalint-required` workflow). False positive escape: `FLEET_GUARD_DISABLED=1`. Process for new violations: operator adds `// fleet-guard: exempt <reason>` line comment.
- **G2 Quality**: All ACs testable with exact `assert.strictEqual()` assertions (see AC4 and Test Template). Tests use `node:test` built-in mocking (no external mock library). Tests ship in same PR.
- **G4 Privacy**: Fleet hosts are on Tailscale VPN (private network only; not internet-accessible). `fleet-call-guard.js` stdout contains only model response + metadata — no credentials, no API keys, no session context. `FLEET_GUARD_DISABLED=1` does not expose any credentials or change network trust scope.
- **G5 Portability**: `FLEET_GUARD_DISABLED=1` bypass for air-gapped operators who have no fleet access at baseline.
- **G6 Resilience**: Bounded retry (max 2) + structured `{fallback_tier:'free-cloud'}` output enables the CALLER to route to the free-cloud failover tier (per global-task-router.js cascade chain). fleet-call-guard.js is a signal-emitter; routing is the caller's responsibility (no tight coupling).
- **G8 Observability**: Structured JSON failure output includes `elapsed_ms` and `fallback_tier`; appends to `~/.megingjord/incidents.jsonl` on timeout with `pattern_id:'fleet-call-timeout'`.
- **G9 Interoperability**: `fleet-call-guard.js` is a Node.js CLI script callable from Claude Code, Codex, and Copilot runtimes without modification. The lint rule applies to all 3 runtimes' script surfaces via the shared `megalint` runner.
- **G10 Maintainability**: All deliverable files ≤100 lines (lint-enforced). Single-purpose scripts.

## Test Implementation Template (G2 reference)

```js
// tests/fleet-call-guard.spec.js — reference assertions
import { test, mock } from 'node:test';
import assert from 'node:assert/strict';
import { callWithGuard } from '../scripts/global/fleet-call-guard.js';

test('success: result.success is true', async () => {
  const result = await callWithGuard({ host: MOCK_OK, timeout: 500 });
  assert.strictEqual(result.success, true);
  assert.ok(result.elapsed_ms < 500);
});

test('timeout: fallback_tier is free-cloud', async () => {
  const result = await callWithGuard({ host: MOCK_SLOW, timeout: 200 });
  assert.strictEqual(result.success, false);
  assert.strictEqual(result.reason, 'timeout');
  assert.strictEqual(result.fallback_tier, 'free-cloud');
});

test('retry_exhaustion: attempts count correct', async () => {
  const result = await callWithGuard({ host: MOCK_FAIL, timeout: 200, maxRetries: 1 });
  assert.strictEqual(result.reason, 'retry_exhaustion');
  assert.strictEqual(result.attempts, 1);
  assert.strictEqual(result.fallback_tier, 'free-cloud');
});
```

## Implementation Notes
- **Composition**: cascade-dispatch.js uses `--max-time` in its curl args and AbortController in fetch paths. fleet-call-guard.js is additive — it guards paths that do NOT go through cascade-dispatch.js. No refactoring in this ticket.
- **G9 cross-runtime**: fleet-call-guard.js has no runtime-specific imports; it works from `node scripts/global/fleet-call-guard.js ...` in any of the 3 runtimes (Claude Code, Codex, Copilot).
- **Lint rule scope**: fleet-call-lint.js checks scripts/ and docs/ only; skips known-guarded files.
- **G6 signal chain**: `fallback_tier:'free-cloud'` output ties into global-task-router.js lane cascade (fleet → free-cloud failover, per #2619).
