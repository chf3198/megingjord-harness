## Research scope

Five threads, all serving #1339 ACs:

### Thread 1 — LLM-consumability of structured logs (serves AC3 + AC12)

- OpenTelemetry GenAI semantic conventions (2025 spec) — applicability to harness event schema v2
- LangSmith / Helicone / Langfuse / Phoenix tracing formats — what do they record per LLM call and per agent step?
- Token cost of LLM reading structured vs free-text logs — quantitative benchmark proposal
- Field-naming conventions LLMs reliably parse (e.g., `severity` vs `level` vs `priority`)
- Whether to add a `summary:` field per event (synthesis prompt-friendly) — cost/benefit
- PII / secret redaction patterns that survive LLM read paths (vs leak via prompt injection of log content)

### Thread 2 — Goal-lens mapping methodology (serves AC2 + AC4)

- How to systematically map a logging surface to G1-G9 goals
- Industry frameworks: SLI/SLO mapping, observability-as-code, "four golden signals" applicability to agentic systems
- Gap-analysis methodology: traceability matrix vs ad-hoc inspection
- Goal Health Score wiring (#1113 actuators) — does each G* need a dedicated sensor or is a composite signal acceptable?

### Thread 3 — Live-streaming pipeline architectures (serves AC13)

- SSE vs WebSocket vs long-polling for `.jsonl` tail → browser
- File-tail watching: `chokidar` / native `fs.watch` reliability
- Backpressure + reconnect semantics for client drift
- Latency budget breakdown: file append → tail detection → SSE emit → DOM update
- Existing `dashboard/sse-handler.js` shape and extension points
- Industry pattern: tail-as-a-service, Grafana Live, Vector / Fluent Bit pipelines

### Thread 4 — Dashboard animation layer (serves AC14)

- Animation primitives appropriate for event-driven visuals (CSS transitions vs requestAnimationFrame vs canvas)
- Accessibility constraints (reduced-motion, contrast, focus)
- Performance budget: how many concurrent live events can the existing dashboard render without jank?
- Visual QA capture pattern for animation evidence (per `visual-qa-governance.instructions.md`)
- Existing dashboard architecture review: Context Flow + anneal queue (#1316) + baton flow panel selection

### Thread 5 — Retention, rotation, redaction (serves AC8 + G4 Privacy)

- Per-surface retention defaults (7d? 90d? indefinite?)
- Rotation mechanism: log-rotate / size-cap / time-window slice
- Append-only-with-archive vs in-place truncation
- Privacy: what fields require redaction at write time vs read time
- Compliance hooks (SOX / DORA / ISO 27001 baselines from Epic #1271 lineage)
