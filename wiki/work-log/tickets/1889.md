# #1889 — Tier-2 anneal: completion-claim-vs-merge-truth validator

> **Source**: github:issue/1889 | **state: OPEN** | **Labels**: type:task, status:backlog, priority:P2, area:governance, lane:code-change, anneal:tier-2

## Body

## Problem

Multiple recurrences observed in 2026-05-18 session: Copilot Team (and the harness in general) accepts comment-based completion claims that aren't backed by shipped artifacts.

**Pattern instances**:
- #1874: ADMIN_HANDOFF cited PR #1880 as merge vehicle; #1880 was actually closed unmerged; PR #1881 silently became the real vehicle. CONSULTANT_CLOSEOUT didn't catch the stale reference.
- #1399 remediation (2026-05-18 17:02:14Z comment): claims six code-change items "completed" but `git log origin/main` shows no new commits since the original `1f2b7fc`. Files exist in working tree, untracked/unstaged, never pushed, no PR opened.

This is the structural failure mode the less-intelligent models running under Copilot 'Auto' select are particularly prone to — they write text confidently regardless of underlying merge state. Instructional governance ("you must verify PRs are merged") is insufficient; programmatic enforcement is needed.

## Goal

Build a `completion-claim-vs-merge-truth` validator that runs on issue comments containing claim language and verifies that cited artifacts are actually on `origin/main`.

## Acceptance Criteria

- [ ] AC1: New validator `scripts/global/megalint/completion-claim-truth.js` detects comments with claim language (regex matching: "remediation actions taken", "shipped", "merged", "completed", "added"/"updated"/"created" + file path).
- [ ] AC2: For each cited file path, validator checks whether that path exists at the SHA of `origin/main` HEAD (not just on disk locally).
- [ ] AC3: For each cited PR `#N`, validator checks `pr view N --json mergedAt` — must be non-null.
- [ ] AC4: Validator runs as a workflow on `issue_comment.created` and `issue_comment.edited`; emits structured advisory comment listing dangling claims.
- [ ] AC5: Tests under `tests/completion-claim-truth.spec.js` covering: false-positive avoidance (prose mentions of paths), legitimate-claim recognition (claim + verifiable merge), dangling-claim detection.
- [ ] AC6: Stress test per Epic #1875 surface matrix (adversarial-input parser; perf budget p99 < 5ms per 100-line comment).
- [ ] AC7: Wired into harness:self-test registry as regression check.

## Test strategy

`tdd-pyramid+stress-test` per Epic #1875 (adversarial input parser, side-effect bearing).

## Why programmatic

Per session note: less-intelligent models in Copilot's auto-mode rely heavily on programmatic governance. Instruction-only enforcement creates a vulnerability where the model can drift confidently from underlying truth. This validator restores ground-truth checking.

## Constraints

- Don't false-positive on legitimate progress reports that include "we'll commit shortly" or similar future-tense markers.
- Don't block — advisory mode with promotion path per Epic #1612 model (replay-eval against 50-PR corpus, promote when FP rate < 10%).
