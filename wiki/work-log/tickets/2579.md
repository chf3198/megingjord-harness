# #2579 — Remove hard-coded codex default from agent-signature.js identity resolution

> **Source**: github:issue/2579 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:scripts, area:governance, lane:code-change, governance:close-without-merge, anneal:tier-2
> Mirror of `gh issue view 2579` (derived; edit the GitHub item, not this page).

## Body

## Problem

`scripts/global/agent-signature.js` hard-codes a team/model default to **codex**:

```js
const team  = (opt.team  || 'codex').toLowerCase();   // line 12
const model = (opt.model || 'gpt-5.4').toLowerCase(); // line 13
```

Any non-codex runtime (Claude Code, Copilot) that invokes the CLI without explicit `--team`/`--model` silently derives a **codex** signature (e.g. `Quill Mason` / `codex:gpt-5.4@local`). This is wrong provenance: a Claude Code Manager should derive `Orla Mason` / `claude-code:opus`.

## Why this matters (goal-lens)

- **G5 Portability** (primary): per-team hard-coding is exactly the coupling the portability contract forbids; identity must be settings/substrate-derived.
- **G1 Governance / OA3 Identity Abuse**: a provenance-signing utility silently mis-attributing the signer team undermines Team&Model signing integrity.
- **G8 Observability**: silent wrong default vs. an explicit, auditable signal.
- **G9 Interoperability**: the utility assumes one runtime rather than being runtime-neutral.

## Evidence

- Surfaced 2026-06-01 while deriving a Manager signature during #2103/#2106/#2254 remediation: `node scripts/global/agent-signature.js --role manager` returned `codex:gpt-5.4@local` in a Claude Code session.
- Sibling `scripts/global/baton-comment-build.js` does the right thing — it **requires** `--team-model`/`TEAM_MODEL` and fails loud (lines 27-31). `agent-signature.js` is the inconsistent one.
- `HAMR_TEAM` is the established team-identity env signal (used by `hamr-activate.sh`, `fleet-claim-client.py`, `merge_claim_client.py`); `agent-signature.js` consults neither it nor `--team` before falling back to codex.
- Blast radius is near-zero: the only repo references (`consultant-checks.js`, `megalint/signer-registry-check.js`) cite the script in *error-message strings*; no CI/hook executes it relying on the codex default.

## Acceptance criteria

- AC1: Remove the hard-coded `codex`/`gpt-5.4` team/model default. Resolve from `--team`/`--model`, then `HAMR_TEAM`/`HAMR_MODEL` env, else **fail loud** (non-zero exit + clear stderr) rather than emit a default-codex signature.
- AC2: Existing explicit-flag behavior is unchanged (`--team claude-code --model opus --role manager` → `Orla Mason` / `claude-code:opus`).
- AC3: `HAMR_TEAM=claude-code HAMR_MODEL=opus` (no flags) resolves to claude-code identity.
- AC4: Add a unit test (`tests/**`) covering: explicit flags, env-derived, and the fail-loud no-identity case.
- AC5: role/substrate defaults (collaborator/local) are preserved — they are role-neutral, not team-coupled.
