# #2888 — Upgrade fleet-critic flaw detection from keyword heuristic to structured tags

## Body

## Scope
Refs Epic #2791 · follow-on from #2797 (cross-family critic).

#2797's `fleet-cross-family-critic.js` classifies critic findings as security/correctness flaws via a
broad keyword regex (`FLAW_RE`), injectable via `deps.classifyFlaw`. Gemini cross-family review (F2)
flagged keyword matching as a false-negative surface for a safety gate: a real flaw phrased without the
vocabulary (e.g. "permits insecure object deserialization", "known gadget chain") could slip to
advisory.

## Strategic upgrade (the injectable seam already exists)
- Have cross-family critics emit STRUCTURED findings (`tags: [security, rce, ...]`, `severity`) via the
  `pr-diff` prompt template + `parseFindings`, instead of free-text verdict lines.
- OR route reject/partial findings through a dedicated fine-tuned flaw-classifier model (fleet/$0 lane).
- Wire either as `deps.classifyFlaw` — no decision-tree change needed (#2797 left the seam).

## Acceptance
- AC1 critic findings carry machine-readable tags/severity (template + parser).
- AC2 `classifyFlaw` consumes tags; keyword regex remains the documented fallback.
- AC3 replay-eval shows reduced false-negative rate vs the keyword baseline on a labeled corpus.
