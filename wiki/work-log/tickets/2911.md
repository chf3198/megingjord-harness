## Body

## Problem
The `cross_runtime_writes` field in MANAGER_HANDOFF is advisory-only (Rule 1.9 from the guardrail assessment). When a ticket involves writing config consumed by another team's runtime (e.g. `.claude/settings.json`), the field is expected but not blocked if absent. The broken hook schema in #1917 was exactly this failure mode.

## Acceptance Criteria
- [ ] In `scripts/global/megalint/manager-handoff.js`, check if PR diff touches cross-runtime paths (`.claude/`, `.codex/`, `.copilot/`) and block if `cross_runtime_writes` field is absent
- [ ] Cross-runtime path list configurable in `config/governance-rules.yaml`
- [ ] Block if `cross_runtime_writes` present but `target_team_sign_off` is `pending` at closeout time
- [ ] Add tests in `tests/`
