# #1546 — Team Worktree Substrate Isolation + VS Code SCM Declutter

## Body

## Problem

The operator VS Code Source Control view is cluttered by linked worktrees from multiple orchestrating teams: Claude Code, Copilot, and Codex. VS Code's current Git/worktree support intentionally displays each detected worktree as a separate Source Control repository, which is useful for manual workflows but risky in a three-orchestrator harness because stale/dirty/locked worktrees can look like active operator work.

## Research Basis

- VS Code docs: Git worktrees are separate checkouts and each worktree appears as a separate repository in Source Control.
- VS Code docs: Repositories view supports single vs multi repository selection mode via `scm.repositories.selectionMode`.
- Git docs: stale worktree metadata should be cleaned with `git worktree remove` and `git worktree prune`, not by manual deletion.

## Objective

Create a governed, non-destructive first pass that helps the operator and teams see which worktrees are active, stale, dirty, locked, or safe to remove, without deleting or moving another team's work.

## Acceptance Criteria

- [ ] Add a dry-run worktree governance inventory command that reports each linked worktree path, branch, HEAD, dirty state, lock/prunable state, upstream divergence, and recommended action.
- [ ] Command must be read-only by default and must not remove worktrees or branches.
- [ ] Add documentation for the recommended three-team substrate model and VS Code SCM declutter settings.
- [ ] Document safe cleanup states: keep, review, quarantine, remove-after-merge.
- [ ] Add tests or fixture coverage for the inventory classifier.
- [ ] Preserve active Claude Code, Copilot, and Codex worktrees; no destructive operations.
- [ ] `npm run lint`, targeted tests, and docs lint pass or blockers are documented.

## Initial Non-Goals

- Do not delete worktrees.
- Do not move team directories.
- Do not change deployed runtime files under `~/.codex`, `~/.copilot`, or `~/.agents`.
- Do not modify active issue branches for other teams.
