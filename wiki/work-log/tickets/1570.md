# #1570 — Lefthook pre-push parity: every required CI gate runs locally

> **Source**: github:issue/1570 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:infra, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 1570` (derived; edit the GitHub item, not this page).

## Body

## Parent

Epic #1568 (AC-1). Research: #1569.

## Goal

Replace the current pre-push hook (only runs `lint:readability:ci`) with a Lefthook config that runs every deterministically-checkable required CI gate in parallel, under ~10s on warm cache. Eliminates the re-roll cycle observed on PR #1558 → #1559.

## Acceptance Criteria

- [ ] AC1 — `lefthook.yml` installed at repo root; replaces or supersedes `.git/hooks/pre-push` shim. Adoption path documented for new clones (`lefthook install` or equivalent).
- [ ] AC2 — Pre-push runs **in parallel**: `npm run lint` (100-line cap), `npm run lint:readability:ci` (readability cap), `npm run lint:js` (eslint), `npm run lint:md`, `npm run lint:py`, `npm run lint:sh`, `node scripts/global/megalint/index.js`, `node scripts/global/test-evidence-validator.js --diff-only`, branch-name regex check.
- [ ] AC3 — Walltime budget: warm-cache pre-push completes in **≤ 10s** at p95. Captured via timing on the operator's box plus a Playwright/CI sanity check that exits in budget.
- [ ] AC4 — Bypass path: `--no-verify` works (operator emergency), but lefthook posts a warning into stdout naming the bypassed gates so they're visible in the terminal scroll.
- [ ] AC5 — CI parity: a smoke test in `.github/workflows/lint.yml` invokes the same `lefthook run pre-push` command (or equivalent script) so that local and CI execute *the identical command*, not just the same intent. Eliminates drift.
- [ ] AC6 — Documentation: `docs/howto/pre-push-gates.md` (or update to existing onboarding) explains what runs, what bypass means, how to add a gate.

## test_strategy

`golden-file` — lefthook config and the CI parity invocation are tested via fixture diff (a fixed input artifact yields a fixed pass/fail set). Plus one `tdd-pyramid` spec for any helper script added.

## Scope notes

- **In scope**: hooks that have only-local inputs (lint, type-check, megalint, test-evidence diff).
- **Out of scope**: hooks that need server state (`merge-evidence-pr-gate`, `evidence-completeness`, `consultant-gate` which scrapes issue comments — these can be moved later if a local API caching layer is added; not in this ticket).
- **Out of scope**: removing existing CI checks. Pre-push and CI both run; CI is the backstop, not the surface.

## Dependencies

- Decision: Lefthook (Go, parallel, ~10× husky) vs Husky+lint-staged (Node, serial, ubiquitous). Recommendation from research #1569: Lefthook. Operator can override on ticket pickup.
- No external service dependencies.

## Source of urgency

PR #1558 → #1559 re-roll burned a full CI cycle on five deterministically-detectable failures (branch-prefix, 100-line cap, markdown bold, two unanchored marker words, missing spec). Every one would have been caught by AC2's parallel pre-push run.
