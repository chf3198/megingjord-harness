## Body

## Research Objective

Survey options for detecting and blocking re-push of branches that already have merged PRs on origin. During Epic #2827, stale worktrees pushed already-merged branches, creating 7 duplicate open PRs (#2859–#2866). Identify the best pre-push hook implementation for AC-I3.

## Scope

- Audit existing worktree hooks in `hooks/` for any pre-push checks
- Enumerate detection approaches:
  1. Pre-push git hook: call `gh pr list --head <branch> --state merged` before push
  2. lefthook pre-push rule checking remote branch status
  3. Session-start validation in `worktree-session-start.sh`
- Evaluate each on: reliability, latency (API call in hot path), G5 portability, CI compatibility

## Deliverable

Findings comment on this ticket:
1. Existing pre-push hook audit — what currently exists
2. Comparison table of detection approaches
3. Selected approach with implementation sketch for AC-I3
