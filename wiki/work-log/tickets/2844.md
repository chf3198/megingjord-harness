## Sandbox test-exec for fleet-proposed changes (P1-0 child of #2802)

Parent: #2802 (P1-0 fleet context-access layer). Phase-0 source: #2792 (D13).

### Problem
The fleet context dogfood loop (#2815/#2817/#2823/#2831) now hands a fleet model
auto-assembled context and dispatches it. The model returns a **proposed change**
(file edits). Trusting that change blind violates G1/G2 and OWASP **OA10 (Code
Execution)** + **OA2 (Tool Misuse)** — a fleet payload could write outside the repo
or break the build. We need a **validation gate**: run the change's tests in an
**isolated sandbox** and only mark it trusted when green.

### Objective
A library slice that takes a fleet model's proposed change (`[{path, content}]`)
plus an **operator-declared** test command, materializes the change in a throwaway
git worktree off HEAD (the real tree is never touched), runs the tests bounded +
captured, and returns a **fail-closed trust verdict**.

### Acceptance criteria
- AC1: `partitionChanges` rejects absolute / `..`-escaping paths (path-traversal
  guard, mirrors #2819) — rejected paths ⇒ `trusted:false`, never written.
- AC2: only the **operator-declared `testCommand`** executes; the fleet payload is
  DATA written to the sandbox, never executed (OA10/OA2 boundary, surfaced in code).
- AC3: isolation via a detached git worktree off HEAD; torn down after (best-effort);
  side-effects (git/exec/fs) injectable so unit + stress tests are process/network-free.
- AC4: fail-closed verdict (G1): unsafe paths, runner error, timeout, or non-zero
  exit ⇒ `trusted:false` with an operator-visible reason (G8).
- AC5: bounded timeout + output clamp (G6 OOM/runaway guard).
- AC6: each new file ≤100 lines (lint); readability snapshot ≤475.
- AC7: tdd-pyramid spec + stress spec (≥1 chaos/fault-injection path, ≥1 p99 budget).

### Gates
lint (≤100 lines/file), lint:js, readability ≤475, test-evidence (tdd-pyramid+stress),
baton-gates, closeout-schema, merge-evidence-pr-gate.

### Lane
lane:code-change. test_strategy: tdd-pyramid+stress-test.

### Non-redundancy (overlap_decision)
No existing module validates fleet-proposed changes in isolation. Reuses the
worktree-isolation PATTERN (Epic #1854) but operates on a fleet payload, not a baton
worktree; reuses the path-traversal guard idiom from `fleet-context-bundle.js` (#2819).
