# #1146 — Design v2 cross-team R&D protocol with iterative debate

## Body

## Problem

Phase-0 R&D for Epic #1112 (productize multi-team parallel R&D collaboration as harness feature). The v1 protocol designed and used in synthesis-1105 and synthesis-1131 failed in 7 distinct ways (see #1131 SYNTHESIS_ABANDONED comment). v1 was modeled on parallel-voting; the operator's intent is iterative-debate-with-evidence per Du et al. 2024 multi-agent debate literature.

Bootstrap constraint: this R&D **cannot use the broken v1 synthesis** — it would compound the very failures it aims to fix. Single-team R&D (Claude Code Team as Manager + Collaborator + Admin + Consultant) producing the protocol document, with operator as design-review consultant.

## v1 failure modes to address (verbatim from #1131 SYNTHESIS_ABANDONED)

1. Substrate-vs-model team identity (#1145)
2. Concurrent-write across shared workspace (worktree isolation didn't fire)
3. Decision numbering collisions (no central allocator)
4. Zero websearch citations (websearch was optional)
5. Zero cross-team challenge (one-shot post + quiesce, not iterative)
6. No active admin facilitation (passive snapshot collection)
7. No baton handoff at synthesis end (quiescence-based termination)

## Operator-stated v2 design requirements

- **No seeded R&D**. Each team starts from scratch — knows the goal/task/issue but not what others have produced.
- **WebSearch mandatory** for opening plans AND for backing debate points.
- **Iterative debate** in waves. Teams read others' work, challenge, defend, concede with evidence.
- **One team owns the full Agile baton** (Manager → Collaborator → Admin → Consultant) for the R&D ticket. Other teams participate ONLY as Collaborators (their R&D + debate). The lead team's Admin and Consultant roles handle proper closeout.
- **Active admin facilitation** during Phase-D. Admin reads everything per wave, posts WAVE_SUMMARY, challenges weak evidence, surfaces missed websearches, calls out stuck points.
