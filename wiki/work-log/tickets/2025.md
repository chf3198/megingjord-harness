## Problem
Currently, there is no impassible automated mechanism that forces parent Epics to synchronize their status with closed child tickets or natively link them using Sub-issues. When a developer or AI agent finishes a task, they might forget to update the parent Epic, leading to governance drift.

## Scope
1. Research existing GitHub Actions, GraphQL webhooks, and local pre-commit/pre-push hooks in the harness to see how we can automatically enforce epic status synchronizations and sub-issue relationships.
2. Outline the design of an automated, impassible validator script (e.g., adding rules to `pre-pr-gate.js` or `harness-self-test-runner.js`) that detects stalled parent Epics or prose-only linkages and blocks builds/merges when drift is found.
3. Establish the architectural specification for this guardrail, including failure conditions and manual override exceptions.

## Acceptance criteria
[ ] Complete research review of other orchestrators and current Megingjord hooks.
[ ] Design draft of an automated validator script or pre-PR gate check enforcing `epic-child-sync`.
[ ] Peer-review report composed by a Consultant with a minimum rubric rating of >= 7 across all G1-G10 categories before this ticket can be closed.
