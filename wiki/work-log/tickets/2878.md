# #2878 — Add merged-branch-guard to lefthook.yml and pretool_guard.py RE_GIT_PUSH handler

## Body

## Summary
Add a merged-branch guard to `lefthook.yml` (pre-push) and a supplementary check in `pretool_guard.py` so that pushes to already-merged branches are detected and blocked (advisory or deny) before the push is accepted.

## Problem
`lefthook.yml` has 9 pre-push commands (branch-name-regex, lint variants, megalint, hydration-lint, closeout-preflight, git-freshness), none of which detect whether the current branch has already been merged. `worktree-session-start.sh` and `pretool_guard.py`'s RE_GIT_PUSH handler also lack this check. An agent can push commits onto a branch whose PR is already merged, potentially re-opening the issue or creating confusing duplicate history. Confirmed by AC-R3 research (#2873).

## Expected Outcome
Before any push, the branch is checked against `gh pr list --head <branch> --state merged`. If a merged PR is found, the push is blocked (Option A: lefthook command) with a deny message including the PR number. Additionally, pretool_guard.py RE_GIT_PUSH handler performs the same check for agent terminal pushes (Option B: belt-and-suspenders).

## Acceptance Criteria
- AC-I3.1: New function `check_merged_pr(branch, cwd) -> Optional[int]` added to `hooks/scripts/live_checks.py`. Returns merged PR number if found, None if not found or gh CLI unavailable. Fail-open on error.
- AC-I3.2: New `merged-branch-guard` command in `lefthook.yml` pre-push section: calls `check_merged_pr()` via a small runner script; exits non-zero if merged PR found; includes PR number in output message.
- AC-I3.3: New check in `pretool_guard.py` RE_GIT_PUSH handler (after existing branch name check): calls `check_merged_pr()` and emits deny if merged PR found.
- AC-I3.4: Unit tests for `check_merged_pr()` covering: merged PR found (returns int), no merged PR (returns None), gh CLI error (returns None / fail-open).
- AC-I3.5: Integration test covering the lefthook deny path and pretool_guard deny path.

## Implementation Entry Point
- `hooks/scripts/live_checks.py`: add `check_merged_pr(branch, cwd)` after `linked_issue_has_manager_handoff()`.
- `lefthook.yml` pre-push section: add `merged-branch-guard` command referencing runner script.
- `hooks/scripts/pretool_guard.py` RE_GIT_PUSH handler: add check after branch name resolved.
