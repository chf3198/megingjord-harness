## Body

## Objective
Create `~/.megingjord/incidents.jsonl` infrastructure with append-only format, rotation policy, and in-memory indexing.

## Acceptance Criteria
- AC1: JSONL appends are atomic (no partial writes on crash)
- AC2: Index rebuild completes in <100ms for 10k events
- AC3: Rotation preserves all events; oldest archive deletable after 90d retention
- AC4: Deduplication works: same pattern in same window produces one entry
- AC5: Suppression entries expire per `suppression_until` timestamp
- AC6: Privacy audit: incidents.jsonl contains NO secrets, NO PII, only ticket #s + pattern IDs

## Implementation Notes
Schema: `{timestamp, pattern_id, window_start, count, evidence[], status, suppression_until}`
- Append-only JSONL format; each line is immutable record
- In-memory index on startup: `Map<pattern_id, List<timestamps>>` for O(1) lookups
- Rotation policy: Archive to `.jsonl.YYYYMMDD` when file > 10MB or > 30 days old; keep last 3 archives
- Nightly cron task: Rotate if needed, rebuild in-memory index
- Backward-compatible: graceful handling if file missing
