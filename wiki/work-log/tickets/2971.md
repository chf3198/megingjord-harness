# #2971 — Require cross-model consensus for research and planning before execution

> **Source**: github:issue/2971 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P2, area:governance, resolution:released, lane:code-change, phase-gate:research-first
> Mirror of `gh issue view 2971` (derived; edit the GitHub item, not this page).

## Body

## Goal
Require that the **research & planning** of every Epic/ticket be validated by **≥2 models (at least
one cross-family)** iterating until they reach a **>93 consensus rating**, before the item may advance
from planning to execution. Enforce it with a harness guardrail and record an auditable artifact (G8).

## Policy
- **Minimum 2 models**, at least one **non-Anthropic / cross-family** (e.g. Gemma/Qwen via Ollama,
  or an OpenRouter free model via `openrouter-free-failover`). Zero-cost lanes preferred (G3).
- Each model emits a **rating 0–100** plus rationale on the plan's quality/completeness/risk.
- **Iterate** (revise plan → re-rate) until **all participating models rate >93** AND their ratings
  agree within a configured band (consensus), or a max-round cap is hit (then escalate to human).
- Record a **CONSENSUS artifact** on the ticket: per-model scores, round count, final verdict, model
  IDs/provenance.

## Design
- Config `hooks/consensus-policy.json`: `{ threshold: 93, min_models: 2, require_cross_family: true,
  agreement_band, max_rounds }`.
- Reuse existing tooling: `cross-family-review`, `research-redteam-loop`, multi-judge orchestrator,
  `fleet-cascade-gate`, `openrouter-free-failover`, Red-Team Reviewer agent.
- Emit a `planning-consensus` event into governance state keyed to the ticket.
- **Guardrail**: a gate that blocks the planning→execution transition (e.g. first code commit / branch
  creation for a ticket) unless a recorded consensus ≥ threshold exists for that ticket. Degradable
  (G6): if no cross-family model is reachable, block with a clear remediation message rather than
  silently passing.

## Acceptance criteria
- [ ] `consensus-policy.json` with documented defaults (threshold 93, min 2, cross-family required).
- [ ] A reusable runner that collects ≥2 model ratings (≥1 cross-family) and computes consensus.
- [ ] Guardrail blocks execution of a ticket lacking a recorded ≥93 consensus; clear message + escape
      hatch for human override (audited).
- [ ] Consensus artifact format defined and recorded on the ticket.
- [ ] Tests (consensus math, gate behavior, degradation) + lint green; docs/CHANGELOG updated.

## Governance note
Applies reflexively: this Epic and HE1 must themselves pass the ≥93 cross-model consensus before
implementation begins.
