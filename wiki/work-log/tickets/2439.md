## Problem

The COLLABORATOR_HANDOFF schema (#2425) will define the doc-coverage block, but it does not require cross-family model rating evidence. A Collaborator from family X (Anthropic, OpenAI, etc.) can rate their own work — creating a self-bias risk. The requirement: the rating must always be done by a model of a **different AI family** than the one that did the coding.

## Objective

Extend the COLLABORATOR_HANDOFF schema (being defined in #2425) to add three required fields that carry the cross-family rating. Add validation in `collaborator-handoff.js` to enforce these fields are present and properly formed for `lane:code-change` tickets.

## Acceptance Criteria

- [ ] COLLABORATOR_HANDOFF template (updated in #2425) gains three required fields for lane:code-change:
  - `cross_family_rating: <N>/100` (integer 1-100; produced by the preflight gate script from C9)
  - `cross_family_reviewer: <model@substrate>` (e.g. `qwen2.5-coder:32b@fleet-tailscale` or `qwen3-coder:free@openrouter`)
  - `cross_family_findings: <one-sentence summary> | none`
- [ ] `scripts/global/megalint/collaborator-handoff.js` validates these three fields for lane:code-change; missing or malformed fields produce a **blocking violation** (ok: false), not advisory
- [ ] Family independence check: the validator extracts the AI family from the `Team&Model` signing field in the COLLABORATOR_HANDOFF and from the `cross_family_reviewer` field; if they share the same family (both Anthropic, both OpenAI, etc.), emit a blocking violation: `cross-family-reviewer-same-family`
- [ ] For lanes other than lane:code-change, these fields are optional (no violation if absent)
- [ ] Unit tests cover: valid cross-family case, missing fields case, same-family reviewer case
- [ ] Family extraction logic is encapsulated in a helper (not duplicated across validators)
- [ ] The CI `baton-gates.yml` collaborator-gate content-verification step (#2427) also checks for the three new fields
- [ ] Documentation update: `instructions/role-baton-routing.instructions.md` MANAGER_HANDOFF schema section gains a note that Collaborator must complete cross-family preflight before handoff
