# #1572 — Model-diversity gate: enforce Admin != Collaborator and Consultant != Admin Team&Model

## Body

## Parent

Epic #1568 (AC-3). Research: #1569.

## Goal

Extend `.github/workflows/baton-gates.yml` to parse `Team&Model:` from each baton artifact and fail the consultant-gate when the critical-path pairs match. Defeats the same-model-as-actor-and-critic self-bias trap quantified in Panickssery et al. (NeurIPS '24/'25) and follow-on 2025–2026 work.

## Acceptance Criteria

- [ ] AC1 — Add `scripts/global/baton-team-model.js` (pure helper): exports `parseTeamModel(artifactBody)` returning the literal value after `Team&Model:` (e.g. `claude-code:opus-4-7@anthropic`). Returns `null` if missing or malformed.
- [ ] AC2 — Add `enforceCriticalPathDiversity({ collaborator, admin, consultant })` that returns `{ ok, violations[] }`. Violations populate when:
  - `collaborator.team_model === admin.team_model` (Admin must differ from Collaborator)
  - `admin.team_model === consultant.team_model` (Consultant must differ from Admin)
  - Manager–Collaborator equality is **allowed** (shared intent acceptable).
- [ ] AC3 — Diversity rule is **cross-model within provider**: literal `Team&Model:` string comparison. Examples:
  - `claude-code:opus-4-7@anthropic` vs `claude-code:sonnet-4-6@anthropic` → **passes** (different models)
  - `claude-code:opus-4-7@anthropic` vs `claude-code:opus-4-7@anthropic` → **fails** (identical)
  - `claude-code:opus@anthropic` vs `copilot:gpt-5.3-codex@github` → **passes** (different providers + different models)
- [ ] AC4 — `.github/workflows/baton-gates.yml` admin-gate (or new consultant-gate step) consumes the helper. Failure mode: blocking on the new gate, with a comment-on-PR explaining which pair matched and how to resolve.
- [ ] AC5 — Override: label `model-diversity:waived` permits a same-model PR to merge (for special cases like solo agent on a tiny tweak). Use is logged with required `WAIVER_REASON` comment.
- [ ] AC6 — Tests: 6 unit-test cases — pass on differ; fail on each critical-path match; pass with waiver label; null-handling on malformed input; Manager–Collab match (must still pass); multi-artifact normalization (different surnames must NOT mask identical models).

## Why "cross-model within provider" (decision rationale)

Operator decision 2026-05-15: strict cross-provider requirement would block every PR until multi-provider tooling is in place (Codex Team uses GPT family; Copilot Team uses GPT; Claude Code Team has Anthropic). Cross-model within provider ships now (Opus reviews Sonnet's work, Sonnet reviews Haiku's; cheaper Haiku triages Opus). The Star Chamber strongest finding (cross-provider preferred) is captured separately in Epic AC-4 (#TBD second-opinion Consultant pass) so the path-of-least-resistance ship and the strongest-independence path coexist.

- Cross-team rollout: applies the same gate to baton artifacts from Codex / Copilot teams since the regex is universal.
