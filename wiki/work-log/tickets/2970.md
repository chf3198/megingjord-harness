# #2970 — External-collaborator governance profile and friction remediation

> **Source**: github:issue/2970 | **state: OPEN** | **Labels**: type:epic, status:backlog, priority:P2, role:manager, area:governance
> Mirror of `gh issue view 2970` (derived; edit the GitHub item, not this page).

## Body

## Goal
Let the harness govern the **operator's** work fully while imposing **light** governance on external/
community repos (e.g. `agentic-classroom`) whose human collaborators don't use the harness. Remediate
the concrete friction observed while bootstrapping such a repo.

## Background (observed friction)
Bootstrapping an external repo under the harness surfaced gates that block legitimate genesis work:
- `ci-green` merge gate returns `unknown` when a repo has **no CI workflow yet**, blocking the very PR
  that adds CI.
- `admin_ops.commit`/`pr_create` baton flags reset when read-only/non-git commands are interleaved;
  working-tree branch silently drifted (active-branch-state reset).
- `repo-scope` is allowlist-only (`default_enabled: true`) with no per-repo profile/opt-out.
- Issue-close gate blocks closing a genesis tracking issue whose work landed as the first
  direct-to-main commit.

## Design
Extend the existing `hooks/governance-profiles.json` with a new profile:
- **`external-collab`** (lite): `commit.ticket_ref` and `commit.branch_naming` = `warn` (not `deny`);
  **disable** `ci_green` merge gate, merge-baton sequencing, and issue-close-needs-merge gate.
  Keep secret-scan + dangerous-command + canonical-main protections (always-on safety).
- Repo→profile resolution + register `/home/curtisfranks/agentic-classroom` → `external-collab`.
- Make `pretool_guard` consult the active profile for the ci-green, merge-sequencing, and issue-close
  gates (currently hardcoded regardless of profile).
- `ci_gate_status`: add a `no-checks-configured` state (treated as pass) distinct from `unknown`.
- `admin_ops`: key to (repo, branch, ticket); resilient to interleaved read-only commands; fix the
  branch-state reset (see memory `feedback_post_restart_active_branch_state_reset`).
- Genesis exception: allow closing a tracking issue whose work is the first commit on `main`.

## Acceptance criteria
- [ ] `external-collab` profile defined; `agentic-classroom` registered under it.
- [ ] Under `external-collab`, a docs/CI PR merges and its issue closes WITHOUT ci-green/merge-baton
      friction; safety gates (secret-scan, dangerous-cmd) still fire.
- [ ] Operator/default repos keep full governance (no regression).
- [ ] `no-checks-configured` distinct from `unknown` (unit test).
- [ ] admin_ops robust to interleaved read-only commands (unit test).
- [ ] Tests + lint green; docs/CHANGELOG updated.
