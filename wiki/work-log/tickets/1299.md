# #1299 — Add AI-suggested acceptance criteria with reconciler validation

## Body

## Problem

Today, Epic acceptance criteria are hand-written by the Manager (or operator) from scratch. This is slow and produces uneven AC quality across Epics — some have 3 ACs, some have 13; some are measurable, some are aspirational. AI-assisted AC suggestion could improve quality and consistency.

However, [Microsoft Research 2026 — Intent Formalization: A Grand Challenge](https://www.microsoft.com/en-us/research/publication/intent-formalization-a-grand-challenge-for-reliable-coding-in-the-age-of-ai-agents/) explicitly warns: *"Informal requirements must be validated by users. There is no oracle for spec correctness."* Naive AI-generated ACs without validation backstop would amplify rather than reduce drift.

This Epic adds AI-suggested ACs **with the reconciler from Epic #1271 child C3 (#1289) acting as the validation backstop**. The reconciler computes truth-status per AC; if AI-suggested ACs don't reconcile against measurable evidence, the suggestion is rejected.

## Goal

Manager writes problem statement; AI proposes 3-7 measurable ACs; Manager edits/approves; reconciler verifies ACs are computable from available evidence sources before they enter the Epic body.

## Phase-0 R&D Gate (required before implementation children)

Mirroring the Epic #1271 pattern: 3 team R&D plans covering (1) prompt-engineering for AC quality; (2) reconciler-backstop integration design; (3) human-in-the-loop UX (CLI vs. GitHub comment vs. dashboard); (4) measurement framework (false-positive rate of AI-suggested ACs that pass reconciler validation).

## Initial acceptance criteria (refined post Phase-0)

- [ ] AC1: Phase-0 R&D Gate complete (3 team plans + cross-team review + synthesis decision)
- [ ] AC2: AI-suggestion implementation lands (LLM call + prompt template + structured output)
- [ ] AC3: Reconciler validation backstop integrated (per [Microsoft 2026 Intent Formalization](https://www.microsoft.com/en-us/research/publication/intent-formalization-a-grand-challenge-for-reliable-coding-in-the-age-of-ai-agents/))
- [ ] AC4: Manager-in-the-loop UX (each suggested AC edit/accept/reject)
- [ ] AC5: Measurement: false-positive rate (AI ACs that pass reconciler but fail in practice) <5%
- [ ] AC6: Documentation: when to use AI suggestion vs. hand-write
- [ ] AC7: Cross-team Consultant CONSULTANT_EPIC_CLOSEOUT signed by independent team
