## Concerns to address (4 file-ticket items)

### #1: Concurrency test coverage for lock acquire

Red-team finding (G2 Quality): "Tests do not cover edge cases such as simultaneous lock acquisition or lease expiry under concurrent operations."

Action: add a spec that spawns 2+ Node processes calling `acquire(rootDir, team, ticket)` simultaneously and verifies exactly one wins. Same for `expireStale()` under concurrent calls.

Files: `tests/worktree-active-session-lock.spec.js` + `tests/worktree-lease-heartbeat.spec.js`.

### #2: Lock-state telemetry to incidents.jsonl

Red-team finding (G8 Observability): "No logging or monitoring for decision-making processes or anomalies in the system."

Action: emit OTel GenAI events on lock acquire/refuse/release/expire to `~/.megingjord/incidents.jsonl` per event-schema-v3. Compose with the same pattern as Epic #1826 harness-self-test telemetry.

Files: `scripts/global/worktree-active-session-lock.js`, `worktree-lease-heartbeat.js`.

### #3: Lock-file tampering mitigation

Red-team findings (bypass #1, #2): "Spoofing PID to acquire a lock" + "Manipulating heartbeat timestamps to extend lease duration."

Action: on `acquire()`, set the lock file mode to 0o600 (owner-only). Add optional content signing (HMAC) gated on env var for paranoid mode. Document the cooperative-agent model limit explicitly in `instructions/sandbox-worktree-governance.instructions.md`.

Files: `scripts/global/worktree-active-session-lock.js`, `instructions/sandbox-worktree-governance.instructions.md`.

### #4: Cross-platform test coverage (Windows + macOS)

Red-team finding (G5 Portability): "Libraries may not be compatible with different operating systems or environments without modification."

Action: add platform-conditional tests for `process.kill(pid, 0)` semantics (Windows differs), `fs.unlinkSync` permission errors (Windows), `fs.chmodSync` no-op on Windows. CI matrix should run lock specs on `windows-latest` + `macos-latest` (currently ubuntu-only).

Files: `tests/worktree-active-session-lock.spec.js`, `.github/workflows/lint.yml` or new matrix workflow.
