## Body

## Context

#1682 (page-level wiki access telemetry, merged via PR #1946 on 2026-05-19) shipped with 18 unit tests covering AC1-AC7 but **no stress-test suite** for the atomic tmp+rename write path. The COLLAB handoff declared `test_strategy: tdd-pyramid` and justified omitting stress as "single-process atomic semantics; no concurrent multi-writer contract claimed."

Per [test-methodology-matrix.instructions.md](../instructions/test-methodology-matrix.instructions.md) stress-applicability criteria, `scripts/wiki-metrics.js` meets:

- **Concurrency**: invoked from dashboard renders + auto-record loops + potential hook callers (multi-process)
- **State mutation**: writes shared `logs/wiki-metrics.json`

The whole purpose of `tmp + writeFileSync + renameSync` is concurrent-writer safety — shipping that claim without an adversarial test that fans out N parallel writers and asserts no lost updates / no torn JSON / no orphan tmps under contention is a Rule 1 governance gap.

## Tier classification

**Tier-2 mid-flight pivot equivalent (post-merge)**: surfaced via direct operator question on whether #1682 shipped with stress coverage. Authority: Manager (post-hoc, since the gap is downstream of a shipped ticket).

## Acceptance criteria

- **AC1**: `tests/stress-wiki-metrics.spec.js` exists, exercises N≥10 parallel `recordAccess` calls against the same file across worker_threads, asserts final `totalAccess` equals N and `pages` map sums match expected (no lost updates).
- **AC2**: Adversarial path — induce write failure mid-tmp (SIGKILL a writer between writeFileSync and renameSync) and assert subsequent loadMetrics returns a valid (possibly stale) value, never a torn `{` parse error.
- **AC3**: p99 latency budget asserted for single `recordAccess` under no-contention (<5 ms on stock Linux runner).
- **AC4**: Stress spec registered in [inventory/harness-self-test-registry.json](../inventory/harness-self-test-registry.json) under `regression` category, AC1-AC3 wired into `npm run harness:self-test`.
- **AC5**: COLLAB handoff on this follow-on ticket declares `test_strategy: tdd-pyramid+stress-test` (composable per Epic #1875).

## Anti-pattern this closes

Self-declared "stress not required" without matrix-criteria check — same trap class as calendar-threshold defaults ([[feedback_calendar_thresholds_in_agentic_systems]]). Decision rule going forward: when a script claims atomic / concurrent-safe semantics, the stress assertion is part of shipping that claim, not a follow-on.
