# #2624 — Add free-cloud usage report to surface paid-spend avoided

> **Source**: github:issue/2624 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P3, area:scripts, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 2624` (derived; edit the GitHub item, not this page).

## Body

## Problem (G8 in service of G3 — make the savings auditable)
#2619 + #2621 added a `free-cloud` lane that executes $0 cloud providers when the fleet is unreachable, emitting `recordTelemetry({lane:'free-cloud', model:<provider>, outcome, latency_ms, execute:true})` to `logs/model-routing-telemetry.jsonl`. But there is **no report** that surfaces this lane — unlike `routing:fallback-report`, `it-ops:usage-report`, and `routing:report`. The G3 savings (paid-Haiku calls avoided by free-cloud failover) are recorded but invisible/unauditable, so they cannot be cited against the premium-share governor or used to confirm the failover is actually firing.

## Objective
Add a free-cloud usage report that aggregates the `lane:free-cloud` telemetry into operator-visible, auditable output: execution count, per-provider breakdown, average latency, and estimated paid-tier calls avoided — completing the observability loop on #2619/#2621.

## Acceptance Criteria
- AC1: `scripts/global/free-cloud-usage-report.js` — reads `logs/model-routing-telemetry.jsonl` (reuse `model-routing-telemetry.readTelemetry`), filters `lane==='free-cloud'`, reports over a `--days N` window (default 7): total free-cloud executions, per-provider breakdown (the `model` field carries the provider), avg latency, and estimated paid-Haiku calls avoided (= successful free-cloud executions) with a rough $-avoided estimate using `policy.models.haiku.costPer1kTokens` and a documented assumed avg tokens/call.
- AC2: npm script `routing:free-cloud-report`.
- AC3: `tests/free-cloud-usage-report.spec.js` — synthetic telemetry → correct count/per-provider/paid-avoided aggregation; empty-log graceful (no crash, zero report).
- AC4: doc note in `hamr-routing.instructions.md` Measurement section (alongside fallback/it-bypass reports) + CHANGELOG fragment.

## Goal Lens
- G8 (primary mechanism): the free-cloud lane becomes operator-visible + auditable.
- G3 (purpose): proves paid-spend avoided; citable against the premium-share governor; confirms the #2619/#2621 failover actually fires.

## Connections / non-redundancy
- Completes observability for #2619 + #2621 (just shipped).
- Mirrors existing aggregators (`routing-fallback-report.js`, `it-bypass-usage-report.js`) — same JSONL-read + window pattern; no overlap (none report the free-cloud lane).
- No conflict: `scripts/global/` only; not Copilot-owned dashboard files.
