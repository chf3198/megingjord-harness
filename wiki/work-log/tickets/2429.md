# #2429 — Add unit tests for doc-coverage blocking validator behavior

## Body

## Problem
When the doc-coverage validators are promoted to hard-blocking (tickets #2424, #2426), there are no unit tests covering the new violation paths. Without tests, regressions back to advisory-only will be invisible.

## Objective
Add unit tests to `tests/` for the upgraded doc-coverage validators.

## Acceptance Criteria
- [ ] Test suite for `scripts/global/megalint/collaborator-handoff.js` covering:
  - Missing doc-coverage block → violation (ok: false)
  - Valid doc-coverage block with all surfaces DONE → ok: true
  - Valid doc-coverage block with N/A + reason → ok: true
  - Bare N/A without reason → violation
  - `lane:docs-research` → ok: true (no block required)
  - `lane:config-only` → ok: true (no block required)
- [ ] Test suite for `scripts/global/megalint/doc-coverage.js` covering:
  - Required surface missing from block → violation
  - Required surface present as DONE → ok
  - Required surface as N/A — reason → ok
  - Required surface as bare N/A → violation
  - Suggested surface missing → ok (advisory only)
- [ ] Tests run via `npm test`
- [ ] All tests green before close
- [ ] Test file(s) ≤100 lines each
