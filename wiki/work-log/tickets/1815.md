## Body

## Why this self-anneal is needed

Operator (Copilot Team session, 2026-05-17) observed live false-positive: prompt `"Complete the work on each of 1794, 1795, and 1796"` emitted

> **Warning from User Prompt Submit hook** — Governance gate: completion requested before required Admin steps were recorded.

This is **identical defect shape to Epic #1798** F2 (phase-blind Admin enforcement) + F3 (over-broad trigger words), but in a hook script Epic #1798 did NOT touch: `hooks/scripts/userprompt_gate.py`.

## Defect specifics

### Bug 1 — FINISH_RE over-broad trigger (Epic #1798 F3 equivalent)

`hooks/scripts/userprompt_gate.py:19-22`:

```python
FINISH_RE = re.compile(
    r"\b(done|finish|finished|complete|completed|close out|wrap up|ship|release)\b",
    re.IGNORECASE,
)
```

The bare word `complete` matches the Manager-phase prompt `"Complete the work on each of 1794, 1795, and 1796"`. The user is asking to BEGIN scoping work on three tickets; the regex misreads it as DECLARING work done.

### Bug 2 — `_admin_missing` is phase-blind (Epic #1798 F2 equivalent)

`hooks/scripts/userprompt_gate.py:27-34`:

```python
def _admin_missing(state: dict) -> list[str]:
    flags, ops = state.get("flags", {}), state.get("admin_ops", {})
    missing = []
    if flags.get("code_touched"):
        missing += [k for k in _ADMIN_KEYS if not ops.get(k)]
    ...
    return missing
```

No `roles.collaborator` guard. Returns "Admin missing" whenever `flags.code_touched` is true, regardless of whether Collaborator has finished. Same shape as the bug Epic #1798 fixed in `stop_checks.py:check_admin_ops` — except this hook was not in scope.

### Why Epic #1798 missed it

The audit identified `stop_checks.py` and `manager_ticket_gate.py` (UserPromptSubmit) but not the SECOND UserPromptSubmit hook `userprompt_gate.py`. The hooks.json wiring for Copilot:

```
"UserPromptSubmit": [
  { ... manager_ticket_gate.py ... },
  { ... userprompt_gate.py ... }   ← Epic #1798 missed this one
]
```

Both run on every prompt submission. Epic #1798's `test_baton_phase_aware.py` covers `manager_ticket_gate.main` but not `userprompt_gate.main`.
