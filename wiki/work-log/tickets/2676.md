## Body

## Problem

A governance gap was observed in the research-first flow: Phase-0 can complete green, yet no deterministic guardrail forces creation of Phase-1 acceptance-criteria and development child tickets.

Evidence:
- Epic #2661 completed Phase-0 via #2662 and closed terminally without spawning a Phase-1 child chain.
- This violates the intended research-first progression contract and creates drift between plan completion and executable implementation scope.

## Objective

Implement a deterministic, cross-runtime guardrail so that when a research-first Phase-0 child is green-complete, the system MUST produce a Phase-1 child-ticket set with explicit ACs (or block closure with machine-readable reason).

## Scope

This ticket is a self-anneal remediation task focused on promotion enforcement, not broad workflow redesign.

In scope:
- Define the canonical "Phase-0 green completion" predicate.
- Enforce automatic creation (or mandatory closure-block) for Phase-1 AC/dev child tickets.
- Ensure parity across Copilot, Codex, Claude Code, and Antigravity adapters.
- Add observability + failure evidence for missed promotions.

Out of scope:
- Rewriting unrelated Epic lifecycle semantics already covered in #2632.
- Broad artifact generation redesign already owned by #2037.

## Why this is not duplicate work

- Related to #2037 but narrower: this ticket targets the specific phase-transition guardrail (Phase-0 -> Phase-1 children).
- Related to #2632 but different: this is transition enforcement + artifact materialization, not state taxonomy.

## Acceptance Criteria

- [ ] AC1: A machine-checkable predicate `phase0_green_complete(issue)` is specified and implemented. Minimum required signals:
  - Research-first parent context present.
  - Phase-0 child terminal status done.
  - Required closeout evidence present for the ticket lane.
- [ ] AC2: On `phase0_green_complete == true`, automation creates Phase-1 child tickets from a canonical template set containing:
  - explicit AC checklist
  - lane label
  - test_strategy
  - dependency references to Phase-0 source
- [ ] AC3: If auto-create fails, parent Epic cannot close as completed; closure is blocked with a structured blocker note that includes cause + remediation path.
- [ ] AC4: Cross-runtime parity tests prove identical behavior for Copilot, Codex, Claude Code, and Antigravity.
- [ ] AC5: Add observability event for promotion decisions (`created`, `blocked`, `deferred`) with reason code.
