## Parent

## Research question

Two coupled questions:

1. **Pre-push parity**: Can/should every CI-required gate run locally before push? What's the empirical tradeoff between developer friction (slow hooks discourage commits) and CI re-roll cost (broken PRs waste cycles + cognitive load)?
2. **Self-annealing review chains**: Can a baton with three review tiers (Collaborator → Admin → Consultant) iterate toward perfect ratings? What's the failure mode of the current architecture? What does the 2025–2026 literature say about same-model vs different-model evaluators?

## Findings

### F1 — Shift-left is the 2026 consensus pattern

Multiple independent 2026 guides converge on the same recommendation: lightweight formatters/linters at pre-commit, slower checks (type-check, lint, full test suite) at pre-push, only genuinely server-side state at CI.

- Lefthook (Go, parallel) is ~10× faster than Husky (Node, serial) on large repos. For a config like this one (~300 JS files + Playwright + Python + Bash), the speed delta is the difference between adoption and being disabled.
- Industry research consistently puts the cost-to-fix multiplier at ~100× per stage of escalation (developer → CI → review → production).
- DevSecOpsSchool's 2026 maturity model: **Beginner** = formatters + linters at pre-commit; **Intermediate** = secret scanning + IaC + slower checks at pre-push; **Advanced** = policy-as-code + metrics collection.

Realistic ceiling: ~90% of CI failures catchable pre-push. The residual ~10% is genuinely server-side state (PR title doesn't exist pre-PR; merge-evidence requires post-merge state; cross-team mutex needs server arbitration).

### F2 — Multi-agent role separation matches every modern framework

The Collaborator → Admin → Consultant pattern is not novel — it converges with every major 2025–2026 framework:

| Framework | Roles | Architecture |
|---|---|---|
| Self-Refine (Madaan et al., NeurIPS '23, still canonical) | Generator → Critic → Refiner | Single model, same context, verbal-feedback loop |
| MAR (arXiv 2512.20845, 2026) | Actor → Debaters → Judge | Single model, n personas, debate-coordinated synthesis |
| Star Chamber (Mozilla.AI, 2025) | Claude → GPT → Gemini | Cross-provider parallel, three-tier consensus aggregation |
| MindStudio (2026) | Builder → Validator → Architect/Sec/QA | Model-diverse, isolated sessions, single-pass triage |
| LessonL (arXiv 2505.23946, 2025) | Lesson-sharing collaborators | Small-LLM team outperforms larger single-LLM |
| **Megingjord baton** | Manager → Collaborator → Admin → Consultant | Single-model-family, isolated baton artifacts, four-tier |

Common thread: **role separation + context isolation + model diversity**. The baton has the first two; it under-enforces the third.

### F3 — Self-bias amplification is the load-bearing limitation

This is the finding that most directly constrains the "eventually perfect ratings" goal:

- Panickssery, Bowman, Feng (NeurIPS '24/'25): *LLM Evaluators Recognize and Favor Their Own Generations*. GPT-4 and Llama-2 distinguish self-vs-other at non-trivial accuracy, and self-favor in scoring even when humans rate equivalent quality the same.
- Multiple 2025–2026 follow-ons (npj AI; ACL '25): self-bias amplifies monotonically across self-refinement iterations. The generator-evaluator-as-same-model exploits the evaluator's blind spots; scores inflate while human-preferred quality stagnates or worsens.
- MAR specifically calls out two failure modes for single-agent Reflexion: **confirmation bias** (same model reinforces its original reasoning) and **mode collapse** (identical buggy solutions repeat across iterations).
- Mitigations with evidence: chain-of-thought before verdict; cross-provider judges; Chatham House anonymization (don't echo builder's reasoning to validator).

Direct implication for the current baton: pursuing "10/10 rubric" via repeated same-model self-critique is *mathematically unsafe*. The route to legitimate high ratings is (a) different model rates, (b) deterministic evidence boxes replace subjective scores.

### F4 — Convergence has natural diminishing returns

- Self-Refine: iterations 0→1 yield strongest gains; iter 1→2 weaker; tapers thereafter. Stopping function (`is_refinement_sufficient`) is task-dependent but always finite.
- MAR: HotPotQA accuracy improves 44%→47% over 5 trials; plateau at trial 5. HumanEval pass@1 improves 76.4%→82.6%, similar plateau.
- Star Chamber debate mode: includes explicit convergence detection to prevent infinite-disagreement loops.

The harness should adopt explicit stopping rules: max 3 iterations OR rubric Δ ≤ 0.5 OR all deterministic gates green. Current `workflow-self-anneal` caps iterations (good) but not score-delta (gap).

### F5 — Cross-provider diversity > intra-provider size diversity

- Star Chamber: routing builder/validator to different providers (Anthropic / OpenAI / Google) yields more genuine independence than rotating model size within one provider.
- Diverse-LLM ensemble (arXiv 2512.12536, 2025): 10–12% higher vulnerability-detection accuracy vs averaging single-model performance; benefit grows with code complexity.
- BUT cross-model within the same provider still catches more than same-model — Claude-Opus vs Claude-Sonnet in isolated sessions detect distinct issues even sharing training base. Star Chamber framing: *"Two Claude instances in separate sessions will already catch more than one, because context isolation removes the specific blind spots introduced during generation."*

User decision (operator, 2026-05-15): adopt **cross-model within provider** as the enforced floor (lower coordination cost, ships now), with cross-provider as an aspirational target captured in Epic #1568 AC-4 (second-opinion Consultant pass with explicitly different-family rater).
