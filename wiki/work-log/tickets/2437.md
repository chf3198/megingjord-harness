## Problem

`agents/pre-merge-review/` has sub-agents for bug-detect, test-coverage, architectural-drift, and security — but **no doc-coverage sub-agent**. The cross-family Red-Team Reviewer cannot specifically evaluate whether documentation surfaces were actually written (present in diff) as part of the Collaborator's work.

## Objective

Create `agents/pre-merge-review/doc-coverage.md` — a cross-family sub-agent spec that evaluates doc surface presence in the PR diff, not just in the handoff declaration.

## Acceptance Criteria

- [ ] `agents/pre-merge-review/doc-coverage.md` exists with the standard JSON Lines output contract matching the other sub-agents (severity/category/file/line/message/suggestion/confidence/sub_agent/trigger)
- [ ] Detection focus covers: missing `.changes/unreleased/<N>.md` fragment in diff; missing `docs/workflow/learnings.md` entry when behavior changed; missing wiki entry when a concept/entity changed; `doc-coverage:` block in COLLABORATOR_HANDOFF cites surfaces that are NOT present in the diff (declared but not written)
- [ ] Auto-escalate trigger: any area:governance change without a corresponding doc surface in diff → severity: high; trigger: "governance-change-no-docs"
- [ ] Confidence calibration section
- [ ] Out-of-scope section (not code quality, not test coverage, not security)
- [ ] File ≤100 lines

## Design Note

The sub-agent is cross-family adversarial — it should err toward flagging missing surfaces rather than silent passes. It complements (does not replace) the programmatic `doc-coverage.js` validator: the validator checks the block schema; this sub-agent evaluates the substantive diff evidence.
