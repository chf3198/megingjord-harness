## Body

## Scope (Phase-0 child of Epic #2150)

Research-and-plan the conversion of fleet/HAMR governance from instruction-layer compliance to deterministic code-level enforcement. Epic body specifies 7 enforcement surfaces (AC1-AC7); Phase-0 validates the slate, identifies the right gate placement per surface, and converges on a Phase-1 implementation sequence.

Refs Epic #2150

## Iteration model

Single iteration acceptable in first pass since Epic body is detailed (7 ACs + 7-child plan + evidence model). Iterate further only if HAMR-routed fleet rater surfaces ≥1 ACCEPT finding that materially changes the slate.

Each iteration:
1. Synthesis comment (gate-placement design + Phase-1 sequencing)
2. HAMR-routed cross-family fleet adversarial red-team via dispatchRedTeam (Epic #2041 #2175)
3. Findings triage (ACCEPT/REJECT/PARTIAL with rationale)
4. Convergence check: zero unresolved ACCEPT AND Consultant rubric ≥8

## Research questions

1. **Gate placement per surface** — which AC enforcements belong as CI workflow vs pretool hook vs pre-push lefthook vs runtime wrapper? Different surfaces have different blast radii.
2. **Router enforcement (AC1)** — how to programmatically detect "fleet-eligible task bypassed fleet/HAMR" without false-positives on legitimate diagnostic/test cases?
3. **Adaptive timeout policy (AC2)** — what's the empirically-grounded timeout table per lane/model/workload? Phase-0 #2174 dog-food saw 907s p99 on qwen2.5-coder:32b under fleet load; current 600s bound was insufficient.
4. **Prompt budget + RAG (AC3)** — what's the chunking threshold for fleet calls? When does RAG fallback fire?
5. **Baton review evidence gate (AC4)** — analog to Epic #2192; how to avoid double-gating?
6. **Replay-eval corpus (AC5)** — which historical fleet/HAMR regressions form the corpus? (P1-7 #2178 sticky-route mis-recording IS one canonical example)
7. **Observability (AC6)** — what new metrics vs reusing cache-stats.jsonl from #1339 / Epic #2041?
8. **Auto-remediation ticketing (AC7)** — threshold + dedup model; integrate with existing anneal protocol per #1308
9. **Sequencing** — which AC is load-bearing for the rest? (likely AC1 + AC2; others stack on top)
