# #2946 — fix: worktree-session-start.sh should create isolated git worktree dir for implementation tickets

> **Source**: github:issue/2946 | **state: OPEN** | **Labels**: type:bug, priority:P1, area:scripts, role:admin, status:testing
> Mirror of `gh issue view 2946` (derived; edit the GitHub item, not this page).

## Body

## Problem

`scripts/worktree-session-start.sh <agent> <task-branch>` creates the task branch **inside the canonical-main checkout** (`/home/curtisfranks/devenv-ops`) via `git switch -c`. It does NOT create an isolated worktree directory via `git worktree add`.

The canonical-main enforcer (#2107) blocks writing new tracked files in that checkout. Any ticket requiring new tracked files (most implementation tickets) will be immediately blocked, even after following the recommended session-start procedure from `sandbox-worktree-governance.instructions.md`.

## Relationship to #2945

#2945 extends the enforcer to block tracked-file edits (not just creation). Together with this ticket, the combined effect is:

- Without #2945 + this fix: partial writes succeed, file creation blocked → dirty canonical-main
- With #2945 alone: all writes blocked immediately → clean fail, but the model has no path forward because the instructions directed it here
- With this fix alone: the script creates an isolated dir → all writes succeed → no failure
- With both #2945 + this fix: belt-and-suspenders coverage

**This ticket (#2946) is the primary fix. #2945 is the enforcer backstop.**

## Acceptance Criteria

- [ ] AC1: When invoked with a task-branch argument, `worktree-session-start.sh` creates an isolated worktree directory at `~/devenv-ops-<ticket-number>` via `git worktree add`, not a branch in canonical-main
- [ ] AC2: `git worktree add` target path is derived from the ticket number in the branch name (e.g., `fix/2933-slug` → `devenv-ops-2933`)
- [ ] AC3: node_modules bootstrap and per-worktree hooks setup run in the NEW isolated directory
- [ ] AC4: Script exits with an error if invoked while `cwd` is the canonical-main checkout AND a task branch is provided (prevents accidental in-place branch creation)
- [ ] AC5: When no task branch is provided (sandbox refresh only), behavior is unchanged — sandbox branch reset in canonical-main is valid
- [ ] AC6: `sandbox-worktree-governance.instructions.md` usage example updated to reflect the new behavior

## Fix Plan

Change the task-branch block in `worktree-session-start.sh` from:
```bash
git switch -c "$task_branch"
bootstrap_node_modules "$root"
configure_per_worktree_hooks "$root"
```
To:
```bash
ticket_num=$(echo "$task_branch" | grep -oP '(?<=/)\d+(?=-)' | head -1)
wt_dir="$HOME/devenv-ops-${ticket_num}"
git worktree add "$wt_dir" -b "$task_branch"
bootstrap_node_modules "$wt_dir"
configure_per_worktree_hooks "$wt_dir"
log "isolated worktree ready at $wt_dir on branch $task_branch"
```
