## Scope
- Regression test: an approved-source key is visible where intended and absent where not intended.
- Lint: flag a new JS/Python credential consumer that reads an env key without hydrating first.

## AC
- AC1: regression spec covers visible-where-intended AND absent-where-not.
- AC2: lint flags an unhydrated new consumer; documented false-positive bound (Phase-0 review flagged FP risk).
- AC3: green in CI.

test_strategy: tdd-pyramid.

## Red-team hardening ACs (#2775)
- AC4 (G1/G10 prevention): the lint fails CI AND pre-commit on (a) a new credential `process.env.*KEY/TOKEN/SECRET` read in a file lacking a hydration call, and (b) any `require('dotenv')`/`import dotenv` outside `load-local-env.js`/`load_local_env.py`.
- AC5 (G4 value-leak): a rule flagging `console.*`/`logger.*`/`print(` whose argument is a credential env read (catches a bypassed shim); documented false-positive bound.
- AC6 (G8 observability): emit a v3 `incidents.jsonl` event `pattern_id: credential-read-before-hydration` on a pre-hydration credential read, and a names-only audit line on hydration failure (never values).

## Red-team iter-2 refinement (#2775)
- AC4 refinement (G1/G10): credential-name match = default suffix set PLUS a project-configurable regex (`config/credential-name-patterns.json`) so non-suffix names are covered.
- AC5 refinement (G4): the value-leak rule also flags a `child_process`/`execFileSync`/`subprocess` call embedding a credential env value directly in args (recommend env pass-through).
- AC7 (G10, advisory): a non-blocking lint listing `.env` keys no consumer declares (clutter-reduction only).
- NOTE (G8, REJECTED finding): no enterprise SIEM integration — `incidents.jsonl` is consumed by the anneal recurrence detector + dashboard SSE (the $0 offline path); SIEM would invert G3/G5.

## Red-team iter-3 refinement (#2775)
- AC8 (G10): a one-time audit of existing non-credential `process.env` reads, flagging names that would become credential-consumers if a `*_KEY`/`*_TOKEN` sibling were added later (collision-risk inventory).
