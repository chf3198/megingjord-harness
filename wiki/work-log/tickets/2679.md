### 2. Historical Pattern

Script scan of all 36 research-first Epics: approximately 50% have Phase-1 children. All Phase-1 children were created by manual agent instruction-following (Claude Code opus-4-7). No automated creation mechanism found anywhere in the codebase. Confirmed via timeline API: Epic #2661 shows only Refs to #2662 (Phase-0) and #2676 (post-hoc anneal). Zero Phase-1 references before close.

### 3. Root Cause

Primary: No auto-create mechanism exists. The Phase-0 to Phase-1 transition is instruction-only.
Secondary: Advisory-only enforcement makes outcomes model-capability-dependent. Frontier models self-enforce; weaker Auto-routed models skip the step.

### 4. Proposed Design

**Component A: phase0_green_complete(epicNumber) predicate** (scripts/global/megalint/)
- Inputs: epic issue number
- Logic: (1) epic has phase-gate:research-first label AND (2) all Phase-0 children (issues referencing the Epic labeled phase-gate:research-first but NOT phase-gate:phase-1) are CLOSED AND (3) at least one has a CONSULTANT_CLOSEOUT comment AND (4) epic has an EPIC_RESCOPE comment
- Returns: { complete: bool, missingPhase1Children: bool, details: string }

**Component B: Auto-materialize script** (scripts/global/)
- Triggered by: phase0_green_complete returning true AND zero Phase-1 children exist
- Action: creates Phase-1 child tickets from template bundle with correct labels, references, and body stubs
- Invocation: GitHub Actions workflow on issues.closed event where closed issue has phase-gate:research-first label

**Component C: Closure-block guard** (modifications to closeout-lint.yml and label-lint-close-protection.js)
- When: Epic with phase-gate:research-first attempts to close
- Check: if phase0_green_complete() returns { complete: true, missingPhase1Children: true } -- emit structured BLOCKER_NOTE and fail workflow
- Form: EPIC_PHASE_GATE_PAUSE comment on Epic plus core.setFailed() exit

**Component D: Cross-runtime parity tests** (tests/)
- Tests covering: Copilot, Codex, Claude Code runtime surfaces all trigger the same enforcement path
- Regression test: recreates #2661 scenario and verifies it is blocked by the new guard
