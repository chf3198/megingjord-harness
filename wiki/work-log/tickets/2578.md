# #2578 — Prevent client arbitration for internal governance conflicts

> **Source**: github:issue/2578 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P1, area:governance, resolution:released, lane:code-change, anneal:tier-2
> Mirror of `gh issue view 2578` (derived; edit the GitHub item, not this page).

## Body

## Problem
Operators still surface internal governance/resource-conflict decisions to the client (for example: asking whether to keep/discard drift caused by cross-runtime sync or team collisions).

This violates the operator contract: the operator must own internal baton/worktree/conflict decisions and only consult the client for design direction or UAT.

## Objective
Make client-arbitration leakage impossible by default.

## Acceptance Criteria
- [ ] Add a blocking guardrail that flags/blocks output patterns that delegate internal governance/worktree conflict decisions to the client.
- [ ] Add a deterministic decision policy for common internal conflicts (sync residue, cross-team lease collisions, branch/worktree drift) so the operator auto-resolves with evidence.
- [ ] Add regression tests for known leakage phrases and scenarios.
- [ ] Add incident emission + anneal hook when the pattern is detected.
- [ ] Update operator-facing governance docs to codify "client never arbitrates internal team conflicts."

## Evidence of Recurrence
- Seen in #2574 delivery path where the client was asked to choose how to handle internal sync-drift conflicts.
- Mirrors existing memory pattern: operator-contract-manual-step-leak.
