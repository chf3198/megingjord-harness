## Goal

Refit `scripts/global/changelog-aggregate.js` to emit Keep-a-Changelog 1.1.0 standard hierarchy (H2 release headers, not H1) and add the T1-T8 mitigations from the Phase-0 threat model. After this child ships, future aggregator runs produce conforming markdown without manual restructuring.

## Acceptance criteria

- **AC1** Aggregator emits `## [<version>] - YYYY-MM-DD` H2 release headers (not `# <version>` H1) when promoting unreleased fragments to a release section.
- **AC2** Aggregator preserves the canonical `## [Unreleased]` section and aggregates fragment content under it (not creating per-fragment `## [Unreleased] — #N` sub-headers as the legacy behavior did).
- **AC3** T3 mitigation: validator rejects fragments containing H1 headers (`^# ` outside code blocks) before aggregation; clear author error message.
- **AC4** T4 mitigation: validator rejects fragments with broken heading hierarchy (H2→H4 skip etc) before aggregation.
- **AC5** T7 mitigation: aggregator no-ops gracefully when `.changes/unreleased/` is empty (no broken output; idempotent).
- **AC6** T8 mitigation: aggregator moves processed fragments to `.changes/v<X>/` archive on release-cut (NOT on dry-run aggregation); re-running on empty unreleased dir is a no-op.
- **AC7** 8+ unit tests at `tests/changelog-aggregate.spec.js` covering T1-T8 (one or more per threat).
- **AC8** Snapshot test `tests/changelog-structure.spec.js` (from C1) continues to PASS after a simulated aggregation cycle (fixture-based).
