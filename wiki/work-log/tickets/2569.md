# #2569 — Prevent agent credential prompts when approved local env already contains the secret

## Body

## Problem
During recent work the operator asked the client for secrets/credentials (a Tavily API key, GitHub login)
that were **already present in the approved local `.env`**. The client must not become the credential
transport layer for operator work (G1), and a locally-stored secret must not be re-exposed in chat (G4).
This is a recurring operator-behavior failure mode (also seen 2026-05-27), so a **durable, deterministic
guard** is required — not a one-off wording fix.

## Now buildable on #2645
#2645 shipped `scripts/global/load-local-env.js` (`loadLocalEnv`/`loadLocalEnvOnce`) — the deterministic
availability primitive: it can tell whether a named key is present in `process.env` or would be hydrated
from the approved local `.env`. This guard consumes it; it does not re-implement env parsing.

## Boundary vs. the hydration cluster (de-conflict — required)
- **#2645 (closed)** owns *hydration* (making `.env` keys visible). This ticket owns *prompt-suppression*
  (never ask the client for a secret that is locally available) and CONSUMES #2645's shim.
- **Epic #2291 / #2292** own the *global* cross-entrypoint hydration normalization + the entrypoint
  *inventory* + the secret-storage-strategy decision. This ticket does NOT re-run that inventory and does
  NOT decide storage strategy — it ships the deterministic availability check + the operator policy.

## Development plan
1. **Availability primitive (G2/G4)** — `scripts/global/credential-availability.js`:
   - `isSecretLocallyAvailable(varName, opts)` → boolean: true if `process.env[varName]` is set OR a
     non-mutating probe of `loadLocalEnv` (into a throwaway env object) would fill it. Never logs values.
   - `classifyCredentialRequest(text)` → `{ isSecretRequest, matched }`: conservative detection of a
     credential-request prompt (credential noun — api key / token / password / secret / login credential —
     AND/OR a known secret var name), so ordinary clarification prompts are NOT flagged (AC3, anti-over-block).
   - `preCredentialPromptCheck(varNames)` → `{ available:[…], absent:[…], action }` where action is
     `use-local` if any available, else `report-absent-no-prompt` (AC2/AC4 — absent ⇒ report + use
     terminal-entry/approved-auth, never request the raw secret in chat).
2. **Policy instruction (G1/G6 — AC6)** — `instructions/credential-prompt-guard.instructions.md`: the
   pre-prompt guard contract (check local availability first; if available use it; if absent, report +
   terminal-entry, never request the raw secret in chat), the catalog of secret-prompt paths (AC1), and
   the recurrence cases. Wired into `CLAUDE.md`.
3. **Tests** — `tests/credential-availability.spec.js`: the named recurrence cases + anti-over-block +
   absent-action; `tests/credential-prompt-guard-doclint.spec.js` (or a drift assertion) that the
   instruction contains the forbidden-behavior clause so the policy stays testable (AC6).

## Acceptance Criteria (testable, goal-mapped)
- [ ] **AC1 (catalog)** The instruction enumerates the prompt paths that can ask the client for a secret
      (chat NL prompt / AskUserQuestion / interactive auth) — concise catalog, not a re-run of #2292.
- [ ] **AC2 (G1 pre-prompt guard)** `preCredentialPromptCheck([names])` returns `use-local` when any
      requested secret is locally available; unit-tested.
- [ ] **AC3 (anti-over-block)** `classifyCredentialRequest` flags a credential request but NOT an ordinary
      non-secret clarification ("which file?", "what color?"); unit-tested both directions.
- [ ] **AC4 (G4/G6 absent behavior)** When a secret is absent locally, the check returns
      `report-absent-no-prompt` (never "request raw secret in chat"); unit-tested.
- [ ] **AC5 (G4 recurrence regression)** `isSecretLocallyAvailable('TAVILY_API_KEY')` and
      `isSecretLocallyAvailable('GITHUB_CLIENT_SECRET')` return true when present in env/.env, false when
      absent — the concrete recurrence cases, unit-tested; **no secret VALUE is ever logged/returned**.
- [ ] **AC6 (G1 policy + testability)** `instructions/credential-prompt-guard.instructions.md` exists,
      wired into `CLAUDE.md`, and explicitly forbids asking the client for a locally-available secret; a
      drift-lint test asserts the forbidden-behavior clause is present.
- [ ] **AC7 (G1 de-conflict)** Boundary section above reflected in the instruction; no duplication of
      #2291/#2292 inventory or storage-strategy scope.
- [ ] **AC8 (lint/cap)** Every new/edited file ≤100 lines; readability ≤ repo baseline.
