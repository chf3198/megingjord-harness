# #2567 — Design VS Code profile-aware MCP registration

> **Source**: github:issue/2567 | **state: OPEN** | **Labels**: type:task, status:backlog, priority:P2, area:scripts, phase-gate:phase-1
> Mirror of `gh issue view 2567` (derived; edit the GitHub item, not this page).

## Body

## Problem
The current Phase-1 plan intentionally targets the default VS Code user profile path `~/.config/Code/User/mcp.json`. Operators using non-default profiles can have profile-specific MCP configuration files, which means default-path registration is not fully portable.

## Objective
Design and validate a profile-aware registration strategy for VS Code MCP configuration, or define an explicit operator override mechanism that removes the default-profile assumption.

## Refs
Refs #2553 (Epic), Refs #2554 (Phase-0 research), Refs #2555 (C1), Refs #2558 (docs)
phase_0_sources: [#2554]

## Acceptance Criteria
- [ ] Identify the supported VS Code profile-specific MCP config locations for the operator environments we care about
- [ ] Decide whether the right fix is profile discovery, an explicit `--mcp-path` override, or both
- [ ] Document the chosen approach and how it interacts with default-profile behavior
- [ ] If implementation is needed, file or link the implementation ticket with clear ACs
- [ ] Update operator docs so non-default-profile users know how to register the server correctly

## Notes
This follow-on exists so the current epic can state its default-profile scope honestly rather than implying multi-profile support that has not been designed yet.
