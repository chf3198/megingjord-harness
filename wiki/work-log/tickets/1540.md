# #1540 — worktree-bootstrap-node-modules lacks main-dir sanity check

> **Source**: github:issue/1540 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:scripts, resolution:completed, lane:code-change

## Body

## Problem

`scripts/worktree-bootstrap-node-modules.sh` (shipped via #1378) checks:
```bash
if [[ -z "$main_root" || ! -d "$main_root/node_modules" ]]; then
  log "node_modules bootstrap: no main checkout node_modules found at $main_root; skipping"
  return 0
fi
```

The `-d` test follows symlinks. If `$main_root/node_modules` is a broken self-referential symlink, `-d` returns true (because the symlink itself exists), the script proceeds, and chains a broken link into every new worktree.

Discovered during #NEW-P1-CROSS-TEAM-BLOCKER triage. Root cause that enabled the cross-team blast radius.

## Proposed fix

Add a sanity check after the existing one:

```bash
# Self-reference guard: catch broken self-symlinks
if [[ -L "$main_root/node_modules" ]] && \
   [[ "$(readlink -f "$main_root/node_modules" 2>/dev/null || echo BROKEN)" == "$main_root/node_modules" || \
      "$(readlink -f "$main_root/node_modules" 2>/dev/null)" == "BROKEN" ]]; then
  log "ERROR: main checkout node_modules is a broken self-symlink at $main_root"
  log "       restore the real directory or run 'npm install' from main first"
  exit 1
fi
```

(Or equivalent — readlink + path-equality check.)
