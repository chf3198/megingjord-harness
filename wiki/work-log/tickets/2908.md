# #2908 — Add consultant rubric internal consistency validator (G-score vs verdict)

## Body

## Problem
A Consultant can post a CONSULTANT_CLOSEOUT with `rubric_rating: 4/10` on individual G-goals but still include `verdict: approve_for_merge`. This internal logical contradiction is not validated (Gap G-04). The closeout-schema validator checks field presence but not internal consistency.

Per the rule: `verdict: approve_for_merge` requires `min(G1..G9) >= 7`. This is currently unchecked.

## Acceptance Criteria
- [ ] In `scripts/global/megalint/consultant-closeout.js`, parse individual G1–G9 scores from `rubric_rating` block
- [ ] Block if `min(G1..G9) < 7` AND `verdict: approve_for_merge`
- [ ] Block if `min(G1..G9) >= 7` AND `verdict: reject`
- [ ] Add tests: `tests/consultant-rubric-consistency.spec.js`
