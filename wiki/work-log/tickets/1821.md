# #1821 — Self-anneal: scope visual_qa requirement to ui_touched (Copilot Team Stop-hook false positive)

## Body

## Why this self-anneal is needed

Operator (Copilot Team session, 2026-05-17) observed live false-positive: editing `tests/routing-policy.spec.js`, `tests/batch-route.spec.js`, `tests/ide-proxy-runtime.spec.js` (test specs — NOT UI code) on PR #1820 emitted

> **Warning from Stop hook** — Hard governance gate. Missing: visual_qa.

This is **the third Copilot governance-hook false-positive in this session** (after #1798 stop_checks phase-blindness and #1815 userprompt_gate phase-blindness). Same defect family: hook scope is too broad.

## Root cause

Two hooks fire `visual_qa` requirement based on **repo classification + any code change**, not on **actual UI path edits**:

1. `hooks/scripts/stop_checks.py:68`:
   ```python
   if repo_type in ("website-static", "web-app") and flags.get("code_touched"):
       ext.append("visual_qa")
   ```

2. `hooks/scripts/pretool_guard.py:84`:
   ```python
   if RE_GIT_TAG.search(joined) and repo_type in ("website-static","web-app") and not ops.get("visual_qa"):
       return emit("deny","Tag blocked: visual QA not recorded for web repo.")
   ```

The Megingjord repo is classified as `"website-static"` because `dashboard/index.html` and `dashboard/css/*.css` exist (per `repo_detection.detect_repo_type`). But the repo is primarily a CLI/governance harness; the dashboard is one sub-component. ANY `.js` edit in a test spec, script, or hook triggers `code_touched`, which then triggers `visual_qa` requirement.

## Fix

Introduce path-scoped `ui_touched` flag (parallel to `extension_touched`):

1. **`classify_path`** — return `"ui"` for: HTML/CSS/TSX/JSX/Vue/Svelte files, paths under `dashboard/`, `public/`, `src/(components|pages|views)/`.
2. **`mark_tool_activity`** — when path classifies as `"ui"`, set BOTH `flags.ui_touched = True` AND `flags.code_touched = True` (parallel to extension handling).
3. **`stop_checks.check_admin_ops`** — change visual_qa condition from `repo_type ∈ {web-app, website-static} AND code_touched` to `ui_touched`.
4. **`pretool_guard`** — same condition change for git-tag deny.
5. **`state_store._default_state`** — add `"ui_touched": False` to flags dict (additive; backward-compatible).
6. **Tests** — extend `tests/hooks/test_baton_phase_aware.py` `UiTouchedScoping` class (5 cases covering test specs NOT marked UI, dashboard paths marked UI, visual_qa firing only when ui_touched).
