# #2272 — Constrain wiki scanner paths to repository root

> **Source**: github:issue/2272 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:knowledge, lane:code-change
> Mirror of `gh issue view 2272` (derived; edit the GitHub item, not this page).

## Body

Problem
The #2054 red-team pass identified a plausible path-boundary risk: shared wiki scanners (`listPages`, `computeWikiHealth`) accept a `wikiDir` argument and currently rely on caller discipline. Even if current callers are trusted, this should be fail-closed at the utility layer.

Objective
Enforce repository wiki-root boundary checks in scanner/write helpers so path traversal or out-of-scope roots are rejected deterministically.

Acceptance Criteria
- `scripts/wiki/wiki-io.js` rejects `wikiDir` values that resolve outside repo `wiki/` root.
- `scripts/wiki/health-contract.js` uses same boundary guard and fails closed on invalid roots.
- Add tests for `../` and absolute-path escape attempts.
- Keep current valid temp-dir test strategy via explicit safe test override mechanism, or refactor tests to use sanctioned path fixtures.

Validation
- Targeted wiki tests pass.
- Lint passes.
