## Body

## Problem

Copilot Auto-mode execution on ticket #2940 drifted from full harness governance rigor mid-flight:
- attempted to proceed without explicit baton artifact continuity at each step,
- attempted cross-family preflight fallback in a way that looked like governance bypass,
- created avoidable confidence loss around role sequencing and parallel team safety.

This is a recurrence-class governance failure risk: Auto model selection must not reduce governance discipline compared to frontier model routing.

## Expected Outcome

Auto-mode sessions enforce identical governance behavior as frontier sessions:
- explicit Manager -> Collaborator -> Admin -> Consultant baton continuity,
- no parallel-unsafe shortcuts,
- transparent blocker handling when gates fail,
- Tier-2 escalation emitted during implementation when recurrence risk appears.

## Acceptance Criteria

- Add guardrail(s) that fail-fast when baton artifact progression is incomplete before privileged admin actions.
- Add/extend test coverage for Auto-mode governance parity with frontier-mode expectations.
- Add operator-facing guidance to prevent preflight/gate bypass-looking behavior.
- Document Tier-2 emission rule in implementation-time checklist for Copilot runtime.

## Verification

- Targeted validator tests pass.
- At least one replay scenario demonstrates Auto-mode governance parity.
