# #2720 — Add /research-redteam-loop skill: web-augmented cross-family consensus gate

## Body

## Problem / objective

The research→adversarial-red-team→rework→consensus loop (just run by hand for Epic #2709 Phase-0 / #2710) is a repeatable, valuable process with no one-shot invocation. Existing skills (`role-red-team-critique`, `cross-family-review`) provide cross-family `/100` rating but **not** (a) web-search augmentation for cutting-edge expertise, nor (b) a gated `>N` consensus *loop* with automatic Manager rework. Operators currently spell the whole loop out in prose each time.

## Deliverable

A user-invocable skill + slash command `/research-redteam-loop` that bundles the documented loop into one line:

```
/research-redteam-loop #N --gate 93 --web --reviewer cross-family
```

## Acceptance criteria

- **AC1** — Skill `skills/research-redteam-loop/SKILL.md` + `.claude/commands/research-redteam-loop.md`, user-invocable, documenting the loop: produce/refresh research deliverable → web-augmented cross-family red-team rating `/100` (detailed issue comments) → if `≤ gate`, Manager reworks with fresh web research → repeat until `> gate` → post ACCEPT verdict + unlock signal.
- **AC2** — **Web augmentation**: the red-team step injects current web-search evidence into the reviewer prompt (the existing fleet skills do not). Source via WebSearch; cite in the issue comment.
- **AC3** — **Fleet→free-cloud failover baked in**: if the fleet reviewer (qwen) is unreachable/returns empty, fail over to a free-cloud **cross-family** reviewer (gemini/groq/cerebras `$0`) before any paid lane, per #2619. (Learned this run: qwen3:32b returned empty; gemini carried it.)
- **AC4** — **Calibrated scoring guard**: instruct the reviewer to give a calibrated holistic `/100`, not a naive per-goal mean, and to withhold ACCEPT only on a *named* unresolved deficiency — preventing the averaging artifact that trapped a genuinely-accept plan at 91 this run. Bounded iteration cap (e.g. 5) with escalation if not converged.
- **AC5** — Posts structured red-team comments per iteration to the target issue for Manager consumption; records the final verdict + per-goal vector; emits the Phase-1 unlock signal only on `> gate`.
- **AC6** — Reuses `scripts/global/fleet-red-team-dispatch.js` + `free-cloud-dispatch.js`; no new provider plumbing.

## Evidence / origin

Hand-run for #2709 Phase-0 (#2710): 4 iterations 78→92→91→94 ACCEPT on gemini-2.5-flash (cross-family, free-cloud failover from a dead qwen3:32b). This ticket productizes that run.

## Goal lens
G2 Quality (rigorous gated review), G3 Zero-Cost (fleet/free-cloud lanes, no paid), G8 Observability (per-iteration audit trail), G10 Maintainability (one-line invocation vs prose).
