# #2910 — Implement authorization profile runtime enforcement in pretool_guard

## Body

## Problem
Authorization profiles (`owner`, `guarded`, `restricted`) exist in `config/authorization-profiles.json` and `instructions/authorization-profile-context.instructions.md` but are not enforced at runtime (Gap G-05). When a session runs with `MEGINGJORD_AUTH_PROFILE=guarded`, the install/privileged capability restrictions are instruction-only — no gate blocks a disallowed tool call.

## Acceptance Criteria
- [ ] Create `scripts/global/auth-profile-enforcer.js` that reads `$MEGINGJORD_AUTH_PROFILE` and returns capability matrix
- [ ] Integrate with `hooks/scripts/pretool_guard.py`: before privileged operations, check profile capability
- [ ] Add `guarded` mode test: `install` tool call returns deny when profile=guarded
- [ ] Add `restricted` mode test: `execute_remote` tool call returns deny when profile=restricted
- [ ] Document in `config/authorization-profiles.json` schema
