## Body

## C3 — Pre-Push Closeout Presence Check

Refs Epic #3026

### Problem
A session can end with a pushed branch and open PR where the linked issue has no CONSULTANT_CLOSEOUT comment. No automated gate catches this at push time, allowing the violation to silently persist until the next session or manual-pull.

### Acceptance Criteria
- [ ] AC1: `scripts/global/open-pr-closeout-check.js` queries GitHub for open PRs on the current repo where: (a) a linked issue exists in the PR body (`Refs #N` or `Closes #N`) and (b) the linked issue has no comment containing `CONSULTANT_CLOSEOUT` or `merge-evidence-deferred-final`. Exits 1 with actionable error message if any such PR is found.
- [ ] AC2: Integrated into `lefthook.yml` as a `pre-push` hook (advisory — exit 0 with warning unless `CLOSEOUT_CHECK_BLOCK=1` env var is set, to allow gradual rollout). Override marker: `[skip-closeout-check]` in latest commit message.
- [ ] AC3: `tests/open-pr-closeout-check.spec.js` covers: clean state passes, open PR with closeout passes, open PR without closeout warns/blocks, override marker respected, no PRs is a clean pass.
- [ ] AC4: `npm run check:closeout` convenience alias added to `package.json`

### Advisory-first rollout
Ship as advisory (`exit 0` + warning message) for one release cycle. Promote to blocking (`CLOSEOUT_CHECK_BLOCK=1` default) in a follow-on ticket once false-positive rate is measured.

### Non-goals
- Does not check ADMIN_HANDOFF or COLLABORATOR_HANDOFF presence (separate concern)
- Does not auto-complete the closeout
