# #2585 — Harden pretool_guard active-ticket gate: worktree-correct + repo-scoped, runtime-neutral across 4 SDKs

## Body

## Goal

Harden the `pretool_guard.py` active-ticket gate so it is (1) **worktree-correct** and (2) **repo-scoped**, with a single runtime-neutral implementation that behaves identically across all four AI agent SDKs (Claude Code, Copilot, Codex, Antigravity). Surfaced 2026-06-02 during #2564/#2579 delivery (operator-local memory `feedback_worktree_edit_ticket_gate_reads_main_state`).

Two child findings:
- **Finding 1 — worktree-blind gate**: the active-ticket check keys on the session `cwd` (main checkout), not the worktree of the edited file, so legitimate worktree edits are falsely denied "no active ticket".
- **Finding 2 — over-broad block**: the no-active-ticket deny fires for *every* Write/Edit, including writes to paths outside any repo (e.g. the `~/.claude` memory store), which are not ticket-gated work.

## Cross-SDK Compatibility Contract (binding on both children)

This Epic exists because the gate is shared infrastructure. Every child MUST satisfy these invariants; the Consultant gate verifies each one with evidence.

1. **Single shared artifact.** `~/.copilot/hooks/scripts/pretool_guard.py` is executed verbatim by all runtimes — Claude (`.claude/settings.json` PreToolUse → `python3 ~/.copilot/hooks/scripts/pretool_guard.py`), Codex (`.codex/runtime-hooks.json` PreToolUse + PermissionRequest), Copilot, and Antigravity. One fix propagates to all four via `deploy.sh`. No per-runtime forks or `if runtime == ...` branches.
2. **Payload-key neutrality.** Tool-input path keys differ per runtime (Claude `file_path`, Antigravity `TargetFile`, Codex `apply_patch`/`Edit`/`Write`, Copilot variants). Children MUST reuse `admin_patterns.iter_paths` (the #2371-hardened `_PATH_PARAM_KEYS` normalizer) to read edited paths — never hardcode a key.
3. **Graceful degradation (G6).** Every new code path falls back to current behavior on any failure (missing path, non-git directory, `git` unavailable, detached HEAD, branch with no ticket). The hook must never crash and must never *newly* block.
4. **Strictly-additive permission change.** Both fixes may only *remove* false denials, never *add* a denial. By construction they cannot regress governance on real repo files or break any runtime.
5. **Runtime-identity-independent.** Behavior derives from payload + filesystem only. The `DEVENT_OPS_RUNTIME` env (set by Codex/Antigravity wiring) MUST NOT alter outcomes.
6. **No new payload fields.** Uses only `tool_name`, `tool_input`, `cwd` — fields every runtime already sends.
7. **Per-runtime parity tests.** Extend the `tests/hooks/test_pretool_guard_antigravity.py` pattern into a 4-runtime matrix (Claude / Copilot / Codex / Antigravity payload shapes) asserting identical decisions for both fixes. `scripts/global/hook-parity-check.js` parity check stays green.

## Test strategy (binding)

`tdd-pyramid + stress-test` — `pretool_guard.py` is a side-effect-bearing gate AND an adversarial-input parser (per `test-methodology-matrix`). Stress corpus MUST include: symlinked/`../`-traversal paths, non-git directories, detached HEAD, multi-path edits spanning different worktrees, and each runtime's payload shape.
