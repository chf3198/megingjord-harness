# #2049 — Support configurable default branch in branch-cleanup-plan.js

> **Source**: github:issue/2049 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:scripts, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 2049` (derived; edit the GitHub item, not this page).

## Body

## Problem

`isMergedToMain()` in `scripts/global/branch-cleanup-plan.js` hardcodes `origin/main`. Operators whose repository default branch is `master`, `trunk`, or any other name get systematically wrong `keep-active` classifications for all merged branches, with no error or warning.

## Expected Outcome

Read default branch from `DEFAULT_BRANCH` env var, falling back to `git symbolic-ref refs/remotes/origin/HEAD` auto-detection, then falling back to `origin/main`.

## Acceptance Criteria

- AC1: `DEFAULT_BRANCH` env var overrides the hardcoded reference
- AC2: Auto-detection via `git symbolic-ref refs/remotes/origin/HEAD` used as secondary fallback
- AC3: Fallback chain documented in a comment
- AC4: Unit test confirms override injection via `plan(overrides)` pattern

## Context

Found during Consultant red-team on #1610. Non-blocking for Megingjord (uses main), but affects portability (G5) for any operator on a non-main default branch.
