## Body

## Context

Follow-on from #2090 (catch-empty lint gate). Adversarial red-team review (Architect, rubric 7/10) identified these advisory gaps.

## Problem

Four advisory findings from post-implementation analysis:

1. **Scope gap (G9)**: `catch-empty-lint.sh` only scans `.github/workflows/`. Empty catches in `scripts/` and `hooks/scripts/` are not covered.
2. **Suppression contract undocumented (G1/G10)**: The `// catch-empty: <reason>` escape hatch is not documented in `instructions/`. Future contributors have no canonical reference.
3. **Error-code specificity (G4 SF-2)**: Annotated sites suppress ALL errors. The catch site should filter only the expected HTTP status code (404/422), not swallow all errors.
4. **Telemetry (G8)**: No step in CI counts or surfaces the number of active suppression sites.

## Acceptance Criteria

- [ ] AC1: `catch-empty-lint.sh` or CI step covers `scripts/global/` and `hooks/scripts/` (or document why exclusion is intentional)
- [ ] AC2: `instructions/` contains a doc entry explaining the `// catch-empty: <reason>` contract, valid uses, and review expectations
- [ ] AC3: At least 2 representative annotated `.catch()` sites updated to filter specific expected HTTP status codes instead of swallowing all errors
- [ ] AC4: CI `lint.yml` includes a step that counts active `// catch-empty:` annotations and emits the count to the job summary
