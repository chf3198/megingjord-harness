# #2619 — Route fleet-unavailable failover to free-cloud before paid Haiku

> Mirror of `gh issue view 2619` (derived; edit the GitHub item, not this page).

## Body

## Problem (G3 lane-order bug)
When the local fleet (Ollama) is **unreachable**, `scripts/global/cascade-dispatch.js` escalates straight to the **paid** Haiku tier (`suggested_tier: 'haiku'`) — at every fleet-availability failure site (`!local.ok`, solo-mode `fleet_unavailable`), and `hooks/scripts/task_router.py` does the same on cascade failure / script-not-found. This violates the G3 cost-ascending lane order: when fleet is down we received **no** answer, so the next-cheapest lane is a **free cloud** provider (OpenRouter-free / gemini / groq / cerebras free-tier, all $0), **not** paid Haiku.

`model-routing-policy.json` already declares `enforceFreeFleetFirst: true` and an `openrouter-free-failover` skill exists, but neither is wired into the cascade escalation signal — so the automated guidance recommends paid spend even though free cloud is available.

Observed this session: the tailscale fleet host was repeatedly DOWN; review work was manually routed to free gemini-2.5-flash, but the automated cascade would have escalated to paid Haiku. Direct, recurring G3 leak.

## Objective
Distinguish **availability failures** (fleet down → no answer) from **capability failures** (fleet answered but quality/judge inadequate). Availability failures escalate to a **free-cloud** tier first; only capability failures step up to paid Haiku.

## Acceptance Criteria
- AC1: `cascade-dispatch.js` — availability failures (`ollama_unreachable`, connection error, solo-mode `fleet_unavailable`) set `suggested_tier: 'free-cloud'`; capability failures (`quality_reason`, `judge_low_score`) keep `suggested_tier: 'haiku'`. Telemetry `escalation` field mirrors the chosen tier.
- AC2: `hooks/scripts/task_router.py` — fleet-unavailable / cascade-script-not-found paths suggest `free-cloud` (availability), consistent with AC1.
- AC3: `model-routing-policy.json` — define the `free-cloud` tier (ordered free cloud providers: openrouter-free, gemini, groq, cerebras) so the new tier is policy-recognized; consumers (`routing_context.py`) pass it through unchanged.
- AC4: Tests — `tests/cascade-free-cloud-failover.spec.js`: fleet-down → `free-cloud` (NOT haiku); quality-fail → `haiku`; judge-fail → `haiku`. Plus a python-path assertion harness for `task_router.py`.
- AC5: Doc — note the free-cloud failover step in the lane-order guidance (`global-task-router.instructions.md` / `hamr-routing.instructions.md`); CHANGELOG fragment.

## Goal Lens
- G3 (primary): fleet-down failover routes to $0 cloud providers before any paid tier — closes a recurring paid-spend leak.
- G6: graceful degradation — explicit availability-vs-capability distinction; free-cloud is itself a failover chain.
- G8: telemetry records the corrected escalation tier for auditability.
