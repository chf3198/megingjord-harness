# #2981 — Inspect and auto-remediate ticket governance drift at zero token cost

> **Source**: github:issue/2981 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P2, area:governance, resolution:released, lane:code-change, phase-gate:research-first
> Mirror of `gh issue view 2981` (derived; edit the GitHub item, not this page).

## Body

## Objective

Build a unified, batch, fleet-wide **ticket governance-drift sweeper** with a
deterministic **auto-remediation** layer that runs at **$0 token cost** (pure
`gh` API JSON + rule evaluation, no LLM). Future drift inspections + safe fixes
should run as a single command — e.g. `npm run governance:drift-sweep -- --fix`
— instead of a human-driven LLM sweep that burns premium tokens.

## Evidence / trigger

A 2026-06-12 manual drift audit of all 127 open tickets (Claude Code, premium
lane) found **8 distinct drift classes** (D1–D8 below). Two of them — fully
unlabeled Epics **#2970** and **#2971** — had slipped past the existing
event-driven `label-lint.yml` entirely. The audit itself cost premium tokens
to perform work that is **fully deterministic**: label-combination legality,
title-convention checks, and terminal-state hygiene need no model reasoning.

## The gap (why existing tooling is insufficient)

The harness already has strong drift **detection**, but it is fragmented and
report-only:

- ~15 scripts: `governance-drift-classifier.js`, `ticket-reconcile.js`,
  `role-baton-audit.js`, `lint-epic-drift.js`, `label-lint-status-cardinality.js`,
  `lint-ticket-redundancy.js`, `coordinator-label-cleanup.js`, etc.
- ~17 workflows: `label-lint.yml`, `role-baton-linter.yml`, `goal-drift-lint.yml`,
  `epic-traceability-lint.yml`, `drift-detection.yml`, etc.

Three gaps remain:

1. **No single batch entry point** — drift logic is spread across many scripts;
   there is no one command that sweeps every open ticket and emits one report.
2. **No auto-remediation** — every existing tool *reports*; none *applies* the
   safe deterministic fix (relabel, demote terminal label, normalize title).
3. **Event-driven, not on-demand fleet-wide** — `label-lint.yml` fires per-issue
   on `issues` events; nothing re-scans the full backlog, so historical drift
   (#2970/#2971) accumulates silently. Manual sweeps are the only fleet-wide
   path today, and they cost premium tokens.

## Scope (Phase-0 research-first)

This Epic is **research-first** and starts with exactly one Phase-0 research
child (see sub-issue). Phase-0 must, before any implementation child is authored:

1. **Inventory** every existing drift-detection script + workflow and map each to
   the D1–D8 classes (reuse-vs-build decision per class). Avoid duplicating
   `governance-drift-classifier.js` / `ticket-reconcile.js` / `role-baton-audit.js`.
2. **Classify** each drift class as **auto-fix-safe** (deterministic, reversible,
   no judgment) vs **propose-only** (needs Manager/Consultant verdict).
3. **Design** the unified CLI: one `--scan` (report) mode and one `--fix` (apply
   safe classes) mode, with a dry-run default and an audit-log of every mutation.
4. **Cost model**: prove the deterministic path is $0 (no LLM). Decide whether any
   ambiguous residue (e.g. title rewrite phrasing, D1 type inference) routes to the
   **free/fleet lane only** — never premium (G3). Most classes should need no model.
5. **Governance fit**: define how `--fix` mutations are themselves auditable (G8),
   reversible, and respect the canonical-main / no-code-remediation lane contract.

## Goal-lens justification

- **G3 Zero Cost (primary driver)**: replaces premium-token manual sweeps with a
  $0 deterministic tool; ambiguous residue caps at the free/fleet lane.
- **G1 Governance**: closes the historical-drift gap that event-driven label-lint
  cannot reach (#2970/#2971 proof).
- **G8 Observability**: every remediation is logged + attributable.
- **G10 Maintainability**: consolidates ~15 fragmented detectors behind one
  command, reducing surface drift.

## Non-goals / boundaries

- Do **not** rebuild detection that `governance-drift-classifier.js`,
  `ticket-reconcile.js`, or `role-baton-audit.js` already do — wrap/reuse them.
- Do **not** auto-fix propose-only classes (D1/D2/D6/D7) without explicit verdict.
- Does not change baton/handoff validators (those gate live work, not metadata).
- Overlap with #2891 (instructions→enforced gates) is complementary: #2891 hardens
  gates on *live* work; this Epic *retroactively* sweeps *existing* ticket metadata.
