# #2425 — Define doc-coverage block schema and update COLLABORATOR_HANDOFF template

> **Source**: github:issue/2425 | **state: OPEN** | **Labels**: type:task, priority:P1, area:governance, status:triage, lane:code-change, phase-gate:phase-1
> Mirror of `gh issue view 2425` (derived; edit the GitHub item, not this page).

## Body

## Problem
There is no defined machine-parseable schema for the `doc-coverage:` block in COLLABORATOR_HANDOFF comments. Contributors have no template showing what to write, and the CI gate has no schema to parse against.

## Objective
Define a formal schema for the `doc-coverage:` block and update the COLLABORATOR_HANDOFF template to pre-populate required surfaces from the ticket's area labels.

## Acceptance Criteria
- [ ] Schema defined in `config/doc-coverage-block.schema.json` or in `config/doc-coverage-matrix.yml` as a schema section
- [ ] Schema requires: `required` array of `{ surface, status }` entries; `status` must be `DONE` or `N/A — <reason>` (reason required, not bare N/A)
- [ ] `suggested` array is optional; entries are informational only
- [ ] COLLABORATOR_HANDOFF template (in `templates/` or `docs/`) updated to include pre-populated `doc-coverage:` block based on ticket area labels
- [ ] Example in template documentation shows DONE and N/A variants
- [ ] Schema documented in `docs/workflow/` or `CONTRIBUTING.md`
- [ ] File stays ≤100 lines
