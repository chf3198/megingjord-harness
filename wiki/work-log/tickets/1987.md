## Problem

C3a (#1968 — formerly C3) produces a mapping table from OWASP Agentic Top 10 to harness goals + per-risk current-coverage classification (enforced / advisory / missing). The mapping is documentation only. Any risk classified "advisory" or "missing" in C3a output remains in that state indefinitely unless promotion work is scoped explicitly.

## Scope

For each OWASP Agentic Top 10 risk that C3a classifies as "advisory" or "missing" in the harness, promote to "enforced" via a deterministic primitive (hook, CI gate, or runtime contract). The set of risks to enforce is determined by C3a's output, not pre-enumerated here.

## Acceptance criteria

- [ ] AC1: every risk C3a labels "advisory" or "missing" has at least one Enforcement primitive in `wiki/concepts/harness-goal-controls.md`, OR has a documented decision-justified deferral with rationale per goal-lens (e.g., out-of-scope until external dependency lands).
- [ ] AC2: each promoted primitive is a concrete artifact (a hook, a gate, a runtime check) — not a documentation entry.
- [ ] AC3: regression test fixture per primitive — proves the gate fires on the violating pattern.
- [ ] AC4: `npm run lint` clean; ≤100 lines per file.
