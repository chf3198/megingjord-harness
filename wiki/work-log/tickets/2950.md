# #2950 — Codex runtime deploy omits scripts/global review pipeline modules

> **Source**: github:issue/2950 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:infra
> Mirror of `gh issue view 2950` (derived; edit the GitHub item, not this page).

## Body

## Problem
The C9 parity verifier (`scripts/global/review-cli-parity.js`, #2935) reports **50% runtime coverage**: the **copilot** runtime is fully synced, but the **codex** runtime (`~/.codex/devenv-ops/scripts/`, deployed via `scripts/global/codex-runtime.js`) is stale —
- `cascade-dispatch.js` → hash-mismatch (older version)
- `fleet-backend-select.js`, `fleet-escalation-policy.js`, `review-stakes-router.js`, `review-cost-telemetry.js`, `review-privacy-gate.js` → **absent**

So the Codex runtime's review skill does NOT get the Epic #2926 cost-ascending review pipeline (C1/C3-C8). The 'one shared CLI, 4 runtimes' D9 goal is met for the single SOURCE (6/6 modules unforked in scripts/global/) but not yet for codex DEPLOYMENT.

## Root cause
`deploy.sh` deploys `scripts/global → ~/.copilot/scripts` via `deploy_files` (line 72/85), but the Codex path goes through `codex-runtime.js deploy` (line 59) which carries a narrower/older manifest that doesn't include the new review modules. (~/.claude and ~/.agents have no stable scripts mirror at all.)

## Fix
Bring `codex-runtime.js`'s deploy manifest in line with the `scripts/global` set (or have it reuse `deploy_files`), and wire the C9 verifier into `hamr:sync-verify` so the coverage gap is caught going forward. Re-run `npm run review:cli-parity:test` evidence after.

## Goal-lens
G9 interoperability (cross-runtime parity) + G5 portability. Surfaced honestly by the C9 verifier rather than hidden behind a fake pass.
