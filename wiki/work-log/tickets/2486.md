# #2486 — Cross-runtime /xteam MCP slash-command surface for synthesis kickoff

## Body

## Goal

Replace the manual 5-step cross-team R&D synthesis kickoff (terminal command + 4 file-paste operations) with a portable MCP slash-command surface that any AI runtime (Claude Code, Copilot, Codex, Antigravity) can use via a single `/xteam` invocation.

## Problem

Per Epic #1112 + #2485 documentation, kicking off cross-team R&D synthesis requires the client to:
1. Open a terminal, run `npm run synthesis:init -- --epic <N>`
2. Find the generated `planning/synthesis-<N>/<team>-prompt.md` files
3. Manually copy contents of each prompt file
4. Paste into each of 4 team chat sessions

This puts terminal + file-system burden on the client (chf3198 = Client role, not Operator). Per Megingjord governance, client role is design + UAT only.

## Proposed solution

Build an MCP (Model Context Protocol) server `megingjord-xteam-mcp` that exposes 3 slash-commands consumable by every MCP-compatible runtime:

```
/xteam <epic-N>        — join synthesis on existing Epic
/xteam ? <description> — create Epic + research ticket from scratch, then kick off
/xteam-status <N>      — query progress without launching
```

The first team session to receive any of these claims LEAD role; subsequent invocations on other team sessions take NON-LEAD roles. The server uses an atomic leader-claim primitive (HAMR merge-claim or GitHub-label-based equivalent per #2479) to handle the race.

MCP `prompts` feature is the 2026 standard for cross-client slash-commands — surfaces natively in Claude Desktop, Cursor, VS Code Copilot, Codex, Antigravity, etc.
