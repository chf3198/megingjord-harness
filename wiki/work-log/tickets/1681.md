## Body

## Context
Part of Epic #1626. Independent. Trivial but high-integrity fix.

`wiki/log.md` contains an entry dated `2026-07-14` — a future date (today is 2026-05-15). The `appendLog()` function in `scripts/wiki/wiki-io.js` writes entries with whatever date is supplied or defaults to `new Date().toISOString().slice(0,10)`. There is no guard preventing caller code or tests from passing future dates, and the anomaly is invisible until a hygiene audit runs.

## Objective
Add a chronology guard to `appendLog()` that rejects any entry whose `date` is more than 1 day in the future (±1 day tolerance for timezone edge cases), and back-correct the existing future-dated entry.

## Implementation notes
- In `appendLog()`, check `new Date(entry.date) > new Date() + 1 day` — if so, throw a descriptive error rather than writing
- Back-correct the `2026-07-14` entry in `wiki/log.md` to `2026-05-15` (or the actual correct date if determinable from context)
- Add a targeted test: `appendLog()` throws when passed a date 2 days in the future

## Acceptance criteria
- [ ] `appendLog()` throws a descriptive error when `entry.date` is > today + 1 day
- [ ] `wiki/log.md` contains no entries with a future date after the back-correction
- [ ] Targeted test for the chronology guard passes
