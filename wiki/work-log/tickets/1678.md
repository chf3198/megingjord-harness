## Body

## Context
Part of Epic #1626. Independent of other children (can run in parallel).

`scripts/wiki/wiki-io.js` `parseFrontmatter()` uses `line.split(':')[0]` and `line.split(':').slice(1).join(':')` to parse YAML-ish frontmatter. This fails silently for any field value containing a colon (e.g. `description: "Routing: HAMR → OpenClaw"`) — the value is truncated at the first colon. With 10+ schema fields, partial metadata loss is routine but invisible.

## Objective
Replace the hand-rolled regex/split parser with the `gray-matter` library, which handles the full YAML spec correctly. This eliminates all colon-in-value breakage and aligns with the Node.js ecosystem standard for frontmatter parsing.

## Implementation notes
- Install `gray-matter` as a production dependency (`npm install gray-matter`)
- Replace `parseFrontmatter()` in `scripts/wiki/wiki-io.js` with `matter(content).data`
- Replace `serializeFrontmatter()` to use `matter.stringify()` — verify output format matches existing page headers
- Run lint + hygiene after migration to confirm no regressions
