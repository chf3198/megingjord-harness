## Body

## Problem
The memory-watchdog (`hooks/scripts/memory-watchdog.sh` or equivalent) guards against `/memories/` write injection. Its current blocking status is unverified — if it is running in advisory/warn-only mode, a malicious or buggy agent can overwrite governance instructions in session memory with no enforcement catch.

This is Gap G-07 from the Phase-0 assessment: ASI04 (Memory Poisoning) + EU Art.12 (Record-keeping integrity).

## Acceptance Criteria
- [ ] Identify the exact memory-watchdog component and its current enforcement mode (blocking vs advisory)
- [ ] If advisory: promote to hard-blocking (exit 1 / deny decision) with a targeted test
- [ ] Add `tests/` coverage: write-to-/memories/-with-injected-content triggers watchdog deny
- [ ] Document blocking status in `research/governance-guardrail-assessment-v2-2026-06-10.md` under G-07
