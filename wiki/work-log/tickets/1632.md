## Parent

## Goal

Real-time cross-team event delivery via GitHub webhooks (sub-second issue/PR/comment events) + repository_dispatch (cross-workflow programmatic triggers) + workflow_dispatch (on-demand). Coordinates with the Projects v2 board (#1630) so a team's claim-event reaches every other team within seconds.

## Portability annotation (per #1628)

- **Webhook receivers require a public endpoint OR a GitHub-hosted Actions runner that accepts the dispatch event.**
- **Air-gapped / local-only operators do NOT have webhook receivers** — they cannot accept push notifications.
- **Mandatory fallback**: polling-based coordination via GitHub API on a configurable interval (default 60s). Workflow_dispatch + repository_dispatch (push-direction) remain available for all operators because they originate from GitHub-hosted infrastructure.
- **Classification**: opt-in *push-receive*; integral *push-send + polling-receive*.
