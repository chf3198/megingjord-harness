# #2154 — Add Tech-Writer sub-phase to the collaborator handoff

## Body

## Scope (Phase-1 child of Epic #2148)

Add a "Tech-Writer" sub-phase to the-collaborator-handoff contract. Before the Collaborator role can post the-collaborator-handoff artifact, the role-collaborator-label holder must complete an explicit doc-coverage step that confirms every applicable doc surface has been updated (or N/A with reason).

Refs Epic #2148
Refs #2149 (Phase-0 source)

## Why agent-speed-appropriate

Round-5 of Phase-0 research identified this as load-bearing process change. Without an explicit step, doc updates depend on per-ticket discretion, which agents skip under time pressure. The step is cheap (a checklist in the-collaborator-handoff body) but eliminates a defect-class (silent doc-drift across merges) that no other role catches.

## Deliverable

Edit `instructions/role-baton-routing.instructions.md` to add a new sub-section under the-collaborator-handoff schema:

```
### Tech-Writer sub-phase (mandatory before posting the-collaborator-handoff)

For each surface in the doc-coverage matrix that applies to this ticket's
diff, the role-collaborator-label holder MUST declare one of:
- UPDATED: <file-path> (link or path)
- N/A: <reason>

Surfaces:
- README.md / vscode-extension/README.md
- CHANGELOG fragment .changes/unreleased/<N>.md
- design docs / dev guide / user guide (per area-label)
- wiki/wisdom/project/, wiki/wisdom/global/, wiki/code/, wiki/work-log/
```
