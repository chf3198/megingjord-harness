# #2142 — Add IT-ops exception path to pretool_guard.py commit gate

## Body

## Problem

`pretool_guard.py` blocks every `git commit` that lacks a `#N` ticket reference — unconditionally. There is no exception path for IT-ops maintenance commits (model pulls, fleet config updates, local-only environment changes) that touch tracked files but don't warrant Agile baton workflow.

Surfaced during session 2026-05-25: adding `qwen3:32b` to `inventory/devices.json` and `model-routing-policy.json` required creating a throwaway tracking issue (#2139) just to satisfy the commit hook.

## Acceptance Criteria

- [ ] `pretool_guard.py` recognizes an IT-ops bypass marker (e.g. `[it-ops]` in commit subject OR `MEGINGJORD_IT_OPS=1` env var) and allows the commit without a `#N` reference
- [ ] Bypass is logged/observable (emits an advisory, not silent)
- [ ] Existing `#N`-required behavior unchanged for all other commits
- [ ] Tests updated to cover the bypass path

## Notes

- Pattern from existing precedent: `MEGINGJORD_REBASE_DISCIPLINE_DISABLED=1` and `MEGINGJORD_HAMR_DISABLED=1`
- Env var approach preferred (consistent with existing bypass convention)
- Consider also allowing `chore(it-ops):` commit type prefix as an alternative marker
