# #1953 — Document two-tier install model and multi-runtime parity

## Body

## Problem

When a new user, contributor, or developer encounters the Megingjord Harness project, the portability architecture is not obvious from the README, ARCHITECTURE.md, or any how-to guide. Specifically:

- The two-tier model (global `~/.copilot/` / `~/.claude/` / `~/.codex/` layer + per-workspace layer) is not explained anywhere.
- The multi-project reuse model (one deploy seeds all projects) is not documented.
- The multi-runtime parity (Copilot, Claude Code, Codex equally supported) is mentioned in passing but not explained as a design goal.
- There is no installation how-to guide.

## Expected outcome

An external observer reading the README, ARCHITECTURE.md, or the docs/howto/ index immediately understands:
1. Install once → global layer seeds all AI extensions on this machine
2. Workspace files are per-project and override global governance
3. All three runtimes (Copilot, Claude Code, Codex) are first-class equal citizens
4. Adding a second project is just `npm run deploy:apply` (idempotent)
