## Body

## Why this self-anneal is needed

Copilot Team stress test on 2026-05-17 (run on branch `feat/1794-g3-cost-routing-hardening`) reported 8 failures across two test sections:

- **C1–C5**: `classify_path` returning wrong classification for UI paths (`dashboard/index.html` → "other"; `dashboard/js/foo.js` → "code"; etc.)
- **E1, E5, E6, E7, E8**: 5 hook scripts differ between repo source and deployed runtime

**Root cause**: the Copilot Team's branch is **5 commits behind main**. Their base predates my #1822 PR merge (UI-scoping fix per #1821). The deployed runtime has my fix (deployed via `npm run deploy:both:apply` post-merge); their branch's repo source does not. All 8 failures collapse to a single coordination signal — the branch needs to rebase onto main.

But the stress test as currently constructed cannot distinguish:
- **(a)** Branch repo source stale (recommend rebase) — what actually happened here
- **(b)** Runtime not yet deployed (recommend `deploy:both:apply`)
- **(c)** Real content drift between repo and runtime (file a bug ticket)

All three look identical: "deployed copy differs from repo source". This is a stress-test signal-to-noise problem.

## Proposed remediation

Add `scripts/global/hook-parity-check.js` that performs a 3-way comparison and produces a discriminated diagnosis:

```
hooks/scripts/*.py  vs  origin/main:hooks/scripts/*.py  vs  ~/.{copilot,codex}/hooks/scripts/*.py
        ↑                          ↑                                  ↑
   current branch          remote main                       deployed runtime
```

| Diagnosis | Repo (branch) | Repo (origin/main) | Deployed | Recommendation |
|---|---|---|---|---|
| `ok` | A | A | A | None |
| `branch-stale` | A | B | B | `git rebase origin/main` |
| `runtime-stale` | A | A | B | `npm run deploy:both:apply` |
| `branch-and-runtime-diverged` | A | B | C | Manual reconciliation; flag in incidents.jsonl |
| `runtime-drift` (real defect) | A | A | A' | File self-anneal ticket |

CLI:

```bash
npm run governance:hook-parity                # default human output
npm run governance:hook-parity -- --json      # machine-readable
```

Exit codes:
- `0` → all ok (or only branch-stale, which is operational, not failure)
- `1` → runtime-stale or runtime-drift (actionable)
- `2` → branch-and-runtime-diverged (manual reconciliation needed)

```

  npm run governance:hook-parity -- --json

Only `runtime-drift` or `branch-and-runtime-diverged` warrants filing a real defect ticket.
```
