# #1875 — Self-anneal Epic: codify stress-testing as required strategy in test-methodology matrix

## Body

## Trigger

Tier-2 self-anneal per `instructions/workflow-resilience.instructions.md`.

Operator pull during Epic #1871 close-out: "Create a ticket that ensures that stress tests are created where applicable in addition to unit testing and that passing the stress test is an integral part of development."

Epic #1871 shipped stress tests for the cross-team conflict-prevention chain and surfaced a real TOCTOU race that unit tests had passed. The pattern needs to propagate to future work as enforceable governance, not as one-off implementation. The current `instructions/test-methodology-matrix.instructions.md` does not include a `stress-test` strategy in the enum; the closeout-schema validator does not check for stress evidence on safety-critical surfaces.

- Severity: medium-to-high (governance gap surfaces real concurrency bugs)
- Recurrence-evidence: Epic #1871 caught race bug that unit tests missed; the same gap class will recur for any future safety-critical module shipped with unit-tests-only coverage.
- Pattern_id: `stress-as-required-strategy-not-just-implementation`

## Objective

Codify stress-testing as a **first-class test_strategy** in the harness matrix, with enforceable evidence requirements and a CI gate that fails when stress is required but missing.

## Scope

### Phase 1 — Matrix extension

Add `stress-test` to the `test_strategy` enum in `instructions/test-methodology-matrix.instructions.md`. Define applicability criteria + required evidence shape.

### Phase 2 — Surface mapping

Extend the Surface → Strategy table:

| Surface | Strategy | Lane default |
|---|---|---|
| Concurrent / file-locking primitives (`scripts/global/worktree-*`, `cross-team-lease*`) | `stress-test` + `tdd-pyramid` | code-change |
| Side-effect-bearing gates (hooks, validators that touch state) | `stress-test` + matched primary | code-change |
| Adversarial-input parsers (detectors, classifiers, schema validators) | `stress-test` (adversarial corpus) + matched primary | code-change |
| Perf-sensitive governance gates (`*-gate.js`, `*-check.js`) | `stress-test` (perf budget) + matched primary | code-change |

Pure-read documentation generation, one-off CLI scripts, trivial config: **NO** stress requirement.

### Phase 3 — Closeout schema enforcement

Extend `scripts/global/megalint/consultant-closeout.js` rubric to verify stress evidence when `test_strategy` declares it. Evidence shapes:
- spec file under `tests/stress-*.spec.js` in PR diff, OR
- `npm run stress:*` invocation cited in `COLLABORATOR_HANDOFF` Pre-handoff verification

Two-track enforcement:
- **NEW surfaces shipped after this Epic lands**: stress-test required from day 0 (blocking). You're shipping new code; ship the stress with it.
- **EXISTING surfaces** (backfill list from Phase 5): advisory until replay-eval per-validator gates pass (>=85% precision against the historical PR corpus, per Epic #1771 pattern). NO calendar window; promotion is replay-eval-gated, not time-gated.

This avoids the calendar-threshold anti-pattern that Epic #1771 killed for soaks and Epic #1827 reframed for branch staleness. In parallel-orchestrator settings, calendar windows simultaneously over-restrict (during high-velocity bursts) and under-restrict (during quiet windows).

### Phase 4 — CI gate

New `test-evidence-stress` workflow runs as required check. Reads `test_strategy` from MANAGER_HANDOFF; if `stress-test` declared, scans PR diff + COLLABORATOR_HANDOFF for evidence; fails when missing.

### Phase 5 — Surface backfill audit

Audit `scripts/global/*.js` against the surface criteria; produce a follow-on ticket list for modules that meet stress-test criteria but lack stress specs today. Prioritize by blast radius.

### Phase 6 — Documentation

Update `instructions/test-methodology-matrix.instructions.md` + `docs/howto/test-strategy-selection.md` (new) with stress-test selection guidance + example patterns from Epic #1871.
