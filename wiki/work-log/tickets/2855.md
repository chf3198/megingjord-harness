## IDE / native-MCP coexistence + port-collision handling (P1-0 child of #2802)

Parent: #2802 (P1-0 fleet context-access layer). Phase-0 source: #2792 (D12, Copilot constraint).
Sibling: #2854 (loop driver), #2847 (tool adapters).

### Problem
#2802 AC3 (D12, Copilot constraint): a fleet MCP loop must NOT collide with a host IDE's native MCP
servers (e.g. `github.copilot.mcp.servers`) — no local **port collisions**, and it must read committed/
saved disk state rather than fight the IDE for a live socket. AC7 requires stress for port-collision +
lock coexistence. Today no coexistence guard exists.

### Objective
A coexistence layer: detect an in-use MCP port / running host-IDE MCP server and degrade gracefully
(ephemeral/alternate port, or disk-state read-only mode), plus a lock so two fleet loops don't double-bind.
Reuse the existing worktree-lease / lock primitives where applicable (no parallel lock system).

### Acceptance criteria
- AC1: detect a bound MCP port / host-IDE MCP server; pick a free ephemeral port or fall back to
  disk-state read-only mode (graceful, G6) — never hard-fail on collision.
- AC2: a coexistence lock prevents two concurrent fleet loops from binding the same resource.
- AC3: tier-graceful — absent IDE / absent MCP is a no-op pass-through (G5).
- AC4: side-effects (port probe, lock) injectable → tests deterministic.
- AC5: <=100 lines/file; readability <=475.
- AC6: tdd-pyramid spec + **stress spec for port-collision + lock coexistence** (satisfies #2802 AC7
  for the MCP-server surface): >=1 concurrency/chaos path + >=1 p99 budget.

### Lane / strategy
lane:code-change. test_strategy: tdd-pyramid+stress-test. overlap_decision: reuse worktree-lease/lock
primitives; no parallel lock system.
