# #1335 — Fix test_strategy validator to honor Manager amendments + handoff lint

> **Source**: github:issue/1335 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:governance, lane:code-change
> Mirror of `gh issue view 1335` (derived; edit the GitHub item, not this page).

## Body

## Problem

Surfaced 4 times during the 2026-05-10 session (#1310, #1311, #1305, #1307). The `test-evidence-validator.js` reads `test_strategy:` via non-global `match()` which returns the **first** occurrence in the issue comment trail. This means:

1. If the Manager filed the ticket with one strategy and later realizes it's wrong, posting a Manager-amendment comment **does not** override — the original wins.
2. The only way to amend `test_strategy` is to edit the ORIGINAL `MANAGER_HANDOFF` comment via `gh api PATCH /repos/.../issues/comments/<id>` — undocumented workaround.
3. If a ticket was filed without a MANAGER_HANDOFF *comment* (only body has the strategy), the validator finds NO match and defaults to `none` — failing on non-permitted lanes.

Live evidence:
- #1310 needed Manager-edit of the original comment to change `tdd-pyramid → drift-lint`
- #1305, #1306, #1307 needed *retroactive* MANAGER_HANDOFF comments because original filings put strategy in body only

## Goal

Make `test_strategy` Manager-amendment a first-class supported flow + reduce the chance of missing-handoff-comment failures.

## Acceptance Criteria

- [ ] **AC1 — Validator honors last occurrence OR explicit marker.** Change `test-evidence-validator.js` to either: (a) match the LAST `test_strategy:` line in trail (most-recent wins), or (b) honor an explicit `MANAGER_HANDOFF_AMENDED:` marker that supersedes prior values. Decide based on operator preference.
- [ ] **AC2 — Lint check for MANAGER_HANDOFF presence.** Add a lint rule (in `label-rules.js` or a new `manager-handoff-lint.js`) that warns when a ticket transitions out of `status:triage` without a `MANAGER_HANDOFF` comment containing `test_strategy:` and `lane:`.
- [ ] **AC3 — Document the amendment pattern.** Update `instructions/role-baton-routing.instructions.md` with a "MANAGER_HANDOFF amendment" subsection explaining the supported flow (whichever AC1 lands on).
- [ ] **AC4 — Golden-file tests.** Cover: original-only, amended-via-comment, amended-via-edit, body-only-no-comment. Each should produce the expected strategy resolution.
- [ ] **AC5 — Regression test on #1310-style amendment.** Re-running test-evidence on #1310's actual comment trail (Manager amendment) should produce `drift-lint`, not `tdd-pyramid`.
