## Body

## Goal

Integrate Tavily AI Search into the Megingjord harness via its MCP server so that all three orchestrators (Claude Code, Copilot, Codex) and fleet models have first-class access to grounded, fresh web research. Optimize the integration against the goal-lens hierarchy with explicit decisions about when Tavily is preferred over the built-in WebSearch tool.

## Why Tavily is a significant gain (goal-lens assessment)

| Goal | Tavily impact | Evidence |
|---|---|---|
| G2 Quality | POSITIVE | Tavily is purpose-built for AI agents: structured ready-to-use data, 180ms p50 latency, 99.99% uptime, 100M+ monthly requests. Reduces hallucinations on package versions, library APIs, fresh CVE data. |
| G3 Zero Cost | MIXED | Free tier 1,000 credits/month covers low-volume use; pay-as-you-go $0.008/credit beyond. Requires HAMR cost-tier routing to stay in free band. Must be cheaper at scale than premium-tier WebSearch invocations. |
| G4 Privacy | STRONG POSITIVE | Zero data retention policy (preserved through Feb 2026 Nebius acquisition). Better than alternatives. |
| G6 Resilience | POSITIVE | Provides a fallback alongside built-in WebSearch — search-tool redundancy. |
| G7 Throughput | POSITIVE | 180ms p50 latency. Faster than most alternatives (Brave 669ms, Exa 1335ms, Parallel Pro 13.6s). |
| G9 Interoperability | STRONG POSITIVE | MCP server makes Tavily available to ALL 3 orchestrators with a single integration. Satisfies HAMR cross-team contract per `instructions/hamr-routing.instructions.md`. |

## Comparison snapshot (2026 benchmarks)

| API | Agent Score | p50 Latency | Strength | Weakness |
|---|---|---|---|---|
| Brave Search | 14.89 | 669ms | privacy-focused, own index, low latency | broader index but less agent-ergonomic |
| Tavily | ~13.9 | 180ms | agent-specific ergonomics, MCP-first, zero-retention | paid above 1K credits/mo |
| Exa | competitive | 1335ms | semantic retrieval, deep-content (Mean Relevance 4.30) | poor on freshness (24% FreshQA) |
| Built-in WebSearch | — | — | always available | not MCP-portable across orchestrators |

Per the benchmarks, no single API dominates. The harness should route by query type — Tavily as primary for time-sensitive + agent-specific queries, Exa for semantic deep-content (potential follow-on Epic), Brave for privacy-critical, built-in WebSearch as fallback.
