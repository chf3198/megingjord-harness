## Body

## Parent / trigger

The Codex Consultant closeout on #1568 reported `gov-006` branch naming failure because the closeout ran from `main`. That is a false positive for issue-only Epic closeout work where no implementation branch or PR is expected.

## Goal

Teach Consultant/governance checks to distinguish implementation work from issue-only Epic/research closeout, so branch-name failures remain meaningful instead of noisy.

## Acceptance Criteria

- [ ] AC1 — Identify the `gov-006` implementation and its inputs for branch context, ticket lane, Epic status, and PR presence.
- [ ] AC2 — Add an explicit issue-only mode or detection rule: if no file changes/PR branch are part of the closeout and the ticket is an Epic/research closeout, `main` is not a branch-naming failure.
- [ ] AC3 — Preserve blocking behavior for code-change tickets that should be on `feat/<ticket>-...` or `fix/<ticket>-...` branches.
- [ ] AC4 — Add regression tests for issue-only Epic closeout, issue-only research closeout, and normal code-change branch enforcement.
- [ ] AC5 — Update Consultant check documentation/output to explain when branch-name checks are skipped.

## Harness goals

Primary: G1 Governance, G2 Quality, G8 Observability, G9 Interoperability.

## Review evidence

#1568 was correctly closed as an issue-only Epic closeout, but the generic Consultant check still counted branch naming as a failure. No remediation ticket existed at review time.
