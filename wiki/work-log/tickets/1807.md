# #1807 — Enforce branch-ticket commit match in pretool guard

## Body

## Problem
Parallel agent workflows can accidentally carry commits from ticket A into branch/PR for ticket B when branch switching occurs around interrupted sessions.

## Objective
Block commits whose ticket reference does not match the branch ticket number (`feat/<N>-...`, `fix/<N>-...`, `hotfix/<N>-...`).

## Acceptance Criteria
- [ ] Pretool guard extracts ticket from current branch name
- [ ] `git commit` is denied when commit message ticket `#M` does not equal branch ticket `#N`
- [ ] Existing commit gate (requires some `#N`) remains intact
- [ ] Hook tests cover match/mismatch/non-ticket branches

## Evidence
- Linked incident: #1793 / PR #1806 cross-ticket commit bleed
- Governance rationale: preserve one-branch-one-ticket traceability in parallel orchestrator sessions
