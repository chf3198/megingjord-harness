# #2157 — Release-notes presence validator for .changes/unreleased fragments

## Body

## Scope (Phase-1 child of Epic #2148)

Build a pre-merge validator that fails any PR carrying a `lane:code-change` label which does not include either (a) a new file at `.changes/unreleased/<N>.md` matching the PR's `Refs #N`, or (b) the literal marker `[skip-changelog]` in the PR description.

Refs Epic #2148
Refs #2149 (Phase-0 source)

## Why agent-speed-appropriate

Round-5 of Phase-0 research kept this as a sub-second presence check. The CHANGELOG fragment pattern (#1132 / #2120) already exists; this child only adds the merge-time enforcement. Cheap, deterministic, fleet-model-compatible.

## Deliverable

- New script: `scripts/global/megalint/changelog-fragment-presence.js`
- New workflow job: `.github/workflows/closeout-schema.yml` adds a `changelog-fragment` job (or extends existing one) calling the script
- Script logic:
  1. Read PR diff via gh api
  2. Extract Refs #N from PR body
  3. If lane:code-change label present AND no `.changes/unreleased/<N>.md` in diff AND no `[skip-changelog]` marker → fail with required-action message

## Acceptance criteria

- AC1: `scripts/global/megalint/changelog-fragment-presence.js` exists; exports a pure function taking PR object and returning {ok, reason}
- AC2: Unit tests in `tests/changelog-fragment-presence.spec.js` cover: fragment-present-pass, fragment-missing-fail, skip-marker-pass, lane:docs-only-skip
- AC3: Workflow integrated (advisory in first ship; promote to required after 2 weeks of green data)
- AC4: `[skip-changelog]` marker documented in `docs/howto/changelog-fragments.md`
- AC5: Lint clean
