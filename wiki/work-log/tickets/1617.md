## Problem

The `Epic #1486 Path D` advisory-first rollout is a valid governance strategy, but it creates a recurring trap at Epic close time:

- AC text written at scoping: e.g. `AC-3 Enforce on PR open: model-diversity gate fails the consultant-gate on mismatch`
- Shipped artifact: advisory-only workflow (does not block; soak window before promotion)
- Epic close action: parent Epic closed `status:done` with the AC ticked as if enforcement landed

This is **observed once** on Epic #1568 (caught by Codex Team's #1612) and **latent across 6 advisory ships** this session: #1554, #1555, #1571, #1572, #1573, #1334 AC1. The pattern affects every Epic that uses Path D rollout AND has AC text that reads `enforce X` rather than `ship advisory; promotion deferred`.

## Goal

Two-layer fix: agent-side discipline (memory written, instruction update) + structural validator (automated check that AC wording matches shipped artifact metadata at Epic close).

## Acceptance Criteria

- [ ] **AC1**: Update `instructions/epic-governance.instructions.md` Epic Close Conditions section with explicit guidance: for every Epic AC containing enforce / required / block / fail-the-gate wording, the Manager at-close must verify the shipped artifact actually enforces, OR rescope the AC text to `ship advisory; promotion deferred` before ticking. New Epic Close Condition: `AC wording matches shipped artifact disposition (advisory vs required)`.

- [ ] **AC2**: Add an advisory validator (helper + standalone workflow following the Epic #1486 Path D pattern) `scripts/global/epic-ac-disposition-check.js`. Inputs: Epic body (AC text), Epic comments (CONSULTANT_EPIC_CLOSEOUT linked-child PRs), and the workflow files those PRs touched. Detect: AC text contains enforcement vocabulary AND the linked PR's shipped workflow YAML is named `*-advisory.yml` AND does not call `core.setFailed`. Output: advisory comment on the Epic flagging the mismatch.

- [ ] **AC3**: Manager pre-close checklist addition: extend the Collaborator self-check helper (#1571) or add a separate Manager-self-check helper that surfaces this disposition mismatch BEFORE the Manager closes the Epic.

- [ ] **AC4**: Tests covering: (a) enforce-wording + advisory-only artifact → advisory triggered, (b) enforce-wording + required-blocking artifact → no trigger, (c) advisory-wording + advisory artifact → no trigger, (d) malformed AC text → safe skip, (e) Epic with NO advisory-shipped children → no trigger.

- [ ] **AC5**: Apply the new instruction guidance retroactively to the 5 advisory-shipped children still affected (Codex's #1612 already handles #1568's case; this AC covers any subsequent Epic touching #1554, #1555, #1571, #1572, #1573, #1334 AC1).

- [ ] **AC6**: Cross-team note posted on Codex's #1612 and any sibling self-anneal Epic indicating this general validator complements their specific-case promotion-or-rescope work.
