# #1790 — Enforce fleet/free routing as hard gate for all orchestrators

> **Source**: github:issue/1790 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P1, area:scripts, resolution:completed, lane:code-change
> Mirror of `gh issue view 1790` (derived; edit the GitHub item, not this page).

## Body

## Problem

Silent fallback to paid providers (Haiku, Premium) violates G3 (Zero Cost) governance. Orchestrators route through global-task-router but have no mandatory cost-minimization gate.

## Goal

Implement a hard gate: all orchestrator routing decisions MUST exhaust free/fleet tiers before falling back to paid. Violation = CI failure + self-anneal trigger.

## Acceptance Criteria

- [ ] AC1: Global task router enforces free/fleet-first cascade (free → fleet → haiku → premium); no silent bypass
- [ ] AC2: All orchestrators (baton-gates, closeout-lint, epic-validator, cross-team-lease, etc.) updated to use router
- [ ] AC3: HAMR wrapper enforces G3 cost check; CI gate fails if production_hamr_utilization < 85% at EOD
- [ ] AC4: Dashboard panel shows live fleet utilization + token spend breakdown
- [ ] AC5: Self-anneal triggers if any workflow falls back to Haiku >3% of calls in rolling 7d window
- [ ] AC6: Docs updated: `instructions/global-task-router.instructions.md`, `docs/cost-minimization-gates.md`

## Harness Goals

Primary: G3 Zero Cost (with G1 Governance, G2 Quality supporting).
