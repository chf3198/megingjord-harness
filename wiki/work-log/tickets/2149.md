## Acceptance criteria

- **AC-R1**: Cutting-edge 2026 industry research, minimum 12 primary sources dated 2024-01-01+, ≥5 dated 2025-2026, spanning:
  - Enforced doc-as-code patterns (changesets / release-please / semantic-release / knope / git-cliff)
  - Conventional Commits + auto-changelog ecosystems
  - Doc-coverage gates (docs-as-tests, Vale, markdownlint baseline-budget patterns)
  - Wiki-as-code + auto-update pipelines (docusaurus + GH Actions, Astro Starlight, mdbook)
  - LLM-agentic governance research: which decisions can be reliably outsourced to non-frontier models
  - AI-agent doc-as-code patterns (post-GPT-5 industry standards, late-2025 / early-2026 references)

- **AC-R2**: Per-surface architectural mapping. For each of the 4 documentation surfaces (GitHub Issues, Release Notes, Project Docs, Wikis), specify:
  - Which gate location (pre-commit / pre-push / PR-open / pre-merge / post-merge / release-tag)
  - What evidence artifact the gate inspects
  - Hard-block vs advisory + fleet-model rationale
  - Existing harness surface(s) that already cover part of it
  - Net-new validators / workflows required

- **AC-R3**: Decision matrix scored against G1-G10 goal-lens (G1 Governance > G2 Quality > G3 Zero Cost > G4 Privacy > G5 Portability > G6 Resilience > G7 Throughput > G8 Observability > G9 Interoperability > G10 Maintainability). For each Phase-1 candidate child, recommend hard-block vs advisory + justify.

- **AC-R4**: Threat model: minimum 8 failure modes the gate set must handle, including:
  - PR with no `Closes #N` / `Refs #N`
  - PR closing issue without baton trail
  - PR shipping code without `.changes/unreleased/<N>.md` fragment
  - PR shipping behavior change without README / design-doc / dev-guide / user-guide update
  - PR shipping infra pattern without wiki entry
  - Operator session-learned memory not promoted to ticket OR wiki
  - Release tag without aggregated changelog
  - Fleet model bypassing instruction-only guards

- **AC-R5**: Fleet-model-compatibility audit. For each proposed gate, classify as:
  - **Schema-checkable** (works for any model — regex / JSON-schema / file-presence)
  - **Reasoning-required** (frontier-only — escalate to human if fleet hits it)

- **AC-R6**: Phase-1 child slate: scoped list of Phase-1 development children with branch/ticket suggestions, lane labels, test_strategy per matrix, hard-block-vs-advisory recommendation.

- **AC-R7**: EPIC_RESCOPE comment on #2148 summarizing chosen architecture, Phase-1 child slate, and Consultant rubric ≥7 score breakdown.
