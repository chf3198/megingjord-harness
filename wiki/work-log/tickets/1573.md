# #1573 — Second-opinion Consultant rating from cross-provider rater (gemma3 or similar)

> **Source**: github:issue/1573 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:governance, resolution:completed, lane:code-change
> Mirror of `gh issue view 1573` (derived; edit the GitHub item, not this page).

## Body

## Parent

Epic #1568 (AC-4). Research: #1569.

## Goal

For every `CONSULTANT_CLOSEOUT`, run a parallel G1–G9 rating from a deliberately different-family rater (e.g. fleet `gemma3:1b`, open-weights local, or a non-Anthropic cloud rater). Append the delta to the closeout. If any goal's |delta| > 1.0, automatically escalate as Tier-3 per `workflow-self-anneal`. Matches the Star Chamber + Diverse-LLMs research findings that cross-provider review yields the strongest independence.

## Acceptance Criteria

- [ ] AC1 — `scripts/global/consultant-second-opinion.js`: pure script that takes a `CONSULTANT_CLOSEOUT` body, extracts the per-G score map, runs a second rater (configurable: local fleet via Ollama, or HAMR-routed cloud call), and returns `{ second_scores: {...}, deltas: {...}, max_abs_delta }`.
- [ ] AC2 — Second-rater **must be a different provider family** than the first Consultant rater. Validated by parsing the closeout's `Team&Model:` field and comparing provider prefix (e.g. `claude-code:` vs `fleet:` vs `copilot:`).
- [ ] AC3 — Output appended to `CONSULTANT_CLOSEOUT` as a structured block:
  ```
  SECOND_OPINION
  rater_team_model: <e.g., fleet:gemma3:1b@ollama>
  scores: G1=8,G2=9,...
  deltas: G1=-2,G2=0,...
  max_abs_delta: 2
  verdict: ESCALATE_TIER_3 / NO_ESCALATION
  ```
- [ ] AC4 — If `max_abs_delta > 1.0` on any G-goal, the script automatically files (or stages for filing) a Tier-3 anneal ticket per `instructions/workflow-resilience.instructions.md`. Same protocol used to file #1373 from Epic #1339's G5=6/10 rating.
- [ ] AC5 — Workflow integration: `.github/workflows/baton-gates.yml` (or a new `consultant-second-opinion.yml` workflow) runs the script on every PR carrying a Consultant closeout. The second-opinion block becomes part of the canonical evidence trail.
- [ ] AC6 — Cost guard: if the configured rater is cloud, the call is wrapped in `cacheHeaders(provider)` per `instructions/hamr-routing.instructions.md`. If rater is fleet (free), no guard needed.
- [ ] AC7 — Tests: golden-file for the appended `SECOND_OPINION` block format; unit test for the delta-threshold escalation logic; integration test against #1339's archival closeout (should reproduce the G5=6/10 → Tier-3 fire that filed #1373).

## test_strategy

`golden-file` + `tdd-pyramid` (helper is pure JS).

## Architecture notes

- **Prefer fleet first**: `gemma3:1b@ollama` is local, free, and provider-distinct from `claude-code:opus@anthropic`. Recommended default.
- **Avoid same family loops**: don't second-rate with another Anthropic model — that's intra-provider, weaker signal per Star Chamber's findings.
- **Anonymization**: pass only the diff + acceptance criteria + Manager handoff to the second rater. Do NOT pass the first Consultant's rationale (Chatham House rule — prevents anchoring).

## Dependencies

- HAMR routing for cloud raters (if non-fleet chosen).
- `instructions/workflow-resilience.instructions.md` Tier-3 escalation protocol (existing).
- Epic #1568 AC-6 deterministic rubric (#TBD) — when rubric is fully deterministic, the second-opinion check collapses to "does the second rater agree on which evidence boxes are checked?", which is much cheaper than re-running judgment.
