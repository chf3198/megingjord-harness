## Body

## Goal

Design and ship governance guardrails that prevent cross-team git collisions in the multi-runtime harness (Claude Code, Copilot, Codex). Today the contract is "do not share this checkout" per `CLAUDE.md` Concurrent-session safety section, but enforcement is documentation-only. Observed practice in 2026-05-21 session: the main checkout `/home/curtisfranks/devenv-ops/` was on another team's `feat/1610-worktree-branch-cleanup` with in-progress uncommitted work while three Claude Code tickets (#1943 #1944 #2048) were shipping — Claude Code had to manually stash + branch-switch + un-stash to commit without the pretool_guard rejecting the commit.

## Gaps to close

1. **Main checkout drift**: main `/home/curtisfranks/devenv-ops/` regularly becomes a working branch checkout. Need to make it canonical-only (read-only by convention + automation that rejects non-main branch switches).
2. **No OS-level mutual exclusion**: another team's process can `cd` into a checkout claimed by Team-N's lease. Need a filesystem lockfile or equivalent that participates with cross-team-lease-registry.
3. **State residue across sessions**: stale state files (`~/.copilot/hooks/state/repo-*.json`) carry `admin_ops` and `flags` from earlier sessions and confuse the stop-hook. Need a session-boundary state reset or per-session state files.
4. **Stash hygiene**: 5+ dangling stashes accumulated. Need an audit + cleanup pass (and prevention via mandatory commit-before-context-switch).
5. **GitHub protected-branch coverage**: confirm `main` is fully protected (no direct push, required reviews, required checks). Confirm other long-lived branches (e.g. `develop` if used) similarly. Confirm CODEOWNERS coverage for governance-critical paths.
6. **Worktree allocation contract**: each runtime team should have a defined worktree-naming convention and an allocation/release protocol so workspaces are unambiguously per-team.
7. **Conflict-detection pre-commit**: pre-commit hook that detects in-progress changes in OTHER checkouts on the same repo and warns/blocks.

## Phase-1 implementation (gated by Phase-0 closeout)

Tentative children (subject to research):
- Per-team branch-namespace enforcement (validator: `<team>/<type>/<N>-<slug>` allowed; flat `<type>/<N>-<slug>` deprecated)
- Filesystem flock on main checkout: any team starting a session must acquire `${CHECKOUT}/.lock-claude|copilot|codex` before any tool call mutates state
- Worktree-allocation manifest: declared `inventory/worktree-allocations.json` mapping team to root path
- Session-boundary state reset: `~/.copilot/hooks/state/*.json` rotated or per-session-scoped on session-start
- Stash-hygiene pre-commit: warn if >2 stashes exist and any are >7 days old
- GitHub ruleset additions: confirm + harden main branch protection; add ruleset for per-team branch namespace prefix
- Cross-checkout-destructive validator promotion: from advisory to required
