# #1673 — research(knowledge): define unified wiki health contract

> **Source**: github:issue/1673 | **state: CLOSED** | **Labels**: type:research, status:done, priority:P1, area:knowledge, lane:research
> Mirror of `gh issue view 1673` (derived; edit the GitHub item, not this page).

## Body

## Context
Part of Epic #1626 — Karpathy Wiki Critical Analysis + Hardening.

The wiki's three health tools (`scripts/wiki/lint.js`, `scripts/wiki/hygiene.js`, `scripts/dashboard-wiki.js`) use independent, divergent health models. This produces contradictory signal: orphan count is 31 in dashboard vs 17 in lint; frontmatter violations are 4 in dashboard vs 18 in lint. Operators cannot trust any single number.

## Objective
Research and design a single canonical health contract that all wiki tooling will share.

## Scope
- Audit all three tools' health metrics: what each computes, how, and which fields/counts diverge
- Define a shared health schema: field names, count semantics, and which source of truth owns each metric
- Specify the module interface for `scripts/wiki/health-contract.js`
- Produce a design doc / ADR entry in `docs/DECISIONS.md`
- Identify any edge cases (e.g. symlinked pages, encoding differences) that caused the divergence

## Out of scope
Implementation. Deliver research findings and interface spec only.

## Acceptance criteria
- [ ] Root cause of orphan count divergence (31 vs 17) is documented with evidence
- [ ] Root cause of frontmatter count divergence (4 vs 18) is documented with evidence
- [ ] A proposed `health-contract.js` module interface (exported function signatures + return schema) is written
- [ ] Findings posted as a comment on Epic #1626
- [ ] ADR entry added to `docs/DECISIONS.md`
