# #2555 — Create scripts/global/xteam-mcp-register.js — idempotent MCP registration for all 4 runtimes

> **Source**: github:issue/2555 | **state: OPEN** | **Labels**: type:task, status:backlog, priority:P2, area:scripts, lane:code-change, phase-gate:phase-1
> Mirror of `gh issue view 2555` (derived; edit the GitHub item, not this page).

## Body

## Problem
`npm run deploy:apply` does not register the xteam-mcp server in any runtime. Users must manually install per runtime.

## Objective
Create `scripts/global/xteam-mcp-register.js` that idempotently writes the `megingjord-xteam` MCP server config to all 4 runtime homes.

## Refs
Refs #2553 (Epic), Refs #2554 (Phase-0 research), Refs #2565 (prompt activation proof), Refs #2567 (VS Code profile discovery follow-on)
phase_0_sources: [#2554]

## Config Path Table

| Runtime | Config File | Key/Method | Scope |
|---------|-------------|------------|-------|
| Claude Code | `~/.claude.json` | root `mcpServers` (JSON merge) | user (cross-project) |
| VS Code Copilot | `~/.config/Code/User/mcp.json` (default profile only in this epic) | `servers` (JSON merge) | default profile |
| Antigravity | `~/.config/Antigravity/User/mcp.json` | `servers` (JSON merge) | default profile |
| Codex | `~/.codex/config.toml` | `[mcp_servers.{name}]` TOML blocks (TOML merge) | global |

**Note**: `~/.claude/` is a directory; `~/.claude.json` is the MCP config file. Do not write into `~/.claude/`.

## Acceptance Criteria
- [ ] Script accepts `--target <copilot|codex|claude|antigravity|all>` flag
- [ ] Script accepts `--root <path>` to resolve absolute bin path (defaults to cwd)
- [ ] `MCP_REGISTER_TEST_HOME` env var overrides `os.homedir()` for test isolation
- [ ] Without `--apply`: dry-run mode prints planned changes and performs no writes
- [ ] With `--apply`: idempotently writes/merges config to target runtime
- [ ] Claude Code: creates/merges `~/.claude.json` with root-level `mcpServers` key (user scope, cross-project); does not touch `~/.claude/` directory
- [ ] VS Code Copilot: creates/merges `~/.config/Code/User/mcp.json` with `servers` key; handles 0-byte file; documents default-profile-only scope for this epic
- [ ] Antigravity: creates/merges `~/.config/Antigravity/User/mcp.json` with `servers` key
- [ ] Codex: creates/merges `~/.codex/config.toml` with `[mcp_servers.megingjord-xteam]` TOML block
- [ ] All runtimes: idempotent — no-op if server already registered (dry-run still reports current state)
- [ ] Write failures are target-scoped: if one runtime write fails, existing config for that target remains unmodified and the error message names the failing target and file path
- [ ] JSON/TOML file updates are atomic or equivalently safe against partial-write corruption
- [ ] Script is ≤100 lines (lint enforced)
- [ ] Existing unrelated entries in all runtimes are preserved (no clobber)

## Design Notes
- This epic intentionally targets the default VS Code profile path only; profile-aware discovery is tracked separately in #2567.
- Prompt activation is not considered proven by config writes alone; runtime proof is tracked in #2565 and consumed by C5 #2559.
