# #1495 — Self-anneal: Signer alias enforcement (Tier-2 recurrence discovery)

> **Source**: github:issue/1495 | **state: CLOSED** | **Labels**: type:task, status:done, priority:P2, area:governance, lane:code-change, governance:close-without-merge
> Mirror of `gh issue view 1495` (derived; edit the GitHub item, not this page).

## Body

## Scope
During #1379–#1474 audit phase, discovered Tier-2 recurrence pattern: baton-artifact signers not enforcing registry-derived aliases (Soren, Orion, Milo for Copilot; Orla, Clio for Claude Code; Quill, Caden for Codex).

## Observation
- inventory/team-model-signatures.json defines aliasSeed + cryptoKeys per role
- Current baton comments use arbitrary human names or GitHub login fallback
- No write-time validation that signed alias matches registry entry
- Same human can appear under multiple inconsistent identities across tickets

## Expected behavior
- Every baton comment must include Team&Model signature
- Signature must use registry-derived alias (not arbitrary name)
- Consultant checks include gov-001-signer-alias-fidelity gate
- Non-compliant artifacts rejected at write-time (CI gate or pre-commit hook)

## Mismatch
Governance registry exists but not enforced; sidesteps Team&Model identity model authority.

## Root cause
Signer alias validation not implemented; tooling assumes human review catches drift.

## Risk
High — drift enables identity confusion in multi-agent scenarios; undermines Team&Model registry authority.

## Proposed changes
1. Create pure-function helper: enforceSignerAlias(teamName, role, input) returns canonical alias or rejects
2. Add unit test coverage: 3 teams × 4 roles = 12 scenarios (valid alias, invalid alias, fallback, missing registry)
3. Integrate into baton-comment helper (shared across teams)
4. Add gov-001-signer-alias-fidelity to consultant-checks.js

## Verification
- All 12 unit tests pass
- Consultant check gov-001 returns PASS on sample baton comments
- Pre-merge lint catches non-compliant aliases
