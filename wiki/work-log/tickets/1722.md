## Parent

Epic #1716. Phase 2 design. Consumes Phase 1.1 inventory output.

## Goal

Define the formal G5 fallback contract for operators who cannot satisfy the rotation rules with their available models.

## Acceptance Criteria

- [ ] AC1: `docs/howto/rotation-g5-fallback.md` defines three operator-side modes:
  - **strict-rotation** (default for multi-family-fleet operators) — all 3 rules enforced.
  - **advisory-only** (auto-detected when fewer than 4 families available) — rules emit advisory comments but do not block.
  - **single-model-fleet** (operator-declared at `npm run hamr:activate` time) — rotation entirely skipped; alias-only artifacts; operator accepts the bias risk.
- [ ] AC2: Defines `MEGINGJORD_MODEL_ROTATION_DISABLED=1` env var as the explicit opt-out (parity with HAMR opt-out).
- [ ] AC3: Auto-detection logic: HAMR queries available providers at activation time and records which mode the operator is eligible for.
- [ ] AC4: Self-classification artifact: when an operator declares `single-model-fleet`, the activation evidence persists in `~/.megingjord/activation.json` with a signed timestamp.
- [ ] AC5: Failover semantics: if a primary family becomes unavailable mid-session (provider 5xx), rotation falls back to next-eligible family OR degrades to advisory-only with audit log entry.
