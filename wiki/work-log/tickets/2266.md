# #2266 — Normalize issue-only closeout evidence for no-code remediation

## Body

Refs Epic #2258

Problem:
Current closeout evidence rules assume PR or merge surfaces and do not cleanly name issue-only remediation exceptions.

Objective:
Update closeout and evidence handling so issue-only remediation can close cleanly without abusing unrelated exception types.

Acceptance criteria:
- Define a clean evidence schema for issue-only remediation.
- Support explicit N/A treatment for PR, merge, CI, and deploy surfaces.
- Preserve consultant-led terminal close requirements.
- Avoid weakening evidence requirements for normal code-change tickets.
