# #2019 — Mitigate runaway Playwright workers + zombie cleanup at SessionStart

> **Source**: github:issue/2019 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P1, area:infra, resolution:released, lane:code-change
> Mirror of `gh issue view 2019` (derived; edit the GitHub item, not this page).

## Body

## Origin

Empirical observation 2026-05-20/21: long-running Claude Code sessions accumulate runaway Playwright worker processes (`playwright/lib/common/process.js`) at 98-200% CPU, surviving hours after their parent test runner exited. Combined with bounded container memory (6.3GiB) and `virtio_balloon: Out of puff` events, these zombies starve the Claude native-binary, causing repeated session terminations that surface to the operator as "Continue from where you left off" prompts.

Live count at investigation time: 9 zombie playwright processes consuming a combined 2-4GB RSS across two ~13h-old PIDs.

Upstream bugs (Playwright side, won't be fixed in our harness):
- microsoft/playwright#27048 (hangs after run)
- microsoft/playwright#34190 (zombie node children)
- microsoft/playwright#26980 (continue-running on early exit)
- microsoft/playwright#38276 (runner hangs after completion)

## Scope

Harness-wide mitigations:

1. New helper `scripts/global/safe-playwright.sh` wrapping `npx playwright test` with `--workers=1 --max-failures=5` and file-capture (`> /tmp/pw-out.txt 2>&1; tail -N`) instead of in-line pipe-tail.
2. New SessionStart hook `hooks/scripts/zombie_cleanup.py` that scans for orphaned playwright/node test workers (>5 min CPU >80%, parent dead) and SIGTERM (5s grace) → SIGKILL. Logs to `~/.megingjord/zombie-cleanup.jsonl`.
3. Documentation update at `docs/howto/playwright-safe-invocation.md` explaining the pattern + the upstream bugs.
4. Stress test `tests/stress-zombie-cleanup.spec.js` (unit + integration):
   - simulate 10 orphaned worker PIDs in a fake /proc tree; assert cleanup hits all
   - simulate live worker (parent alive); assert NOT killed
   - timing: SIGTERM ≤5s, SIGKILL escalation only on stall
   - p99 cleanup-scan runtime ≤500ms
