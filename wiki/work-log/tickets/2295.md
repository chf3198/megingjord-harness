## Origin

Cross-team operator observation (2026-05-27, Codex Team mid-flight): the local closeout gate REQUIRES `Closes #N` in PR body, while the canonical PR template + instructions say `Refs #N` (with explicit warning that auto-close should NOT be used because Consultant retains close authority).

Codex's verbatim observation:

> *"The local closeout gate now wants the PR body to include `Closes #2250`, even though the template warns against auto-close. Since Consultant review is already recorded and the gate is now explicitly using auto-close as merge evidence, I'm updating the PR body to match the current enforced rule."*

This is **documentation-vs-enforcement drift** — a class of governance defect where:
- The **template / instruction / wiki** says one thing
- The **gate / hook / validator** enforces something else
- Operators (cross-team) discover the contradiction at runtime and obey the gate (because it blocks) over the docs (which only warn)

That choice is reasonable; the underlying contradiction is the defect.

## Scope (research-first)

Multi-phase Epic to:
1. **Detect** all current governance-rule conflicts in the harness (docs ↔ gates, gate ↔ gate, instruction ↔ instruction, template ↔ workflow)
2. **Reconcile** existing conflicts with operator UAT (which rule wins gets human design decision)
3. **Prevent** future drift via a continuous rule-consistency-check mechanism (single-source-of-truth contract + automated parity validator)

Phase-0 research+planning child (this Epic's first deliverable) will:
- Inventory ALL governance-rule source surfaces (instructions/, .github/workflows/, scripts/global/megalint/, hooks/scripts/, PR templates, issue templates, lefthook.yml, CLAUDE.md, AGENTS.md)
- Map known + discovered contradictions with explicit rule-pair citations
- Survey 2026 best-practice for documentation-vs-enforcement parity in CI/CD policy systems (OPA bundles, Cedar policy stores, policy-as-code single-source contracts)
- Propose: (a) which rule-class wins by default, (b) how a parity-validator detects future drift, (c) how to migrate existing conflicts to the resolved state
