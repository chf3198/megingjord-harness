# #2708 — Audit and remediate all accumulated documentation debt

> **Source**: github:issue/2708 | **state: OPEN** | **Labels**: type:epic, priority:P1, role:manager, area:knowledge, status:dormant, phase-gate:research-first
> Mirror of `gh issue view 2708` (derived; edit the GitHub item, not this page).

## Body

## Problem

Significant documentation debt has accumulated in the Megingjord harness because
the `doc-coverage` gate was advisory-only throughout the period when most core
development work shipped. Code changes that introduced new APIs, CLI flags, config
keys, governance contracts, architecture decisions, and workflow behaviors did so
without corresponding updates to README, AGENTS.md, CLAUDE.md, docs/howto/,
ARCHITECTURE.md, wiki pages, design docs, or contributor guides.

No comprehensive audit of this debt has been done. The scope is unknown — it
may span dozens of tickets and hundreds of lines of missing or stale documentation.

## Objective

Produce a **complete, prioritized documentation remediation backlog** through a
structured Phase-0 audit, then execute the highest-priority documentation work as
governed Phase-1 children. After this Epic closes, every shipped feature, API, and
governance contract in the harness has accurate, current documentation.

## Expected outcome

- A manifest of every documentation gap found: what is missing, what is stale, what
  shipped code lacks any documentation, ranked by developer/operator impact.
- All P1-tier gaps resolved: README, AGENTS.md, CLAUDE.md, ARCHITECTURE.md, and
  active howto guides are accurate and current for all shipped features.
- All P2-tier gaps resolved: wiki pages, skill documentation, and developer guides
  for all public APIs exist and are correct.
- P3-tier gaps tracked as follow-on backlog in Epic #GUARDRAILS or filed as
  standalone doc tickets.
- A freshness policy documented: each surface has an owner, a `freshness_window`,
  and a defined trigger for update (code change type that requires it).

## Acceptance criteria

### Phase-0 — Audit (must close before any Phase-1 children open)

- **AC-R1**: Enumerate all documentation surfaces in the repo: every file under
  `docs/`, `wiki/`, `instructions/`, `skills/*/SKILL.md`, root `README.md`,
  `AGENTS.md`, `CLAUDE.md`, `.codex/AGENTS.md`, `.github/copilot-instructions.md`,
  `CONTRIBUTING.md`, `SECURITY.md`, `ARCHITECTURE.md`, and per-area `docs/ARCHITECTURE.md`.
  Record last-modified date and a one-sentence description of what it covers.

- **AC-R2**: Cross-reference the enumeration from AC-R1 against all PRs merged since
  the repo was created (or a pragmatic cutoff — last 6 months minimum). For each
  significant code-change PR, identify: (a) what documentation it should have produced,
  (b) whether that documentation was actually updated, (c) severity of the gap
  (P1 operator-blocking / P2 developer-confusing / P3 incomplete).

- **AC-R3**: Audit the following specific high-risk surfaces known to have accumulated
  debt from the session history:
  - `README.md` — new scripts, new exported functions, new CLI flags
  - `AGENTS.md` / `CLAUDE.md` / `.codex/AGENTS.md` — new governance contracts,
    new baton rules, new workflow requirements
  - `docs/howto/` — new capabilities with no howto entry
  - `instructions/` — instruction files whose content diverges from actual behavior
  - `wiki/wisdom/global/` — concepts/syntheses/skills that reference deprecated behavior
  - `docs/workflow/learnings.md` — learnings that were identified in closeouts but
    not actually written to the file
  - `ARCHITECTURE.md` / `docs/ARCHITECTURE.md` — architecture changes that shipped
    without corresponding doc updates

- **AC-R4**: For each gap found in AC-R3, produce a structured record:
  ```
  surface: <path>
  gap_type: missing | stale | wrong | incomplete
  severity: P1 | P2 | P3
  last_accurate_sha: <sha or unknown>
  required_update: <one paragraph describing what needs to be written>
  source_prs: [#N, ...]
  ```

- **AC-R5** (this AC): Synthesize the audit into a MANAGER_HANDOFF-ready rescope
  comment on this Epic. Propose Phase-1 child tickets grouped by surface area and
  priority tier. Include total estimated documentation volume (pages/files).

### Phase-1 — Remediation (gated on all Phase-0 ACs closed; children filed per AC-R5)

- **AC1 (P1)**: All P1 gaps resolved — README, AGENTS.md, CLAUDE.md accurate for all
  shipped features and governance contracts.
- **AC2 (P1)**: `docs/howto/` complete for all active capabilities:
  baton workflow, doc-coverage enforcement, fleet routing, OTel GenAI conformance
  (including new `isValidGenAI()` API), wiki ops, and any other capabilities without
  a howto guide.
- **AC3 (P2)**: `wiki/wisdom/global/` pages audited and updated for all governance
  concepts that have drifted from current behavior.
- **AC4 (P2)**: `docs/workflow/learnings.md` populated with all learnings identified
  in CONSULTANT_CLOSEOUT artifacts from recent sessions that were not actually recorded.
- **AC5 (P2)**: All `skills/*/SKILL.md` files accurate for their current implementation;
  deprecated behaviors or options removed.
- **AC6 (P3)**: `ARCHITECTURE.md` updated for all architectural decisions that shipped
  without corresponding documentation.
- **AC7 (governance)**: Each Phase-1 child PR carries a fully compliant `doc-coverage:`
  block with governed N/A reasons (demonstrating compliance with Epic #GUARDRAILS
  even before that Epic's implementation ships).

## Non-goals

- Writing documentation for deprecated/removed features (mark as archived, not updated).
- Retroactively producing baton artifacts for already-closed issues.
- Rewriting documentation that is accurate but has a different style or format.
