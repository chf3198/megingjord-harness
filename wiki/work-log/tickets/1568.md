# #1568 — Shift-left governance: pre-push parity + model-diverse review tiers

## Body

## Problem

Current governance enforcement is server-side-heavy: the developer pushes, CI runs ~24 checks, 5 of them fail on first attempt, the PR is re-rolled. This session alone (PR #1558 → #1559) burned one full CI cycle on issues — branch-prefix, 100-line cap, markdown bold on `test_strategy:`, two unanchored marker words, missing spec — every one of which was detectable before push.

Three structural gaps make the baton model less robust than its parts:
1. **Pre-push hooks under-cover what CI enforces.** Local hook runs only `lint:readability:ci`; the 100-line cap, megalint suite, test-evidence validation, baton-artifact regex checks all run only in CI.
2. **All four baton roles can use the same underlying model.** Signer-independence is enforced by surname only (Mason/Harper/Reyes/Vale); `Team&Model:` can be identical across roles. Per Panickssery et al. (NeurIPS '24/'25) and follow-on 2025–2026 work, same-model evaluator scores inflate monotonically while quality stagnates or worsens — a self-bias amplification trap.
3. **Consultant rubric is subjective.** "G2=9/10 because…" is judgment-driven rather than evidence-driven. Subjective scores converge to inflation when iterated; deterministic checklists converge to truth.

## Goal

Shift every deterministically-checkable gate left to pre-push; enforce model diversity on the critical review path; replace subjective rubric scores with evidence boxes so "perfect rating" becomes mechanical (not lenient).

## Acceptance Criteria (Epic-level)

- [x] **AC-1 — Pre-push CI parity**: `lefthook.yml` (or equivalent) runs every required CI gate that has only-local inputs, in parallel, under ~10s on warm cache. Tracked in **#1570**.
- [x] **AC-2 — Collaborator self-review checklist**: A fixed, deterministic checklist runs before `COLLABORATOR_HANDOFF` is posted. Each item is a pasted exit code or diff inspection, not a judgment call. Tracked in **#1571**.
- [x] **AC-3 — Model-diversity gate**: Parse `Team&Model:` from each baton artifact. Enforce on PR open: `COLLABORATOR_HANDOFF.team_model ≠ ADMIN_HANDOFF.team_model` AND `ADMIN_HANDOFF.team_model ≠ CONSULTANT_CLOSEOUT.team_model`. **Cross-model within the same provider passes** (e.g. `claude-code:opus-4-7` vs `claude-code:sonnet-4-6` is acceptable); same `Team&Model:` literal across either critical-path pair fails. Manager–Collaborator may match. Tracked in **#1572**.
- [x] **AC-4 — Second-opinion Consultant pass**: For every Consultant rating, a second independent rater (different provider family — e.g. fleet `gemma3:1b` or open-weights local) emits a parallel G1–G9 score; published delta is appended to `CONSULTANT_CLOSEOUT`. If |delta| > 1.0 on any goal, escalates as Tier-3. Tracked in **#1573**.
- [x] **AC-5 — Anneal stopping rules**: `workflow-self-anneal` enforces a stop condition: max 3 iterations OR `|rubric_delta| ≤ 0.5` OR all deterministic gates green. Tracked in **#1574**.
- [x] **AC-6 — Deterministic G1–G9 rubric**: Each goal's score = (boxes_checked / boxes_total) × 10, where every box is a verifiable evidence check (file path exists, command exit code, regex match). Subjective phrasing forbidden in the rubric template. Tracked in **#1575**.
- [x] **AC-7 — Research ticket closed**: Literature synthesis + decision record. Tracked in **#1569**.
