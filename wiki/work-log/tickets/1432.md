## Objective
Extend `.github/workflows/epic-traceability-lint.yml` (currently exits early when issue is not `type:epic`) so that when a CHILD ticket is closed, the workflow resolves and validates the parent Epic.

## Rationale
Current Copilot implementation fires on all issue events but only validates when the issue itself is an Epic. Side effect: if a child closes and the parent Epic body never had the child reference, no automated event triggers re-validation, so the drift goes unnoticed until someone manually edits the Epic.

My closed PR #1431 had this behavior (`resolveEpic()` helper). The work is in branch `feat/1423-epic-traceability-body-ac` (already deleted; can be reconstructed from PR #1431 history if useful).
