# #1951 — fix: repair .claude/settings.json hook schema regression from #1917

> **Source**: github:issue/1951 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P1, area:hooks, resolution:completed
> Mirror of `gh issue view 1951` (derived; edit the GitHub item, not this page).

## Body

## Problem

PR #1917 (commit `5074559`) introduced the `hooks` block to `.claude/settings.json` using the **flat** hook entry format:

```json
"SessionStart": [
  { "type": "command", "command": "..." }
]
```

The Claude Code extension schema requires each event-array item to be wrapped as a `{ matcher, hooks: [...] }` object (see `.claude/settings.json.template`). The flat format causes the extension to read `.hooks` as `undefined`, producing:

> Settings file failed to parse: `.claude/settings.json` — Expected array, but received undefined. Permission rules and other settings from this file are not in effect.

A secondary pre-existing issue (commit `276821d`, Apr 26) was also present: `Bash(awk '{if \\($1+0 <= 100\\) print}')` fails the `permissionRule` schema pattern due to shell metacharacters.

Both issues combined caused the VS Code Claude Code extension to fail entirely on startup.

## Expected Outcome

`.claude/settings.json` validates against `https://json.schemastore.org/claude-code-settings.json` with no errors, and the Claude Code extension loads permission rules and hooks correctly.

## Acceptance Criteria

- [ ] All hook event entries in `.claude/settings.json` use `{ matcher, hooks: [{type, command}] }` wrapper format
- [ ] The invalid `Bash(awk ...)` permission rule is replaced with a schema-valid equivalent
- [ ] `python3 -c "from jsonschema import ..."` reports VALID against the published schema
- [ ] No regression to existing permission entries
- [ ] Claude Code extension loads without parse error after file is deployed

## Root Cause

Ticket #1917 (Claude hook adapter) was authored by Codex (Quill Reyes / codex:gpt-5.4@openai) which used the wrong hook schema variant — flat `{type,command}` instead of the required `{matcher, hooks:[...]}` wrapper. The template at `.claude/settings.json.template` shows the correct structure but was not referenced during implementation.
