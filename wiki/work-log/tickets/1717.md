## Body

## Parent

Epic #1716. Phase 1 residual research.

## Goal

Inventory the models actually available across HAMR-routed providers (Anthropic, OpenAI-compatible, Ollama, OpenRouter, LiteLLM) and verify the 3 rotation rules can be satisfied with current routes.

## Research questions

1. What providers are HAMR-active for the harness today (per `routing-provider-adapters.json`)?
2. What models does each provider expose? Group by `family` (anthropic vs openai vs google vs meta vs ...).
3. For each baton role transition, can the rotation rules be satisfied? Specifically: Manager → Collaborator (different family); Collaborator → self-check (different family); Collaborator → Admin (different family from both prior); Admin → Consultant (different family from all 3 prior).
4. Identify operators who cannot satisfy the rules with current routes (single-family fleet; air-gapped).

## Acceptance Criteria

- [ ] AC1: `research/hamr-model-family-inventory-2026-05-15.md` lists all HAMR-routed providers and their model families.
- [ ] AC2: Table mapping each baton transition to viable model-family choices.
- [ ] AC3: Identification of operator profiles that cannot satisfy the rules (e.g., Anthropic-only, OpenAI-only, Ollama-only).
- [ ] AC4: Recommendation to Phase 2.4 (G5 fallback design) on which operator profiles get advisory-only vs full opt-out.
