# #1891 — Tier-2 anneal: lease-worktree-path-consistency validator

> **Source**: github:issue/1891 | **state: OPEN** | **Labels**: type:task, status:backlog, priority:P2, area:governance, lane:code-change, anneal:tier-2

## Body

## Problem

The cross-team lease protocol requires each team work in a dedicated worktree to prevent concurrent-session contamination (Epic #1854). The lease comment carries `ticket` + `worktree` fields, but these are not validated for consistency.

**Pattern instance** (2026-05-18):
- Copilot Team's lease comment on #1399 declared `worktree: /home/curtisfranks/devenv-ops-copilot-1874`
- The worktree path embeds ticket number `1874` — but the lease is for ticket `1399`
- This is documented evidence the Copilot Team's worktree separation did NOT isolate #1399 work from concurrent claude-code session on #1612
- Direct downstream effect: the commit message for #1399 was tagged `(#1612)` — cross-team hook context bled in because the worktree was effectively shared with prior session state

The tier-4 worktree isolation from Epic #1854 expects exclusive worktree-per-ticket. Implicit assumption was that the worktree path would semantically match the ticket. No validator enforces this.

## Goal

Pre-flight check: when a cross-team-lease comment is created/edited, validate that the `worktree` field path contains the `ticket` field number (or matches one of the documented exception patterns).

## Acceptance Criteria

- [ ] AC1: `scripts/global/lease-worktree-consistency.js` validator parses `CROSS_TEAM_LEASE_CREATE` comment body, extracts `ticket: #N` + `worktree:` fields, asserts the worktree path contains `N`.
- [ ] AC2: Exception patterns documented (e.g. shared-pool worktrees for trivial lanes) — must be explicitly opted-in via lease field `worktree_exception: true` with rationale.
- [ ] AC3: Workflow runs on `issue_comment.created` and pattern-matches `CROSS_TEAM_LEASE_CREATE` marker.
- [ ] AC4: Failed validation emits structured advisory comment on the affected issue + appends to `incidents.jsonl`.
- [ ] AC5: Stress test: 100 synthetic lease comments (50 consistent + 50 mismatched) verify precision/recall.
- [ ] AC6: Composes with existing worktree-isolation stress suite (Epic #1854).
- [ ] AC7: Tests + harness:self-test registry entry.

## Test strategy

`tdd-pyramid+stress-test` per Epic #1875.

## Why programmatic

Less-intelligent models in Copilot auto-mode reuse prior session paths (worktree from #1874 carried into #1399 lease) without semantic check. Programmatic enforcement closes the gap that allowed hook-context contamination.
