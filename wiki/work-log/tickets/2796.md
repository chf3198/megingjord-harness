Escalation-rate governor with hysteresis and shadow re-promotion.

## Acceptance criteria
- AC1: track per-**task-class** escalation rate over a velocity-relative window (N attempts, not
  calendar). Task-class unit defined (default: area-label; documented).
- AC2: **demote** class at >40% escalation; **re-promote** only at <25% (hysteresis, no flapping).
  Demotion emits Tier-2 anneal (`fleet-dev-class-mis-profiled`) + a re-profile ticket; observable (G8).
- AC3: **shadow re-promotion** — demoted class still run on fleet in shadow (output discarded, scored
  vs accepted result) so re-promotion is data-driven after a model refresh (D6).
- AC4: tests cover demote / hold / re-promote transitions + anti-flap.
