## Body

## Problem

Epic #2362 shipped a CLIENT-side advisory detector in Megingjord hooks (hooks/scripts/antigravity_signer_guard.py). But the Antigravity team uses its own runtime, NOT Megingjord's hooks. The client-side detector cannot actually catch Antigravity's own runtime commits — it only catches the rare case where a Megingjord session happens to commit with an Antigravity signature.

The actual problem (Antigravity committing direct on main) requires SERVER-SIDE detection.

## Goal

Add a GitHub Action that runs on every push to main and inspects commit authorship for Antigravity team signer pattern.

## AC

- AC1: .github/workflows/antigravity-direct-main-guard.yml
- AC2: Triggers on 'push' to main
- AC3: For each commit, check if signer alias matches Antigravity team pattern (Apollo Mason/Harper/Reyes/Vale)
- AC4: If matched AND commit doesn't have associated merged PR, post advisory comment to a tracking issue + append event to a JSONL artifact
- AC5: Advisory mode (post comment, no revert) per Epic #2362 Phase-2 escalation pattern
- AC6: Document this is the SERVER-SIDE companion to the client-side guard from #2471

## Why this ticket exists

Discovered during 2026-05-31 client review of Epic #2362 deferrals. The client correctly identified that the passive detector only runs in Megingjord-deployed sessions, which excludes the actual Antigravity runtime — so the original detection design was incomplete.
