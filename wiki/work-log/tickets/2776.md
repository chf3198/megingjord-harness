## Problem

Five documentation files are artificially capped at exactly 100 lines by the
repo's lint rule. This causes critical sections to be silently omitted, making
the harness harder to understand, onboard into, and contribute to:

- `CONTRIBUTING.md` — 100/100 lines; missing: env setup, bug reporting, community standards, first-PR guide
- `docs/ARCHITECTURE.md` — 100/100 lines; missing: runtime view, quality requirements, security model, ADR table, glossary
- `ARCHITECTURE.md` (root) — 98/100 lines; governance-baton section truncated; no skills/security/crosscutting sections
- `wiki/WIKI.md` — 100/100 lines; missing: three-wiki typology v2, namespace isolation, content-trust scoring
- `AGENTS.md` — 100/100 lines; missing: agent catalog, memory guidance, tool permissions, session startup detail

## Solution

Split each truncated document into a suite of ≤100-line linked pages following
arc42 section conventions, C4 hierarchy, and GitHub community-health standards.
Each primary file becomes the navigation overview; companion files hold the
expanded content.
