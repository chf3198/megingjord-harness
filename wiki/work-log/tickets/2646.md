# #2646 — Review dispatchers must fail over fleet→free-cloud when fleet is down

> **Source**: github:issue/2646 | **state: CLOSED** | **Labels**: type:bug, status:done, priority:P2, area:scripts, lane:code-change, governance:close-without-merge, anneal:tier-2

## Body

## Self-anneal origin
Surfaced 2026-06-03 (Tier-2 anneal). The review-dispatch path is **fleet-only** and does not honor the #2619 cost-ascending failover: `scripts/global/fleet-red-team-dispatch.js#dispatchRedTeam` and `scripts/global/collaborator-preflight.js#runFleetReview` call Ollama via `wrapProviderCall` and return `{hamrStats.ok:false}` when the fleet host's Ollama is down — they do **not** cascade to the `$0` free-cloud tier. Only `cascade-dispatch.js` (task routing) fails over (#2621). Verified 2026-06-06: `grep -c free-cloud` on both dispatchers = **0**.

**Impact (G3):** with fleet down, *every* baton cross-family review (collaborator preflight, red-team) returns empty unless the operator manually calls `free-cloud-dispatch.js`. That manual workaround was used ~6× across the 2026-06-05/06 session. The failure mode pushes the operator toward either (a) skipping the cross-family review (G1/G2 governance loss) or (b) escalating to a paid tier (G3 violation) — the exact anti-pattern #2619 exists to prevent.

## Root cause
Review dispatch predates the #2619/#2621 free-cloud-failover execution path and was never wired to it. The two dispatchers each own a bespoke fleet call instead of routing through the shared cascade.

## Acceptance Criteria
- [ ] **AC1 — failover trigger (availability only):** `dispatchRedTeam` catches a fleet **availability** failure (connection refused / DNS / bounded-timeout / Ollama 5xx) and calls `dispatchFreeCloud`, returning a real cross-family verdict envelope. A **capability** failure (fleet answered but judge/quality inadequate) does **not** trigger free-cloud — preserves the #2619 availability-vs-capability distinction.
- [ ] **AC2 — cross-family validity preserved:** the free-cloud reviewer MUST be non-Anthropic (cross-family vs the claude-code author). Envelope records `substituted_reviewer`, `substitution_reason: fleet-unreachable`, `tier: free-cloud`.
- [ ] **AC3 — preflight consumption:** `collaborator-preflight.js#runFleetReview` consumes the failover; the emitted `cross_family_reviewer` / `cross_family_rating` / `cross_family_findings` fields reflect the substituted free-cloud reviewer so the COLLABORATOR_HANDOFF stays schema-valid.
- [ ] **AC4 — telemetry (no new surface):** the substitution emits one `lane:free-cloud` row to `logs/model-routing-telemetry.jsonl` (reuse #2624 surface) so the failover is auditable via `npm run routing:free-cloud-report` against the premium-share governor.
- [ ] **AC5 — graceful degradation (G6), per sub-case:** the free-cloud path can fail three distinct ways — each returns the advisory `{ok:false, suggested_tier:'free-cloud', degraded_reason:<...>}` envelope with **no crash and no paid escalation**, and the caller surfaces "review-unavailable":
  - (a) **no key configured** → `degraded_reason: free-cloud-unconfigured`;
  - (b) **all free-cloud providers error** (rate-limit/5xx across the ordered chain) → `degraded_reason: free-cloud-exhausted`;
  - (c) **free-cloud also times out** (bounded per the #2626 contract) → `degraded_reason: free-cloud-timeout`.
- [ ] **AC6 — backward-compat:** the fleet-reachable path is behavior-unchanged (byte-identical envelope when Ollama is up).
- [ ] **AC7 — idempotency:** one review request triggers at most one fleet attempt + at most one free-cloud attempt (no retry storm / double-dispatch).
- [ ] **AC8 — docs + supersede workaround:** update the review-dispatch howto; supersede the operator-memory manual-workaround note (`feedback_review_dispatch_env_and_freecloud_failover`); cross-link the #2619/#2621/#2624 contract.
