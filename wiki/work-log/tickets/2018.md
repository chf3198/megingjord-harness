# #2018 — AC6: Promote Copilot/Antigravity shared signing keys T4 → T1 persistent (Epic #2014)

## Body

**Overview**
Promote the shared Copilot/Antigravity team role-signing keys from T4 (Ephemeral-in-Memory, regenerated per session) to T1 (Persistent, ed25519 GPG-bound stored at `~/.megingjord/keys/`). This eliminates the ephemeral key spoofing vector identified in the adversarial red-team audit.

**Context**
Antigravity operates under the shared `copilot` governance lane (no separate shim files). Key promotion under `--team copilot` covers both orchestrators simultaneously — consistent with the harness goal of minimal, centrally managed governance.

**Acceptance Criteria**:
- [ ] Run `npm run signing:bootstrap -- --team copilot` to generate persistent ed25519 keypairs for all 4 roles (manager, collaborator, admin, consultant).
- [ ] Verify key files written to `~/.megingjord/keys/copilot-*.key` with mode 0600.
- [ ] Copy the emitted public key JSON block into `inventory/team-model-signatures.json` under the `cryptoKeys` array.
- [ ] Run `npm run governance:verify` and `npm run harness:self-test` — all checks must pass.
- [ ] Confirm signing tier is now T1 (no ephemeral regeneration on restart).
