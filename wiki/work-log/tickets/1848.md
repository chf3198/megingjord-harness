## Objective

Implement VS Code extension-host guardrails for orchestrator side effects, ensuring sensitive actions are explicitly validated and user-approved in the harness UX flow.

## Rationale (VS Code AI extensibility guidance)

- VS Code chat participant guidance: request explicit user consent for costly operations and irreversible edits/deletes.
- VS Code chat output guidance: command links must use trusted command allowlists to mitigate command-injection vectors.
- VS Code LM/chat patterns: route checks near tools/actions, not only at top-level prompt handling.

## Scope

- Add guard checks for side-effect actions in orchestrator interaction surfaces (chat commands, tool calls, shell-affecting actions).
- Enforce trusted command ID allowlist for Markdown command links used in chat responses.
- Add "costly/irreversible action requires confirmation" policy check where relevant.
- Emit actionable diagnostics with location and remediation.

## Acceptance Criteria

- [ ] AC1: Guard detects and blocks untrusted command URI usage in chat response generation paths.
- [ ] AC2: Guard enforces confirmation policy for costly/irreversible operations in orchestrator flows.
- [ ] AC3: Tests cover allowlist pass/fail and approval-required scenarios.
- [ ] AC4: Documentation explains guard intent and extension-author remediation.
