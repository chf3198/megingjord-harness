# #1501 — Self-anneal: Process drift — self-annealing tickets not filed during review/analysis phases

## Body

## Observation
During the 2026-05-13 session, review and analysis phases (covering #1379, #1473, #1474, #1488 audit) identified improvement opportunities. These were not formalized as self-annealing tickets during the session — they were deferred as 'next session notes' without issue creation. The gap was caught by the operator, not the agent.

## Expected behavior
Per the workflow-self-anneal skill trigger conditions:
- Same failure pattern ≥2 times in 7 days
- Pre-merge gate requires hardening evidence
- Commits exist without linked GitHub issues

The Consultant phase of each baton cycle should invoke the self-anneal skill and file tickets for any identified improvements before marking the cycle complete. No improvement observation should exit as a free-floating note.

## Mismatch
Agent treated 'next session recommendation' as an acceptable artifact. Ticket-first governance requires every actionable observation to have a linked issue.

## Root cause
Ambiguity in Consultant exit criteria: Consultant produces a critique and scorecard but has no explicit gate requiring self-anneal ticket creation before baton close.

## Risk
High — recurrence likely across sessions. Improvements silently evaporate between sessions unless operator intervenes.

## Proposed changes
1. **Research:** Review Consultant exit criteria in role-baton-routing.instructions.md, role-consultant-critique SKILL.md, and consultant-checks.js — identify all gaps
2. **Remediate:** Add explicit Consultant exit gate: 'If any improvement identified during critique, at least one self-anneal ticket must be filed before baton close'
3. **Enforce:** Add check to consultant-checks.js (e.g. gov-004-anneal-ticket-presence): scan session for improvement observations, verify at least one linked open issue exists
4. **Verify:** Confirm anneal tickets created for all future sessions where improvements are noted

## Verification
- role-baton-routing.instructions.md updated with Consultant exit gate
- gov-004 check exists and returns PASS when anneal ticket present, FAIL when not
- Unit test: session with improvement observation but no ticket → FAIL; with ticket → PASS
