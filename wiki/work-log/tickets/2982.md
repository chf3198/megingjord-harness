## Research questions

1. **Tooling inventory + reuse map** — for each of the ~15 existing drift scripts
   (`governance-drift-classifier.js`, `ticket-reconcile.js`, `role-baton-audit.js`,
   `lint-epic-drift.js`, `label-lint-status-cardinality.js`,
   `lint-ticket-redundancy.js`, `coordinator-label-cleanup.js`, …) and ~17
   workflows (`label-lint.yml`, `role-baton-linter.yml`, `epic-traceability-lint.yml`,
   `goal-drift-lint.yml`, `drift-detection.yml`, …): what drift class does it cover,
   is it report-only or mutating, and is it batch or per-issue-event?

2. **D1–D8 coverage gap matrix** — map each seed drift class (D1–D8 in the Epic
   body) to existing-tool coverage. Output: build-vs-reuse decision per class.

3. **Auto-fix-safe vs propose-only classification** — for each class, decide
   whether the fix is deterministic + reversible (auto) or needs a role verdict
   (propose). Define the reversibility/audit contract for `--fix`.

4. **Zero-cost proof** — confirm which classes need *zero* LLM calls. For any
   residue (D1 type inference, D3 title rephrasing), specify free/fleet-lane-only
   routing with a hard premium-lane prohibition. Estimate token cost of the
   current manual sweep vs the proposed tool to quantify the G3 win.

5. **CLI + governance design** — propose the unified command surface
   (`--scan` report / `--fix` apply / dry-run default), the audit-log schema for
   mutations (G8), and how batch `--fix` respects the `lane:no-code-remediation`
   issue-only contract and canonical-main policy.

## Acceptance criteria

- AC-R1: Tooling inventory table (script/workflow → drift class → report|mutate →
  batch|event) committed to `research/`.
- AC-R2: D1–D8 coverage-gap matrix with explicit build-vs-reuse decision per class.
- AC-R3: Auto-fix-safe vs propose-only classification with reversibility contract.
- AC-R4: Zero-cost proof + free/fleet-lane routing spec for any ambiguous residue;
  premium lane explicitly prohibited. Quantified token-cost delta vs manual sweep.
- AC-R5: Unified CLI + audit-log design, including `lane:no-code-remediation`
  compatibility and canonical-main compliance.
- AC-R6: Cross-family adversarial review (non-Anthropic fleet/free-cloud model);
  ACCEPT with min(G1..G10) ≥ 7.
