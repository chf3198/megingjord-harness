# #1574 — Anneal stopping rules: max 3 iter, |delta| <= 0.5, or all deterministic gates green

## Parent

Epic #1568 (AC-5). Research: #1569.

## Goal

Add explicit stopping rules to `workflow-self-anneal` so the anneal loop converges instead of inflating scores. Self-Refine + MAR both show diminishing returns after ~2–3 iterations; current `workflow-self-anneal` caps iteration count but doesn't cap on rubric delta. Without a delta cap, repeated same-model self-critique inflates scores while quality stagnates.

## Acceptance Criteria

- [ ] AC1 — Update `skills/workflow-self-anneal/SKILL.md` (or equivalent) to declare the stopping function explicitly:
  ```
  stop iff:
    deterministic_gates_all_green
    OR iterations >= 3
    OR |rubric_mean_delta_from_previous_iter| <= 0.5
  ```
- [ ] AC2 — Implementation: `scripts/global/anneal-stop.js` exports `shouldStop({ iterations, prev_rubric_mean, current_rubric_mean, deterministic_gates_ok })` returning `{ stop: bool, reason: 'gates'|'iter-cap'|'delta-cap' }`.
- [ ] AC3 — `workflow-self-anneal` skill consumes the helper and refuses to continue after a stop. Continuation requires explicit operator override comment `ANNEAL_OVERRIDE_CONTINUE: <reason>` and is logged as an event in `~/.megingjord/incidents.jsonl` with `event: kill-switch-bypass`.
- [ ] AC4 — Telemetry: each anneal iteration appends to `~/.megingjord/incidents.jsonl` with `iteration_n`, `rubric_mean`, `delta_from_prev`. Visible to the existing dashboard for trend detection.
- [ ] AC5 — Tests: unit tests for `shouldStop` covering 8 cases — first-iter (no prev), gates-green-stop, iter-cap-3-stop, delta-cap-0.5-stop, just-above-delta-cap-continue, just-above-iter-cap-stop, multiple-trips-pick-correct-reason, malformed-input-safety.

## Why these specific numbers

- **Max 3 iterations**: Self-Refine's "Refinement Impact" chart shows iter 0→1 yields most gain; iter 1→2 weaker; iter 2→3 marginal. MAR plateaus at iter 5 but with majority of gain by iter 3. 3 is the smallest cap that captures the gain curve.
- **|delta| ≤ 0.5**: Half a point on a 10-point rubric is below the noise floor of subjective scoring per the self-bias literature. Smaller deltas indicate inflation, not improvement.
- **Deterministic-gates green**: trumps everything — if every box is checked, iteration only inflates scores.

## Bounded-loop interaction (#1339 era)

Current kill-switch guards in `workflow-resilience.instructions.md`:
- Max 1 active pivot per session
- Max 3 pivots per 24h per session
- Max 5 anneal tickets per 7-day window per pattern_id
- Step counter aborts >50 tool calls

This ticket adds the **score-delta** stopping rule, which is orthogonal to the existing rate-limit guards. They compose: tier-2 pivot can fire (rate-limit permitting), but each pivot's anneal loop terminates per the new function.
