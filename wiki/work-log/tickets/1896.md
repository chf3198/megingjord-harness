## Body

## Problem

The harness governance contract (memory `feedback_all_baton_artifacts_before_pr.md`) requires posting all 4 baton artifacts (MANAGER, COLLABORATOR, ADMIN, CONSULTANT) on the linked issue BEFORE creating a PR. This is enforced reactively via `closeout-schema` + `evidence-completeness` after PR creation, but the post-failure feedback loop is expensive and the trap has now recurred multiple times in 24 hours:

- PR #1880 (Copilot Team's #1874 work) — failed closeout-schema, closed unmerged
- PR #1885 (claude-code Team's #1612 work) — failed closeout-schema before retroactive fix
- PR #1887 (claude-code Team's #1259 work) — same trap, fixed mid-flight
- PR #1895 (Copilot Team's #1399 follow-up) — currently failing, baton not advanced past Manager on #1888

This is the structural gap: GitHub-side validators only run after PR creation. A LOCAL pre-PR gate would catch this before push.

## Goal

Pre-PR validator that scans the linked issue's comments for all 4 baton artifact comments BEFORE `gh pr create` is allowed.

## Acceptance Criteria

- [ ] AC1: `scripts/global/pre-pr-baton-completeness.js` script that:
  - reads the current branch's lead ticket (from `feat/<N>-...`)
  - queries `gh issue view N --json comments`
  - asserts all 4 artifacts (MANAGER_HANDOFF, COLLABORATOR_HANDOFF, ADMIN_HANDOFF, CONSULTANT_CLOSEOUT) are present
  - exit 0 on pass, exit non-zero with structured violation list on fail
- [ ] AC2: Integrated into `lefthook.yml` pre-push hook (or new `pre-pr-create` hook).
- [ ] AC3: Honors override via `PRE_PR_BATON_BYPASS=1` env var or `merge-evidence-override:approved` label on issue, with audit trail in `~/.megingjord/incidents.jsonl`.
- [ ] AC4: Tests covering: all-4-present pass, missing-Collaborator fail, missing-Admin fail, missing-Consultant fail, override path.
- [ ] AC5: Stress test (adversarial: 50 historical issues, measure precision/recall).
- [ ] AC6: harness:self-test registry entry.

## Why programmatic

Less-intelligent models in Copilot auto-mode skip ahead to `gh pr create` without verifying the baton was completed. Local pre-PR enforcement removes the dependency on model attention and converts a recurring failure into a fast-fail.

## Composition

- Memory note `feedback_all_baton_artifacts_before_pr.md`
- Existing `closeout-schema` (post-PR) — this validator is the local-pre-PR sibling
- Composes with #1889 (completion-claim-vs-merge-truth) as a paired pre-merge audit
