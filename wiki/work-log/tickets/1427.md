## Body

## Summary
Optimize and harden harness scripts and tools to reduce token spend across core orchestrating services (Claude Code, Copilot, Codex) without degrading execution quality or governance compliance.

## Problem Statement
Core orchestrating services (Claude Code, Copilot, Codex) consume tokens on context that the harness could pre-filter, compress, or cache. Identified waste vectors include:
- Repeated injection of large skill/instruction payloads already loaded in context
- Non-summarized or verbose governance audit output piped to agents
- Redundant prompt construction (role preambles, baton headers, constitution reminders)
- Over-broad context windows when scoped task context would suffice
- Raw JSON telemetry passed to agents rather than pre-digested summaries

## Objective
Equip the harness with token-reduction tooling that measurably reduces average token spend per agent session while maintaining correctness and governance compliance.

## Gate Policy
No ACs or development child-tickets may be created until Phase-0 R&D ticket is completed and signed off.

## Delivery Phases
- Phase-0: R&D synthesis (required gate) — identify waste vectors, quantify baseline, design controls
- Phase-A: prompt compression + context scoping tools
- Phase-B: caching and deduplication controls
- Phase-C: telemetry integration and automated measurement
