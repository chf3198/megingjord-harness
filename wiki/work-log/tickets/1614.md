# #1614 — Self-anneal: harden governance parsing for Refs ordering and structured-field prose collisions

> **Source**: github:issue/1614 | **state: OPEN** | **Labels**: type:task, status:backlog, priority:P2, area:governance, lane:code-change
> Mirror of `gh issue view 1614` (derived; edit the GitHub item, not this page).

## Body

## Parent / trigger

Review of Claude Code Team work on Epic #1568 found repeated CI churn caused by parsers matching human prose or the wrong linked issue.

Observed evidence from #1572 terminal notes:

- PR #1577 needed 3 CI iterations.
- One failure came from a `Refs #N` parser choosing the wrong linked issue when the PR body contained parent/research references.
- Another failure came from closeout/schema parsing matching prose that included a colon-bearing structured field phrase such as `Team&Model:`.

These were worked around manually, but the pattern is recurring enough to deserve a parser-level fix.

## Goal

Make governance parsers prefer explicit machine-readable blocks over loose prose matching, so agent teams can write normal review notes without accidentally triggering structured-field validators.

## Acceptance Criteria

- [ ] AC1 — Replace first-match `Refs #N` linkage logic in affected workflows/scripts with deterministic ticket selection: prefer `Closes #N`, then first `Refs #N` under a dedicated linkage block, and ignore `Refs Epic #N` for child-ticket validation.
- [ ] AC2 — Update structured-field parsers so prose references to field names do not masquerade as actual baton fields. Prefer anchored lines inside known baton artifacts.
- [ ] AC3 — Add regression fixtures for the #1572 cases: wrong parent/research issue selected; prose phrase containing `Team&Model:`; markdown/backtick variants.
- [ ] AC4 — Update handoff/template guidance so teams know the canonical machine-readable linkage block.
- [ ] AC5 — Post compatibility guidance to #1568, #1572, and any validator-owning tickets touched by the fix.

## Harness goals

Primary: G1 Governance, G2 Quality, G6 Resilience, G7 Throughput, G9 Interoperability.

## Review evidence

The Claude Code Team correctly diagnosed the issue in terminal evidence, but operationalizing the fix in parser code will reduce future re-rolls for all three teams.
