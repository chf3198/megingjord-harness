# #2613 — Operationalize governance-bundle: orchestrator producer-push to HAMR KV (makes #2094 deliver)

## Body

## Problem (G3 — operationalize #2094)
Epic #2094 shipped the governance-bundle assembler (`scripts/global/governance-bundle.js`), the read-only `tool:governance-bundle` HAMR dispatch, and the `closeout-schema` fleet parity check. But nothing **produces and publishes** a bundle: there is no orchestrator step that gathers the live governance fields for an issue, builds + signs the bundle, and pushes it to HAMR KV at `governance-bundle:<issue>`. Until that exists, `tool:governance-bundle` always returns `present:false` and a fleet model still cannot self-attest a CLOSEOUT — so the free-fleet consultant parity (the Epic's G3 win) is dormant infrastructure.

## Objective
Add the orchestrator producer-push that makes #2094 deliver: gather the CLOSEOUT fields for an issue, build the bundle (reuse `governance-bundle.js`), sign + POST it to HAMR for KV storage — mirroring the existing `substrate-health-push.js` producer pattern.

## Acceptance Criteria
- AC1: `scripts/global/governance-bundle-push.js` — given an issue number, gathers fields (consultant-checks + rubric-score outputs + a wiki_health snapshot), calls `governance-bundle.buildBundle`, signs (Ed25519 via `OPERATOR_KEY_SEED_B64`, as substrate-health-push does), and POSTs to HAMR.
- AC2: HAMR write route (mirror `/substrate-health`) that validates DPoP + writes `governance-bundle:<issue>` to KV, so the existing read dispatch serves it.
- AC3: `npm run hamr:governance-bundle-push -- --issue N` script.
- AC4: Unit tests (mocked subprocess + fetch): field-gathering assembles the allow-listed fields; push POSTs the signed bundle; graceful no-op when tools/keys unavailable.
- AC5: CHANGELOG fragment; docs note in hamr-routing (producer side).

## Goal Lens
- G3 (primary): turns the dormant #2094 capability into actual free-fleet consultant reviews (no paid-cloud escalation).
- G4: reuses the bundle's allow-list + redaction; signed push (no raw secrets).
- G6: graceful no-op if a tool or the operator key is unavailable.
