# #2488 — Tier-1 GitHub-native primitives for HAMR Layer 2 coordination routes

## Body

## Goal

Provide GitHub-native primitives for every HAMR Layer 2 coordination route used in Tier 1 cross-team communication (single-workspace, multi-AI-team coordination). HAMR remains available as opt-in accelerator and as the substrate for Tier 2/3 cross-workspace coordination (separate future exploration, see follow-on tracking ticket).

## Tier-1 scope (this Epic)

Single-workspace, multi-AI-team coordination — 4 AI agents (Claude Code, Copilot, Codex, Antigravity) all working in the SAME workspace, coordinating on tickets in that workspace's repo. User has write access to the workspace repo.

This is the harness's primary use case today.

## Out of scope (deferred to future exploration)

- Tier 2: Cross-workspace single-user (user has 2+ workspaces; AI team in Workspace A coordinates with AI team in Workspace B). See tracking ticket below for the design questions.
- Tier 3: Cross-user / cross-org coordination. Requires genuinely neutral substrate; HAMR is purpose-fit.
- Layer 1 enforcement (provider wrapper, governance-context injection, fleet-direct-block, bypass detection) — stays as-is. This Epic does NOT touch Layer 1.

## Problem

HAMR Layer 2 routes (mailbox, merge-claim, bundle, MCP-dispatch, quota, cache-stats, substrate-health, review-run, rotation-check) require CloudFlare Workers + KV/R2. For Tier-1 use cases:
- G5 portability: forces operator to maintain a Cloudflare account
- G3 zero-cost: Cloudflare is free at our scale but adds an account dependency
- G6 resilience: when CloudFlare has an outage, Tier-1 coordination breaks even though GitHub is up

GitHub already provides every primitive needed for Tier 1 (per the research synthesis at research/github-cross-team-communication-2026-05-31.md — to be authored as Phase-0 child).
