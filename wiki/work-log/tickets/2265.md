## Body

Refs Epic #2258

Problem:
A reduced-role lane is unsafe unless the repo can reliably deny it when implementation or admin activity occurs.

Objective:
Implement machine-checkable enforcement for the no-code remediation lane.

Acceptance criteria:
- Deny the lane when commits, pushes, PR creation, merges, deploys, or other admin_ops are detected after Manager pickup.
- Deny the lane when repo diffs or implementation uncertainty indicate the work is not issue-only.
- Ensure accepted cases can move Manager -> Consultant without false positives from the skipped roles.
- Cover label/status invariants for the reduced path.
