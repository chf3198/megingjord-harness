# #3019 — Canonical-main enforcer: relative ../ traversal from worktree bypasses write-guard

> **Source**: github:issue/3019 | **state: OPEN** | **Labels**: type:bug, status:backlog, priority:P3, area:hooks
> Mirror of `gh issue view 3019` (derived; edit the GitHub item, not this page).

## Body

## Problem
The canonical-main write-guard in `pretool_guard.py` (hardened in #2945 for absolute paths) does not catch a **relative** path that escapes a worktree CWD into canonical-main, e.g. a write to `../devenv-ops/README.md` issued from `~/devenv-ops-<ticket>/`. The new check only fires when `is_main_checkout(cwd)` OR the target `path_val.startswith('/')` and resolves inside main-root; a relative `..` traversal matches neither branch and is skipped.

## Context
- Pre-existing gap (the original guard only fired on `is_main_checkout(cwd)`); #2945 strictly ADDED absolute-path coverage and introduced no regression. Surfaced by the Admin + Consultant cross-family fleet review of #2945.

## Expected
Normalize relative target paths against CWD before the canonical-main containment check, so `../devenv-ops/<tracked>` from a worktree is evaluated against main-root and denied if tracked.

## Acceptance criteria
- AC1: a relative `../devenv-ops/<tracked-file>` write from a worktree CWD is DENIED.
- AC2: relative writes within the worktree itself remain ALLOWED (no false-positive).
- AC3: unit test covering both cases added to tests/canonical-main-enforcer.spec.js.

Refs #2945.
