# #1575 — Deterministic G1-G9 rubric: replace subjective scores with evidence-box checklists

## Parent

Epic #1568 (AC-6). Research: #1569.

## Goal

Replace subjective Consultant rubric scoring ("G2=9/10 because…") with deterministic evidence-box checklists. Each goal's score becomes `(boxes_checked / boxes_total) * 10` where every box is a verifiable evidence check — file existence, command exit code, regex match, etc. Removes the surface area where self-bias inflates scores. Pursuing perfect ratings becomes pursuing more evidence boxes, not more lenient judgment.

## Acceptance Criteria

- [ ] AC1 — Author `inventory/rubric-g1-g9-v2.json` defining each goal's evidence-box set. Schema:
  ```json
  {
    "G1": {
      "title": "Governance",
      "boxes": [
        { "id": "g1-baton-all-four",
          "check": "issue has MANAGER_HANDOFF + COLLABORATOR_HANDOFF + ADMIN_HANDOFF + CONSULTANT_CLOSEOUT",

        { "id": "g1-signer-independence", "check": "...", "evidence_command": "..." },
        ...
      ]
    },
    "G2": { ... }, ...
  }
  ```
- [ ] AC2 — `scripts/global/rubric-score.js`: takes the PR + issue trail + diff + closeout, runs every box's `evidence_command`, returns `{ G1: {boxes_checked, boxes_total, score}, G2: ..., mean }`.
- [ ] AC3 — `instructions/role-consultant-critique.instructions.md` updated: subjective phrasing ("because the test coverage is good") is **forbidden** in the rubric block. The Consultant pastes only the output of `rubric-score.js`. Free-form rationale moves to a separate `### Rationale` section that does NOT affect the score.
- [ ] AC4 — Compatibility: closeout-schema validator accepts both the new structured rubric and the legacy v1 subjective format for a 7-day transition window. After window, v1 advisory becomes required-fail.
- [ ] AC5 — Coverage: every box in the JSON is testable in isolation. Comprehensive set of test fixtures (one per goal × pass + fail) verifies the score arithmetic.
- [ ] AC6 — Per-goal mins: each goal must have ≥3 evidence boxes (so a single missing box yields 6.67/10, not 0/10). Caps subjective collapse risk.
- [ ] AC7 — Score "10" becomes mechanical: 10 iff *all* boxes in the goal pass. Self-bias inflation has no surface — the box either passes or doesn't.

## Worked example (one box from G2 Quality)

Current subjective: "G2 Quality: 9/10 because tests cover most cases."

New deterministic:
```
G2 Quality
- [x] g2-tests-green: npm test exit code == 0
- [x] g2-new-spec-in-diff: git diff --name-only origin/main matches tests/.*\.spec\.(js|ts)
- [x] g2-no-skipped-tests: grep -c "test.skip" in PR diff == 0
- [ ] g2-coverage-delta-positive: coverage delta vs main >= 0
- [x] g2-lint-readability-no-regression: warnings <= main baseline
score: 4/5 = 8.0
```
Score is now deterministic. Operator can dispute by adding more boxes (more rigor) or removing boxes (less rigor) but not by adjusting subjective weighting.

## Why this kills self-bias amplification

Per Panickssery et al. (NeurIPS '24/'25) and follow-ons: same-model rater inflates scores monotonically during iteration. Their proposed mitigations include CoT-before-verdict and cross-provider judges. **A deterministic checklist is the strongest mitigation of all** — there is no judgment surface for bias to land on. The same model rating the same PR with the same evidence will always produce the same score.

This collapses Epic AC-4 (second-opinion Consultant pass) to a much cheaper check: "does the second rater agree on which evidence boxes are checked?" — usually yes, because the boxes are mechanical.
