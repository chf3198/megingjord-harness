## Self-anneal origin
Surfaced 2026-06-03 (Epic #2398 session, now CLOSED — historical context, **not a live parent**).
New chat sessions cannot run harness review/provider dispatch because **nothing auto-sources the
repo-root `.env`**. All provider keys (`ANTHROPIC` / `OPENROUTER` / `GOOGLE_AI_STUDIO` / `GROQ` /
`CEREBRAS`) live there, but dispatch scripts read `process.env` only, so an interactive session sees
zero keys and wrongly concludes "no review substrate." Verified root cause:
[`free-cloud-dispatch.js:57`](../blob/main/scripts/global/free-cloud-dispatch.js#L57) reads
`(opts.env || process.env)[spec.envKey]` with no dotenv load; no SessionStart hook hydrates it.

Anneal anchor: operator-memory `feedback_review_dispatch_env_and_freecloud_failover`
(pattern: "nothing auto-sources `.env`; review dispatchers are fleet/free-cloud only").

## Goal-lens framing (G3 primary)
This is a **G3 Zero-Cost** unblocker: without auto-hydration the zero-cost lanes (fleet via Ollama,
and free-cloud failover: openrouter-free / gemini / groq / cerebras) are **unreachable**, forcing
escalation to a paid lane or a client credential prompt. Sourcing `.env` is the precondition for the
entire free/fleet review stack. Secondary goals enforced below: G4 (secret handling), G5/G6
(portability/resilience when `.env` is absent or malformed), G8 (auditable hydration), G9 (all four
runtimes), G10 (single shared shim, no per-script duplication).

## Root cause
`.env` is gitignored and only loaded if the operator manually runs `set -a; . ./.env; set +a`.
No SessionStart hook or dispatcher dotenv-load step exists.

## Development plan (Option B primary + Option A complementary — decided, not open)
1. **Shared shim (G10)** — add `scripts/global/load-local-env.js` exporting `loadLocalEnv(opts)`:
   - Lazily parses repo-root `.env` (configurable via `MEGINGJORD_DOTENV_PATH`).
   - **Precedence (G2):** never overrides an already-set `process.env` var (CI / real-env / explicit
     export always wins); only fills absent keys.
   - **Privacy (G4):** loads into the current process only; never writes values to disk/logs; routes
     any emitted line through `log-redaction.js`; honors `MEGINGJORD_NO_DOTENV=1` disable flag.
   - **Resilience (G5/G6):** missing file → no-op pass-through; malformed/partial line → skip-and-warn
     (var NAME only), never throw into the dispatch path.
   - **Observability (G8):** emit one structured line `env-hydrate: filled=[NAME,...] source=.env`
     (NAMES only, count) so hydration is auditable/attributable.
2. **Dispatch wiring (G3/G9)** — `free-cloud-dispatch.js`, `cascade-dispatch.js`, and
   `hamr-provider-wrapper.js` call `loadLocalEnv()` once before key lookup when the needed key is
   absent. Env-based, so it works identically across Claude Code / Codex / Copilot / Antigravity.
3. **SessionStart hook (Option A, complementary)** — advisory hook hydrates the interactive session
   env via the same shim (so direct-CLI usage also benefits); never logs values; offline-safe.
4. **Docs + promotion** — add the auto-hydration step to onboarding + the HAMR activation runbook;
   promote operator-memory `feedback_review_dispatch_env_and_freecloud_failover` to an instruction.
