# #2709 — Unbreakable governance chains: no link may depend on operator discretion

> **Source**: github:issue/2709 | **state: CLOSED** | **Labels**: type:epic, status:done, priority:P2, area:hooks, area:governance, resolution:released, lane:code-change, phase-gate:research-first

## Body

## Meta-invariant (the goal of this Epic)

> **No governance-critical link may depend on operator discretion.** Every link in a
> governance chain MUST be machine-guaranteed — either **(a) auto-emitted** by a hook on the
> triggering event, or **(b) fail-closed** by a guard that blocks forward progress until the
> link exists. "The operator is trusted to remember" is never a valid link type.

This extends the fail-closed property the **baton layer already has** (you cannot push without the
four artifacts — `closeout-preflight` enforces it) *up* into the **meta-governance layer**, which
today still trusts the operator for several critical links.

## The anti-pattern: discretionary links

A governance chain (`detect → record → escalate → remediate`) is only as strong as its weakest
link. Any link that is *voluntary* makes the whole chain breakable by construction — independent of
operator skill. Origin incident: #2703 / PR #2705 — the self-anneal chain broke at a voluntary link
(Tier-1 emission), so an operator-caused CI failure + rework never escalated. See #2706 (the
first concrete instance) and the Tier-1 incident `admin-bypass-merge-without-pre-recorded-exception`.

## Known discretionary (BREAKABLE) links to inventory

Non-exhaustive seed list for Phase-0 — the research must find the complete set:

- **Tier-1 anneal emission** — operator must remember to append to `incidents.jsonl`.
- **Flaw-recognition decision** (CLAUDE.md) — operator must record `file-ticket | log-incident-only | memory-note-only | no-action-justified`.
- **Exception-before-bypass** (Epic #2517) — enforced POST-merge only; no pre-flight guard (#2706).
- **`active_ticket` state-setting** — operator sets it so worktree edit-gates pass.
- **Stale-advisory cleanup** — operator must notice/clear false-positive governance labels.

## Enforcement philosophy (Client design direction, this session)

**Fail-closed + auto-queue** — priority order **G1 > G6 > G7**:
1. Block the risky action when a required chain-link is absent.
2. If the link genuinely cannot be persisted (GitHub/disk unreachable), **auto-write a LOCAL deferred
   marker** and reconcile on reconnect — offline work is never dead-ended, but the chain can never be
   *silently skipped*.
3. Prefer auto-emit for links the system can complete itself (Tier-1, state markers); reserve
   hard-block for links that require an operator judgment the system cannot synthesize.

## Phase-0 research scope (research-first gate)

- **AC-R1**: Inventory EVERY governance chain in the harness (baton, anneal, bypass-exception,
  state-gating, advisory-cleanup, cross-team sign-off, …). For each, enumerate its links.
- **AC-R2**: Classify each link: `enforced (fail-closed)` | `auto-emitted` | `operator-discretionary (BREAKABLE)`.
- **AC-R3**: For each BREAKABLE link, propose the guardrail (auto-emit hook vs fail-closed gate) and
  its offline fail-closed-+-auto-queue behavior; cite the existing enforcement point it should mirror.
- **AC-R4**: Define the machine-checkable meta-invariant + a CI validator that detects a newly-added
  governance chain shipping a discretionary critical link (so the invariant is self-enforcing, not prose).
- **AC-R5**: Cross-family adversarial review (fleet, non-Anthropic) of the inventory + classification.

## Phase-1 (deferred until Phase-0 closes)

Implement the guardrails per the AC-R3 proposals, highest-risk chains first. #2706 is the pilot
instance (admin-bypass pre-flight guard + auto-Tier-1 bridge) and validates the pattern end-to-end.

## Goal lens

G1 Governance (chains are non-negotiable controls — they must not be skippable), G6 Resilience
(unbreakable end-to-end + graceful offline degradation), G3 Zero-Cost (avoidable rework from broken
chains burns tokens), G8 Observability (auto-emit makes the chain self-recording), G2 Quality.

## Anti-pattern guard for this Epic itself

This Epic must NOT solve the problem with another discretionary link (e.g., "the operator should run
a checklist"). The deliverable is structural enforcement (hooks + a CI meta-validator), per the
self-anneal-scope rule that code work routes through the normal baton.
