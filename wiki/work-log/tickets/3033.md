# #3033 — C5: Atomic coordination primitives for shared state + lease liveness

## Body

## Problem (F-LS1, F-LS2, F-LS4, F-WT1, F-LG2, F-LG3)
state_store.save_state, cross-team-lease-registry write, and JSONL emitters are non-atomic read-modify-write (no flock/atomic-rename) -> lost updates / torn files / lost leases under concurrent teams. Crashed-team lease blocks others ~24h. No cross-team lease pre-push gate (#2916). Log-rotation rename->write window drops emits.

## ACs
- AC1: ONE reusable atomic primitive (atomic-rename / with-file-lock, timeout + fail-closed) consumed by state_store, cross-team-lease-registry, JSONL emitters (DRY).
- AC2: ship the cross-team-lease pre-push blocking gate (#2916).
- AC3: bounded, observable heartbeat reaper for crashed-team leases.
- AC4: log-rotation creates the empty hot file before rename (no mid-emit drop).

## Success metric: N-writer stress test -> zero lost updates / torn files; lock p99 sub-ms.
test_strategy: tdd-pyramid+stress-test
