## Epic Summary

Empirical diagnostics (this session, `tmp/it-copilot-diagnostics.md`) proved that **VS Code GitHub Copilot in Agent mode on a BYOK model (Gemini 2.5 Flash) is already a capable, governed harness participant** ‚Äî it has ~50 tools (file ops, terminal, `gh`, `github_repo`/`github-pull-request_*`, `runSubagent`, 13 Playwright MCP), reads the governance instructions correctly, returns real repo data (verified #3021 title), and is subject to the harness ticket-gate (it hit `File edit blocked: no active ticket` and created #3040 + a branch trying to satisfy it).

**The gap is NOT capability ‚Äî it is harness integration.** Copilot BYOK is missing the bridges that make a runtime a *first-class* Megingjord citizen. This Epic upgrades the harness to close them so the Copilot Team operates at **full parity** with Claude Code under BYOK, at $0.

## Confirmed gaps to close (from the diagnostic)

1. **No HAMR capability bridge** ‚Äî Copilot is aware of HAMR (reads `hamr-routing.instructions.md`) but has **no direct tool** to fetch the `governance-bundle` or invoke `/mcp` capabilities (review:run, bundle:fetch). HAMR `/mcp` uses Ed25519 DPoP, not standard MCP, so it needs a **HAMR‚ÜíMCP adapter**.
2. **HAMR observability blind spot** ‚Äî the Copilot extension's BYOK provider calls bypass HAMR's cost/observability wrapper entirely (no metering/attribution of Copilot model usage).
3. **Signer / Team&Model parity** ‚Äî Copilot-on-Gemini artifacts declare `copilot:gemini-2.5-flash`; the alias must be registry-derivable or admin-gate rejects it (the #3021 signer-mismatch class, now extended to BYOK model families).
4. **Ticket-activation friction** ‚Äî Copilot hit the same `no active ticket` gate I keep hitting and got *stuck*; there is no clean, Copilot-usable mechanism to activate a ticket/worktree for edits. Cross-runtime state (`active_ticket`) is not wired for Copilot.
5. **Tool-use reliability** ‚Äî when Copilot does NOT engage its tools it hallucinates (the "E123 / ML Model Finalization" turn). Needs harness-side nudges (instruction directives / required-tool patterns) to force tool use for governed tasks.
6. **Skills are reference-only** ‚Äî `~/.copilot/skills` are docs; Copilot can `runSubagent` but the harness skills are not wired as Copilot-invokable executables.
7. **Enforcement-hook reach** ‚Äî the ticket-gate fired on Copilot, but it is unclear which guards reach Copilot's own tool calls vs only terminal/git; needs an explicit cross-runtime enforcement contract.
