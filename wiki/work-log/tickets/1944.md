## Body

## Problem

The 'complete ticket ####' process repeatedly stalls after Collaborator phase implementation. Agents implement code changes, run tests, and commit locally — but do NOT proceed through Admin (push/PR/merge) and Consultant (review/closeout/close) phases to fully deliver work.

Example: Issue #1933 remediation completed Collaborator implementation (all tests pass, code committed locally), but stalled before Admin push/PR/merge gate.

## Root Causes to Investigate

1. **Baton-to-Admin handoff unclear**: No explicit gate signal or artifact name triggers Admin phase progression
2. **PR creation assumed**: Admin phase may assume PR already exists; no automation
3. **Consultant gate missing**: Post-merge Consultant critique not consistently invoked
4. **Token budget overflow**: Long sessions may exhaust context mid-handoff
5. **Role-switching latency**: Manual transition between roles (code edit → git push → PR → merge → review) may cause state loss

## Expected Behavior

After Collaborator posts COLLABORATOR_HANDOFF:
- Admin automatically: pushes branch, creates PR, runs CI, waits for approval
- After merge: Consultant automatically critiques code, posts CONSULTANT_CLOSEOUT
- After Consultant approval: Issue closes atomically with evidence linkage

## Acceptance Criteria

- AC1: Document exact Admin gate entry condition (what artifact/state triggers Admin phase start)
- AC2: Document exact Consultant gate entry condition (what artifact/state triggers Consultant phase start)
- AC3: Measure 'complete ticket' workflow latency (Collaborator start → Consultant CLOSEOUT)
- AC4: Identify and remove any manual hand-offs in the baton cycle (target: zero manual steps)
- AC5: Create a test case/mock that executes full Manager→Collaborator→Admin→Consultant cycle end-to-end
- AC6: File follow-up tickets if blockers found (e.g., PR approval delays, test failures, Consultant override rules)
