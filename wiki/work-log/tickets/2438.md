## Problem

There is no programmatic quality gate that a Collaborator runs BEFORE writing a COLLABORATOR_HANDOFF. Currently a Collaborator can write the handoff comment without running lint, tests, doc-coverage validation, or cross-family review. The user's requirement: "the collaborator is responsible for heavy testing, linting, 1-100 rating of their work, and all other quality and governance checks before the PR — as much of this as possible should be programmatic."

## Objective

Create `scripts/global/collaborator-preflight.js` (and an npm script `npm run collaborator:preflight`) that orchestrates the full Collaborator quality gate in sequence and produces a structured exit report. This script must run and pass before the Collaborator writes their COLLABORATOR_HANDOFF.

## Acceptance Criteria

- [ ] `scripts/global/collaborator-preflight.js` exists (≤100 lines; delegates to helpers)
- [ ] Gate sequence (ordered, exits on first hard failure): (1) lint (`npm run lint`), (2) tests (`npm test`), (3) megalint doc-coverage validator (requires doc-coverage block + surfaces in diff), (4) changelog-fragment-presence validator (requires `.changes/unreleased/<N>.md` in staged files), (5) cross-family fleet review via `multi-model-critique.js` or `fleet-red-team-dispatch.js` — produces a 1-100 rating
- [ ] On success: prints structured JSON report with fields: `lint_ok`, `tests_ok`, `doc_coverage_ok`, `changelog_fragment_ok`, `cross_family_rating` (int 1-100), `cross_family_reviewer` (model@substrate), `cross_family_findings` (brief summary or "none")
- [ ] The cross-family reviewer MUST use a free fleet model: priority order = Tailscale Ollama (`qwen2.5-coder:32b` via fleet-red-team-dispatch.js) → OpenRouter free cascade. Routing follows `scripts/global/model-routing-policy.json` red-team lane
- [ ] `npm run collaborator:preflight` added to `package.json` scripts
- [ ] The script accepts `--ticket=N` and `--diff-files=path1,path2...` args (or reads from git staging when not supplied)
- [ ] Help text printed on `--help`; non-zero exit code on any gate failure
- [ ] Unit tests in `tests/` cover: all-pass case, each individual gate failure case
- [ ] File ≤100 lines (delegates step logic to existing validators; does not reimplement them)

## Context Feeding for Cross-Family Reviewer

The cross-family reviewer should receive:
1. The git diff (via `git diff --staged` or `git diff HEAD~1`)
2. The `doc-coverage:` block content (if present in staged content or passed as arg)
3. Relevant wiki pages from `wiki/` (the `[[concepts]]` pages referenced by the changed files' area labels)

This gives the reviewer the context to produce a substantive 1-100 rating.

## Design Constraint

G3 Zero Cost is critical: all reviewer model calls must use FREE resources (fleet Ollama via Tailscale, or OpenRouter free-tier failover). No paid model invocations.
