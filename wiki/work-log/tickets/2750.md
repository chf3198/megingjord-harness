## Goal
Implement a GitHub-native mailbox primitive that replaces HAMR `/mailbox/{read,write}` for Tier-1 single-workspace cross-team coordination.

## Design (from Phase-0 synthesis #2489)
- One pinned `coordination` Issue per repo acts as the append-log
- `write` = post a comment to that issue (append-only, no write conflicts)
- `read` = GET `.../comments` with `If-None-Match: <etag>` cursor (304 responses are rate-budget free)
- Authorship and ordering are native GitHub; Team&Model signing block in comment body
- Client auto-detects or creates the coordination issue on first use

## Acceptance Criteria
- AC1a: `scripts/global/github-mailbox.js` (≤100 lines) exposes `writeMessage(repo, body)` and `readMessages(repo, since)` using `@octokit/request` or `gh` CLI subprocess
- AC1b: `readMessages` uses ETag/If-None-Match header to avoid counting free-304 responses
- AC1c: Auto-creates/finds the pinned coordination issue if absent (label `coordination-mailbox`)
- AC1d: `tests/github-mailbox.spec.js` with ≥80% coverage (mock `gh` or Octokit)
- AC1e: Cross-family review: fleet dispatch (qwen or gemma) → ACCEPT ≥7/10

## Implementation Notes
- Use `gh api` subprocess (no new npm dep) OR `@octokit/request` if already in package.json
- ETag stored in a local state file (`.dashboard/mailbox-etag.json`) — not in git
- `since` cursor = last-read comment `id` field (persisted alongside ETag)
