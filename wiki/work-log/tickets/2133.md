# #2133 — Pull reasoning model onto 36gbwinresource for zero-cost Consultant inference

## Body

## Problem

`36gbwinresource` carries ~28 GB available RAM and a Quadro T2000 GPU but only
has coding-specialized models (`qwen2.5-coder:*`). Consultant critique (red-team
review) requires reasoning-capable models — coding-specialized models produce
structurally correct but governance-nuance-deficient closeouts.

G3 (Zero Cost) > G7 (Throughput). CPU-offloaded speed (~5-10 tok/s for a 32B
model) is acceptable when the result is $0 cost AND reasoning quality. The
current routing (`recommended-lane: premium`) incurs Copilot premium requests
for every Consultant pass when a zero-cost fleet alternative exists.

## Acceptance Criteria

- AC1: A reasoning-capable model (preferred: `qwen3:32b` Q4_K_M ~19.85 GB;
  fallback: `deepseek-r1:32b` if unavailable in Ollama registry) is pulled
  on `36gbwinresource` via `ollama pull`.
- AC2: `inventory/devices.json` updated — new model added to `ollamaModels`
  and `recommendedModels` for `36gbwinresource` with `use: "reasoning and
  critique (Consultant role)"`.
- AC3: `scripts/global/model-routing-policy.json` updated — Consultant
  critique task class routed to `36gbwinresource` with the new reasoning
  model instead of the `escalation_direct_to_premium` list.
- AC4: `skills/role-consultant-critique/SKILL.md` updated — `recommended-lane`
  changed from `premium` to `fleet` with rationale citing G3 > G7.
- AC5: All existing tests pass; lint clean.

## Constraints

- Do not remove `qwen2.5-coder:32b` from `36gbwinresource` — it serves the
  coding lane.
- RAM budget: total loaded models must leave ≥ 20% free (~6.4 GB of 32 GB).
  Loading both `qwen2.5-coder:32b` (19.85 GB) and `qwen3:32b` (19.85 GB)
  simultaneously exceeds budget — document that they are mutually exclusive
  (only one loaded at a time via `OLLAMA_KEEP_ALIVE`).
- Verify `qwen3:32b` is available in the Ollama registry before pulling;
  fall back to `deepseek-r1:32b` if not.
