# #2574 — Guardrail-first CI completion across all teams

> **Source**: github:issue/2574 | **state: CLOSED** | **Labels**: type:epic, priority:P1, area:governance, status:cancelled, lane:code-change, anneal:tier-2
> Mirror of `gh issue view 2574` (derived; edit the GitHub item, not this page).

## Body

## Objective
Make CI monitoring/wait behavior a hard guardrail (not optional instruction) across Copilot, Codex, and Claude Code team flows.

## Problem
Current behavior can allow intervention churn while required checks are still pending. This should be impossible by default unless monitoring reports unrecoverable errors.

## Completion-Truth Requirement (Non-Optional)
Work is **not complete** until terminal closure conditions are met.
For deferred-final merge evidence, completion requires all of:
- merged PR evidence on main,
- CONSULTANT_CLOSEOUT artifact,
- explicit issue terminal-finalize action (`status:done` + issue closed),
- no active execution role labels remaining.

## Success Criteria
- Pending-only required-check state enforces wait-only behavior.
- Session proceeds autonomously until completion when recovery is possible.
- Session stops and reports only on unrecoverable errors (with explicit evidence).
- Policy is encoded in guardrail scripts/validators and covered by tests.

## Child Scope
- #2573 — strict wait-for-green loop and merge-attempt gating.
- #2575 — enforce consultant terminal close after merged deferred-final PR.
- Add cross-runtime/operator guidance + validator enforcement and regression tests.

## Acceptance Criteria
- [ ] AC1: State machine for required checks (`pending-only`, `failing`, `green`) implemented in guardrail path.
- [ ] AC2: `pending-only` permits read-only monitoring operations only.
- [ ] AC3: `failing` permits remediation paths; `green` permits merge path.
- [ ] AC4: `unrecoverable` classification emits explicit stop/report artifact with error evidence.
- [ ] AC5: CI + local tests cover all transitions and prevent regressions.
- [ ] AC6: Team-facing docs/instructions updated to make this non-optional and obvious.
- [ ] AC7: Add a blocking "completion-truth" validator for deferred-final flow:
  - merged PR + consultant closeout + issue still open => FAIL/alert (cannot claim complete).
- [ ] AC8: Add auto-remediation or required admin/consultant finalize step to close issue once AC7 preconditions are met.
