### Added
- Closeout recurrence guard + session-end closeout advisory (#3025, lead child of Epic #3026, G1). The consultant-closeout validator now emits a hard-blocking `memory-note-only-on-recurring-pattern` violation when a `CONSULTANT_CLOSEOUT` classifies a flaw as `memory-note-only` for a `pattern_id` already recorded in `~/.megingjord/incidents.jsonl` — recurrence proves a memory note is insufficient, so the correct classification is `file-ticket` (`scripts/global/closeout-recurrence-guard.js`). Detection is factored into a shared `scanOpenPRs` so sibling #3028's pre-push lefthook hook reuses it instead of reimplementing (anti-duplication, G2/G10). Covered by `tests/closeout-recurrence-guard.spec.js`, `tests/open-pr-closeout-scan.spec.js`, and `tests/stress-closeout-recurrence.spec.js` (adversarial corpus + p99 budget), all wired into the quality-gates deterministic CI list.
