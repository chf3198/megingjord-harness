### Governance — flaw-capture replay-eval promotion gate (#3434, Epic #3425 P1-g, terminal)

`scripts/global/flaw-capture-replay-eval.js` + the labelled corpus `tests/eval/flaw-capture-corpus.json`
(built by `flaw-capture-corpus-build.js` per the #3426 AC-R6 methodology) gate the flaw-capture
advisory→blocking promotion — **replay-eval-gated, never calendar-gated**. `npm run flaw-capture:replay-eval`
reports per-class + aggregate precision; promotion requires aggregate ≥0.85 **and** no friction class
below 0.70, subject to anti-skew sample floors (≥200 total, ≥15 per class, ≥30 per role). Promotion is
**per-class** (a data-poor class stays advisory) and **auto-revoking** (recomputed every run, so a
regression below the bar reverts to advisory with no manual re-arm). Only **high-confidence F6**
contradictions are ever blocking-eligible; `FLAW_CAPTURE_DISABLED=1` is a hard rollback no-op. The seed
corpus ships **below** the ≥200 floor by design, so the whole flaw-capture system correctly stays
advisory until the corpus is populated from real history — closing Epic #3425.
