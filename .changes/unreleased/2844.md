### Added

- Fleet sandbox test-exec (#2844, P1-0 child of #2802): validate a fleet model's proposed change by
  materializing it in a throwaway detached git worktree off HEAD and running an operator-declared test
  command there, returning a fail-closed trust verdict. The real working tree is never touched; the
  fleet payload is treated as data (never executed) and unsafe/`..`-escaping paths are rejected before
  anything is written (OWASP OA10/OA2). New: `scripts/global/fleet-sandbox-exec.js`,
  `scripts/global/fleet-sandbox-runner.js`.
