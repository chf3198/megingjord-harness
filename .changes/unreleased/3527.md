### Added
- Fleet-wide **cross-repo epic-aging rollup** (#3527, Phase-1b of Epic #3520, per ADR-020 §D2/D3/D4/D5): a scheduled report of every open `type:epic` across all non-archived `chf3198` repos with `age_days` (createdAt), `idle_days` (updatedAt), and `child_completion` %. New `scripts/global/epic-aging-rollup.js` (pure, fixture-testable core `classifyEpic`/`buildRollup` + thin `gh` GraphQL IO). Reads child-completion via native `subIssuesSummary { total completed percentCompleted }` **with the `GraphQL-Features: sub_issues` header** (omitting it silently returns null — the #3354 trap; a test asserts the header is sent). Surfaces the **silently-parked** signal — `child_completion == 100 && state == open` (the `copilot-global-skills#1` scenario: all children done, epic still open) or `idle_days > 30`. On a null `subIssuesSummary` it falls back to REST `GET /repos/{o}/{r}/issues/{n}/sub_issues` (`completed/total`), and if that is unavailable emits `child_completion: null` — never a wrong 0%. Every open epic is written to `incidents.jsonl` + surfaced on the #1611 dashboard card. Rate-limit safe via the shared #2930 circuit-breaker; `repos_without_epic_read` coverage + `>100-epic` cursor-pagination truncation are both logged (no silent drop, #2090/#2895). Ships a fixture self-test (`tests/epic-aging-rollup.spec.js`, #1893) covering parked/healthy/null-subissue/zero-epic + the header-sent assertion. Runs `47 */6 * * *` (#2405 6-hourly snapshot shape) + `workflow_dispatch`. $0 (G3).
