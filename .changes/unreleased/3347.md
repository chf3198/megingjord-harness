### Fixed
- `merge-bypass-gates` no longer false-flags every clean self-merge as an admin-bypass (#3347). `admin-merge-exception.js#detectAdminBypass` previously returned `true` whenever `reviewApproved === false`, but this single-operator harness governs via baton artifacts + required status checks, not GitHub PR-review approvals — so every merge tripped `admin-merge-without-exception`, defeating the gate's purpose (a real bypass was indistinguishable from the noise, G1/G8) and false-blocking the local `pretool_guard` merge gate. Fix: `reviewApproved` is now gated on a new `reviewRequired` input (only a bypass when review is actually required), and the workflow computes a new pure helper `computeRequiredChecksGreen(checkRuns, requiredContexts)` over only the branch's **required** contexts (derived from the ruleset/branch-protection) instead of all `check_runs` — so advisory checks and the gate's own run no longer self-perpetuate the failure. Genuine `--admin` override detection (Epic #2517) is preserved. tdd-pyramid: `tests/merge-bypass-gates.spec.js`.
