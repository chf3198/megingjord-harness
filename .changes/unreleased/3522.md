### Added
- Fleet-wide **watcher-health monitor** (#3522, Phase-1a of Epic #3520, per ADR-020 §D1/D3/D4/D5): a scheduled job that *watches the watchers* — enumerates every non-archived `chf3198` repo's scheduled governance/validator workflows and detects four distinct failure signatures: `failed-since-inception` (last min(10,total) runs all failed — the `copilot-global-skills#1` silent-audit case), `n-consecutive-failure` (3 failures after a prior success), `auto-disabled-inactivity` (GitHub disabled it after 60d idle), and `stale-heartbeat` (dead-man's-switch: no run within `schedule_interval + 6h` jitter buffer). New `scripts/global/watcher-health.js` (pure, fixture-testable core `classifyRuns`/`classifyWorkflow`/`buildReport` + thin `gh` IO). Alert routing per §D3: `incidents.jsonl` always → #1611 dashboard card → `governance:needs-triage` issue (deduped by `<!-- watcher-health:{repo}/{workflow} -->`) for `failed-since-inception`/`auto-disabled-inactivity` only. Rate-limit safe via the shared #2930 circuit-breaker (trips after 3 secondary-limits, records a `rate-limited` incident, never silent) + #3483 content-SHA workflow-file cache. False-pos/neg controls (§D5): `watcher-health.allow.json` allow-list with required reason strings + `stale-allowlist` orphan detection, and a `coverage`/`partial-coverage` record every run (no `.catch(()=>{})`/`|| true`, #2090/#2895). Ships a fixture self-test (`tests/watcher-health.spec.js`, #1893) covering all four signatures + a healthy control + edge cases (zero-workflow repo, archived repo, allow-list suppression, stale-allowlist). Runs `17 */6 * * *` + `workflow_dispatch`, self-exempt from inactivity-disable. $0 (G3).
