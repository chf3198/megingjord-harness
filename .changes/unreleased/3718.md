### Fixed
- `wiki-reconcile-cron.yml` daily reconcile no longer aborts on a gitignored `git add` path (#3718). The commit step staged `wiki/ dashboard/events.jsonl`, but `dashboard/events.jsonl` is gitignored, so under `bash -e` the `git add` errored and aborted the whole job — freezing 1170/1176 work-log mirrors at `updated: 2026-06-17` (170 still showing `OPEN` for closed issues) for 6 consecutive days, and silencing the wiki-health alarm (which writes to that same gitignored path). Fix drops the gitignored path (`git add wiki/`; never `-f` a runtime log into the repo), adds an `if: failure()` durable GitHub-issue alert so a silent daily-cron failure surfaces, and pins a golden-file regression test asserting the git-add pathspec excludes gitignored paths.
