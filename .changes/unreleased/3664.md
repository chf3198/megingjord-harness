### Fixed
- `hooks/scripts/live_checks.py` `ci_gate_status()` now fails only on **branch-protection required** red checks, not on non-required advisory ones (#3664). `gh pr checks --json` carries no `isRequired` field, so the gate resolved the base branch's required-context set (`_required_contexts`) and filters advisory reds out (`filter_to_required`) before classification. Unresolved required set falls back to the legacy all-checks (fail-closed) behavior; required reds stay enforced. This stops the local pre-merge gate (`pretool_guard.py`) from false-blocking `gh pr merge` on advisory checks such as `worktree-governance-required` / `Doc update required`.
