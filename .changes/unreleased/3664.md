### Fixed
- `hooks/scripts/live_checks.py` `ci_gate_status()` now fails only on **branch-protection required** red checks, not on non-required advisory ones (#3664). Two root causes fixed: (1) the gh query requested an invalid `conclusion` field (`gh pr checks --json` exposes `name,state` only, no `conclusion`), so every call silently exited non-zero with empty stdout and fell through to the exit-code fallback that counts *all* reds — the query is now `name,state`; (2) `gh pr checks` carries no `isRequired` field, so the gate now resolves the base branch's required-context set (`_required_contexts`) and filters advisory reds out (`filter_to_required`) before classification. Unresolved required set falls back to legacy all-checks (fail-closed); required reds stay enforced. This stops the local pre-merge gate (`pretool_guard.py`) from false-blocking a PR-merge on advisory checks such as `worktree-governance-required` / `Doc update required`.
