### Added
- One-time governed backfill of the two empty Three-Wiki data-stores (#3065, Epic #3063): new `scripts/wiki/backfill-work-log.js` mirrors OPEN + last-90d-closed issues to `wiki/work-log/tickets/<N>.md` and merged PRs to `wiki/work-log/prs/<N>.md`, each with #2052 provenance frontmatter (`source_path`, `source_sha256`, `content_hash`, `last_updated`, `generated_by_run`), `log-redaction` applied to bodies, and validate-at-write (frontmatter schema + source-hash parity aborts a drifted entry). Seeds 1172 ticket + 961 PR mirrors.

### Fixed
- `scripts/wiki/ingest-code.js` `computeTrustScore` referenced an undefined variable `s` and threw a `ReferenceError` on every call — the root cause of Wiki A (`wiki/code/`) never populating. Fixed the binding and routed output into the storage layout: `scripts/global` → `wiki/code/symbols/` (structural), `instructions` → `wiki/code/concepts/` (semantic). Backfills 423 symbol + 42 concept pages with provenance frontmatter.

### Changed
- `lint:md` now excludes the generated `wiki/work-log/**` and `wiki/code/**` mirror trees (governed by `wiki:lint`, consistent with the existing `wiki/wisdom/global/sources` + `syntheses` exclusions).
- The advisory Vale prose-lint step in `.github/workflows/lint.yml` is now `continue-on-error: true` — reviewdog's `github-pr-check` reporter fetches the PR diff, which GitHub caps at 300 files, so a large PR (this 2608-file backfill) made it 406 and broke the required `lint-required` job on an infra error rather than a prose finding. Robust large-diff handling tracked as follow-on #3079.
