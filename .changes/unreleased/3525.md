### Added
- `resolution:superseded` closure-reason taxonomy (F6 of Epic #3517, per ADR-020 §D3): work overtaken-by-events now has an honest terminal reason, distinct from `duplicate` (not identical) and `completed` (not performed). New pure decision function `scripts/global/superseded-resolution.js` (`decideSupersededResolution`) gates the apply-step with six false-positive controls — two-signal rule, evidence guard, self-supersession block, reversibility (auto-reopen on `#M` reopen), appeals path (`signal:contested-superseded` → #2990 propose-only queue), and an acyclic/DAG guard — upholding the surface-only invariant I0 (never auto-closes on uncertainty; degraded input fails closed). Adds `signal:contested-superseded` label. Detection remains owned by the #3398/#3420 lane (not rebuilt). Operator guide: `docs/howto/superseded-resolution.md`. Tests: `tests/superseded-resolution.spec.js`.
