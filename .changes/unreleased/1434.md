### Changed
- Readability gate is now **diff-aware** (#1434): `lint:readability:diff` scans only the JS files changed versus the merge-base with `main` and fails solely on **net-new** warnings (per-file delta against the base revision), instead of failing on an absolute repo-wide warning count. This operationalizes Stage 2 ("block regressions on touched files") of the readability rollout model and resolves the recurring baseline-drift breakage where the absolute threshold had been ratcheted 420 → 475 → 486 to sit exactly at the live count, re-breaking every unrelated PR (documented admin-overrides + `PUSH_GATES_BYPASS` across cycles 4–7). The local pre-push hook (`lefthook.yml`) and CI (`.github/workflows/lint.yml`, now `fetch-depth: 0`) both route through diff-aware mode; when the base ref is unavailable (shallow checkout, bogus ref) the gate degrades gracefully to the absolute ceiling (`--max-warnings=486`) rather than crashing or failing open. New unit coverage: `tests/readability-diff-gate.spec.js`.
