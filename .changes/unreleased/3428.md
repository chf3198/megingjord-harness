### Governance — per-review-point flaw capture, foundation (#3428, Epic #3425 P1-a)

Generalized the Consultant-only `mid_flight_flaws` block into a `flaws_recognized:` block on
**all four** baton artifacts (MANAGER / COLLABORATOR / ADMIN / CONSULTANT). A new **advisory**
megalint validator (`scripts/global/megalint/flaws-recognized.js`) checks the block on each
artifact present on an issue: it must be a bare `none` or one entry per disposed candidate whose
`decision:` is in the `judgment-gate.js` `FLAW_DECISIONS` enum with a decision-typed `artifact:`
(`#N` / `pattern_id` / memory-path / rationale). No new enum, no new event schema. The field is a
`block` (not required) field in `baton-artifact-schema.js`, preserving the historical baton replay
corpus and the cross-runtime byte-identity invariant; live-artifact requiredness is enforced by the
validator. Ships advisory pending the replay-eval promotion gate (#3434).
