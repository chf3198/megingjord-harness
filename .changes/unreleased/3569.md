fix(hooks): docs-lane close/Stop no longer false-blocked by phantom merge demand (#3569)

Generalizes the #3266 report-only-lane carve-out to `lane:docs-research` and `lane:docs-only`.
Both are PR-less/merge-less "Manager+Consultant" reduced batons, yet a lingering/false
`code_touched` flag (or a genuine docs-only session) could still trip the two governance surfaces
that issue 3266 only taught about `lane:research`:

- **Docs-lane carve-out** — a new sibling `active_ticket_is_docs_lane()` (matching
  `lane:docs-research` / `lane:docs-only`) joins `active_ticket_is_research_lane()` at BOTH the
  `pretool_guard` issue-close block and the Stop-hook Admin-op gate (`stop_checks` /
  `admin_patterns.required_admin_ops`). When the ticket is a docs lane AND the working tree is
  clean, zero Admin ops are required and close/Stop is permitted with no merge.
- **Anti-abuse guardrail (clean-tree condition is load-bearing)** — docs work often DOES produce a
  legitimate tracked docs diff, so a REAL repo diff on a docs lane is denied and told to re-route
  to `lane:code-change`; only a genuinely-empty tree is exempt.
- The shared downstream exemption flag is renamed `research_clean_exempt` → `report_only_clean_exempt`
  (honest lane-agnostic name); `lane:research` / `lane:no-code-remediation` / `lane:code-change`
  behaviour is unchanged.
