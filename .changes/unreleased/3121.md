### Changed

- **doc-coverage gate now verifies declared `UPDATED` surfaces appear in the PR diff** (#3121, Epic
  #2707). The collaborator-gate fetches the PR file list and the doc-coverage validator advisory-flags
  any surface declared `UPDATED`/`DONE` that no changed file actually touches — closing the gaming hole
  where a handoff could claim a doc was updated without touching it. This **wires in the previously
  unused `doc-coverage-diff-verify.js` module** (shipped by #2716 but never called by any gate — a
  phantom-completion), reusing it as the single source of truth rather than adding a duplicate. Ships
  **advisory-first** (won't block in-flight PRs); promotion to blocking follows the replay-eval-gated
  model. Local pre-push validation (no PR file list) skips the check gracefully.
