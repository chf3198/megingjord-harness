### Added
- Positive epic close-eligibility signal (F1 of Epic #3517, per ADR-020 §D1): new weekly `epic-close-eligible.yml` + pure detector `scripts/global/epic-close-eligible.js` that surfaces `signal:close-eligible` when every child of an OPEN `type:epic` is terminal. Complements the #3350 close-time veto (surface-only invariant I0 — it NEVER closes an epic). Reads children via GraphQL sub_issues **with the `GraphQL-Features: sub_issues` header** and a REST cross-check that must agree — closing the #3354 fails-open trap: any detection degradation fails closed (no signal + `SUBISSUE_DETECT_DEGRADED` incident). A 2-sweep circuit-breaker (`CLOSE_ELIGIBLE_PENDING`) prevents flap, and the label auto-clears (`CLOSE_ELIGIBLE_CLEARED`) if a child reopens. Kill-switch: `CLOSE_ELIGIBLE_DISABLED=1`. Tests incl. a regression proving the signal cannot bypass or weaken the #3350 veto. Uses the `signal:*` namespace (D-0002).
