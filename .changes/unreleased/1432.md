### Changed
- `epic-traceability-lint.yml` now re-validates the **parent Epic** when a child ticket closes (#1432). Previously the workflow early-returned on any non-Epic issue, so a child close never re-triggered parent traceability validation — the drift (Epic body missing a child reference) went unnoticed until someone manually edited the Epic. Parent resolution is **native Sub-issue parent first, prose fallback second** (`Refs Epic #N` | `Epic: #N` | `Parent: #N`), via the new pure module `scripts/global/megalint/epic-parent-resolve.js`. This corrects drift in the original 2026-05-12 AC, which predated the Sub-issues primitive and named only the prose forms. The Epic-edited/opened path is unchanged. tdd-pyramid coverage: `tests/epic-parent-resolve.spec.js` (14 cases). Advisory mode preserved (promotes to required after soak).
