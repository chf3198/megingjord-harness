## fix(governance): worktree tool boundary — restrict file-editing tools to registered workspace (#3243)

Adds `instructions/worktree-tool-boundary.instructions.md` (all-teams scope)
codifying that file-editing tools (`view_file`, `replace_file_content`, etc.)
are permitted **only** within the registered workspace path. Any git worktree
at a sibling path must use shell commands (`sed -i`, `cat`, `patch`) to avoid
triggering VS Code "Allow in Workspace?" authorization dialogs on the client.

**Root cause (#3243)**: During session 684fe057, Aria Mason used file-editing
tools on `/home/curtisfranks/devenv-ops-2826` (outside the workspace),
triggering authorization dialogs across dozens of edits. No rule prohibited it.
