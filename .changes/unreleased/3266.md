fix(hooks): research-lane close/Stop no longer false-blocked by phantom merge demand (#3266)

`lane:research` sessions are report-only and produce no PR or merge by design, yet the
governance hooks demanded an Admin merge and hard-blocked a clean research close/Stop. Two
root causes fixed together:

- **Honest `code_touched`** — `tool_activity.mark_tool_activity` no longer flips a touch flag
  for mutations OUTSIDE the tracked repo working tree (scratchpad / `/tmp` / `~/.claude`);
  only a real in-repo change flips it (fail-safe when the repo root cannot be resolved).
  Extends #2978 to the remaining non-repo / scratchpad execution paths.
- **Research-lane carve-out** — a new `active_ticket_is_research_lane()` mirrors the
  no-code-remediation carve-out at BOTH surfaces: the `pretool_guard` issue-close block and
  the Stop-hook Admin-op gate (`stop_checks` / `admin_patterns.required_admin_ops`). When the
  ticket is `lane:research` AND the working tree is clean, zero Admin ops are required and
  close/Stop is permitted with no merge.
- **Anti-abuse guardrail** — if a REAL repo diff exists while the ticket is `lane:research`,
  the close is denied and told to re-route to `lane:code-change` (the merge gate is skipped
  only when there is genuinely nothing to merge).
