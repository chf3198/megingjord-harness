### Added
- `scripts/global/governance-qa-cache.js` + `npm run qa:cache` (#3142, Epic #3137 T4): `get(question)` / `put(question, answer)` over a normalized-hash key (reuses `memory-dedup-lint` `normalizeKey` + a full-content sha256 suffix so same-prefix questions don't collide), so a repeated governance question is a cache **hit** rather than a re-inference (saves tokens on both lanes). Case/whitespace-insensitive; bounded store (drop-oldest); resilient to an absent/corrupt store; a miss is a clean `null` pass-through. The embedding/cosine version is deferred (replay-eval-gated). Deterministic, local, $0 — no embeddings.
