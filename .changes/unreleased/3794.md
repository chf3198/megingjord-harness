### Added

- `scripts/global/mock-mcp-resource-server.js` (#3794, Epic #3789) — a $0/Tier-0 local mock
  OAuth 2.1 / MCP resource server: serves RFC 9728 Protected Resource Metadata at
  `/.well-known/oauth-protected-resource` and a DPoP-gated `/mcp` endpoint that enforces the
  RFC 6750/9449 error taxonomy (`401 invalid_token`, `403 invalid_target`).
- `scripts/global/mock-mcp-dpop.js` — Ed25519 DPoP + token primitives (mint/verify, JWK
  thumbprint binding via `cnf.jkt`); `scripts/global/mock-mcp-verify.js` — the end-to-end
  driver that runs the four mandatory security negatives (forged-proof → 401, replay → 401,
  wrong-audience → 403 `invalid_target`, no-key-leak) and emits the
  `ephemeral_verification_evidence` block (per-test PASS + `transcript_sha256`) consumed by #3795.
- `tests/mock-mcp-resource-server.spec.js` (tdd-pyramid, AC1–AC6) and
  `tests/stress-mock-mcp-resource-server.spec.js` (stress-test: 2000-case adversarial-input
  fault-injection + a p99 latency budget) — supplies the rejection counterparty that made these
  security negatives unverifiable headless (Epic #3789 Phase-0 §2).
