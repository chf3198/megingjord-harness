### Added

- Fleet-dev execution path with a two-part objective gate + one-tier escalation (#2795, P1-2 of Epic
  #2791): `executeFleetDev` dispatches a fleet implementation, then runs BOTH a functional gate
  (lint+tests) AND a security gate (real local secret-exposure scan; dep-review/SAST injectable) — only
  when both pass is the result accepted (the G3 zero-cost win). Any failure escalates EXACTLY one tier,
  never silent-to-Premium: a capability failure steps fleet→haiku→premium; an availability failure (fleet
  unreachable) fails over to the $0 free-cloud lane first (#2619). The escalation reason (capability vs
  availability) is recorded for the #2796 governor + telemetry. Fail-closed: a gate that can't run is a
  FAIL. New: `scripts/global/fleet-dev-execute.js`, `scripts/global/fleet-dev-gates.js`.
