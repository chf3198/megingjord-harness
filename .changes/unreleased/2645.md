### Added
- `scripts/global/load-local-env.js` — a shared, secret-safe shim that auto-hydrates the gitignored repo-root `.env` into `process.env` so zero-cost review/provider dispatch (fleet + free-cloud) finds keys with no manual `set -a; . ./.env` step (#2645, G3 enabler). Fill-don't-override precedence (an already-set var always wins), never logs values (one redaction-safe `env-hydrate:` audit line of NAMES only), graceful no-op on missing/malformed `.env`, disable via `MEGINGJORD_NO_DOTENV=1`, relocate via `MEGINGJORD_DOTENV_PATH`.

### Changed
- `free-cloud-dispatch.js`, `cascade-dispatch.js`, and `hamr-provider-wrapper.js` now call the shared `loadLocalEnvOnce()` shim before provider-key lookup, so a fresh session reaches the fleet + free-cloud zero-cost lanes without manual env sourcing (#2645). Promoted operator-memory guidance into `instructions/hamr-routing.instructions.md`. Scope boundary vs Epic #2291 (global cross-entrypoint hydration + secret-storage strategy) documented on the ticket.
