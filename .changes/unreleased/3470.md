Fixed the diff-aware readability lint gate falsely flagging GitHub ticket refs as magic numbers. An inline trailing `// #NNNN` comment on a code line is now stripped (quote-aware, preserving `https://` inside strings) before the magic-number scan, and a digit run adjacent to a hyphen/underscore/word-char (`F6-3424`, `ticket3424`) is no longer treated as a standalone numeric literal. Genuine magic numbers (`5000`) are still flagged. (#3470)
