- **Merge-gate real-PR verification (#3344):** the pretool_guard admin-merge gate no longer false-blocks a genuine, CI-green PR when the cwd-keyed `admin_ops.pr_create` flag is lost to cwd-churn. When the flag is unrecorded it verifies a real OPEN PR for the merge ref via a read-only lookup (`live_checks.open_pr_for_ref`) and allows the merge only on a confirmed-OPEN PR; it fails closed (retains the block) on no-PR or an indeterminate gh result, so the gate stays honest while no longer stranding a legitimate Admin merge.
