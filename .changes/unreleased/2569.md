### Added
- Credential prompt guard (#2569, G1/G4): `scripts/global/credential-availability.js` — a deterministic, value-free guard so the operator never asks the client for a secret already available locally. `isSecretLocallyAvailable(name)` probes `process.env` + the approved local `.env` (via #2645's `loadLocalEnv`, non-mutating); `classifyCredentialRequest(text)` conservatively distinguishes a credential request from ordinary clarification (anti-over-block); `preCredentialPromptCheck([names])` returns `use-local` when any is available else `report-absent-no-prompt` (absent ⇒ report + terminal-entry, never request the raw secret in chat). New policy `instructions/credential-prompt-guard.instructions.md` (wired into `CLAUDE.md`) codifies the forbidden behavior; unit + drift-lint tests cover the Tavily/GitHub recurrence cases.
