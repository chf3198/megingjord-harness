### Governance — none-vs-candidate reconciler, anti-checkbox-fatigue (#3433, Epic #3425 P1-f)

`scripts/global/none-vs-candidate-reconciler.js` implements the load-bearing rule: a
`flaws_recognized: none` is valid only when the per-review-point checkpoint surfaced **zero** candidates
(or only un-escalated low-severity ones). A `none` contradicted by a medium/high candidate is a
**violation** — a lazy `none` now fails against machine-produced evidence, not passes. The AC-R3 state
machine: `none` + 0 → pass; + low-sev only → advisory; + ≥3 same-`pattern_id` low-sev (recurrence-escalated
to medium) → violation; + any medium/high → violation. Carve-out: a **medium-confidence F6** contradiction
never forces a violation (advisory-only, dodging API-latency false-positives). Wired into
`megalint/flaws-recognized.js` when a surfaced-candidate feed is supplied (`candidatesBySurface`), lenient
otherwise. Ships **advisory** until the P1-g replay-eval promotion gate (#3434).
