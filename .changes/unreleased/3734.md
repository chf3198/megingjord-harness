fix(infra): derive Sandbox Launcher Sync branch list from the runtime-catalog SSoT (#3734)

The `post-merge-sandbox-sync.yml` workflow force-synced only three launcher refs
(copilot/codex/claude-code) while `worktree-governance-audit.js` validated all
five catalog runtimes. `sandbox/antigravity` and `sandbox/cursor` fell behind
`main` on every merge, so `worktree-governance-required` failed and
`baton-authority/merge` blocked **all** PRs. Both consumers now derive their
launcher set from one shared helper (`scripts/global/sandbox-launcher-targets.js`)
reading the Epic #3411 runtime-catalog SSoT, with a golden-file regression test
(`tests/sandbox-launcher-sync-golden.spec.js`) asserting the sync list equals the
audit's `validTargets` so the two can never drift again.
