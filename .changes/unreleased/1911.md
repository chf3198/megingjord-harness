### Added

- `scripts/global/post-merge-sweep.js` (#1911, Epic #1894): post-merge sweep that force-closes ticket drifters left OPEN when GitHub's auto-close-via-PR-keyword misfires (selective-miss across multi-keyword bodies, or commit-message-only keywords that GitHub ignores). Parses the full GitHub close-keyword set (`close[sd]?`/`fix(e[sd])?`/`resolve[sd]?`, superset of AC2) from the PR body AND every commit message, de-duplicates citations, polls each cited issue's state for up to ~30s, and force-closes survivors with an audit comment. Dependency-injected GitHub client, clock, and audit-writer keep the core pure (tests never touch real issues or sleep). A state-read error yields `skipped-errored` (never a close) — OA2 mass-close guard.
- `.github/workflows/post-merge-sweep.yml` (#1911): advisory-mode workflow on `pull_request: closed` + `workflow_dispatch`. Reports `would-force-close` by default; set repo variable `POST_MERGE_SWEEP_ENFORCE=1` to enforce (Epic #1612 promotion model). Pinned-SHA actions; `issues: write` least-privilege.
- `tests/post-merge-sweep.spec.js` + `tests/stress-post-merge-sweep.spec.js` (#1911): 7 unit (AC7) + 4 stress (fault-injection chaos paths + p99 parse-latency budget, per test-methodology-matrix). Registered via `npm run test:post-merge-sweep` / `npm run stress:post-merge-sweep` and `harness:self-test` registry entry `post-merge-sweep` (AC8). Audit surface: `~/.megingjord/post-merge-sweep.jsonl` (event-schema-v3).
