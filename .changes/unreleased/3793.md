## [#3793] Narrow default-OFF MCP adapter opt-in gate (Epic #3789)

**Added** `scripts/global/mcp-adapter-gate.js` — the single source of truth for the
`MEGINGJORD_MCP_ADAPTER_ENABLED` opt-in flag (default-OFF, fail-closed). `mcpAdapterEnabled()`
returns `{enabled, flag, advisory}` and emits a non-silent OFF advisory (G8) when the adapter is
consulted while dark, suppressible via `MEGINGJORD_QUIET_RESOLVER=1`.

**Added** a new `opt-in-enabler` classification to `scripts/global/baton-bypass/env-flag-classifier.js`
and registered `MEGINGJORD_MCP_ADAPTER_ENABLED` there (`ci_authority: false`). This resolves audit
finding **C-8**: dark-merging the HAMR MCP adapter must NOT reuse the global `MEGINGJORD_HAMR_DISABLED`
kill-switch, which would collaterally disable every HAMR Tier-2 subsystem.

The `hamr-mcp-adapter.js` wiring (one `require` of this gate) lands with PR #3050 on child #3796; the
gate ships and is unit-tested here on `main` independently (`tests/mcp-adapter-gate.spec.js`, 7 cases).
