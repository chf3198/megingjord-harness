# HAMR v3 — Harness-Aware Mesh Routing

Ticket #873 (parent EPIC #860). Lane: docs-research. 2026-05-04. Builds on v1 (#861) + v2 (#863). Acronym: **HAMR — Harness-Aware Mesh Routing**.

v1 picked CF Worker + R2 + KV + MCP. v2 hardened (embedded floor, Cosign, per-user JWT, 8-provider caching, A2A mailbox, Yjs wiki). v3 optimizes 5 axes — replacing v2 only where primary sources show a strictly better path.

## 1. Security — supply-chain to runtime

- **SLSA v1.0 Build L3 for the Worker.** `slsa-framework/slsa-github-generator` reusable workflow signs provenance via sigstore, uploads to release assets ([Build L3 reqs](https://slsa.dev/spec/v1.0/levels#build-l3); [generator](https://github.com/slsa-framework/slsa-github-generator)). Consumers run `slsa-verifier verify-artifact` before trusting a new bundle. v2 signed the artifact; v3 attests **how it was built**.
- **OIDC-federated CI publishing.** Drop long-lived CF API tokens. [`cloudflare/wrangler-action@v3`](https://github.com/cloudflare/wrangler-action) + [GitHub OIDC](https://docs.github.com/en/actions/deployment/security-hardening-your-deployments/about-security-hardening-with-openid-connect) → CF workload identity ([CF OIDC](https://developers.cloudflare.com/cloudflare-one/identity/idp-integration/github/)). Same for npm via [trusted publishing GA 2025-09](https://github.blog/changelog/2025-09-23-trusted-publishing-with-npm/). Zero secrets in repo settings.
- **Cosign Bundle 1.0** ([spec](https://github.com/sigstore/cosign/blob/main/specs/BUNDLE_SPEC.md)) packs `messageSignature` + cert chain + tlog inclusion proof into one `*.sigstore` JSON. Offline-verifiable; saves Rekor round-trip per verify. **Change from v2.**
- **MCP OAuth 2.1 + DPoP** ([MCP §2.4](https://modelcontextprotocol.io/specification/2025-06-18/basic/authorization); [RFC 9449](https://datatracker.ietf.org/doc/html/rfc9449)) binds tokens to a holder-of-key proof per request — defeats keychain bearer-only theft. **Change from v2.**
- **Capability manifests in the bundle.** Each MCP-exposed tool carries an explicit allowlist:
  ```json
  {"tool":"github.repo.read","scopes":["repo:ro"],"surfaces":["claude-code"],
   "max_calls_per_session":50,"data_classes":["public-code"]}
  ```
  A `bundle-fim-5kb` profile requesting `github.repo.write` gets 403 at the MCP wire. Maps to Anthropic [`tool_use input_schema`](https://platform.claude.com/docs/en/agents-and-tools/tool-use/overview).
- **Secret scanning at the bundle layer.** [gitleaks](https://github.com/gitleaks/gitleaks) `--redact` runs in `prepack`; CI re-runs on the published bundle. Fleet-generated wiki drafts pass through [TruffleHog `--only-verified`](https://github.com/trufflesecurity/trufflehog) before R2 queue write — `.env` / GH PATs / OpenAI keys never reach R2.
- **Anthropic content-blocks narrowed.** Governance-critical paths use `tool_choice:{type:"tool",name:"<allowed>"}` + `disable_parallel_tool_use:true` ([forcing tool use](https://platform.claude.com/docs/en/agents-and-tools/tool-use/overview#forcing-tool-use)). No free-form output.

## 2. User-friendliness

```
$ npx megingjord init
[1/5] Detecting harness…  claude-code  found
[2/5] Identity… browser opening for GitHub device flow [Y/n]
[3/5] Bundle…  fetched  bundle-fim-5kb-3a9c.json  (4.7 KB, 60ms)
[4/5] MCP…  registered  harness-context  (cap: read-only)
[5/5] Ready.  npm run hamr:status  to inspect.
```

- **One command** writes `.claude/`, `CLAUDE.md`, `.github/instructions/`, registers MCP, writes `~/.megingjord/identity.json`. Idempotent. Pattern from [Vercel CLI](https://vercel.com/docs/cli) and [Stripe CLI](https://docs.stripe.com/stripe-cli).
- **Magic-link OAuth** = GitHub device flow ([`gh auth login`](https://cli.github.com/manual/gh_auth_login)). No client secret, no redirect URI, no localhost. 8-char code paste.
- **Progressive disclosure.** `--profile` / `--tier` / `--substrate` resolve from harness + task + `substrate-health.json`. `--explain` prints resolved config + provenance.
- **`hamr:status`** single line:
  ```
  hamr  ok  substrate=cf-worker  identity=ok(23h-left)  bundle=fim-5kb-3a9c  cache=82%  age=4h
  ```
  Non-zero exit on red. Equivalent to `wrangler whoami` + `gh auth status` collapsed.
- **Failure surfacing — never silent**:
  ```
  hamr  warn  substrate=embedded-floor  reason="cf-worker:5xx"  bundle_age_days=2
        falling back; cascade-dispatch disabled
  ```
- **Diátaxis docs** ([framework](https://diataxis.fr/)). `docs/hamr/{tutorial,how-to,reference,explanation}/`. v2 had unstructured docs.
- **Schema drift / self-update.** Bundle [JSON Schema `$id`](https://json-schema.org/draft/2020-12/json-schema-core#name-the-id-keyword): `https://schemas.megingjord.dev/bundle/2026-05.json`. Client refuses unknown major; `hamr:doctor` polls daily.

## 3. Token-use minimization (no quality loss)

The largest projected win. Anchored by Anthropic [Effective context engineering, 2025-09](https://www.anthropic.com/engineering/effective-context-engineering-for-ai-agents) and [multi-agent research, 2025-06](https://www.anthropic.com/news/built-multi-agent-research-system).

- **Per-tier sub-bundles** (single biggest lever):

  | Profile | Size | Use | ~Tokens |
  |---|---|---|---|
  | `bundle-fim-5kb` | 5 KB | autocomplete (qwen2.5-coder) | 1.2K |
  | `bundle-coder-15kb` | 15 KB | single-file refactor (Haiku) | 3.5K |
  | `bundle-reasoner-30kb` | 30 KB | architecture / debug (Sonnet/Opus) | 7K |
  | `bundle-governance-12kb` | 12 KB | baton + ticketing | 2.8K |

  Hook reads task class, picks profile. v2's monolithic 30 KB is wasteful on FIM. **-60% tokens for fleet-lane FIM.**
- **Differential bundles.** Embedded floor pinned at `floor_sha`; server returns `bundle-diff-<floor>-<head>.json` as [JSON Patch RFC 6902](https://datatracker.ietf.org/doc/html/rfc6902). Median diff < 2 KB. **-85% network bytes.**
- **Distilled constitution.** 14 instruction files ≈ 5K preamble tokens. Offline LLM-as-compressor → ~500-token "constitution" ([Anthropic constitutional AI](https://www.anthropic.com/news/claudes-constitution); [LLMLingua-2 EMNLP-2024](https://arxiv.org/abs/2403.12968)). Verification: 100-prompt rule-coverage suite must reproduce baton-routing decisions on ≥97% before ship. **-75% governance preamble**, version-pinned in bundle as `constitution.txt`.
- **Structured outputs over CoT prose.** Handoff artifacts, status updates, evidence blocks via Anthropic [forced tool use](https://platform.claude.com/docs/en/agents-and-tools/tool-use/overview) or [OpenAI strict JSON Schema](https://platform.openai.com/docs/guides/structured-outputs). **30–60% shorter** vs free-form.
- **Just-in-time wiki.** Bundle ships only the index (~3 KB of `[[wikilink]]→uri`); full body via `resources/read` on dereference. Pattern from Anthropic context-engineering §"just-in-time". **-90% wiki bytes per cold start.**
- **Sub-agent isolation** ([sub-agents](https://platform.claude.com/docs/en/agents-and-tools/sub-agents)) for long-horizon governance — fresh window, structured return. v2 silent.
- **Compact memory in bundle.** `memory_digest` ≤500 tokens summarizing the operator's last-7-days terminal tickets. Pattern from [Anthropic memory cookbook](https://github.com/anthropics/anthropic-cookbook/tree/main/skills/memory).
- **LLMLingua-2 at runtime: rejected.** Tokenization cost ≈ savings under a 30-KB bundle. Use **only offline** in constitution distillation.

**Stack effect** (governance-heavy session): v2 ≈ 8K preamble × 50 calls = 400K tok. v3 ≈ 1.5K × 50 = 75K. **~80% reduction**, unchanged rule-coverage.

## 4. Paid-token + rate-limit compliance

- **Anthropic Batch** ([Message Batches](https://platform.claude.com/docs/en/build-with-claude/batch-processing)) — 50% off, 24h SLA — for nightly wiki anneal, weekly profile-governance audit, periodic constitution rule-coverage. Not for live baton. **OpenAI Batch** ([batch](https://platform.openai.com/docs/guides/batch)) and **Gemini batch** ([batch mode](https://ai.google.dev/gemini-api/docs/batch-mode)) — same 50%/24h. `harness-batch-router.js` picks cheapest per job class; falls back live if SLA misses.
- **Sticky cache routing.** Anthropic [prompt cache](https://platform.claude.com/docs/en/docs/build-with-claude/prompt-caching) reads at 0.1×. Place `cache_control` boundary deterministically right after the bundle; route same `bundle_sha256` to same workspace. Hit rate target ≥80% via `usage.cache_read_input_tokens`. v2 mentioned caching; v3 measures and alerts on hit-rate < 60%.
- **Anthropic context-editing** ([context management](https://platform.claude.com/docs/en/build-with-claude/context-management)) lets bundle stay in cache while older turns truncate. Enable for sessions > 30 turns.
- **Background mode** ([streaming](https://platform.claude.com/docs/en/api/messages-streaming)) for wiki-anneal CRDT merge — releases foreground.
- **Header-driven rate-limit spillover.** `harness-router.js` reads per-call:
  - Anthropic: `anthropic-ratelimit-{requests,tokens}-{remaining,reset}`
  - Groq: `x-ratelimit-{remaining,reset}-*` ([Groq](https://console.groq.com/docs/rate-limits))
  - OpenAI: `x-ratelimit-*` ([OpenAI](https://platform.openai.com/docs/guides/rate-limits))
  - OpenRouter: `x-ratelimit-*` per upstream
  Spillover when `remaining < 10%` OR `reset > task budget`. Order: Groq → Cerebras → OpenRouter free → Gemini Flash → Haiku. **Change from v2:** v2 named tiers; v3 reads headers and decides per-call.
- **Token-budget-per-task.** Each baton role declares budget in the handoff (`Manager: 4K`, `Collaborator: 30K`, `Admin: 6K`, `Consultant: 8K`). Hook fails loudly on exceed. From [Anthropic multi-agent](https://www.anthropic.com/news/built-multi-agent-research-system) ("subagents track their own budgets").
- **Free-tier first.** `npx megingjord init` detects workspace tier via `/v1/organizations/me`; defaults model to `claude-haiku-4-5` until upgraded.
- **`hamr:quota`.** Queries [Anthropic count_tokens](https://platform.claude.com/docs/en/api/messages-count-tokens) preflight; [OpenAI usage](https://platform.openai.com/docs/api-reference/usage) aggregates; Groq via response headers. Reports remaining budget per provider into `substrate-health.json`.

## 5. Maintenance

- **Wrangler 4.x** ([install/update](https://developers.cloudflare.com/workers/wrangler/install-and-update/)). `wrangler deploy --dry-run` = reproducible bundle hash for SLSA. `wrangler deployments list` gives drift detection.
- **Worker observability.** [Tail Workers](https://developers.cloudflare.com/workers/observability/logs/tail-workers/) → sibling Worker → [Workers Analytics Engine](https://developers.cloudflare.com/analytics/analytics-engine/) (free 10M datapoints/month). [Logpush](https://developers.cloudflare.com/logs/logpush/) only on paid. AE feeds `hamr:status`.
- **R2 lifecycle** ([object lifecycles](https://developers.cloudflare.com/r2/buckets/object-lifecycles/)): `mailbox/*` 7 days, `wiki-queue/*` 30 days, `bundle/*` keep latest 50 + 90-day age. Prevents storage drift.
- **Bundle schema versioning.** [JSON Schema `$id` + `$schema`](https://json-schema.org/draft/2020-12/json-schema-core#name-the-schema-keyword); `additionalProperties: true` = old client / new bundle degrades silently. Major bumps dual-publish 30 days.
- **Drift detection.** `hamr:doctor` diffs `~/.megingjord/snapshot.json` vs `bundle/latest`; reports added/removed instructions, wiki staleness > 30 days, identity TTL < 24h. Pattern from [Doppler CLI](https://docs.doppler.com/docs/cli) `secret status`.
- **Dependabot scope.** Worker source under `cloud/worker/` is its own npm workspace; independent patch cadence.
- **Migration paths.** R2 price change → bundle storage flips to GitHub release assets (free, unmetered); Worker keeps caching/auth only. CF down >24h → `hamr:emergency` flips health to `embedded-only`, surfaces banner. No code change required for failover.

## What changes from v2 vs what stays

**Stays (v2 still right):** CF Worker + R2 + KV; embedded-floor + release-asset + Pages tiers; Cosign keyless principle; Yjs CRDT; A2A mailbox; per-provider caching matrix; substrate-health probe; wiki via GitHub App PRs.

**Changes:**
1. Cosign artifact → **Cosign Bundle 1.0** single-file format.
2. MCP bearer → **OAuth 2.1 + DPoP**.
3. Manual CF/npm tokens → **OIDC trusted publishing**.
4. Build → **SLSA L3 provenance** attached.
5. Monolithic bundle → **4 per-tier profiles**.
6. Full-bundle delivery → **Differential JSON Patch**.
7. Full instruction files → **Distilled constitution** (offline, rule-coverage gated).
8. Wiki content in bundle → **JIT retrieval** (index only).
9. Implicit caching → **Sticky-cache + measured hit-rate + context-editing**.
10. Tier-named routing → **Header-driven rate-limit spillover**.
11. Long governance jobs → **Anthropic / OpenAI / Gemini Batch** (50%).
12. Bare tool exposure → **Capability manifests** with allowlists.
13. None → **gitleaks/trufflehog at prepack + R2 ingress**.
14. Implied CLI → **`init / status / doctor / quota`** canonical.
15. Unstructured docs → **Diátaxis** four-quadrant.

## Risk register

| # | Risk | Mitigation | Residual |
|---|---|---|---|
| 1 | DPoP key compromise on host | OS keychain; 24h rotation; revoke endpoint | 24h window |
| 2 | Distilled constitution drops a rule | 100-prompt ≥97% gate; periodic re-verify | Long-tail edge cases |
| 3 | Wrong sub-bundle profile picked | `--profile=auto` + decision log + manual override | -1 round-trip |
| 4 | SLSA verifier failure blocks CI | Cache last-good provenance; embedded-floor fallback | Slower hot-fix |
| 5 | Differential patch corrupts client | Patch + full-hash check; refetch on mismatch | One round-trip |
| 6 | Batch SLA miss >24h | `harness-batch-router` re-routes live after 26h | Cost upside missed |
| 7 | Rate-limit header misreport | Tail Worker logs every 4xx/429; alert > 5/h | Brief over-spill |
| 8 | Capability manifest blocks legit tool | `--grant <cap>` once; logged | Onboarding friction |
| 9 | OIDC trust policy drift | Codified in `wrangler.toml` + `slsa-verify` gate | Opaque CF console state |
| 10 | gitleaks false positive blocks publish | Versioned `--allow` rules; manual override label | Noisy publish day |

## Acronym

**HAMR — Harness-Aware Mesh Routing** confirmed. Captures: bundle is harness-aware (per-profile, per-provider), topology is a mesh (Worker + R2 + KV + Pages + release-asset + embedded floor + mailbox), routing is the central function (substrate health + rate-limit spillover + provider caching). Alternatives: HARM (rejected — negative valence); MERIDIAN (too long).
