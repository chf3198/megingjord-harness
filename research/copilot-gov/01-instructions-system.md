# 1. Copilot Custom Instructions System

## 1.1 Repository-wide: copilot-instructions.md

- Lives at `.github/copilot-instructions.md`
- Auto-appended to every chat request in the workspace
- Supported: VS Code, JetBrains, Visual Studio, github.com, CLI
- Priority: Personal > Repository > Organization
- Can be auto-generated by Copilot cloud agent
- **Governance value**: Defines project standards (advisory only)

## 1.2 Path-specific: .instructions.md files

- Live in `.github/instructions/` (recursive subdirectories)
- YAML frontmatter with `applyTo` glob (e.g., `**/*.py`)
- Optional `excludeAgent`: `"code-review"` or `"cloud-agent"`
- Applied when matching files are in agent context
- Can also match via semantic `description` field

## 1.3 AGENTS.md

- Open standard (agentsmd/agents.md, 20.1k stars)
- Auto-detected by VS Code, Copilot CLI, cloud agent
- Nearest `AGENTS.md` in directory tree takes precedence
- Experimental: nested AGENTS.md in subfolders
- Also supports CLAUDE.md, GEMINI.md for cross-agent compat
- **Governance value**: Single source for all AI agents

## 1.4 Agent Skills (SKILL.md)

- Open standard (agentskills.io) — portable across VS Code, CLI, cloud
- Stored: `.github/skills/`, `.claude/skills/`, `~/.copilot/skills/`
- YAML: `name`, `description`, `user-invocable`, `disable-model-invocation`
- Three-level loading: discovery → instructions → resources
- Invocable as slash commands (`/skill-name`)
- Auto-loaded by model based on description relevance
- Extensions can contribute via `chatSkills` contribution point

## 1.5 Instruction Priority (confirmed)

1. Personal instructions (user-level, highest)
2. Repository instructions (copilot-instructions.md or AGENTS.md)
3. Organization instructions (lowest)
4. All combined; conflicts resolved by priority order
