# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please
report it responsibly. **Do not open a public issue.**

### Preferred: GitHub Private Vulnerability Reporting

1. Go to the [Security tab](https://github.com/chf3198/devenv-ops/security)
2. Click **Report a vulnerability**
3. Provide a description, steps to reproduce, and impact assessment

### Alternative: Direct Contact

Contact the maintainer via the information listed on
[curtisfranks.com](https://curtisfranks.com).

## Response Timeline

- **Acknowledgment**: Within 72 hours
- **Assessment**: Within 7 days
- **Fix or mitigation**: Best effort, typically within 30 days

## Scope

This policy covers the devenv-ops repository, including:

- Skills, agents, hooks, and instructions
- Dashboard (Alpine.js web app)
- Deploy and sync scripts
- Plugin manifest and cross-tool symlinks

## Out of Scope

- Third-party dependencies (report to upstream maintainers)
- Issues in VS Code, GitHub Copilot, or other host platforms
- Personal fleet infrastructure (Tailscale, Ollama, OpenClaw)

## Supported Versions

Only the latest release on the `main` branch is supported.
